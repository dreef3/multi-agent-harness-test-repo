Implementation Plan: Codebase Structure Analysis

Context
- Repository: https://github.com/dreef3/multi-agent-harness-test-repo.git
- Repository ID: d8a593e2-8cdc-40c5-9a6a-e50f1d497095
- Goal: perform a read-only, reproducible analysis that produces inventory.json and report.md describing the repo structure, languages, manifests, CI, tests, Docker, scripts, and prioritized recommendations.

Overall approach
- Implement an automated, repeatable script or sequence of commands that produces a small set of artifacts:
  - artifacts/inventory.json (machine-readable)
  - artifacts/report.md (human-readable summary)
  - artifacts/summary.txt (1-2 line summary)

Tasks (detailed)

1) Checkout & baseline snapshot
- Purpose: create a working copy and baseline file listing.
- Exact steps/commands:
  - git clone --depth 1 https://github.com/dreef3/multi-agent-harness-test-repo.git repo || exit 1
  - cd repo
  - mkdir -p artifacts
  - tree -a -L 2 > artifacts/tree_L2.txt
  - find . -maxdepth 2 -type f -print | sort > artifacts/files_top2.txt
- Files to inspect at this step (examples): README.md, LICENSE, .gitignore, CODE_OF_CONDUCT.md, CONTRIBUTING.md, CHANGELOG.md
- Deliverables: artifacts/tree_L2.txt, artifacts/files_top2.txt
- Estimated time: 2–5 min

2) Language detection & LOC counts
- Purpose: detect primary languages and rough size.
- Commands (preferred):
  - cloc --json . > artifacts/cloc.json
  - If cloc is not installed: use tokei, or fallback: find . -name "*.py" | wc -l etc.
- Expected output: artifacts/cloc.json with languages and line counts.
- Deliverables: artifacts/cloc.json and a short summary extracted into artifacts/lang_summary.txt
- Estimated time: 2–5 min

3) Manifest discovery & parsing
- Purpose: find and parse dependency/declarative files.
- Files and paths to check (scan these at repo root and first level):
  - package.json, package-lock.json, yarn.lock
  - pyproject.toml, setup.py, requirements.txt, Pipfile, Pipfile.lock
  - go.mod
  - Cargo.toml
  - pom.xml, build.gradle
  - Makefile
  - Dockerfile(s)
  - requirements-dev.txt, dev-requirements
- Commands:
  - for package.json: jq '{name:.name,version:.version,scripts:.scripts,dependencies:.dependencies,devDependencies:.devDependencies}' package.json > artifacts/package_meta.json
  - for pyproject.toml: python -c "import tomllib,sys;print(tomllib.loads(open('pyproject.toml','rb').read()))" > artifacts/pyproject.json  (or use toml library)
  - grep -E "^name|^version|^scripts|dependencies" package.json -n || true
- Deliverables: parsed manifest snippets in artifacts/, and a table of languages-to-manifests mapping in artifacts/manifests_summary.txt
- Estimated time: 10–20 min (depends on languages)

4) CI / workflow inspection
- Purpose: detect and parse CI config and extract build and test steps and matrix.
- Files/paths to inspect:
  - .github/workflows/*.yml
  - .gitlab-ci.yml
  - .circleci/config.yml
  - Azure Pipelines: azure-pipelines.yml
- Commands:
  - grep -R "runs-on\|steps\|jobs\|strategy" .github/workflows -n || true
  - for each workflow: yq eval -o=json <workflow.yml> > artifacts/ci_<workflow>.json (or simple grep/extract)
- Deliverables: artifacts/ci_summary.md listing CI files and top-level jobs/steps.
- Estimated time: 10–15 min

5) Docker and deployment artifacts
- Purpose: locate Dockerfiles, docker-compose, k8s manifests, Terraform.
- Files/paths:
  - Dockerfile, **/Dockerfile, docker-compose.yml, docker/**, k8s manifests under deploy/ or k8s/
  - Terraform under infra/ or terraform/
- Commands:
  - rg "^FROM\b" -n || true
  - find . -type f -name Dockerfile -print > artifacts/dockerfiles.txt
  - rg "docker-compose|kubernetes|apiVersion: apps/" -n || true
- Deliverables: artifacts/dockerfiles.txt and artifacts/deploy_summary.md
- Estimated time: 5–10 min

6) Tests, test runners, and coverage
- Purpose: identify where tests live, test framework, and basic counts.
- Files/paths:
  - tests/, test/, src/**/tests, pytest.ini, tox.ini, .coveragerc, jest.config.js
- Commands:
  - find . -type d -name tests -print
  - find . -name "*_test.py" -or -name "test_*.py" | wc -l > artifacts/test_count.txt
  - for JS: rg "describe\(|it\(|test\(" -n || true
- Deliverables: artifacts/test_locations.txt, artifacts/test_count.txt, artifacts/test_summary.md
- Estimated time: 5–15 min

7) Scripts, entrypoints, and developer tooling
- Purpose: list Makefile targets, package.json scripts, scripts/*.
- Files/paths:
  - Makefile, package.json (scripts), scripts/, bin/, tools/
- Commands:
  - awk '/^[a-zA-Z0-9_\-]+:/ {print $1}' Makefile | sort > artifacts/make_targets.txt || true
  - jq '.scripts' package.json > artifacts/package_scripts.json || true
  - ls -la scripts || true
- Deliverables: artifacts/scripts_summary.md
- Estimated time: 5–10 min

8) Static health checks & quick dependency health
- Purpose: flag obvious risks: missing LICENSE, outdated lockfile, stale CI, large binary blobs, secrets.
- Checks:
  - Check for LICENSE file
  - Check for large files (>10MB): git rev-list --objects --all | grep $(git verify-pack -v .git/objects/pack/*.idx | sort -k3 -n | tail -10)
  - Search for possible secrets: rg "(API_KEY|PASSWORD|SECRET|BEGIN RSA PRIVATE KEY)" -S || true
  - Look for common linter/formatter configs: .eslintrc*, .prettierrc, .flake8
- Deliverables: artifacts/health_checks.md with flags and severity tags (low/medium/high)
- Estimated time: 10–20 min

9) Aggregate report & recommendations
- Purpose: synthesize all findings into artifacts/report.md and artifacts/inventory.json.
- Composition:
  - Top summary (1 paragraph)
  - Top-level tree snapshot and language breakdown
  - Manifests found with parsed metadata
  - CI and Docker summary
  - Tests and coverage summary
  - Tooling and scripts
  - Health checks and risks
  - Top 3 prioritized recommendations with rough effort estimates (S/M/L)
- Example structure of artifacts/inventory.json (fields):
  {
    "repo": "multi-agent-harness-test-repo",
    "languages": {"Python": 12000, "JavaScript": 4500},
    "top_level": ["src","tests","docs",".github","Dockerfile"],
    "manifests": {"package.json": {...}, "pyproject.toml": {...}},
    "ci": [".github/workflows/ci.yml"],
    "dockerfiles": ["Dockerfile"],
    "tests": {"locations": ["tests/"], "count": 42},
    "health_flags": [{"type":"secret","path":"...","severity":"high"}]
  }
- Deliverables: artifacts/inventory.json, artifacts/report.md, artifacts/summary.txt
- Estimated time: 15–30 min

Acceptance criteria
- artifacts/inventory.json present and parsable
- artifacts/report.md includes clear recommendations
- No write operations to the repository other than writing artifacts under the local working copy

Operational notes
- If cloc, jq, or yq are not available in the environment, provide fallback commands (find + wc, python toml parsing, grep) in the runbook section.
- Use temporary branch or local checkout only; do not push to remote.

Estimated total time: 1–2 hours for a single engineer or automated job (depending on repo size).

Hand-off
- Attach artifacts to an issue or PR and summarize the top findings in the PR description. Mark areas recommended for follow-up with a suggested ticket template (title, description, severity, owner, ETA).
