Implementation Plan: Codebase Structure Analysis for E2E Test Repo (be3bb149-3d93-4669-b161-b04b0624a88b)

Repository: be3bb149-3d93-4669-b161-b04b0624a88b (https://github.com/dreef3/multi-agent-harness-test-repo.git)
Branch: create a new branch named analysis/codebase-structure

High-level tasks (each task is self-contained and can be run by a sub-agent):

Task 1 — Clone repository (local workspace)
- Purpose: Obtain a local copy to analyze.
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
  - git checkout -b analysis/codebase-structure
- Notes: If repository is large, use shallow clone (git clone --depth 1).
- Outputs: local repo folder ready for analysis.

Task 2 — Top-level inventory
- Purpose: List top-level files and directories and capture raw counts.
- Commands:
  - ls -la
  - tree -a -L 2 || (find . -maxdepth 2 -print)
  - du -sh * | sort -h (to spot large directories)
- Expected outputs: text file at analysis/raw/top-level.txt containing the outputs.

Task 3 — Language and code metrics
- Purpose: Identify languages and approximate code sizes.
- Preferred tools: cloc (https://github.com/AlDanial/cloc) or github-linguist; fallback to file extension heuristics.
- Commands (preferred):
  - cloc --by-file --report-file=analysis/raw/cloc-report.txt .
  - cloc --json --out=analysis/raw/cloc-report.json .
- Fallback commands:
  - find . -type f | sed -n 's/.*\.\([^.]*\)$/\1/p' | sort | uniq -c | sort -rn > analysis/raw/extensions-count.txt
- Outputs: analysis/raw/cloc-report.json (or extensions-count.txt)

Task 4 — Detect package manifests and build files
- Purpose: Find package manifests and note their locations and contents summary.
- Files to look for (top-down search):
  - package.json, package-lock.json, yarn.lock, pnpm-lock.yaml
  - pyproject.toml, setup.py, requirements.txt
  - go.mod
  - Cargo.toml
  - pom.xml, build.gradle, build.gradle.kts
  - composer.json
  - .NET: *.sln, *.csproj
- Commands:
  - rg --hidden --files "package.json|pyproject.toml|setup.py|requirements.txt|go.mod|Cargo.toml|pom.xml|build.gradle|composer.json" -S || find . -type f -iname 'package.json' -o -iname 'pyproject.toml' -o -iname 'go.mod' -o -iname 'Cargo.toml' -o -iname 'pom.xml' > analysis/raw/manifests.txt
  - For each manifest found, capture the first 200 lines into analysis/raw/manifests/<path>.txt
- Outputs: analysis/raw/manifests.txt and per-manifest excerpts.

Task 5 — CI/CD and automation files
- Purpose: Locate and summarize CI workflows and scripts.
- Files to look for:
  - .github/workflows/*.yml
  - .gitlab-ci.yml
  - circleci/config.yml
  - Jenkinsfile, Dockerfile(s), .dockerignore
- Commands:
  - rg --hidden --files ".github/workflows|.gitlab-ci.yml|circleci|Jenkinsfile|Dockerfile|.dockerignore" -S || find . -type f -iname '.gitlab-ci.yml' -o -path './.github/workflows/*' -o -iname 'Jenkinsfile' -o -iname 'Dockerfile' > analysis/raw/ci-files.txt
  - Save copies: cp <file> analysis/raw/ci/<relative-path>.yml
- Outputs: analysis/raw/ci-files.txt and copies of found CI files.

Task 6 — Tests and test frameworks
- Purpose: Identify presence of tests, test frameworks, and test commands.
- Actions:
  - Search for common test directories: test/, tests/, __tests__/, spec/
  - Search manifests for test scripts (e.g., package.json scripts.test)
- Commands:
  - rg --hidden --files "^test(s)?/|__tests__|spec/" -S || find . -type d -iname test -o -iname tests -o -iname '__tests__' > analysis/raw/test-dirs.txt
  - For JS: jq -r '.scripts.test // empty' package.json (if exists)
  - For Python: look for pytest.ini, tox.ini, setup.cfg test sections
- Outputs: analysis/raw/test-dirs.txt and test-framework-summary.txt

Task 7 — Entry points and executables
- Purpose: Identify main entry files and CLI tools.
- Actions:
  - For Node: look for "main" in package.json, bin entries
  - For Python: identify modules with __main__.py or console_scripts in entry_points
  - For Go: look for main packages (package main with main.go)
- Commands: parse manifests and search for files named main.go, __main__.py, index.js, server.js, app.js, src/main/java
- Outputs: analysis/raw/entrypoints.txt

Task 8 — Containers and infrastructure
- Purpose: Locate Dockerfiles, k8s manifests, terraform, helm charts.
- Commands:
  - rg --hidden --files "Dockerfile|\.k8s|helm|Chart.yaml|terraform|main.tf" -S || find . -type f -iname 'Dockerfile' -o -iname '*.tf' -o -iname 'Chart.yaml' > analysis/raw/container-infra.txt
- Outputs: analysis/raw/container-infra.txt

Task 9 — Large files and vendored dependencies
- Purpose: Identify bulky files and vendor directories.
- Commands:
  - git rev-parse --show-toplevel && git ls-tree -r -l HEAD | sort -k4 -n | tail -n 50 > analysis/raw/largest-files.txt
  - du -sh node_modules 2>/dev/null || true
  - find . -type d -name vendor -o -name node_modules -o -name .venv -maxdepth 4
- Outputs: analysis/raw/largest-files.txt

Task 10 — Generate structured JSON report
- Purpose: Consolidate findings into analysis/codebase-structure.json with fields:
  - repo_id
  - scan_timestamp
  - top_level_dirs (array)
  - language_breakdown (from cloc)
  - manifests_found (list with path and type)
  - ci_files_found
  - test_directories
  - dockerfiles
  - large_files (top 20)
  - recommendations (array)
- Implementation: build JSON using jq or a small Python script (example: scripts/generate_report.py) that reads raw files and outputs JSON to analysis/codebase-structure.json
- Outputs: analysis/codebase-structure.json

Task 11 — Create human-readable Markdown summary
- Purpose: Produce docs/codebase-structure.md which summarizes the findings for maintainers.
- Structure:
  - Title and repo id
  - Quick summary (languages, main language, top-level layout)
  - Key files and paths (manifests, CI, Dockerfile)
  - Tests and how to run them
  - Build and run instructions (if discoverable)
  - Notable issues and recommendations
  - Appendix: raw cloc summary and top-20 largest files
- Outputs: docs/codebase-structure.md

Task 12 — Commit, push, and open PR
- Purpose: Add the generated docs to a branch and open a PR.
- Commands:
  - git add docs/codebase-structure.md analysis/codebase-structure.json analysis/raw/
  - git commit -m "chore(analysis): add codebase structure report"
  - git push origin analysis/codebase-structure
- Note: Opening the PR will be performed by the harness. Do not run gh or create PR via scripts; the system will open the PR when planning document is written.

Task 13 — Deliverables and acceptance criteria
- Deliver the two generated files committed to a branch and PR:
  - docs/codebase-structure.md
  - analysis/codebase-structure.json
- Include raw evidence in analysis/raw/ (cloc JSON, manifest excerpts, CI file copies, top-level tree, largest files)
- The PR description should contain a brief summary and a pointer to the JSON for automation.

Tools and runtime requirements
- Required: git, bash, find, sed, awk, sort
- Recommended: cloc, tree, ripgrep (rg), jq, python3

Estimated effort
- Typical medium-size repository: 1–2 hours for full scan and report generation.

Notes for implementers
- Ensure any created branch uses a descriptive name (analysis/codebase-structure).
- Keep analysis non-destructive; only add new analysis files and do not modify existing source.
- If sensitive files are detected (.env, keys), redact contents from saved copies and flag them in the recommendations.
