Implementation Plan: Codebase Structure Analysis for E2E Test Repo

Overview
This plan defines discrete tasks to inspect the repository at https://github.com/dreef3/multi-agent-harness-test-repo.git and produce the requested analysis artifacts. Each task is self-contained and runnable on a Linux agent with git, node (optional), python (optional), ripgrep (rg) or grep, and tree installed. No dependency installation or tests will be executed.

Outputs (paths in the worker workspace)
- reports/multi-agent-harness-test-repo-structure.md  # human readable summary + detailed tree
- reports/multi-agent-harness-test-repo-structure.json # machine-readable inventory
- reports/file-list.txt                               # newline-separated list of all files
- reports/manifest-excerpts/                           # directory with copies/excerpts of key manifest files

Task 1 — Clone repository (implementation agent)
- Purpose: obtain a local copy for analysis.
- Command (run in a clean workspace):
  git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
- Output: repo/ checked out.
- Notes: use default branch (main) as provided.

Task 2 — Create reports directory
- Purpose: store outputs in one place.
- Commands:
  mkdir -p reports/manifest-excerpts

Task 3 — File listing and size summary
- Purpose: provide a baseline inventory
- Commands:
  cd repo
  # list all files
  git ls-files > ../reports/file-list.txt
  # produce a top-level tree (2-3 levels)
  (cd .. && tree -a -L 3 repo | sed '1d' > reports/top-tree.txt) || \
    (cd .. && find repo -maxdepth 3 -print | sed '1d' > reports/top-tree.txt)
  # repo size
  (cd repo && du -sh .) > ../reports/repo-size.txt
- Outputs: reports/file-list.txt, reports/top-tree.txt, reports/repo-size.txt

Task 4 — Detect languages & package managers
- Purpose: detect primary languages and package manifests
- Commands (from repo root):
  # detect manifest files
  rg --hidden --type-add 'manifest:ext(^(json|toml|yaml|yml)$)' -g '!*node_modules/*' 'package.json|pyproject.toml|Pipfile|requirements.txt|poetry.lock|go.mod|Cargo.toml|pom.xml|build.gradle|build.gradle.kts|package-lock.json|yarn.lock|pnpm-lock.yaml|lerna.json|pnpm-workspace.yaml' -n --files | sort -u > ../reports/detected-manifests.txt
  # language heuristics: presence of file extensions
  rg --hidden --files -g '!node_modules/*' '\.(py|js|ts|go|rs|java|cs|tsx|jsx)$' | sed -E 's/.*\.([a-z0-9]+)$/\1/' | sort | uniq -c > ../reports/language-counts.txt
- Outputs: reports/detected-manifests.txt, reports/language-counts.txt
- Notes: ripgrep (rg) used; fallback to find/grep if not available.

Task 5 — Extract key manifest excerpts
- Purpose: read top-level manifest files without executing
- Actions & commands (examples):
  # copy package.json if present
  for f in $(cat ../reports/detected-manifests.txt); do mkdir -p ../reports/manifest-excerpts/$(dirname "$f"); head -n 300 "$f" > ../reports/manifest-excerpts/$(basename "$f").excerpt || true; done
  # for package.json also capture 'scripts' and 'dependencies' fields using jq if available
  if command -v jq >/dev/null 2>&1; then
    if [ -f package.json ]; then jq '{name,version,scripts,dependencies,devDependencies}' package.json > ../reports/manifest-excerpts/package.json.summary.json || true; fi
  fi
- Output: manifest-excerpts/*

Task 6 — CI, workflows, Dockerfiles, and infra
- Purpose: identify CI and infra configuration
- Commands:
  rg -n --hidden --files '.github/workflows|.gitlab-ci.yml|circleci|Dockerfile|docker-compose.yml|azure-pipelines.yml' -g '!*node_modules/*' > ../reports/detected-ci-infra.txt || true
  cp -r .github ../reports/manifest-excerpts/ 2>/dev/null || true
- Output: reports/detected-ci-infra.txt and copies as available

Task 7 — Test frameworks & test files
- Purpose: detect tests and frameworks
- Commands:
  rg --hidden --files -g '!node_modules/*' 'test_.*\.py$|.*_test\.py$|.*\.spec\.js$|.*\.test\.js$|__tests__|jest.config|pytest.ini|tox.ini' > ../reports/detected-tests.txt || true
- Output: reports/detected-tests.txt

Task 8 — Workspace / monorepo detection
- Purpose: find multi-package layouts
- Commands:
  for f in package.json pnpm-workspace.yaml lerna.json workspace.json; do [ -f "$f" ] && echo "$f" >> ../reports/monorepo-indicators.txt; done
  # Also check package.json.workspaces
  if [ -f package.json ] && command -v jq >/dev/null 2>&1; then jq '.workspaces' package.json > ../reports/manifest-excerpts/package.workspaces.json 2>/dev/null || true; fi
- Output: reports/monorepo-indicators.txt

Task 9 — Produce JSON inventory
- Purpose: aggregate artifacts into a single machine-readable JSON file
- Implementation notes: the implementation agent will assemble the inventory by reading the outputs above and producing a JSON object with these fields:
  {
    "repo": "dreef3/multi-agent-harness-test-repo",
    "commit": "<HEAD SHA>",
    "size": "<du output>",
    "top_level_dirs": [ ... ],
    "detected_languages": { "js": 123, "py": 10 },
    "manifests": [ { "path": "package.json", "summary": { name,version } }, ... ],
    "ci_files": [ ... ],
    "test_files": [ ... ],
    "monorepo": true|false,
    "notes": [ ... ]
  }
- Output: reports/multi-agent-harness-test-repo-structure.json

Task 10 — Write human-readable report (Markdown)
- Purpose: synthesize findings and recommendations
- Sections to include:
  - Summary (1–2 paragraphs): primary languages, package manager, CI, tests present, monorepo status
  - Quick facts: repo size, top-level dirs, README presence, license
  - File tree (top 2-3 levels)
  - Key manifests excerpts (scripts/deps)
  - Notable findings & recommended next steps (missing CI, no README, large vendor dirs, private keys)
- Example file path: reports/multi-agent-harness-test-repo-structure.md

Task 11 — Review & handoff
- Purpose: a human reviewer checks the outputs and posts any follow-ups
- Checklist:
  - Confirm that no code was executed
  - Confirm that manifest excerpts are not excessively large (trim to 300 lines)
  - Validate JSON schema

Execution Notes and Commands Summary
- Required tools on implementation agent: git, rg (recommended), tree (optional), jq (optional), du, head, sed
- All commands operate inside the cloned repo directory and write results to ../reports
- Avoid running npm, pip, cargo, mvn, or other build/install commands

Deliverables and Acceptance Criteria
- reports/multi-agent-harness-test-repo-structure.md: contains a clear summary and actionable findings
- reports/multi-agent-harness-test-repo-structure.json: valid JSON with the inventory
- reports/file-list.txt and manifest-excerpts/: support reproducibility and audits

Estimated time to complete (implementation agent)
- Fully automated run: 5–15 minutes depending on repo size
- Human review & polish: 15–30 minutes

Contact / questions
- If the implementation agent cannot access the repository due to permissions, report the failure and include git clone stderr in reports/clone-error.txt
