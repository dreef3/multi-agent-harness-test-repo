Implementation Plan: Step-by-step tasks to analyze repository structure

Summary
- This plan provides task-by-task instructions that an executor (CI job, developer, or sub-agent) can run without modifying the repo. The plan produces two outputs in the repository: analysis/structure-report.md (human-readable analysis) and analysis/raw-outputs.tar.gz (raw command outputs and probed files).

Global assumptions
- The executor will run commands from the repository root.
- Tools assumed available: bash, find, sed, awk, jq, tar, gzip. Optional but useful: tree, rg (ripgrep), cloc, python3 with pip.
- If a required helper (e.g., toml library) is missing, fallback text parsing steps are provided.

Tasks

Task 0 — Prepare working paths (local, no changes)
- Create analysis directory for outputs.
- Commands:
  mkdir -p analysis/raw-outputs

Task 1 — Top-level inventory
- Goals: list files and directories at repo root and sizes.
- Commands:
  ls -la > analysis/raw-outputs/ls-root.txt
  find . -maxdepth 2 -mindepth 1 -printf "%p\n" | sort > analysis/raw-outputs/tree-level-2.txt
  # If tree available:
  tree -L 3 -a > analysis/raw-outputs/tree-l3.txt || true
- Deliverables: analysis/raw-outputs/ls-root.txt, tree-level-2.txt, tree-l3.txt (optional)

Task 2 — Language detection and file counts
- Goals: detect primary languages by extension and count files/lines per extension.
- Commands (fast, portable):
  # File extension counts
  find . -type f -not -path './.git/*' -printf '%f\t%p\n' | awk -F. 'NF>1{ext=$NF;count[ext]++} END{for (e in count) print count[e], e}' | sort -rn > analysis/raw-outputs/extension-counts.txt
  # If cloc available for language breakdown:
  cloc --hide-rate --report-file=analysis/raw-outputs/cloc-report.txt . || true
- Deliverables: extension-counts.txt, cloc-report.txt (optional)

Task 3 — Detect common manifest and config files
- Goals: find package.json, pyproject.toml, requirements.txt, Pipfile, setup.py, go.mod, Cargo.toml, pom.xml, composer.json, Gemfile, Dockerfile, tsconfig.json, yarn.lock, package-lock.json, pnpm-lock.yaml
- Command:
  rg -n --hidden --glob '!node_modules' '(^|/)(package.json|pyproject.toml|requirements.txt|Pipfile|setup.py|go.mod|Cargo.toml|pom.xml|composer.json|Gemfile|Dockerfile|tsconfig.json|package-lock.json|yarn.lock|pnpm-lock.yaml)' || true > analysis/raw-outputs/manifest-locations.txt
- After discovery, cat each file into analysis/raw-outputs/ (see Task 4)

Task 4 — Extract metadata from manifests (examples)
- For each manifest found, copy raw content into analysis/raw-outputs and extract key fields.
- Examples (package.json):
  if [ -f package.json ]; then jq '{name: .name, version: .version, scripts: .scripts, dependencies: .dependencies, devDependencies: .devDependencies}' package.json > analysis/raw-outputs/package-json-summary.json || cp package.json analysis/raw-outputs/package.json; fi
- Examples (pyproject.toml):
  if [ -f pyproject.toml ]; then cp pyproject.toml analysis/raw-outputs/; python3 - <<'PY'
import tomli, json, sys
with open('pyproject.toml','rb') as f:
    data=tomli.load(f)
print(json.dumps({'tool': data.get('tool', {}), 'project': data.get('project', {})}, indent=2))
PY
  > analysis/raw-outputs/pyproject-summary.json || true
- Examples (requirements.txt):
  if [ -f requirements.txt ]; then cp requirements.txt analysis/raw-outputs/; awk 'NF && $1!~/^#/{print $1}' requirements.txt > analysis/raw-outputs/requirements-list.txt; fi
- Note: tomli python module may need to be installed. If unavailable, recommend fallback of copying raw files for later parsing.

Task 5 — CI and workflow detection
- Goals: identify CI configuration (.github/workflows, .gitlab-ci.yml, .circleci)
- Commands:
  find . -type f -path './.github/workflows/*' -print > analysis/raw-outputs/github-workflows.txt || true
  [ -f .gitlab-ci.yml ] && cp .gitlab-ci.yml analysis/raw-outputs/ || true
  [ -d .circleci ] && find .circleci -type f > analysis/raw-outputs/circleci-files.txt || true

Task 6 — Tests and test runners
- Goals: detect tests directories, e2e test frameworks, and test-related scripts in manifests.
- Commands:
  rg -n --hidden --glob '!node_modules' '(^|/)(tests|test|__tests__|spec|e2e|integration)(/|$)' || true > analysis/raw-outputs/test-paths.txt
  # Check package.json scripts for test commands
  if [ -f package.json ]; then jq '.scripts?' package.json > analysis/raw-outputs/package-scripts.json || true; fi
  # Python typical patterns
  rg -n --hidden -S 'pytest|unittest|nose|tox' || true > analysis/raw-outputs/test-framework-hints.txt

Task 7 — Large files, binary artifacts, and ignored items
- Goals: find files over threshold (e.g., 5MB) and list large directories like node_modules or build outputs.
- Commands:
  find . -type f -not -path './.git/*' -printf '%s %p\n' | sort -nr | awk 'NR<=50{print}' > analysis/raw-outputs/largest-files.txt
  du -sh * 2>/dev/null | sort -hr > analysis/raw-outputs/dir-sizes.txt

Task 8 — Quick heuristics for issues
- Goals: find TODO/FIXME, missing README or LICENSE
- Commands:
  rg -n --hidden --glob '!node_modules' 'TODO|FIXME|HACK' || true > analysis/raw-outputs/todo-fixme.txt
  [ -f README.md ] || echo 'MISSING' > analysis/raw-outputs/readme-missing.txt
  [ -f LICENSE ] || echo 'MISSING' > analysis/raw-outputs/license-missing.txt

Task 9 — Assemble the human-readable report
- Goals: create analysis/structure-report.md summarizing findings: top-level summary, tree snippet, languages, important manifests with extracted fields, detected CI and tests, large files, and recommendations.
- Template (paths and placeholders). Create file analysis/structure-report.md with sections:
  - Repo identification and date
  - Top-level summary (one-paragraph): primary language(s), presence of CI, tests
  - Representative tree (small excerpt from tree-l3.txt)
  - Manifests found and short JSON/TOML-extracted summaries (inline)
  - CI files detected and their paths
  - Tests and how to run them (if detectible)
  - Notable large files and directories
  - Issues and recommended next steps

Example skeleton (shell heredoc):
cat > analysis/structure-report.md <<'MD'
# Repository Structure Report

Repository: <repo-url>
Date: $(date -u)

## Top-level summary
- Primary languages: (from cloc or extension counts)
- CI: (found or not)
- Tests: (found or not)

## Representative tree
(see analysis/raw-outputs/tree-l3.txt)

## Manifests
- package.json: (name/version) — details in analysis/raw-outputs/

## CI
- files: (list)

## Tests
- paths and hints

## Large files
- analysis/raw-outputs/largest-files.txt

## Issues & Recommendations
- (list)
MD

Task 10 — Package raw outputs
- Commands:
  tar -czf analysis/raw-outputs.tar.gz -C analysis raw-outputs
  # Leave both analysis/structure-report.md and analysis/raw-outputs.tar.gz in repo

Task 11 — Sanity check and deliver
- Inspect the two generated files and commit them in a branch/open PR following repository contribution guidelines.
- Expected outputs: analysis/structure-report.md, analysis/raw-outputs.tar.gz (or .tar.gz inside analysis)

Extras (optional but useful)
- Generate a dependency tree for Node (npm ls --json), Python (pipdeptree), Go (go list -m all), Rust (cargo metadata --format-version 1) when manifest detection finds those ecosystems. Commands should be guarded with file existence checks.
- Generate a simple language breakdown script (Python) to count lines by extension and produce percentages.

Time estimate
- Inventory and manifest extraction: ~2–5 minutes for small repos, up to 10–20 minutes for large ones.
- Report assembly: ~5 minutes.

Deliverables
- analysis/structure-report.md (human readable)
- analysis/raw-outputs.tar.gz (raw outputs)
- analysis/raw-outputs/ (raw command outputs retained in the working branch)

Notes and risks
- If the repo contains very large files (gigabytes) the plan will list them but should avoid copying them into the archive unless necessary.
- Some parsing steps rely on optional tools (cloc, jq, tomli). If not available, raw files will be copied for offline parsing.
