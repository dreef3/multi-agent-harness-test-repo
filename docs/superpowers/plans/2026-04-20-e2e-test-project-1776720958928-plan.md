# Implementation Plan: Codebase Structure Analysis

Objective
- Run a reproducible set of static inspection steps against the repository root and produce docs/codebase-structure.md summarizing the codebase structure.

High-level tasks (ordered)
1. Repository inventory: list top-level files and directories.
2. Language & package manager detection.
3. Manifest & dependency inspection.
4. Source tree & module mapping.
5. Tests & CI inspection.
6. Tooling, linters, and formatters.
7. Container & deployment artifacts.
8. Output report assembly.
9. Deliverables and PR notes.

Detailed task list with exact commands and file targets

Task 1 — Repository inventory
- Purpose: get a top-level snapshot of repo contents.
- Commands (run in repo root):
  - ls -la
  - find . -maxdepth 2 -type d -print | sed 's|^\.|/|'
- Files to capture: top-level listing.
- Output file: artifacts/inventory/top_level.txt

Task 2 — Language & package manager detection
- Purpose: detect primary languages and package managers.
- Commands:
  - Detect common manifests (stop when found):
    - ls package.json 2>/dev/null && echo "package.json found"
    - ls pyproject.toml 2>/dev/null && echo "pyproject.toml found"
    - ls setup.py 2>/dev/null && echo "setup.py found"
    - ls go.mod 2>/dev/null && echo "go.mod found"
    - ls Cargo.toml 2>/dev/null && echo "Cargo.toml found"
    - ls Gemfile 2>/dev/null && echo "Gemfile found"
  - Optionally run 'cloc --csv --out=artifacts/cloc.csv .' or 'git-linguist' if available.
- Output file: artifacts/language-detection.txt

Task 3 — Manifest & dependency inspection
- Purpose: read package manifests and extract key fields (name, version, scripts, dependencies, devDependencies).
- Files to read if present: package.json, package-lock.json, yarn.lock, pnpm-lock.yaml, pyproject.toml, requirements.txt, go.mod, Cargo.toml
- Use repository file-reading (cat/read) to capture sections.
- Example extract commands:
  - jq '{name,version,scripts,dependencies,devDependencies}' package.json > artifacts/manifests/package-summary.json
  - python -c "import toml,sys; print(toml.load(open('pyproject.toml'))['tool'])" 2>/dev/null > artifacts/manifests/pyproject-tool.txt
- Output files: artifacts/manifests/*

Task 4 — Source tree & module mapping
- Purpose: identify primary source directories and map packages/modules to paths.
- Commands:
  - Common source paths to check: src/, lib/, app/, packages/, modules/, cmd/, internal/
  - find src lib packages -maxdepth 3 -type d -print > artifacts/source_dirs.txt
  - For monorepos: check pnpm-workspace.yaml, lerna.json, turbo.json, workspace.json
- Output file: artifacts/source-map.md (list of source dirs and short note per dir)

Task 5 — Tests & CI inspection
- Purpose: confirm presence and layout of tests and CI workflows.
- Commands and paths to read:
  - find . -name "test*" -o -name "*spec*" -type d
  - Read .github/workflows/*.yml and any CI config in .gitlab-ci.yml, .circleci/config.yml
  - Check for jest/mocha/pytest/Go test usage in package.json or manifests
- Output files: artifacts/tests/listing.txt and artifacts/ci/workflows.md (short summaries of each workflow file)

Task 6 — Tooling, linters, and formatters
- Purpose: find developer tooling configurations.
- Files to check: .eslintrc*, .prettierrc*, .editorconfig, .rubocop.yml, .flake8, tox.ini, Makefile, Dockerfile
- Commands: for each pattern, run 'ls <pattern> 2>/dev/null' and capture files.
- Output file: artifacts/tooling/tooling.md

Task 7 — Container & deployment artifacts
- Purpose: locate Dockerfiles, k8s manifests, helm charts, terraform folders.
- Files to check: Dockerfile*, k8s/, charts/, terraform/
- Output: artifacts/deploy/deploy.md

Task 8 — Output report assembly
- Purpose: collate all artifacts and produce final summary docs/codebase-structure.md
- Structure for docs/codebase-structure.md:
  - Executive summary (1 paragraph)
  - Top-level tree (excerpt)
  - Languages & package managers
  - Key manifests and notable dependencies
  - Source layout and module mapping
  - Tests & CI (what exists, recommended next steps)
  - Tooling & developer experience notes
  - Container/deploy artifacts
  - Risks & technical debt hotspots
  - Recommended next steps (onboarding, running tests, coverage, CI fixes)
- Example snippet to include in doc:
  - "Detected languages: JavaScript (package.json), Python (pyproject.toml), Go (go.mod) — [list exact files found]"
- Output files:
  - docs/codebase-structure.md (main report)
  - artifacts/ (all intermediate captures)

Task 9 — Deliverables and PR notes
- Add a README note for future runs: scripts/analyze-codebase.sh (executable) that re-runs the commands and regenerates artifacts.
- Create PR body with a summary and attach docs/codebase-structure.md

Permissions, assumptions, and constraints
- This plan assumes read access to the repository files. It does not require network access or running unit tests.
- Do not modify source files; only create new docs and artifact captures under an artifacts/ folder and docs/.

Estimated effort and timeline
- Inventory & detection: 15–30 minutes
- Manifest inspection: 15–30 minutes
- Source & tests mapping: 30–45 minutes
- Report assembly: 30–60 minutes
- Total: ~1.5–3 hours

Implementation notes for the executor
- Use repository root as working directory.
- Use provided commands and capture outputs to artifacts/.
- When reading JSON/TOML files, extract only top-level summaries to keep artifacts concise.
- If monorepo patterns are detected, add a section listing packages and their relative paths.

Next steps after merge
- Optionally dispatch tasks to run tests per-language and generate coverage reports.
