Implementation Plan: Step-by-step tasks to analyze the repository structure

Overview
- This plan breaks the repository analysis into small, parallelizable tasks. Each task is self-contained and only reads files. Tasks should be run in the repository root (default branch 'main'). Each task's expected outputs and example commands are provided. Outputs should be collected into a folder report/ and include both human-readable (Markdown) and machine-readable (JSON) summaries.

Global output locations (in repo working copy)
- report/summary.md - human-readable summary
- report/details.md - per-file/directory details and findings
- report/report.json - machine-readable summary with schema below

report.json schema (recommended)
{
  "repoId": "ebf38f5c-42d9-4f9f-b549-664880d67743",
  "primaryLanguages": [ {"language": "JavaScript", "percent": 72.3, "loc": 12345 } ],
  "topLevelDirs": ["src","test",".github"],
  "packageManagers": ["npm","pip"],
  "ciFiles": [".github/workflows/ci.yml"],
  "testFrameworks": ["jest","pytest"],
  "hasDockerfile": true,
  "readmeExists": true,
  "largeFiles": [{"path":"path/to/big.bin","sizeMb":45}],
  "notes": "free-text"
}

Tasks

Task 1 — Top-level inventory
- Purpose: list top-level files & dirs, show file counts, disk usage
- Commands:
  - ls -la
  - find . -maxdepth 2 -type d -print
  - du -sh * | sort -h
- Expected outputs: top-level listing, sizes. Save to report/top_level.txt
- Output example file: report/top_level.txt

Task 2 — Detect languages & package managers
- Purpose: detect languages and package managers by presence of signature files and run cloc for LOC breakdown.
- Files to check (in order): package.json, yarn.lock, pnpm-lock.yaml, pyproject.toml, requirements.txt, setup.py, Pipfile, go.mod, Cargo.toml, Gemfile, composer.json, build.gradle, pom.xml
- Commands:
  - for f in package.json pyproject.toml requirements.txt setup.py go.mod Cargo.toml Gemfile build.gradle pom.xml; do [ -f $f ] && echo FOUND:$f; done
  - cloc --json --out=report/cloc.json . || (echo "cloc not installed")
- Expected outputs: report/cloc.json, report/package_detection.txt

Task 3 — CI and automation files
- Purpose: find CI workflows, GitHub actions, GitLab CI, CircleCI, Azure pipelines
- Paths to check:
  - .github/workflows/*.yml
  - .gitlab-ci.yml
  - .circleci/config.yml
  - azure-pipelines.yml
  - Jenkinsfile
- Commands:
  - ls -la .github/workflows || true
  - [ -f .gitlab-ci.yml ] && echo FOUND:.gitlab-ci.yml
  - rg "name:|on:|jobs:" .github/workflows -n || true
- Expected outputs: report/ci_files.txt and brief descriptions of each workflow (jobs, triggers)

Task 4 — Tests and test frameworks
- Purpose: locate tests, frameworks, and entry points (package.json test script, pytest.ini, jest.config.js)
- Paths to check:
  - test/, tests/, spec/, __tests__/
  - package.json (scripts.test)
  - pytest.ini, tox.ini, jest.config.js, mocha.opts
- Commands:
  - rg "pytest|unittest|mocha|jest|ava|nose" -n || true
  - jq .scripts test package.json || true
  - find tests -type f | sed -n '1,200p' || true
- Expected outputs: report/tests.txt listing test files and detected frameworks

Task 5 — Source layout and entry points
- Purpose: identify source directories (src/, lib/, cmd/, pkg/), main binaries, and library structure
- Commands:
  - find src lib -maxdepth 3 -type f -name "*.*" | sed -n '1,200p' || true
  - rg "package main|def main\(|if __name__ == '__main__'" -n || true
- Expected outputs: report/source_layout.md

Task 6 — Dependency & lockfile inspection
- Purpose: list direct dependencies declared and lockfile presence
- Commands/examples:
  - [ -f package.json ] && jq '.dependencies + .devDependencies' package.json > report/npm_deps.json || true
  - [ -f pyproject.toml ] && sed -n '1,200p' pyproject.toml > report/pyproject_head.txt || true
- Note: only read files; do not run install

Task 7 — Static attention checks (README, LICENSE, CONTRIBUTING)
- Purpose: verify presence and basic content of documentation files
- Files: README.md, LICENSE, CONTRIBUTING.md, CODE_OF_CONDUCT.md
- Commands:
  - [ -f README.md ] && sed -n '1,200p' README.md > report/README_head.md || true
- Expected outputs: report/docs_presence.txt

Task 8 — Large files and vendor directories
- Purpose: find files >5MB and presence of vendor/ or node_modules/
- Commands:
  - find . -type f -size +5M -printf '%s %p\n' | sort -nr > report/large_files.txt || true
  - [ -d node_modules ] && du -sh node_modules > report/node_modules_size.txt || true
- Expected outputs: report/large_files.txt

Task 9 — Licensing and contributors
- Purpose: extract LICENSE text and list authors from git metadata (if available)
- Commands:
  - [ -f LICENSE ] && sed -n '1,200p' LICENSE > report/LICENSE_head.txt || true
  - git shortlog -sne | sed -n '1,200p' > report/contributors.txt || true
- Note: git commands require git history; if not available, skip contributors step

Task 10 — Compose the final report
- Purpose: aggregate all partial outputs into report/summary.md and report/report.json
- Steps:
  - Use the cloc.json and detection outputs to fill primaryLanguages and topLevelDirs
  - Summarize CI files, test frameworks, README presence, Dockerfile, and large file warnings
  - Write a short (5-10 bullet) top findings section
- Expected outputs: report/summary.md and report/report.json

Execution notes
- Agents executing tasks should not modify repository files. Output files may be written under report/ to collect artifacts.
- If a command/tool is not available (cloc, jq, rg), the agent should note that in the task output and provide an alternative (use wc and file name inspection).
- Time estimates: each task 1–10 minutes depending on repo size. Total < 60 minutes for a normal-sized repo (<100k LOC).

Acceptance Criteria
- report/summary.md exists and contains a concise top-level summary with primary languages, top-level dirs, CI status, tests summary, and 3-5 recommended next steps.
- report/report.json exists and conforms to the schema above.
