Overview
- This plan breaks the analysis into discrete, self-contained tasks. Each task contains exact commands to run in the repo root, expected outputs, output file paths to create, and simple parsing instructions. Do not modify source code; write outputs under analysis/.

Task 1 — Repo file tree (shallow)
- Purpose: capture top-level layout and a shallow tree to understand project shape.
- Commands to run (in repo root):
  - find . -maxdepth 3 -path './.git' -prune -o -type f -print | sed 's|^./||' | sort > analysis/tree.txt
  - head -n 200 analysis/tree.txt > analysis/tree_top200.txt
- Expected output: analysis/tree.txt (full file list limited to depth 3), analysis/tree_top200.txt
- Notes: If repo is extremely large, increase -maxdepth or prune large folders (node_modules, .venv) using -path patterns.

Task 2 — Detect language(s) and common manifest files
- Purpose: identify languages and package manifests.
- Commands:
  - Define search list of manifest filenames:
    manifests=("package.json" "yarn.lock" "pnpm-lock.yaml" "pnpm-lock.yml" "pyproject.toml" "requirements.txt" "Pipfile" "Pipfile.lock" "setup.py" "setup.cfg" "requirements-dev.txt" "poetry.lock" "poetry.toml" "go.mod" "Cargo.toml" "pom.xml" "build.gradle" "build.gradle.kts" "Gemfile" "composer.json" "Makefile" "Dockerfile" "docker-compose.yml" "composer.lock")
  - for m in "${manifests[@]}"; do find . -maxdepth 4 -type f -name "$m" -print; done | sed 's|^./||' > analysis/manifest_locations.txt
  - Run a lightweight language scan fallback (if cloc exists, optional):
    - if command -v cloc >/dev/null 2>&1; then cloc --json . > analysis/cloc.json || true; else echo "cloc not available" > analysis/cloc_missing.txt; fi
  - Also produce a filename-based heuristic breakdown:
    - awk -F. '{print $NF}' analysis/tree.txt | sort | uniq -c | sort -rn > analysis/extension_counts.txt
  - Combine results into analysis/lang-manifest.json with fields: manifests (list), extension_counts (top 50), cloc (if present)
- Expected output: analysis/manifest_locations.txt, analysis/extension_counts.txt, optional analysis/cloc.json, and analysis/lang-manifest.json

Task 3 — CI and workflow inspection
- Purpose: find CI config files and list workflow names.
- Commands:
  - find . -type f \( -path './.github/workflows/*' -o -name '.circleci' -o -name 'circleci' -o -name 'Jenkinsfile' -o -name 'appveyor.yml' -o -name 'azure-pipelines.yml' -o -name 'gitlab-ci.yml' \) -print | sed 's|^./||' > analysis/ci_candidates.txt
  - If .github/workflows exists, list files and extract 'name:' fields from YAML (grep -E "^name:\s" -R .github/workflows || true) >> analysis/ci_workflow_names.txt
- Expected output: analysis/ci_candidates.txt, analysis/ci_workflow_names.txt

Task 4 — Tests and test frameworks
- Purpose: enumerate test directories and common test file patterns.
- Commands:
  - find . -type d \( -name 'test' -o -name 'tests' -o -name '__tests__' -o -name 'spec' -o -name 'e2e' \) -print | sed 's|^./||' > analysis/test_dirs.txt
  - find . -type f \( -name '*test*.js' -o -name '*spec*.js' -o -name '*_test.py' -o -name 'test_*.py' -o -name '*_spec.rb' -o -name '*_test.go' \) -print | sed 's|^./||' > analysis/test_files_sample.txt
- Expected output: analysis/test_dirs.txt, analysis/test_files_sample.txt

Task 5 — Dependency extraction (manifest excerpts)
- Purpose: extract dependency sections from manifests without resolving them.
- Commands/examples (create analysis/dependencies.json):
  - For package.json: if [ -f package.json ]; then jq '{path: "package.json", dependencies: .dependencies, devDependencies: .devDependencies}' package.json > analysis/dependencies_package.json || true; fi
  - For pyproject.toml: if [ -f pyproject.toml ]; then sed -n '/\[tool.poetry.dependencies\]/,/^\[/p' pyproject.toml > analysis/dependencies_pyproject_poetry.txt || true; fi
  - For requirements.txt: if [ -f requirements.txt ]; then awk 'NF && $1 !~ /^#/' requirements.txt > analysis/requirements_list.txt || true; fi
  - For go.mod: if [ -f go.mod ]; then sed -n '1,200p' go.mod > analysis/go_mod_excerpt.txt || true; fi
  - Combine all manifest-specific outputs into a single JSON (dependencies.json) with keys per-manifest. A small helper (jq) or manual assembly is fine.
- Expected output: analysis/dependencies_package.json, analysis/requirements_list.txt, analysis/dependencies.json
- Notes: Avoid running package managers; only parse text.

Task 6 — README, license, and contributor information
- Purpose: capture README top lines and license presence.
- Commands:

  - for l in LICENSE LICENSE.md LICENSE.txt; do if [ -f "$l" ]; then cp "$l" analysis/LICENSE_COPY || true; break; fi; done
- Expected output: analysis/README_head.txt, analysis/LICENSE_COPY
