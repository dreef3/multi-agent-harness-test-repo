Implementation Plan: Codebase Structure Analysis for E2E Test Repo

Overview
- This plan breaks the analysis work into self-contained tasks. Each task is written so a sub-agent or engineer can run it independently on a local clone of the repo (or in CI). Commands assume a POSIX shell environment and standard tools (git, jq, rg/ripgrep, tree, python, node/npm, cargo, go). The commands are explicit and reference exact files/paths to inspect.

Repository
- ID: 04a1c36c-59d9-4748-8a45-ab40b8ff7d29
- URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Tasks (self-contained)

Task 1 — Clone repository and record top-level listing
- Goal: get a snapshot of the repository root.
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
  - ls -la > /tmp/repo_root_listing.txt
  - tree -L 2 -a -I 'node_modules|.venv|venv' > /tmp/tree_L2.txt
- Output files:
  - /tmp/repo_root_listing.txt
  - /tmp/tree_L2.txt
- Estimated time: 1–2 minutes

Task 2 — Detect primary languages and manifests
- Goal: detect ecosystem(s) and collect manifest files.
- Commands:
  - rg --hidden --files --glob '!**/.git/**' --glob '!node_modules/**' "package.json|pyproject.toml|requirements.txt|setup.py|go.mod|Cargo.toml|pom.xml|build.gradle|Makefile|Dockerfile" || true
  - For each found manifest, copy to artifacts/manifests/ (e.g., cp package.json artifacts/manifests/)
- Expected files to look for (exact names): README.md, LICENSE, package.json, pyproject.toml, requirements.txt, setup.py, setup.cfg, pyproject.toml, go.mod, Cargo.toml, pom.xml, build.gradle, Dockerfile, Makefile, .github/workflows/*.yml, tests/, src/ or lib/.
- Output files:
  - artifacts/manifests/*
- Estimated time: 2–5 minutes

Task 3 — Parse Node.js manifest (if present)
- Trigger: package.json exists
- Commands:
  - jq . package.json > artifacts/manifests/package.json.json
  - jq -r '.name, .version, .main, .scripts' package.json > artifacts/manifests/package-summary.txt
  - npm --version || true
  - If Node.js present, run: npm ci --prefer-offline --no-audit --no-fund (in CI use a separate cache)
  - List top dependencies: jq -r '.dependencies // {} | to_entries[] | "\(.key) \(.value)"' package.json | sort > artifacts/manifests/dependencies-node.txt
- Notes: do not run npm install in long-running environments unless necessary. Use --prefer-offline and --no-audit.

Task 4 — Parse Python manifests (if present)
- Trigger: pyproject.toml / requirements.txt / setup.py exists
- Commands:
  - python --version
  - If pyproject.toml exists: cat pyproject.toml > artifacts/manifests/pyproject.toml
  - If requirements.txt: cp requirements.txt artifacts/manifests/
  - For top-level dependencies (requirements.txt): sort -u requirements.txt > artifacts/manifests/dependencies-py.txt
  - If tox.ini or pytest.ini present, copy them as well.
- Notes: do not create virtualenv side effects in the repo root; create venv under /tmp when running tests.

Task 5 — Parse Go / Rust / Java manifests (if present)
- For go.mod: cat go.mod > artifacts/manifests/go.mod; go list -m all > artifacts/manifests/go-deps.txt
- For Cargo.toml: cargo metadata --format-version 1 --no-deps > artifacts/manifests/cargo-metadata.json
- For Java: cat pom.xml or build.gradle to artifacts/manifests/

Task 6 — Discover source layout
- Goal: identify top-level source and test directories and main entry points.
- Commands:
  - rg --hidden --files --glob '!**/.git/**' "package.json|pyproject.toml|setup.py|go.mod|Cargo.toml" || true
  - If package.json: jq -r '.main // ""' package.json; look for src/, lib/, index.js
  - If Python: rg -n "if __name__ == ['\"]__main__['\"]" || true
  - List top source directories: ls -la src lib app pkg cmd > artifacts/tree_src.txt || true
- Output: artifacts/tree_src.txt

Task 7 — Tests: discovery and run (safe)
- Goal: find tests and run them in an isolated environment.
- Discovery commands:
  - rg --hidden --glob '!**/node_modules/**' "\bdescribe\(|\bit\(|^def test_|^class Test|pytest" -S --line-number || true
  - tree -a -I 'node_modules|.venv|venv' | rg 'test|spec' || true
- Running tests (only if artifacts and a safe environment available):
  - Node: npm test --if-present (or npx jest --version)
  - Python: python -m venv /tmp/venv && . /tmp/venv/bin/activate && pip install -r requirements.txt --no-deps --no-build-isolation || true && pytest -q --maxfail=1 || true
- Capture outputs to artifacts/tests/test-run.txt
- Note: tests may require network or credentials — run with --maxfail or skip slow suites.

Task 8 — CI inspection
- Goal: collect CI workflows and summarize what they run.
- Commands:
  - tree -a .github/workflows -P '*.yml' -L 1 || true
  - For each workflow YAML, print header lines and the jobs: yq e '.name, .jobs' .github/workflows/*.yml || true (or grep -n "steps:" -n)
  - Copy workflows to artifacts/ci/
- Output: artifacts/ci/ci-workflows-summary.txt

Task 9 — Linting, static analysis, formatting configs
- Goal: find ESLint, Prettier, flake8, mypy, bandit, etc.
- Commands:
  - rg --hidden --files "\.eslintrc|\.prettierrc|mypy.ini|flake8|pyproject.toml|setup.cfg" || true
  - Copy found config files to artifacts/lint/

Task 10 — Produce final report
- Goal: consolidate findings into final-report.md with sections:
  - Summary (one paragraph)
  - Top-level layout (list directories and purpose)
  - Manifests and major dependencies (top 10)
  - Tests (location, how to run, run summary)
  - CI (workflows found and what they do)
  - Observations and suggested next steps / improvements
- Example skeleton for final-report.md (to be filled by implementer):
  - # Summary
  - # File & Directory Overview
  - # Dependency Manifests
  - # Tests
  - # CI
  - # Observations & Recommendations
- Commands to create: gather outputs from artifacts/* and format into final-report.md; copy summary to summary.txt
- Expected outputs: final-report.md, artifacts/*

Reporting format
- All artifacts go into an artifacts/ directory at the repo root. final-report.md should be readable in plain markdown.
- Include small machine-readable JSON: artifacts/summary.json with keys: languages[], top_directories[], has_ci (bool), tests_found (bool), primary_manifest (string).

Risks and mitigations
- Private submodules or Git LFS content: the clone step will surface this; record blockers in artifacts/clone-blockers.txt.
- Long-running installs or tests: limit installs with flags and use --maxfail and short timeouts. Report any tests skipped due to environment.

Time estimates
- Full run (including running tests): 15–45 minutes depending on test suites and dependency installation.
- Fast inspection (manifests + tree + CI): 2–5 minutes.
