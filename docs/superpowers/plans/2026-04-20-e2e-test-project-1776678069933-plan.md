Implementation Plan: Task-by-task plan to analyze the codebase structure for E2E Test Repo (id: 8054c555-f5da-4b82-bc63-ea61f8776c2f)

Overview
- The implementation is broken down into 5 independent tasks. Each task is self-contained and can be executed by a sub-agent. After the plan PR is merged, dispatch tasks to perform the actual repo analysis.

Tasks

Task 1: Clone & Top-level Inventory
- Purpose: Clone the repo and produce a top-level listing (files, dirs) and a tree (depth 2). Identify presence/absence of README, LICENSE, .gitignore, and other standard files.
- Exact steps/commands:
  1. git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  2. cd repo
  3. ls -la
  4. tree -a -L 2 || find . -maxdepth 2 -print
  5. echo "--- README.md ---" && sed -n '1,200p' README.md || true
- Files to collect: output of ls, tree, and first 200 lines of README.md
- Deliverable: top-level inventory section for the final report containing a list of top-level dirs and files and short README summary

Task 2: Language & LOC Breakdown
- Purpose: Determine primary languages and code size
- Exact steps/commands:
  1. Install/run cloc if not present: sudo apt-get install -y cloc || curl -LO https://github.com/AlDanial/cloc/releases/latest/download/cloc && chmod +x cloc
  2. cloc --vcs=git --by-file --exclude-dir=node_modules,venv,.venv --quiet . > cloc-by-file.txt
  3. cloc --vcs=git --quiet . > cloc-summary.txt
  4. Provide top 10 largest files by size: find . -type f -printf "%s %p\n" | sort -nr | head -n 10
- Files to collect: cloc-summary.txt, cloc-by-file.txt, list of top 10 largest files
- Deliverable: languages and approximate LOC in final report

Task 3: Dependency Manifests & Package Managers
- Purpose: Locate dependency manifests and list direct dependencies
- Files to search for (in order): package.json, yarn.lock, pnpm-lock.yaml, package-lock.json, pyproject.toml, requirements.txt, setup.py, Pipfile, poetry.lock, go.mod, Makefile, Cargo.toml, Gemfile, composer.json
- Exact steps/commands (examples):
  1. rg -n "package.json|pyproject.toml|go.mod|Cargo.toml|Gemfile|requirements.txt|Pipfile|Dockerfile|Makefile" || true
  2. For each manifest found run appropriate command to list dependencies:
     - Node: cat package.json; npm --prefix . ls --json --depth=0 || cat package.json
     - Python: cat pyproject.toml; grep -n "^\s*dependencies\|^\s*requires" pyproject.toml || pip-compile --dry-run
     - Go: cat go.mod; go list -m all
     - Rust: cat Cargo.toml; cargo metadata --no-deps --format-version 1
  3. Record presence of lockfiles (package-lock.json, yarn.lock, pnpm-lock.yaml, poetry.lock)
- Deliverable: manifest inventory and dependency summaries in report, with explicit file path references and suggested commands to update/install

Task 4: CI, Workflows, and Automation
- Purpose: Identify CI pipeline files and describe test/build commands the CI runs
- Files/locations to inspect: .github/workflows/*, .gitlab-ci.yml, .circleci/config.yml, Jenkinsfile, Dockerfile, Makefile
- Exact steps/commands:
  1. rg -n "\.(github/workflows)|gitlab-ci.yml|.circleci|Jenkinsfile|Dockerfile|Makefile" || true
  2. For each .github/workflows/*.yml: sed -n '1,200p' .github/workflows/*.yml
  3. Summarize: which OS, matrix strategy, steps that run tests, build, lint, publish
- Deliverable: CI summary including where to run tests locally and any secrets/credentials required per CI files

Task 5: Tests, E2E, and Quality
- Purpose: Locate tests and e2e suites, list test command(s), and record if coverage config exists
- Files/locations: tests/, spec/, e2e/, src/__tests__, jest.config.js, pytest.ini, tox.ini, coverage.yml, sonar-project.properties
- Exact steps/commands:
  1. rg -n "\bdescribe\b|\bpytest\b|__tests__|@Test\b|mocha\b|jest\b" || true
  2. Check common test commands in package.json scripts or Makefile
  3. If present, run test commands in a sandboxed environment (this will be a separate execution task, optional)
- Deliverable: tests summary: location, command to run, estimated time, and whether coverage is enforced in CI

Final report assembly
- Combine outputs from Tasks 1-5 into a single markdown report with these sections:
  1) Executive summary (languages, LOC, primary package manager, CI status)
  2) Top-level file/dir inventory
  3) Languages & LOC (cloc summary)
  4) Dependency manifests & important dependencies
  5) CI/workflows summary and test commands
  6) Tests and coverage
  7) Actionable recommendations and next steps
  8) Appendix: raw outputs collected (ls, tree, cloc, manifest snippets, workflow snippets)

Reporting format and examples
- Provide example snippets to include, e.g.:
  - package.json scripts section: "test": "jest --runInBand"
  - .github/workflows snippet: name, runs-on, steps that checkout, setup-node, run: npm install, run: npm test

Notes and safety
- The analysis tasks are read-only (no commits or changes). Running tests is optional and will be done in controlled sub-agent environments only.
- If secrets or tokens are referenced in CI files, redact them in the report and note possible misconfiguration.
