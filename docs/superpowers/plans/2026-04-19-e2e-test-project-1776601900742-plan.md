Implementation plan: repository structure analysis for multi-agent-harness-test-repo

Overview
- Implement a sequence of small, self-contained tasks that clone and inspect the repository, extract structural and tooling information, and write a human-readable REPORT.md in the repository root. Each task includes exact file paths, shell commands, expected outputs, and estimated time.
- The tasks are suitable to be executed by a single agent in order; each produces artifacts that are committed in a single PR (handled by the harness).

Tasks

1) Clone the repository (workspace path)
- Goal: get a local copy to inspect
- Command (run from /workspace):
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git
- Expected result: repo checked out to /workspace/multi-agent-harness-test-repo
- Time: 30s–2min (depends on repo size)
- Notes: If private or inaccessible, stop and report error.

2) Create a reproducible top-level tree listing
- Goal: produce a concise listing of top-level files and directories
- Commands:
  - cd /workspace/multi-agent-harness-test-repo
  - find . -maxdepth 2 -mindepth 1 -type d -printf "%p\n" | sort
  - ls -la
- Output file: report/top_level_tree.txt
- Time: 10s

3) Detect languages and package managers
- Goal: detect primary languages and package manifests
- Commands (examples):
  - rg --hidden --files --no-ignore-vcs "package.json|pyproject.toml|poetry.lock|Pipfile|requirements.txt|go.mod|Cargo.toml|Gemfile|mix.exs|composer.json|build.gradle|pom.xml|Cargo.lock|yarn.lock" || true
  - find . -type f -name "*.py" -or -name "*.js" -or -name "*.ts" -or -name "*.go" -or -name "*.rs" -or -name "*.java" -or -name "*.swift" -or -name "*.cs" | sed -E 's|^./||' | sort | head -n 200
- Expected outputs: list of manifest files and sample source file list
- Output file: report/language_manifest_summary.txt
- Time: 30s

4) Inspect CI and workflows
- Goal: summarise GitHub Actions workflows and other CI
- Commands:
  - ls -la .github/workflows || true
  - for f in .github/workflows/*.y*ml; do echo "--- $f"; sed -n '1,200p' "$f"; echo; done
- Expected output: list of workflow files and brief summary of jobs (build, test, matrix)
- Output file: report/ci_workflows_summary.txt
- Time: 1–2min

5) Locate Dockerfiles and container config
- Goal: find Dockerfiles and Docker Compose
- Commands:
  - rg -n "^FROM " --hidden --glob '!node_modules' || true
  - ls -la Dockerfile* docker-compose* || true
- Output file: report/docker_summary.txt
- Time: 30s

6) Identify tests and test frameworks
- Goal: find test directories and test commands
- Commands:
  - rg --hidden --files "__tests__|tests/|Test.*\.java|*_test.go|*_spec.rb" || true
  - for f in package.json pyproject.toml; do if [ -f "$f" ]; then echo "---- $f"; sed -n '1,200p' $f; fi; done
- Note: extract npm test, pytest, go test, cargo test commands
- Output file: report/tests_summary.txt
- Time: 1–5min (depends on repo)

7) Summarize entry points and library structure
- Goal: find main entry points (e.g., src/index.js, cmd/, main.go) and top-level packages
- Commands:
  - rg --hidden --files "main\(|if __name__ == \"__main__\"|func main\(|module.exports|export default" || true
- Output: report/entrypoints_summary.txt
- Time: 30s

8) Run linters and test (optional, gated)
- Goal: run quick lint and test to check build health. These are optional and should be run with timeouts.
- Example commands per language (run only after confirming resource/time limits):
  - npm ci && npm test --if-present --silent (Node)
  - pip install -r requirements.txt && pytest -q (Python)
  - go test ./... (Go)
- Capture outputs: report/test_run_summary.txt and report/test_failures.log
- Time: variable (minutes)
- Safety: run inside ephemeral container or with timeouts.

9) Produce final REPORT.md and artifacts
- FILE: /workspace/multi-agent-harness-test-repo/REPORT.md (content built from other report/*.txt files)
- Sections to include:
  - Summary (one-paragraph)
  - Top-level tree
  - Languages and package managers
  - CI/workflows
  - Docker/containers
  - Tests and how to run them
  - Entry points and main components
  - Recommendations and next steps
- Example header for REPORT.md:
  - Title: Repository Structure Analysis for multi-agent-harness-test-repo
  - Repo URL and commit hash
- Time: 20–30min

10) Commit the report and open PR (handled by harness)
- Files to commit:
  - REPORT.md
  - report/*.txt (optional)
  - output/deps.png (optional)
- The harness will create a PR. Include summary in PR body.
- Time: 5–10min

- Full analysis including test runs and graphs: 30–60+ minutes depending on repo size

Notes and follow-ups
- If the repo is a monorepo, adapt tests discovery per package manager (yarn workspaces, pnpm, lerna, monorepo toolchain).
- If the repo contains submodules or LFS objects, ensure tools are present to fetch them.
- Suggest adding CI badges and a short CONTRIBUTING.md if missing.

Deliverable
- The REPORT.md will be the main deliverable and will answer the original request (analysis of codebase structure) with concrete suggestions and reproducible commands.
