Implementation Plan: Analyze codebase structure for E2E Test Repo (id: c4447e85-ad42-4d8d-ab33-f5e286fe6b1f)

Overview
- The following tasks are intended to be executed inside a checked-out copy of the repository (repo root). Each task is self-contained and produces artifacts (files or short text outputs). Tasks may be run in parallel where indicated. When all tasks complete, a final aggregation task will collect the outputs and produce the requested analysis report.

Repository reference
- Repo: https://github.com/dreef3/multi-agent-harness-test-repo.git
- Repository ID: c4447e85-ad42-4d8d-ab33-f5e286fe6b1f
- Default branch: main

Task 1 — Top-level file & directory inventory (fast, parallel)
- Purpose: get high-level structure, detect presence of common manifest files
- Run in repo root
- Commands:
  - ls -la
  - tree -a -L 2 || (find . -maxdepth 2 -printf "%P\n")
  - find . -maxdepth 3 -type f -name 'package.json' -o -name 'pyproject.toml' -o -name 'requirements.txt' -o -name 'Pipfile' -o -name 'go.mod' -o -name 'Cargo.toml' -o -name 'pom.xml' -o -name 'build.gradle' -o -name 'Gemfile' -o -name 'Makefile' -o -name '.github' -o -name '.gitlab-ci.yml'
- Artifacts produced (place in /tmp/planning-artifacts or attach in agent workspace):
  - top_level_tree.txt (output of tree or find)
  - manifest_list.txt (list of matching manifest files)

Task 2 — Read README, LICENSE, CONTRIBUTING
- Purpose: capture project purpose, install/run instructions, license
- Files to read (if present): README.md, README.rst, LICENSE, CONTRIBUTING.md
- Commands:
  - head -n 200 README.md || true
  - sed -n '1,200p' LICENSE || true
- Artifacts:
  - README_excerpt.txt (first 200 lines)
  - LICENSE.txt

Task 3 — Language & package-manager detection
- Purpose: detect primary languages and package managers
- Commands:
  - if [ -f package.json ]; then cat package.json | jq '{name,version,scripts,dependencies,devDependencies}' ; fi
  - if [ -f pyproject.toml ]; then sed -n '1,200p' pyproject.toml; fi
  - if [ -f requirements.txt ]; then head -n 200 requirements.txt; fi
  - if [ -f go.mod ]; then cat go.mod; fi
  - if [ -f Cargo.toml ]; then sed -n '1,200p' Cargo.toml; fi
  - if [ -f pom.xml ]; then sed -n '1,200p' pom.xml; fi
- Artifacts:
  - package_manifests/ (files above copied)

Task 4 — CI and automation discovery
- Purpose: identify CI workflows and scripts used by the repo
- Paths:
  - .github/workflows/*
  - .gitlab-ci.yml
  - .circleci/config.yml
  - .github/actions/
- Commands:
  - ls -la .github/workflows || true
  - sed -n '1,200p' .github/workflows/*.yml || true
- Artifacts:
  - ci_workflows/ (copies of workflow files)

Task 5 — Test discovery and dry-run
- Purpose: locate tests and discover how to run them. If CI or scripts define test commands, capture them.
- Commands:
  - rg -n "describe\(|it\(|pytest|unittest|test_|^@Test|^func Test" --hidden || true
  - find . -type d -name test -o -name tests -maxdepth 4 || true
  - if [ -f package.json ]; then jq -r '.scripts | to_entries[]?.key + " => " + .scripts[.key]' package.json || true; fi
  - Capture top-level Makefile targets: if [ -f Makefile ]; then sed -n '1,200p' Makefile; fi
- If safe and allowed by runner policy, run test commands in a sandbox (this plan only recommends commands; actual execution must be in a CI or ephemeral container):
  - npm ci && npm test (Node.js projects)
  - pip install -r requirements.txt && pytest -q (Python projects)
  - go test ./... (Go projects)
  - cargo test (Rust projects)
- Artifacts:
  - test_locations.txt
  - test_commands.txt
  - test_run_output.txt (if tests were executed)

Task 6 — Static metrics
- Purpose: collect code size and language stats
- Tools/commands:
  - du -sh .
  - cloc --quiet --by-file --out=cloc-by-file.txt . || (find . -name "*.py" -o -name "*.js" -o -name "*.go" -o -name "*.rs" | xargs wc -l > sloc.txt)
- Artifacts:
  - repo_size.txt
  - cloc-summary.txt

Task 7 — Notable scripts and entrypoints
- Purpose: identify scripts, binaries, and major entrypoints (Dockerfile, k8s manifests)
- Files to check:
  - Dockerfile, docker-compose.yml, k8s/ or manifests/
  - src/, cmd/, bin/
- Commands:
  - rg -n "FROM .*\|EXPOSE\|ENTRYPOINT\|CMD" Dockerfile || true
  - ls -la Dockerfile || true
- Artifacts:
  - docker_and_entrypoints.txt

Task 8 — Security and licensing quick checks
- Purpose: detect secrets, license compatibility, public vulnerability scanning hints
- Commands:
  - rg -n "AKIA|PRIVATE_KEY|BEGIN RSA PRIVATE KEY|BEGIN PRIVATE KEY" --hidden || true
  - rg -n "GPL|MIT|Apache|Proprietary" README.md LICENSE || true
- Artifacts:
  - secrets_scan.txt
  - license_findings.txt

Task 9 — Aggregate findings and produce analysis report
- Purpose: collect all artifacts and produce a human-readable structured report summarizing repository structure
- Inputs: outputs from Tasks 1–8
- Output: analysis_report.md containing sections:
  - Executive summary (1 paragraph)
  - Top-level structure (directories and files)
  - Detected languages and package managers
  - Tests and how to run them
  - CI workflows and how CI runs tests/builds
  - Notable files (Dockerfile, scripts)
  - Size & cloc metrics
  - Security/license notes
  - Recommended next steps
- Template snippets (example content to fill in):
  - Top-level files: copy contents of top_level_tree.txt
  - Detected languages: list based on presence of package.json, pyproject.toml, go.mod, etc.
  - Test commands: copy from test_commands.txt; include instructions to run in ephemeral container

Execution notes and requirements
- Tasks that run tests (Task 5) should be executed in ephemeral containers or CI runners with resource limits. They may modify the environment; isolate them.
- If running pip/npm installs, prefer ephemeral virtualenvs and --no-audit or --no-save flags when available.
- Keep all artifacts in a planning-artifacts/ subdirectory in the workspace to be uploaded or attached to the PR.

Deliverables
- analysis_report.md (final, human-readable report)
- planning-artifacts/ (raw outputs from discovery tasks)

Who should run these tasks
- A sub-agent or human engineer with network access to clone the repository and run shell commands. The tasks are intentionally low-risk; no destructive operations.

Estimated timeline
- Discovery tasks (1–3): 5–10 minutes
- CI & tests (4–5): 10–30 minutes depending on project and tests
- Metrics & aggregation (6–9): 5–15 minutes
