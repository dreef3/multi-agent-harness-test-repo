Implementation plan: Task-by-task plan to analyze repository structure for 'E2E Test Repo 0' (id: d16f98b0-7188-46fd18f33525)

Summary
- The plan is a sequence of self-contained tasks that inspect the repository working tree and produce artifacts under a /reports directory. Each task includes the exact commands to run, files to read, and expected output files.
- Tasks are small and independent so they can be dispatched to sub-agents.

Repository identifier
- repositoryId: d16f98b0-7188-46b4-9599-74fd18f33525

Common environment and conventions
- Working directory: the repository root.
- Reports output directory: ./reports (create if missing).
- All commands are POSIX/sh-compatible. Use installed tools: ls, find, sed, jq, python (3.x), node (if present), cloc (if present). If cloc not installed, fallback to simple file-count heuristics.
- Avoid printing secrets: skip any file paths matching **/secrets/**, **.env**, **.env.***, or files >5MB.

Tasks

Task 1 — Top-level file tree
- Purpose: capture the top-level layout (files and directories) and 2-level deep tree for context.
- Commands:
  - mkdir -p reports
  - find . -maxdepth 2 -mindepth 1 -not -path "./.git*" -printf "%p\n" | sed 's|^./||' > reports/file-tree.txt
- Expected outputs:
  - reports/file-tree.txt (human readable list)

Task 2 — Language breakdown
- Purpose: determine primary languages and LOC.
- Commands (prefer cloc):
  - if command -v cloc >/dev/null 2>&1; then cloc --json --quiet . > reports/language-stats.json; else use a fallback python script to count files by extension and approximate lines.
- Fallback Python approach (one-liner):
  - python3 - <<'PY'\nimport os, json\nstats={}\nfor root,_,files in os.walk('.'):\n  if root.startswith('./.git'): continue\n  for f in files:\n    if f.startswith('.'): continue\n    p=os.path.join(root,f)\n    try:\n      with open(p,'r',errors='ignore') as fh:\n        n=sum(1 for _ in fh)\n    except:\n      continue\n    ext=f.split('.')[-1] if '.' in f else ''\n    stats.setdefault(ext,{'files':0,'lines':0})\n    stats[ext]['files']+=1; stats[ext]['lines']+=n\nprint(json.dumps(stats,indent=2))\nPY > reports/language-stats.json
- Expected outputs: reports/language-stats.json

Task 3 — Identify package/dependency manifests
- Purpose: find manifests and extract primary dependencies.
- Files to look for (top-level and recursive to depth 3): package.json, yarn.lock, pnpm-lock.yaml, requirements.txt, pyproject.toml, setup.py, Pipfile, go.mod, pom.xml, build.gradle, composer.json, Gemfile, Cargo.toml
- Commands:
  - find . -maxdepth 3 -type f \( -name package.json -o -name yarn.lock -o -name pnpm-lock.yaml -o -name requirements.txt -o -name pyproject.toml -o -name setup.py -o -name Pipfile -o -name go.mod -o -name pom.xml -o -name build.gradle -o -name composer.json -o -name Gemfile -o -name Cargo.toml \) -not -path './.git/*' -print > reports/manifests.list
- For each manifest found, add a short parser command to extract top-level info:
  - package.json: jq '{name: .name, version: .version, scripts: .scripts, dependencies: .dependencies, devDependencies: .devDependencies}' package.json > reports/package.json.summary.json
  - requirements.txt: copy first 200 lines to reports/requirements.txt.preview
  - go.mod: copy first 200 lines to reports/go.mod.preview
  - pyproject.toml: copy first 200 lines to reports/pyproject.toml.preview
- Expected outputs: reports/manifests.list, reports/*.summary.json or *.preview

Task 4 — CI / Automation
- Purpose: find CI config and summarize what runs in CI.
- Files to search: .github/workflows/*.yml, .circleci/config.yml, .gitlab-ci.yml, Jenkinsfile, .travis.yml
- Commands:
  - find . -type f \( -path './.github/workflows/*.yml' -o -name '.circleci' -o -name '.gitlab-ci.yml' -o -name 'Jenkinsfile' -o -name '.travis.yml' \) -print > reports/ci-files.list
  - For each GitHub workflow file: extract 'name' and top-level jobs list with jq-like parsing using python/ruamel or simple grep heuristics and produce reports/ci-summary.md
- Expected outputs: reports/ci-files.list, reports/ci-summary.md

Task 5 — Tests layout and test commands
- Purpose: identify test directories (tests/, spec/, __tests__) and test commands in package manifests or CI.
- Commands:
  - rg -n "(^pytest|^unittest|jest|mocha|ava|pytest|python -m pytest|pytest\b|npm test|yarn test)" -S --hidden || true
  - find . -type d \( -iname 'tests' -o -iname 'test' -o -iname 'spec' -o -iname '__tests__' \) -print > reports/test-dirs.list
  - If package.json present, extract scripts.test to reports/npm-test-command.txt
- Expected outputs: reports/test-dirs.list, reports/npm-test-command.txt (if applicable), reports/test-summary.md with instructions to run tests found.

Task 6 — Docker and Infrastructure
- Purpose: locate Dockerfiles, k8s manifests, Terraform, Helm charts.
- Commands:
  - find . -type f \( -iname 'Dockerfile' -o -iname 'docker-compose.yml' -o -iname '*.tf' -o -iname 'Chart.yaml' -o -iname '*.yaml' -a -path './deploy/*' \) -print > reports/infra-files.list
  - For each Dockerfile, produce a short preview: head -n 200 Dockerfile > reports/Dockerfile.<n>.preview
- Expected outputs: reports/infra-files.list, reports/Dockerfile.*.preview

Task 7 — Notable scripts and Makefiles
- Purpose: find Makefile, build scripts, and top-level scripts in scripts/ folder.
- Commands:
  - find . -maxdepth 2 -type f \( -name 'Makefile' -o -path './scripts/*' -o -name 'build.sh' -o -name 'start.sh' \) -print > reports/scripts.list
  - Create small summaries by capturing first 200 lines.
- Expected outputs: reports/scripts.list, reports/*script*.preview

Task 8 — README, CONTRIBUTING, LICENSE
- Purpose: extract the repository's human-facing description.
- Commands:

  - Compose CODEBASE_SUMMARY.md by combining README preview, language-stats, manifests summary, test summary, and CI summary.
- Expected outputs: reports/README.md.preview, reports/CONTRIBUTING.md.preview, reports/LICENSE.preview, reports/CODEBASE_SUMMARY.md

Task 9 — Package/module entry points and service discovery
- Purpose: for monorepos or multi-package repos, list packages under packages/, services/, apps/, modules/.
- Commands:
  - for d in packages services apps modules libs; do [ -d $d ] && find $d -maxdepth 2 -type f -name package.json -print; done > reports/monorepo.packages.list
  - For each package.json found, extract name and main/module fields into reports/monorepo.packages.summary.json
- Expected outputs: reports/monorepo.packages.list, reports/monorepo.packages.summary.json

Task 10 — Produce final human summary
- Purpose: assemble CODEBASE_SUMMARY.md and a short action checklist (how to run, where tests live, important TODOs).
- Contents to include:

  - Languages and LOC summary (top 3)
  - Main manifests and dependency managers
  - CI presence and how to run locally
  - Tests and commands to run them
  - Docker/infra notes
  - Recommended next steps (run cloc, run tests, run static analysis)
- Expected outputs: reports/CODEBASE_SUMMARY.md
