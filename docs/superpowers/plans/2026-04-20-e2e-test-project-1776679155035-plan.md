Implementation plan: step-by-step tasks to analyze the repository "E2E Test Repo 1" (id: c6ca4ca8-5660-4067-a980-6f25fd520cf7, URL: https://github.com/dreef3/multi-agent-harness-test-repo.git).

Overview
- The plan is a sequence of discrete tasks. Each task is self-contained, declares exact file paths or shell commands to run, expected outputs, and the deliverable format. The agent performing the tasks should commit results to a folder named analysis-output/ at the repository root (or return them to the caller if running outside the repo).

Task 0 — Prepare workspace
- Description: Clone the repository and change into it.
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
- Deliverable: none (precondition for following tasks). Note: if cloning is not permitted, use available repo copy or GitHub API to fetch files.

Task 1 — Top-level listing
- Description: Enumerate top-level files and directories.
- Commands:
  - ls -la
  - find . -maxdepth 2 -type d -print
- Expected output:
  - analysis-output/top-level.txt: plain text list of top-level files and directories.

Task 2 — Detect common manifest and config files
- Description: Look for common project files. Exact file paths to check (if they exist):
  - README.md, CONTRIBUTING.md, LICENSE
  - package.json, yarn.lock, pnpm-lock.yaml
  - requirements.txt, Pipfile, pyproject.toml, setup.py, setup.cfg
  - go.mod, Gopkg.toml
  - Cargo.toml
  - pom.xml, build.gradle, settings.gradle
  - Cargo.toml
  - Makefile
  - Dockerfile
  - .github/workflows/*.yml
  - .gitlab-ci.yml, .circleci/config.yml
  - tsconfig.json, .eslintrc*, .prettierrc
  - .env.example
- Commands:
  - for f in README.md LICENSE package.json pyproject.toml go.mod Cargo.toml Dockerfile Makefile; do [ -f "$f" ] && echo "FOUND: $f"; done > analysis-output/manifests.txt
  - ls -la .github/workflows || true >> analysis-output/manifests.txt
- Deliverable: analysis-output/manifests.txt listing which of the above files are present, with a brief note about each.

Task 3 — Language and size breakdown
- Description: Determine languages used and approximate code size by language.
- Commands (examples):
  - Use cloc if available: cloc --json . > analysis-output/cloc.json
  - Fallback: count files by extension: find . -type f | sed -n 's/.*\.\([^.]*\)$/\1/p' | sort | uniq -c | sort -nr > analysis-output/file-ext-counts.txt
- Deliverable: cloc.json or file-ext-counts.txt

Task 4 — Locate source roots and entry points
- Description: Find obvious source roots and entry points.
- Paths/commands to check in order:
  - src/, lib/, app/, cmd/, pkg/
  - Look for main files: src/index.js, src/main.ts, main.go, cmd/*/main.go, __main__.py, setup.py entry points
- Commands:
  - rg "package main" -n || true
  - rg "if __name__ == .__main__." -n || true
  - rg "module.exports|export default|ReactDOM.render|createApp\(" -n || true
- Deliverable: analysis-output/entrypoints.txt listing located entrypoints and source roots with file paths.

Task 5 — Tests and CI
- Description: Find test directories and CI pipeline configs and infer test commands.
- Commands:
  - rg "pytest|unittest|mocha|jest|ava|go test|cargo test" -n || true
  - ls -la tests/ spec/ || true
  - cat .github/workflows/* || true > analysis-output/github-workflows.txt
- Deliverable: analysis-output/tests-and-ci.md describing test frameworks found and suggested commands to run.

Task 6 — Dependency extraction
- Description: Extract declared dependencies from manifest files.
- Commands (per manifest type):
  - If package.json exists: jq '.dependencies, .devDependencies' package.json > analysis-output/package-deps.json
  - If pyproject.toml or requirements.txt: cat them into analysis-output/python-deps.txt
  - If go.mod: cat go.mod > analysis-output/go-mod.txt
  - If Cargo.toml: cat Cargo.toml > analysis-output/cargo-toml.txt
- Deliverable: analysis-output/*-deps.* files.

Task 7 — Build and run commands
- Description: Try to determine build and start commands from manifests.
- Commands:
  - For Node: jq '.scripts' package.json > analysis-output/npm-scripts.json
  - For Python: look for setup.py setup.cfg pyproject.toml for commands
  - For Go: look for Makefile or go.mod
- Deliverable: analysis-output/build-commands.md with recommended commands to build, run, and test.

Task 8 — Static checks and linters
- Description: Check for linter configs (ESLint, flake8, golangci-lint) and report presence.
- Paths to check: .eslintrc*, .prettierrc, .flake8, .golangci.yml
- Deliverable: analysis-output/linters.txt

Task 9 — Produce human-readable summary
- Description: Collate findings into a concise markdown report with these sections:
  - Repo metadata (URL, default branch if known)
  - Top-level layout summary
  - Languages and LOC
  - Key files (manifest, CI, Dockerfile)
  - Entry points and how to run
  - Tests and CI
  - Dependencies
  - Issues / Missing items (README, license, tests)
  - Suggested next steps
- Deliverable: analysis-output/summary.md (2–4 pages)

Task 10 — Machine-readable manifest
- Description: Produce a JSON file with discovered files, types, languages, entrypoints, and dependency files.
- Example schema:
  {
    "repo": "multi-agent-harness-test-repo",
    "files": [
      {"path":"package.json","type":"node-manifest"},
      {"path":"src/index.js","type":"entrypoint","language":"javascript"}
    ],
    "languages": {"JavaScript": 1234, "Python": 100},
    "entrypoints": ["src/index.js"],
    "ci": [".github/workflows/ci.yml"]
  }
- Deliverable: analysis-output/manifest.json

Task 11 — Recommendations and next steps
- Description: Short bullets recommending immediate improvements (missing docs, add CI, add tests, tidy dependencies).
- Deliverable: analysis-output/recommendations.md

Output packaging

- Create a top-level zip: zip -r analysis-output.zip analysis-output/
