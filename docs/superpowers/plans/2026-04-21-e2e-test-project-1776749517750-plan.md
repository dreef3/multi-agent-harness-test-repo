Implementation Plan: Detailed task-by-task plan to analyze the repository structure for d585b833-b17d-4df6-ad8b-52dfc9d49758 (https://github.com/dreef3/multi-agent-harness-test-repo.git)

Overview
- This plan breaks the analysis into 7 discrete tasks. Each task lists exact file paths to inspect, recommended commands, expected outputs, acceptance criteria, and estimated effort.

Repository root assumption
- All commands below assume you are operating from the repository root after cloning the repository URL above.

Task 1 — Create a canonical file tree
- Purpose: Capture the repository layout (top-level dirs and files) so further analysis can target relevant locations.
- Files to inspect: top-level (./)
- Commands / procedure (examples):
  - tree -a -L 3 > artifacts/file-tree.txt
  - find . -maxdepth 2 -type d -print
- Expected artifact: artifacts/file-tree.txt (committed)
- Acceptance criteria: file-tree.txt exists and shows directory names such as README.md, package.json, src/, tests/, .github/, Dockerfile, etc.
- Estimated time: 5–10 minutes

Task 2 — Detect languages and code size
- Purpose: Determine primary languages and approximate code size.
- Files to inspect: all source files; typical language markers: package.json, pyproject.toml, go.mod, Cargo.toml, Gemfile, pom.xml
- Commands / procedure (examples):
  - Install/run cloc: cloc --json . > artifacts/language-summary.json
  - Alternatively, use tophash or linguist if available.
- Expected artifact: artifacts/language-summary.json with lines of code per language.
- Acceptance criteria: language-summary.json lists at least one primary language and LOC counts.
- Estimated time: 5–15 minutes

Task 3 — Identify package manifests and extract metadata
- Purpose: Find manifest files and extract project metadata (name, version, entrypoint, scripts, dependencies).
- Files to inspect (check presence and open):
  - Node: package.json, package-lock.json, yarn.lock, pnpm-lock.yaml
  - Python: pyproject.toml, setup.cfg, setup.py, requirements.txt, Pipfile
  - Go: go.mod
  - Java: pom.xml, build.gradle
  - Rust: Cargo.toml
  - PHP: composer.json
  - Others: Gemfile
- Commands / procedure (examples):
  - For JSON manifests: jq '{name:.name, version:.version, main:.main, scripts:.scripts}' package.json > artifacts/package-json-metadata.json
  - For pyproject.toml: tomlq or python -c 'import tomllib,sys;print(tomllib.load(open("pyproject.toml")))'
  - For go.mod: sed -n '1,200p' go.mod > artifacts/go-mod-head.txt
- Expected artifacts: artifacts/dependency-manifests.md (human-readable summary), and per-manifest JSON where appropriate in artifacts/
- Acceptance criteria: dependency-manifests.md lists identified manifest files and key metadata (project name, version, main/entrypoints, test script commands).
- Estimated time: 15–30 minutes

Task 4 — Inspect CI, build, and container configs
- Purpose: Understand automated processes for build/test/deploy.
- Files to inspect:
  - .github/workflows/*
  - .gitlab-ci.yml
  - .circleci/config.yml
  - Dockerfile, docker-compose.yml
  - Makefile
- Commands / procedure (examples):
  - ls -la .github/workflows
  - sed -n '1,200p' .github/workflows/*.yml > artifacts/ci-workflows-heads.txt
  - Check for matrix builds, test steps, cache usage.
- Expected artifact: artifacts/ci-summary.md containing a concise description of each workflow and the primary build/test steps.
- Acceptance criteria: ci-summary.md identifies whether CI runs tests, builds artifacts, and any deployment steps.
- Estimated time: 20–40 minutes

Task 5 — Locate and summarize tests and quality tools
- Purpose: Find test suites, test frameworks, linting, and coverage tools.
- Files to inspect: tests/, test/, __tests__/, jest.config.js, pytest.ini, tox.ini, .eslintrc*, .prettierrc, coverage.yml
- Commands / procedure (examples):
  - rg "pytest|jest|mocha|unittest|go test|mvn test|gradle test" -n || true
  - tree -a tests -L 2 > artifacts/tests-tree.txt
  - If package.json: jq '.scripts' package.json > artifacts/package-scripts.json to spot test scripts
- Expected artifacts: artifacts/tests-tree.txt, artifacts/tests-summary.md (frameworks, location, how to run)
- Acceptance criteria: tests-summary.md states where tests live and provides commands to run them locally.
- Estimated time: 20–40 minutes

Task 6 — Document developer setup and run instructions

  - Extract setup sections and create artifacts/setup-steps.md summarizing steps (prereqs, install commands, run commands).
- Expected artifact: artifacts/setup-steps.md
- Acceptance criteria: setup-steps.md contains at least one working sequence of commands to install deps and run the project locally (language-specific).

Task 7 — Produce the final report and recommendations

  - Top-level file tree excerpt
  - Manifest and dependency summary
  - CI summary
  - Tests and quality tool summary
  - Actionable recommendations (e.g., add CI if missing, add CONTRIBUTING, pin deps, add tests)
- Commands / procedure (examples):
  - Collect artifacts/ into REPORT.md
  - git add artifacts/ REPORT.md && git commit -m "analysis: repository structure report" && push to a topic branch (per harness process)
- Expected artifact: REPORT.md in repository root and artifacts/ directory with supporting files.
- Acceptance criteria: REPORT.md covers all sections above and is reviewed by a maintainer.
