Implementation plan: Task-by-task plan to analyze repository e6b7e30e-3d6e-47d4-b853-b0471b7ad216 (https://github.com/dreef3/multi-agent-harness-test-repo.git)

Overview
This plan breaks the analysis into discrete tasks that can be executed by sub-agents (or a CI job) with repository access. Each task is self-contained and produces artifacts saved under analysis/ in the repo or workspace. Expected outputs: analysis/report.md, analysis/summary.json, analysis/raw/ (command outputs).

Task 1 — Repository inventory (priority: high, est: 5–10m)
- Purpose: Record top-level files and directory tree; identify manifest files and CI configs.
- Commands (run from repo root):
  - ls -la
  - tree -a -L 2 || find . -maxdepth 2 -print
  - rg -n --hidden --glob "!.git" "^" --files | sed -n '1,200p' (or use git ls-files)
- Detect manifest files (presence check): package.json, package-lock.json, yarn.lock, pnpm-lock.yaml, pyproject.toml, requirements.txt, setup.py, pipfile, Pipfile.lock, go.mod, Cargo.toml, Gemfile, composer.json, build.gradle, pom.xml, Makefile, Dockerfile, workspace config files (pnpm-workspace.yaml, lerna.json, nx.json).
- Detect CI files: .github/workflows/*, .gitlab-ci.yml, azure-pipelines.yml, circleci/config.yml.
- Output files:
  - analysis/raw/inventory.txt (command outputs)
  - analysis/summary.json (start filling manifest presence array)

Task 2 — Language & LOC metrics (priority: high, est: 10–30m)
- Purpose: Quantify languages, file counts and lines of code.
- Commands:
  - cloc --json . --quiet > analysis/raw/cloc.json (or use scc if cloc not installed)
  - If cloc not available, fallback to: find . -type f -name "*.py" | wc -l etc., and wc -l for line counts per extension.
- Output:
  - analysis/raw/cloc.json
  - Update analysis/summary.json with per-language LOC and file counts.

Task 3 — Dependency manifests parsing (priority: high, est: 10–30m)
- Purpose: Extract direct dependencies from primary manifests.
- Commands/examples:
  - Node: jq '.dependencies, .devDependencies' package.json > analysis/raw/package-deps.json
  - Python: parse pyproject.toml or requirements.txt; for pyproject use toml parser to extract [project] or [tool.poetry]
  - Go: parse go.mod -> go list -m all (if environment allows) or parse file
  - Rust: parse Cargo.toml [dependencies]
  - Java: parse pom.xml for <dependencies>
- Output:
  - analysis/raw/dependencies/ (one file per ecosystem, e.g. node.json, python.txt)
  - Update analysis/summary.json with counts and major direct dependencies.

Task 4 — Tests and CI (priority: medium, est: 10–20m)
- Purpose: Locate tests, test frameworks, and CI workflows.
- Commands:
  - rg -n "pytest|unittest|mocha|jest|ava|tap|rspec|go test|pytest" || search for tests/ directories
  - ls -la .github/workflows || rg -n "runs-on" .github -S
- Output:
  - analysis/raw/tests.txt (list of test directories and detected frameworks)
  - analysis/raw/ci-files.txt (CI workflows content)

Task 5 — Architecture & entrypoints (priority: medium, est: 15–30m)
- Purpose: Identify likely application entrypoints, server frameworks, and package layout.
- Approach & commands:
  - For Node: look for "main" in package.json, scripts like start, bin entries
  - For Python: find __main__.py, setup.py entry_points, or CLI scripts
  - For Go: look for package main and func main in .go files
  - For Java: look for src/main/java and classes with public static void main
  - Inspect Dockerfile(s) for CMD/ENTRYPOINT and exposed ports
- Output:
  - analysis/raw/entrypoints.txt
  - architecture diagram description in analysis/report.md

Task 6 — Large files and hotspots (priority: low, est: 5–15m)
- Purpose: Find oversized files and directories that dominate repo size.
- Commands:
  - du -sh * | sort -h | tail -n 20
  - find . -type f -exec du -h {} + | sort -h | tail -n 20
- Output:
  - analysis/raw/large-files.txt

Task 7 — Governance and metadata (priority: low, est: 5–10m)
- Purpose: Check for LICENSE, README, CODE_OF_CONDUCT, SECURITY.md, CONTRIBUTING, CHANGELOG
- Commands: 
  - for f in LICENSE README* CHANGELOG* CODE_OF_CONDUCT* SECURITY* CONTRIBUTING*; do [ -e "$f" ] && echo "$f"; done
- Output:
  - analysis/raw/governance.txt

Task 8 — Produce final report (priority: high, est: 20–40m)
- Purpose: Combine artifacts into analysis/report.md: executive summary, inventory, languages & LOC, dependency summary, entrypoints & architecture notes, tests & CI, governance, large files, and recommended next steps.
- Steps:
  - Compose analysis/report.md from collected artifacts.
  - Ensure analysis/summary.json is populated with keys: {repo: {id, url}, inventory: [...], languages: {...}, manifests: {...}, dependencies: {...}, tests: {...}, ci: {...}, entrypoints: [...], large_files: [...], governance: [...], recommendations: [...]}

Task 9 — Output verification and PR (priority: high, est: 5–10m)
- Purpose: Ensure artifacts are present and open a PR (or attach to issue). The harness will handle PR creation.
- Steps:
  - Run quick validation script that checks existence of analysis/report.md and analysis/summary.json

Outputs & File paths
- analysis/report.md (human-readable summary)
- analysis/summary.json (machine-readable summary)
- analysis/raw/cloc.json
- analysis/raw/inventory.txt
- analysis/raw/dependencies/*.json or *.txt
- analysis/raw/tests.txt
- analysis/raw/ci-files.txt
- analysis/raw/entrypoints.txt
- analysis/raw/large-files.txt
- analysis/raw/governance.txt

Notes and constraints
- This plan assumes read access to the repository files. Tasks should be executed in a clean checkout at the repository root.
- Avoid making any code changes; write artifacts under analysis/ only.
- If the repository is very large (>1GB), consider sampling or limiting cloc depth.

Who should run these tasks
- A sub-agent with repository checkout ability and standard tooling installed: bash, rg (ripgrep), cloc (or scc), jq, python (for toml parsing) where needed.

Estimated total time
- Small repo: ~45 minutes
- Medium repo: ~1.5 hours
- Large repo: ~3+ hours

Sample commands
- cloc --json . --quiet > analysis/raw/cloc.json
- tree -a -L 2 > analysis/raw/inventory.txt
- jq '.dependencies + .devDependencies' package.json > analysis/raw/dependencies/node.json || true
