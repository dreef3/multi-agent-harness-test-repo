Tasks (self-contained, ready to dispatch to implementers)

Task 1 — Add repo-inspector tool (scripts only)
- Purpose: produce the analysis artifacts from a checked-out repo.
- Files to create (planning branch):
  - tools/repo-inspector/README.md  (documentation on usage and requirements)
  - tools/repo-inspector/analyze_repo.py  (main inspector script)
  - tools/repo-inspector/requirements.txt  (if Python used)
- Script responsibilities (pseudo-commands / examples):
  - Accept input: repository path (local) and output directory.
  - Produce directory tree: using os.walk or `git ls-tree -r --name-only`.
  - Compute language breakdown: count bytes by file extension; for better accuracy, use github-linguist or pygments (optional).
  - Detect config files: check for common filenames (package.json, pyproject.toml, go.mod, pom.xml, Cargo.toml, .github/workflows/*, Dockerfile, Makefile, .gitmodules, .gitattributes).
  - Detect tests: look for test directories (tests, spec) and common test deps.
  - Detect CI: list .github/workflows and other CI files.
  - Detect LFS: parse .gitattributes and git config.
  - Output: reports/<repo-id>-analysis.md and reports/<repo-id>-summary.json
- Example analyze_repo.py outline (do NOT implement here; include in task so implementer can copy into the PR):
  - argparse to parse inputs
  - walk the tree and collect stats
  - simple heuristics to detect languages
  - write Markdown report and JSON summary
