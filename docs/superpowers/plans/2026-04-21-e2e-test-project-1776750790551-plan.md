Implementation Plan: Tasks to analyze repository structure

Overview
This plan breaks the work into self-contained tasks that can be dispatched to sub-agents (or run by maintainers). Each task lists the exact commands to run, expected outputs, files to create, and where to place them in the repository.

Global constants
- REPORT_DIR=reports
- TOOL_DIR=tools
- TIMESTAMP format: $(date -u +%Y%m%dT%H%M%SZ)

Deliverables (file paths)
- tools/analyze_repo.sh (reproducible analysis script)
- reports/tree.txt (top-level tree listing)
- reports/language_summary.json (language LOC breakdown)
- reports/dependencies.md (list of dependency manifests and extracted deps)
- reports/ci_summary.md (CI/config files discovered)
- reports/docker_summary.md (Dockerfiles found)
- reports/tests_summary.md (test directories & noteworthy test frameworks)
- reports/structure_report.md (human-readable consolidated summary)
- reports/metrics.json (machine-readable metrics: file count, LOC, top languages)

Task 1 — Create tools/analyze_repo.sh
- Goal: single script that performs the repo scan and writes outputs to REPORT_DIR.
- Commands (bash script body):
  #!/usr/bin/env bash
  set -euo pipefail
  REPORT_DIR="${REPORT_DIR:-reports}"
  mkdir -p "$REPORT_DIR"
  TS=$(date -u +%Y%m%dT%H%M%SZ)

  # 1. Full file tree (limited depth 3 for overview, full list too)
  command -v tree >/dev/null 2>&1 && tree -a -L 3 > "$REPORT_DIR/tree.txt" || find . -maxdepth 3 -print > "$REPORT_DIR/tree.txt"
  find . -type f | sed 's|^\./||' | sort > "$REPORT_DIR/file_list.txt"

  # 2. Language breakdown (prefer cloc, fallback to simple extension counts)
  if command -v cloc >/dev/null 2>&1; then
    cloc --json . > "$REPORT_DIR/language_summary.json"
  else
    # fallback: count lines per extension
    awk 'BEGIN{FS="."} NF>1{ext=$NF; a[ext]+=1} END{for (i in a) print i, a[i]}' <(sed 's|^\./||' file_list.txt) > "$REPORT_DIR/language_by_file.txt"
  fi

  # 3. Dependency manifests
  # Look for common manifests
  MANIFESTS=("package.json" "yarn.lock" "pnpm-lock.yaml" "pyproject.toml" "requirements.txt" "Pipfile" "go.mod" "build.gradle" "pom.xml" "Cargo.toml" "Gemfile")
  echo "Dependency manifests found:" > "$REPORT_DIR/dependencies.md"
  for m in "${MANIFESTS[@]}"; do
    if [ -f "$m" ]; then
      echo "- $m" >> "$REPORT_DIR/dependencies.md"
    fi
  done

  # 4. CI files
  echo "CI/config files found:" > "$REPORT_DIR/ci_summary.md"
  if [ -d .github/workflows ]; then
    ls -1 .github/workflows >> "$REPORT_DIR/ci_summary.md" || true
  fi
  for ci in .gitlab-ci.yml circle.yml .circleci config.yml; do
    [ -f "$ci" ] && echo "- $ci" >> "$REPORT_DIR/ci_summary.md"
  done

  # 5. Dockerfiles
  echo "Dockerfiles found:" > "$REPORT_DIR/docker_summary.md"
  find . -type f -iname Dockerfile -o -iname '*Dockerfile*' | sed 's|^\./||' >> "$REPORT_DIR/docker_summary.md" || true

  # 6. Tests
  echo "Test directories and files:" > "$REPORT_DIR/tests_summary.md"
  find . -type d -name test -o -name tests -o -name spec | sed 's|^\./||' | sort >> "$REPORT_DIR/tests_summary.md" || true
  rg -n "pytest|unittest|mocha|jest|describe\(|it\(" || true >> "$REPORT_DIR/tests_summary.md" 2>/dev/null || true

  # 7. Metrics - file count, total lines (using wc)
  FILE_COUNT=$(wc -l < <(find . -type f | wc -l) ) || FILE_COUNT="N/A"
  TOTAL_LOC=$(awk 'BEGIN{c=0} {cmd="wc -l < \""$0"\""; cmd | getline l; close(cmd); c+=l} END {print c}' $(find . -type f) 2>/dev/null || echo "N/A")
  cat > "$REPORT_DIR/metrics.json" <<EOF
  {"timestamp": "$TS", "file_count": "$FILE_COUNT", "total_loc": "$TOTAL_LOC"}
  EOF

  # 8. Consolidated human-readable summary
  {
    echo "Repository structure report\nGenerated: $TS\n"
    echo "Top-level directories:"; ls -1 | sed -n '1,200p'
    echo "\nSee detailed files in: $REPORT_DIR"
  } > "$REPORT_DIR/structure_report.md"

- Expected outputs: files listed in Deliverables. Script must be executable (chmod +x).
- Place file at: tools/analyze_repo.sh

Task 2 — Run the analysis script and collect outputs
- Preconditions: repository cloned locally and the script committed.
- Commands (to run in repo root):
  chmod +x tools/analyze_repo.sh
  REPORT_DIR=reports tools/analyze_repo.sh
- Expected outcome: reports/ populated with report files.

Task 3 — Add a summary README and CI integration (optional)
- Create reports/README.md that explains the reports and how to regenerate them (commands above).
- If desired, add a CI job (e.g., .github/workflows/analyze.yml) that runs tools/analyze_repo.sh on demand or schedule and uploads artifacts.

Task 4 — Manual review & recommendations
- A reviewer will inspect reports/structure_report.md and:
  - Confirm main languages and frameworks
  - Confirm whether tests are runnable and whether CI covers them
  - Note any missing or outdated dependency files
  - Recommend next steps: add license, add CI, add container publishing, address dependency issues
- Place review notes in reports/review_notes.md

Task 5 — Produce final consolidated report for maintainers
- Combine the various findings into reports/structure_report.md with sections:
  - Overview and top-level tree
  - Languages and LOC
  - Package/dependency manifests
  - CI and workflows
  - Docker/containers
  - Tests and test frameworks
  - Scripts and developer utilities
  - Recommendations and follow-ups
- Use content from the other reports to populate this file.
