Implementation plan: Analyze repository structure for "E2E Test Repo 0" (id: 019595aa-3665-4a36-82b5-b06c9b785fbd)

Overview
This plan breaks the work into small, self-contained tasks that can be executed by sub-agents or a human reviewer. Each task includes exact file paths to inspect, shell commands to run, expected outputs, and where to write artifacts. All artifacts should be written into analysis/ at the repository root.

Task 0 — Preparation
- Goal: Prepare workspace and tools
- Actions:
  - Ensure standard CLI tools are available: git, find, du, sort, head, wc, sed, grep
  - Optional but recommended: cloc or tokei for language statistics
- Estimated time: 1–2 minutes

Task 1 — Clone repository
- Goal: fetch repository locally for analysis
- Command(s):
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
- Artifacts: local checkout at ./repo
- Notes: Use shallow clone if desired: git clone --depth 1 <url>

Task 2 — Top-level listing and tree
- Goal: produce a human-readable top-level file tree (first 4 levels)
- Commands:
  - mkdir -p analysis
  - find . -maxdepth 4 -type d -print | sed 's|^\./||' | sort > analysis/tree_dirs.txt
  - find . -maxdepth 4 -print | sed 's|^\./||' | sort > analysis/tree.txt
  - head -n 200 analysis/tree.txt > analysis/tree_head.txt
- Files to inspect manually: README.md, CONTRIBUTING.md, LICENSE, CODE_OF_CONDUCT.md, .gitignore

Task 3 — Top-level files and manifests
- Goal: enumerate common manifest and config files
- Files to check (paths relative to repo root):
  - README* (README.md, README.rst)
  - LICENSE, COPYING
  - package.json, package-lock.json, yarn.lock
  - pyproject.toml, setup.py, requirements.txt, Pipfile
  - go.mod, Gopkg.toml
  - Cargo.toml
  - Gemfile
  - composer.json
  - Dockerfile
  - Makefile
  - .github/workflows/*
  - .gitlab-ci.yml, .circleci/config.yml
  - tsconfig.json, nx.json, angular.json (JS monorepo indicators)
- Command(s):
  - for f in README* LICENSE* package.json package-lock.json yarn.lock pyproject.toml setup.py requirements.txt Pipfile go.mod Cargo.toml Gemfile composer.json Dockerfile Makefile tsconfig.json .gitlab-ci.yml .circleci/config.yml; do [ -f "$f" ] && echo "$f"; done > analysis/manifests_found.txt
  - For each found manifest copy it to analysis/manifests/ (mkdir -p analysis/manifests; cp <file> analysis/manifests/)
- Artifact: analysis/manifests_found.txt, analysis/manifests/

Task 4 — Language breakdown and LOC
- Goal: get languages and line counts
- Commands (prefer cloc if installed, fallback to tokei or wc):
  - if command -v cloc >/dev/null 2>&1; then cloc --json . > analysis/languages.json; else if command -v tokei >/dev/null 2>&1; then tokei --output json . > analysis/languages.json; else find . -name "*.*" -type f | sed 's|^\./||' | xargs -d '\n' -r awk 'FNR==1{files++} {total++} END{print files, total}' > analysis/languages_fallback.txt; fi
- Also produce a simple file-count by extension:
  - find . -type f | sed 's|.*\.||' | sort | uniq -c | sort -rn > analysis/file_extensions_count.txt
- Artifact: analysis/languages.json, analysis/file_extensions_count.txt

Task 5 — Disk usage (sizes)
- Goal: find largest directories and files
- Commands:
  - du -h --max-depth=2 2>/dev/null | sort -hr | head -n 100 > analysis/sizes_top.txt
  - find . -type f -printf '%s %p\n' | sort -nr | head -n 50 > analysis/largest_files.txt
- Artifact: analysis/sizes_top.txt, analysis/largest_files.txt

Task 6 — CI and workflows
- Goal: enumerate CI configurations and summarize their jobs
- Files to inspect: .github/workflows/*.yml, .gitlab-ci.yml, .circleci/config.yml
- Commands:
  - if [ -d .github/workflows ]; then for f in .github/workflows/*.{yml,yaml}; do echo "==== $f ===="; sed -n '1,200p' "$f"; echo; done > analysis/ci_workflows_raw.txt; fi
  - if [ -f .gitlab-ci.yml ]; then sed -n '1,200p' .gitlab-ci.yml > analysis/ci_gitlab_raw.txt; fi
- Summarize: parse top-level job names (grep '^[[:space:]]*[^#].*:' may find job keys) and write analysis/ci_summary.md

Task 7 — Tests and test commands
- Goal: identify test directories and common test commands
- Paths to check: tests/, test/, src/*/tests, spec/, __tests__/, jest.config.js, pytest.ini, tox.ini
- Commands:
  - find . -type d -name tests -o -name test -o -name __tests__ -print > analysis/test_dirs.txt
  - grep -R "pytest\|jest\|mocha\|ava\|go test\|cargo test\|mvn test\|gradle test" -n --exclude-dir=.git > analysis/test_commands_candidates.txt || true
- Artifact: analysis/test_dirs.txt, analysis/test_commands_candidates.txt, analysis/tests.md

Task 8 — Dependency parsing (per ecosystem)
- Goal: extract declared dependencies from known manifests
- Files/commands per ecosystem:
  - Node.js: package.json -> jq '.dependencies, .devDependencies' > analysis/manifests/package_dependencies.json
  - Python: requirements.txt (copy), pyproject.toml (parse [tool.poetry.dependencies] or [project.dependencies]), setup.py may require heuristics
  - Go: go.mod -> list modules (grep '^require' -A 20)
  - Rust: Cargo.toml -> [dependencies]
  - Ruby: Gemfile -> list gems
- Example commands (Node):
  - if [ -f package.json ]; then jq '{dependencies: .dependencies, devDependencies: .devDependencies}' package.json > analysis/manifests/package_dependencies.json; fi
- Artifacts: one file per manifest with extracted dependencies under analysis/manifests/

Task 9 — Entry points and build files
- Goal: locate common entry points and build instructions
- Files to search: main.go, cmd/, src/index.js, bin/, setup.py (entry_points), Makefile, Dockerfile, package.json (scripts.start, scripts.build)
- Commands:
  - grep -R "^package main$\|^func main()\|module.exports\|export default\|scripts" -n --exclude-dir=.git . | head -n 200 > analysis/entrypoint_candidates.txt
  - if [ -f Dockerfile ]; then sed -n '1,200p' Dockerfile > analysis/Dockerfile.txt; fi
- Artifact: analysis/entrypoint_candidates.txt, analysis/Dockerfile.txt

Task 10 — Produce human summary and recommendations
- Goal: synthesize the findings into analysis/summary.md
- Contents:
  - Repo metadata: url, clone size, top languages
  - Top-level layout and likely entry points
  - Where dependencies live and how to install/build
  - CI and test coverage summary
  - Notable risks or missing pieces (no README, no license, no CI, huge vendor dir)
  - Recommended next actions (add README, add CI, run linters, add CONTRIBUTING)
- Commands: assemble previous artifacts into summary.md. Example skeleton (to be filled by analyzer):
  - echo "# Summary" > analysis/summary.md
  - echo "- URL: https://github.com/dreef3/multi-agent-harness-test-repo.git" >> analysis/summary.md
  - echo "- Top languages: $(jq -r 'keys | .[]' analysis/languages.json | paste -sd ", ")" >> analysis/summary.md
  - append other artifact snippets (head -n 50 of tree.txt, sizes_top.txt, etc.)

Task 11 — Package artifacts for review
- Goal: ensure analysis/ contains everything and produce a short changelog/README for the analysis
- Commands:
  - tar -czf analysis_artifacts.tar.gz analysis/
  - ls -l analysis/ > analysis/manifest.txt

Deliverables (final files inside repo/analysis)
- summary.md
- tree.txt (or tree_head.txt)
- manifests_found.txt and manifests/ (copied manifests and parsed dependencies)
- languages.json or languages.csv
- file_extensions_count.txt
- sizes_top.txt, largest_files.txt
- ci_workflows_raw.txt, ci_summary.md
- test_dirs.txt, tests.md
- entrypoint_candidates.txt
- analysis_artifacts.tar.gz (packaged)

Execution notes
- Each task is independent and can be executed in order; the manifest parsing tasks (8) depend on presence of those files.
- Use caution when parsing setup.py (it may execute code). Prefer static parsing or pip-tools/pipdeptree if available.
- When cloc/tokei are not available, the fallback produces coarser results.

Estimated timeline
- Total wall time: 5–30 minutes depending on repository size and whether language tools are installed.
