Implementation Plan: Repository analysis for f6503b47-5c27-4789-bad7-af9154233752

Overview
- The implementation will be a sequence of small, self-contained inspection tasks executed in the repository workspace. Each task is designed to be run by a sub-agent or CI job and writes its output into a dedicated analysis directory: analysis/codebase-inspection/.
- The final outputs are:
  - analysis/codebase-inspection/CODEBASE_STRUCTURE.md (human-readable)
  - analysis/codebase-inspection/codebase_summary.json (machine-readable)
  - analysis/codebase-inspection/raw-outputs/ (command outputs for auditing)

Tasks (ordered)

1) Prepare workspace
- Purpose: ensure a consistent working directory and create analysis output directories.
- Commands:
  - mkdir -p analysis/codebase-inspection/raw-outputs
  - git rev-parse --abbrev-ref HEAD > analysis/codebase-inspection/raw-outputs/branch.txt

2) Top-level listing
- Purpose: capture the repository root contents.
- Commands:
  - ls -la > analysis/codebase-inspection/raw-outputs/ls-root.txt
  - tree -a -L 2 > analysis/codebase-inspection/raw-outputs/tree-L2.txt
- Files to examine in top-level: README.md, LICENSE, .gitignore, .gitattributes, package.json, pyproject.toml, setup.py, requirements.txt, Pipfile, go.mod, Cargo.toml, Gemfile, Makefile, Dockerfile, .github/

3) Detect language/manifests
- Purpose: identify primary languages and package managers.
- Commands (examples):
  - [Node] if [ -f package.json ]; then jq '{name,version,dependencies,devDependencies,scripts}' package.json > analysis/codebase-inspection/raw-outputs/package.json.parsed.json || cp package.json analysis/codebase-inspection/raw-outputs/; fi
  - [Python] if [ -f pyproject.toml ]; then sed -n '1,200p' pyproject.toml > analysis/codebase-inspection/raw-outputs/pyproject.toml.head; fi
  - [Go] if [ -f go.mod ]; then sed -n '1,200p' go.mod > analysis/codebase-inspection/raw-outputs/go.mod.head; fi
  - [Rust] if [ -f Cargo.toml ]; then sed -n '1,200p' Cargo.toml > analysis/codebase-inspection/raw-outputs/Cargo.toml.head; fi
  - [Ruby] if [ -f Gemfile ]; then sed -n '1,200p' Gemfile > analysis/codebase-inspection/raw-outputs/Gemfile.head; fi

4) CI and workflow detection
- Purpose: list CI workflows and key steps.
- Commands:
  - if [ -d .github/workflows ]; then ls -la .github/workflows > analysis/codebase-inspection/raw-outputs/workflows-list.txt; for f in .github/workflows/*.y*ml; do echo "--- $f" >> analysis/codebase-inspection/raw-outputs/workflows-all.txt; sed -n '1,200p' "$f" >> analysis/codebase-inspection/raw-outputs/workflows-all.txt; done; fi
  - Also check for other CI files: .gitlab-ci.yml, circle.yml, azure-pipelines.yml

5) Docker/Container artifacts
- Purpose: find Dockerfiles and image build contexts.
- Commands:
  - rg --hidden --glob '!node_modules' -n "^FROM" -S || true
  - find . -maxdepth 3 -type f -iname Dockerfile -print > analysis/codebase-inspection/raw-outputs/dockerfiles.txt

6) Source tree and common layout detection
- Purpose: detect common source directories and conventions.
- Commands:
  - for d in src lib app cmd pkg test tests e2e examples; do if [ -d "$d" ]; then echo "$d" >> analysis/codebase-inspection/raw-outputs/source-dirs.txt; fi; done
  - for d in $(ls -d */ 2>/dev/null); do echo $d; done > analysis/codebase-inspection/raw-outputs/topdirs.txt

7) Test and coverage detection
- Purpose: find test frameworks, CI test steps, and coverage config.
- Commands:
  - rg --hidden -n "pytest|mocha|jest|go test|cargo test|mvn test|gradle test|ruby -I|rspec" || true > analysis/codebase-inspection/raw-outputs/test-detections.txt
  - rg --hidden -n "coverage|codecov|coveralls|pytest-cov|nyc|istanbul" || true > analysis/codebase-inspection/raw-outputs/coverage-detections.txt

8) Static analysis for README and LICENSE
- Purpose: make sure basic docs exist and capture project description.
- Commands:
  - if [ -f README.md ]; then sed -n '1,200p' README.md > analysis/codebase-inspection/raw-outputs/README.head.md; fi
  - if [ -f LICENSE ]; then sed -n '1,200p' LICENSE > analysis/codebase-inspection/raw-outputs/LICENSE.head; fi

9) Aggregate findings into JSON
- Purpose: create codebase_summary.json summarizing detected artifacts.
- Strategy: use a small shell script or python one-liner to populate fields by testing for existence and reading parsed snippets.
- Example pseudo-command (POSIX sh / python):
  - python3 - <<'PY'
import json,os
root='analysis/codebase-inspection/raw-outputs'
s={'branch':open(os.path.join(root,'branch.txt')).read().strip() if os.path.exists(os.path.join(root,'branch.txt')) else None}
# presence checks
s['has_readme']=os.path.exists('README.md')
s['has_license']=os.path.exists('LICENSE')
# check manifests
for m in ['package.json','pyproject.toml','go.mod','Cargo.toml','Gemfile','Pipfile','requirements.txt','Dockerfile','Makefile']:
    s.setdefault('manifests',{})
    s['manifests'][m]=os.path.exists(m)
open('analysis/codebase-inspection/codebase_summary.json','w').write(json.dumps(s,indent=2))
PY
