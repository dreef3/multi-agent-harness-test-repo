# Implementation Plan: Codebase Structure Analysis for E2E Test Repo

Repository: E2E Test Repo
Repository ID: 4f32690d-46cd-4399-a144-f98f1759d609
URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Overview

This plan lists concrete, ordered tasks to be executed inside a checked-out copy of the repository to produce the analysis artifacts listed in the spec. Each task includes exact commands, required tools, expected outputs (file names under reports/), and time estimates.

Environment prerequisites

- Linux/macOS with: git, tree, cloc, jq, ripgrep (rg), find, sed, awk, Python 3, Node.js/npm (optional), Rust/cargo (optional), Go (optional), Java (Maven/Gradle) (optional), graphviz (dot) (optional).
- Assumes a network connection for cloning the repo.

Top-level plan and artifact layout

- Clone repo: ./workspace/<repo>
- Create reports/: reports/tree.txt, reports/cloc.json, reports/manifest-summary.md, reports/ci-files.md, reports/dependency-*.json/txt, reports/test-summary.md, reports/todos.md, reports/summary.md

Detailed tasks

Task 1 — Checkout (manual or automated runner)
- Purpose: ensure a local copy to analyze.
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
- Output: working directory ready
- Time: 30s–2m

Task 2 — Top-level tree and file counts
- Purpose: capture the repository layout and top-level items.
- Commands:
  - mkdir -p reports
  - tree -a -I 'node_modules|venv|\.git' -L 2 > reports/tree.txt || (find . -maxdepth 2 -print > reports/tree.txt)
  - find . -type f | sed 's|^\./||' | awk -F/ '{print $1}' | sort | uniq -c | sort -rn > reports/top-level-counts.txt
- Outputs: reports/tree.txt, reports/top-level-counts.txt
- Time: 10s

Task 3 — Code size and language metrics
- Purpose: quantify code size by language and files.
- Commands:
  - cloc --json . > reports/cloc.json
  - cloc --by-file-by-lang --yaml --out=reports/cloc-by-file.yaml . || true
- Outputs: reports/cloc.json, reports/cloc-by-file.yaml
- Time: 10–30s

Task 4 — Detect manifests and CI configs
- Purpose: list common manifests (package managers) and CI files.
- Commands (single-run discovery):
  - printf "# Manifests\n" > reports/manifest-summary.md
  - for f in package.json pyproject.toml requirements.txt Pipfile setup.py setup.cfg build.gradle pom.xml go.mod Cargo.toml composer.json yarn.lock Gemfile Podfile Dockerfile docker-compose.yml; do if [ -f "$f" ]; then echo "- $f" >> reports/manifest-summary.md; fi; done
  - printf "\n# CI Configs\n" >> reports/manifest-summary.md
  - for f in .github/workflows/*.yml .gitlab-ci.yml .circleci/config.yml Jenkinsfile; do if [ -f "$f" ]; then echo "- $f" >> reports/manifest-summary.md; fi; done
- Outputs: reports/manifest-summary.md
- Time: 10s

Task 5 — Dependency lists per ecosystem (guarded by manifest presence)
- Purpose: produce per-language dependency snapshots.
- Commands (run only if the corresponding manifest exists):
  - Node (package.json): npm --quiet --no-audit --no-update-notifier ls --all --json > reports/npm-deps.json || jq -r '.dependencies|keys[]' package.json > reports/npm-manifest-deps.txt
  - Python (pyproject.toml or requirements.txt): if pyproject.toml exists, run python -c 'import toml,sys, json;print(json.dumps(toml.load(open("pyproject.toml")),indent=2))' > reports/pyproject.json || if [ -f requirements.txt ]; then cat requirements.txt > reports/requirements.txt; fi
  - Go (go.mod): if [ -f go.mod ]; then go list -m all > reports/go-deps.txt; fi
  - Rust (Cargo.toml): if [ -f Cargo.toml ]; then cargo metadata --format-version=1 --no-deps > reports/cargo-metadata.json; fi
  - Java/Maven (pom.xml): if [ -f pom.xml ]; then mvn -q dependency:tree -DoutputFile=reports/maven-deps.txt || true; fi
  - PHP (composer.json): if [ -f composer.json ]; then composer show --format=json > reports/composer-deps.json || echo "composer.json present" > reports/composer-manifest.json; fi
- Notes: Some commands require toolchains (npm, go, cargo, mvn). If tool is missing, fallback to recording the manifest file content into reports/ for manual inspection.
- Outputs: reports/npm-deps.json, reports/requirements.txt, reports/go-deps.txt, reports/cargo-metadata.json, reports/maven-deps.txt, reports/composer-deps.json
- Time: 1–5 min depending on env and registries

Task 6 — CI/workflow scanning
- Purpose: extract and summarize CI workflows and pipeline triggers.
- Commands:
  - mkdir -p reports/ci
  - if [ -d .github/workflows ]; then for f in .github/workflows/*.yml; do echo "-- $f"; sed -n '1,120p' "$f" > reports/ci/$(basename "$f"); done; fi
  - if [ -f .gitlab-ci.yml ]; then cp .gitlab-ci.yml reports/ci/.gitlab-ci.yml; fi
  - echo "Files:" > reports/ci-files.md
  - ls -1 .github/workflows 2>/dev/null | sed 's/^/- .github\/workflows\//' >> reports/ci-files.md || true
  - [ -f .gitlab-ci.yml ] && echo '- .gitlab-ci.yml' >> reports/ci-files.md || true
- Outputs: reports/ci-files.md, reports/ci/*
- Time: 20s

Task 7 — Tests and coverage discovery
- Purpose: detect test directories, test frameworks and count tests.
- Commands:
  - printf "# Test Summary\n" > reports/test-summary.md
  - find . -type d -name test -o -name tests -o -name __tests__ | sed 's/^\.\///' | tee -a reports/test-summary.md
  - find . -type f -name '*_test.py' -o -name 'test_*.py' | wc -l > reports/test-count.txt || true
  - rg --hidden --glob '!node_modules' "@pytest|pytest\.main|unittest\.|mocha|jest" -S --no-ignore -n || true >> reports/test-framework-hints.txt
- Outputs: reports/test-summary.md, reports/test-count.txt, reports/test-framework-hints.txt
- Time: 20s

Task 8 — Collect TODOs, FIXMEs, and technical debt markers
- Purpose: show concentrated areas of unfinished work or notes.
- Commands:
  - rg -n --hidden --no-ignore "TODO|FIXME|XXX|HACK" || true > reports/todos.md
- Outputs: reports/todos.md
- Time: 10s

Task 9 — License and contributor info
- Purpose: capture license and author files.
- Commands:
  - for f in LICENSE LICENSE.md LICENSE.txt COPYING AUTHORS CONTRIBUTING.md README.md README.rst; do [ -f "$f" ] && cp "$f" reports/ || true; done
  - if [ -f LICENSE ]; then head -n 40 LICENSE > reports/license-snippet.txt; fi
- Outputs: reports/LICENSE*, reports/license-snippet.txt
- Time: 5s

Task 10 — Optional: Dependency graph visualization
- Purpose: create a simple dot graph for top-level dependencies (Node/Rust/Go) if metadata available.
- Commands (example for npm):
  - jq -r '.dependencies | keys[]' package.json | awk '{print "\"$(basename $PWD)\" -> \""$0"\""}' > reports/dep_edges.txt
  - printf "digraph deps {\n" > reports/dep.dot
  - sed 's/.*/  &;/' reports/dep_edges.txt >> reports/dep.dot
  - printf "}\n" >> reports/dep.dot
  - dot -Tsvg reports/dep.dot -o reports/deps.svg || true
- Outputs: reports/dep.dot, reports/deps.svg (optional)
- Time: 30s

Task 11 — Synthesize findings into summary.md
- Purpose: produce a human-readable summary that includes: top languages, lines of code, major manifests, CI presence, tests presence and counts, notable TODOs, recommended next steps.
- Example content (to be generated by a script or manually):
  - Top-level summary lines (1–3 lines)
  - Table: language | files | LOC (from cloc)
  - Manifests found and recommended follow-ups
  - CI present: yes/no + which files
  - Test coverage: tests found, how to run tests
  - Recommendations: add CI if missing, add CONTRIBUTING.md, add codeowners, add dependency scanning, increase tests, etc.
- Commands:
  - python3 scripts/summarize_reports.py --reports reports --out reports/summary.md || echo "Manual: compile reports/* -> reports/summary.md"
- Outputs: reports/summary.md
- Time: 2–10 min (scripted)

Task 12 — Packaging artifacts
- Purpose: ensure all artifacts are stored under reports/ and optionally archived.
- Commands:
  - tar czf ../repo-analysis-$(date +%Y%m%dT%H%M%S).tgz reports
- Outputs: repo-analysis-<timestamp>.tgz
- Time: 10s

Deliverables

- reports/ directory with the files listed in each task. A single reports/summary.md that synthesizes the findings and recommended follow-ups.

Estimated total time

- 5–15 minutes for basic discovery (Tasks 1–9) on most machines; additional time (up to 10–20 minutes) if dependency enumeration commands need to resolve registries or large dependency trees.

Notes for implementers

- If a manifest is present but the corresponding tool chain (npm, go, cargo, mvn) is not available in the execution environment, capture the raw manifest into reports/ and mark dependency enumeration as skipped in reports/summary.md.

- Keep the analysis read-only. Do not modify repository files.

Next steps

- After these reports are produced, a follow-up task should parse the artifacts and create an executive summary with prioritized remediation steps (e.g., add CI, resolve TODOs, increase tests coverage, add CONTRIBUTING). That follow-up will be a separate task.
