Implementation plan: Task-by-task codebase analysis for E2E Test Repo 1 (01f654cd-db55-449e-8d8e-56f0298a87eb)

Overview

This plan defines a sequence of independent tasks that a sub-agent or operator can run to analyze the repository. Each task includes exact commands, expected file paths, and artifacts to produce. Tasks are ordered: discovery -> metadata -> tests & linting (optional) -> report generation.

Global notes

- Default branch: main. If main is not present, fall back to the repository default branch.
- Use a sandboxed environment when running tests or executing repository code.
- Output artifacts are written into repo root under ./analysis-artifacts/.

Task 00: Clone repository (operator or sub-agent)

- Purpose: create a local copy for analysis.
- Commands:
  git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  cd repo
  git checkout main || true

Task 01: Top-level file and directory listing

- Purpose: identify README, LICENSE, manifests, source dirs, tests, CI configs, Dockerfiles, Makefile.
- Commands:
  mkdir -p analysis-artifacts
  (ls -la | sed -n '1,200p') > analysis-artifacts/top_level_listing.txt
  tree -a -L 2 . | sed 's/^[\t ]*//' > analysis-artifacts/tree_L2.txt || (find . -maxdepth 2 -printf "%P\n" > analysis-artifacts/tree_L2.txt)
- Expected files to note (if present): README.md, CONTRIBUTING.md, LICENSE, package.json, pyproject.toml, requirements.txt, Pipfile, setup.py, Cargo.toml, go.mod, build.gradle, pom.xml, Gemfile, src/, lib/, app/, tests/, .github/, Dockerfile, Makefile

Task 02: Language detection and LOC

- Purpose: produce language breakdown and approximate LOC per language.
- Commands:
  if command -v cloc >/dev/null 2>&1; then cloc --json --out=analysis-artifacts/cloc.json .; else echo 'cloc not installed' > analysis-artifacts/cloc-not-installed.txt; fi
  # Fallback simple heuristic: count extensions
  rg --no-ignore --hidden --files | sed -n '1,20000p' | awk -F. '/\./{print $NF}' | sort | uniq -c | sort -nr > analysis-artifacts/file-extension-counts.txt

Task 03: Detect dependency manifests and capture dependency trees

- Purpose: locate manifests and extract dependency info.
- Commands (per language):
  # Node/npm
  if [ -f package.json ]; then jq . < package.json > analysis-artifacts/package.json.pretty.json || true; npm --no-audit --no-fund ls --all --json > analysis-artifacts/npm-deps.json 2>/dev/null || true; fi

  # Python
  if [ -f pyproject.toml ] || [ -f requirements.txt ] || [ -f setup.py ]; then cp pyproject.toml analysis-artifacts/ 2>/dev/null || true; cp requirements.txt analysis-artifacts/ 2>/dev/null || true; python3 -m pip freeze > analysis-artifacts/pip_freeze.txt 2>/dev/null || true; fi

  # Go
  if [ -f go.mod ]; then cp go.mod analysis-artifacts/; go list -m all > analysis-artifacts/go-mod-list.txt 2>/dev/null || true; fi

  # Rust
  if [ -f Cargo.toml ]; then cp Cargo.toml analysis-artifacts/; cargo metadata --format-version 1 > analysis-artifacts/cargo-metadata.json 2>/dev/null || true; fi

  # Java/Maven/Gradle
  cp pom.xml analysis-artifacts/ 2>/dev/null || true; cp build.gradle analysis-artifacts/ 2>/dev/null || true

Task 04: CI/CD and workflow inspection

- Purpose: find CI config and describe workflows.
- Commands:
  find . -maxdepth 4 -type f \( -name "*.yml" -o -name "*.yaml" \) -path "./.github/*" -print > analysis-artifacts/ci_files.txt || true
  cp -r .github analysis-artifacts/ 2>/dev/null || true
  cp .gitlab-ci.yml analysis-artifacts/ 2>/dev/null || true
  cp .circleci/config.yml analysis-artifacts/ 2>/dev/null || true
  rg "name:|jobs:|steps:|runs-on:" .github -n || true > analysis-artifacts/ci_keywords.txt

Task 05: Test discovery and optional test run

- Purpose: locate tests and, if operator-approved, run them in sandbox to gather pass/fail and coverage.
- Commands:
  rg --hidden --glob '!node_modules' -n "\bdescribe\(|\bpytest\b|unittest\.|@Test\b|it\(" || true > analysis-artifacts/test_grep.txt
  find . -type d \( -name test -o -name tests -o -name spec -o -name __tests__ \) -print > analysis-artifacts/test_dirs.txt || true

  # Optional: run tests (operator flag RUN_TESTS=true)
  if [ "$RUN_TESTS" = "true" ]; then
    # Node
    if [ -f package.json ]; then npm install --no-audit --no-fund && npm test --silent > analysis-artifacts/npm_test_output.txt 2>&1 || true; fi
    # Python
    if [ -f pyproject.toml ] || [ -f setup.py ] || [ -f requirements.txt ]; then python3 -m pip install -r requirements.txt >/dev/null 2>&1 || true; pytest -q --maxfail=1 --junitxml=analysis-artifacts/pytest_results.xml > analysis-artifacts/pytest_output.txt 2>&1 || true; fi
  fi

Task 06: Static analysis and basic security checks (optional)

- Purpose: run linters and dependency audits where feasible, gated by operator flag RUN_AUDITS=true.
- Commands:
  if [ "$RUN_AUDITS" = "true" ]; then
    # npm audit (CI-safe)
    if [ -f package.json ]; then npm audit --json > analysis-artifacts/npm_audit.json 2>/dev/null || true; fi

    # pip-audit (if installed)
    if command -v pip-audit >/dev/null 2>&1; then pip-audit -f json -o analysis-artifacts/pip_audit.json || true; fi

    # Static linters (if configured)
    if [ -f .eslintrc* ] || rg -n "eslint" package.json >/dev/null 2>&1; then npm run lint --silent > analysis-artifacts/eslint_output.txt 2>&1 || true; fi
    if rg -n "flake8|pylint|mypy" >/dev/null 2>&1; then python3 -m pytest --dry-run >/dev/null 2>&1 || true; fi
  fi

Task 07: Generate structured report

- Purpose: produce CODEBASE_ANALYSIS.md (human-readable) and codebase_summary.json (machine-readable) in repo root.
- Template contents (to be filled by the script from artifacts):
  - Summary: top-level description (one paragraph)
  - Files & dirs discovered: list or table pointing to analysis-artifacts files
  - Languages & LOC: cloc.json or file-extension-counts.txt
  - Dependency manifests found and dependency summaries with top-level direct deps
  - CI status: files found and brief description
  - Tests: test locations, results (if run)
  - Issues & risks: missing README, no tests, pinned deps, outdated CI, large transitive dependency trees
  - Recommendations: add README, improve CI, run audits, pin dependencies, add tests and coverage

- Example generation command (script to be provided):
  ./scripts/generate_codebase_report.sh analysis-artifacts > CODEBASE_ANALYSIS.md
  python3 - <<'PY'
import json,os
summary={}
summary['top_level_listing']=open('analysis-artifacts/top_level_listing.txt').read() if os.path.exists('analysis-artifacts/top_level_listing.txt') else None
# (extend to gather other artifacts)
open('codebase_summary.json','w').write(json.dumps(summary,indent=2))
PY

Task 08: Optional: produce dependency graph visuals

- Purpose: generate DOT/SVG graphs for complex dependency graphs (npm, pip via pipdeptree, cargo) and save to analysis-artifacts.
- Commands (examples):
  if [ -f package.json ]; then npx depcruise --output-type dot src | dot -Tsvg -o analysis-artifacts/npm-deps.svg || true; fi
  if command -v pipdeptree >/dev/null 2>&1; then pipdeptree --json > analysis-artifacts/pipdeptree.json || true; fi

Task 09: Finalize and summarize

- Purpose: produce top-level SUMMARY.txt with recommended next steps and risk level.
- Commands:
  echo "See CODEBASE_ANALYSIS.md and analysis-artifacts/ for full details" > SUMMARY.txt
  echo "Repository: 01f654cd-db55-449e-8d8e-56f0298a87eb" >> SUMMARY.txt

Permissions and safety

- By default RUN_TESTS and RUN_AUDITS are false. Tests and audits can be enabled by setting environment variables when running the tasks, but should only be run in a controlled sandboxed environment.

Execution notes for the harness

- When implementing as sub-agent tasks, include the repository id in the subtask description and ensure the sub-agent clones the repository fresh.
