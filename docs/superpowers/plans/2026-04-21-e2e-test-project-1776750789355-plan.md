Implementation plan: step-by-step tasks to analyze repository structure and produce reports.

Repository: https://github.com/dreef3/multi-agent-harness-test-repo.git
Repository UUID: 0fdf7599-6b35-4ed5-99e5-1adaade7bb71
Default branch: main

Overview
- We will create a short script and a sequence of bash commands to inventory files and extract metadata. The plan is split into discrete tasks that a sub-agent can run independently. Each task is self-contained and writes outputs to the workspace.

Task 1 — Checkout (preparation)
- Goal: clone repository at main into working directory.
- Commands:
  git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  cd repo
  git checkout main
- Output: repository files available under ./repo

Task 2 — Top-level inventory
- Goal: list top-level files and directories, show sizes and types.
- Commands:
  cd repo
  ls -la
  tree -L 2 --dirsfirst || (find . -maxdepth 2 -print)
- Expected files to highlight (if present): README.md, LICENSE, package.json, pyproject.toml, requirements.txt, setup.py, Cargo.toml, go.mod, pom.xml, build.gradle, .github, Dockerfile, docker-compose.yml, Makefile, src/, lib/, tests/, e2e/
- Output file: ./analysis/top_level.txt

Task 3 — Language and manifest detection
- Goal: detect primary languages and package managers by checking for known manifest files and language-specific source dirs.
- Commands:
  cd repo
  echo '{}' > ./analysis/manifest_detection.json
  (for f in package.json package-lock.json yarn.lock pnpm-lock.yaml pyproject.toml requirements.txt setup.py Pipfile Pipfile.lock Cargo.toml go.mod pom.xml build.gradle Gemfile Gemfile.lock composer.json); do if [ -f "$f" ]; then echo "Found: $f"; fi; done > ./analysis/manifests_found.txt
  # Also search for source directories
  [ -d src ] && echo 'src dir found' >> ./analysis/manifests_found.txt
  rg --hidden --glob '!node_modules' --files --glob '*/**/*.py' -S || true
- Expected output: list of manifests and likely language(s). Save to ./analysis/languages.json

Task 4 — CI / workflows / automation
- Goal: find CI configuration files and Dockerfiles.
- Commands:
  cd repo
  find . -maxdepth 3 -type f -name '*.yml' -o -name '*.yaml' | rg '\.github|gitlab' -n || true
  ls -la .github || true
  find . -type f -name 'Dockerfile' -print > ./analysis/dockerfiles.txt || true
  [ -f .github/workflows ] && tree -L 2 .github/workflows || true
- Output: ./analysis/ci_files.txt, ./analysis/dockerfiles.txt

Task 5 — Tests and E2E
- Goal: locate tests (unit and E2E), test frameworks, and test run commands.
- Commands:
  cd repo
  rg "\bdescribe\(|\bit\(|@pytest|unittest|TestCase|test_.*\.py|\.spec\.js|\.e2e\.js|cypress" --hidden --glob '!node_modules' -n || true > ./analysis/test_grep.txt
  find . -type d -name 'tests' -o -name 'test' -o -name 'e2e' -o -name 'spec' > ./analysis/test_paths.txt || true
  # Check package.json scripts for test commands
  if [ -f package.json ]; then jq '.scripts' package.json > ./analysis/package_scripts.json || cat package.json > ./analysis/package.json; fi
- Output: ./analysis/test_paths.txt, ./analysis/test_grep.txt, ./analysis/package_scripts.json
