Implementation Plan: Analyze codebase structure for repo id 96ad3d3d-7633-4afa-9f75-f779007952f1 (https://github.com/dreef3/multi-agent-harness-test-repo.git)

Overview
- This plan breaks the analysis into discrete tasks that are safe and read-only. Each task includes exact shell commands to run in the repository root, the files to capture, and the expected deliverable.
- Assign each task to a sub-agent or CI job that has the repository checked out.

Tasks

Task 1 — Top-level inventory
- Purpose: Capture top-level layout and immediate files.
- Commands:
  - ls -la
  - find . -maxdepth 2 -type d -printf "%p\n" | sed 's|^./||'
- Artifacts to save: top_level_ls.txt (stdout of ls -la), top_level_dirs.txt (directories list)
- Expected deliverable: list of top-level files and directories

Task 2 — Full file tree (depth 4)
- Purpose: Provide a human-readable file tree for navigation.
- Commands:
  - find . -maxdepth 4 -print | sed 's|^./||' > file_tree_depth4.txt
- Artifacts: file_tree_depth4.txt

Task 3 — Identify languages and package manifests
- Purpose: Detect primary languages and package managers.
- Commands (run and capture if present):
  - [ -f package.json ] && echo package.json && cat package.json > package.json.txt || true
  - [ -f pyproject.toml ] && echo pyproject.toml && cat pyproject.toml > pyproject.toml.txt || true
  - [ -f requirements.txt ] && echo requirements.txt && cat requirements.txt > requirements.txt.txt || true
  - [ -f go.mod ] && echo go.mod && cat go.mod > go.mod.txt || true
  - [ -f Cargo.toml ] && echo Cargo.toml && cat Cargo.toml > Cargo.toml.txt || true
  - [ -f Gemfile ] && echo Gemfile && cat Gemfile > Gemfile.txt || true
  - rg --hidden --glob '!node_modules' --files --glob "*.lock" || true
- Artifacts: copies of any manifest files; a languages.txt summary
- Expected deliverable: languages and package managers list (e.g., Node/npm, Python/Poetry)

Task 4 — Inspect README and docs
- Purpose: Capture project description, getting started, and contributing guidelines.
- Commands:
  - [ -f README.md ] && sed -n '1,200p' README.md > README_head.md || true
  - [ -f CONTRIBUTING.md ] && cat CONTRIBUTING.md > CONTRIBUTING.md.txt || true
  - [ -d docs ] && find docs -type f -maxdepth 2 -print -exec sed -n '1,200p' {} \; > docs_snippets.txt || true
- Artifacts: README_head.md, CONTRIBUTING.md.txt, docs_snippets.txt

Task 5 — CI and workflows
- Purpose: Identify CI providers, workflow files, and matrix builds.
- Commands:
  - [ -d .github/workflows ] && find .github/workflows -type f -maxdepth 2 -print -exec sed -n '1,240p' {} \; > workflows.txt || true
  - [ -f .gitlab-ci.yml ] && cat .gitlab-ci.yml > gitlab-ci.yml.txt || true
  - [ -f azure-pipelines.yml ] && cat azure-pipelines.yml > azure-pipelines.yml.txt || true
- Artifacts: workflows.txt, gitlab-ci.yml.txt, azure-pipelines.yml.txt

Task 6 — Docker and containerization
- Purpose: Find Dockerfiles and container-related configs.
- Commands:
  - find . -type f -iname Dockerfile -print -exec sed -n '1,240p' {} \; > dockerfiles.txt || true
  - rg --hidden --glob '!node_modules' "FROM .*" -n || true
- Artifacts: dockerfiles.txt

Task 7 — Tests and test frameworks
- Purpose: Find test directories and test runners (jest, pytest, go test, etc.).
- Commands:
  - find . -type d -name test -o -name tests -maxdepth 4 -print > tests_dirs.txt || true
  - rg --hidden --glob '!node_modules' "pytest|jest|mocha|AVA|go test|cargo test" -n || true
  - If package.json present: jq '.scripts' package.json > package_scripts.json || true
- Artifacts: tests_dirs.txt, package_scripts.json (if applicable)

Task 8 — Build and run scripts
- Purpose: Collect project scripts and Makefiles.
- Commands:
  - [ -f Makefile ] && sed -n '1,240p' Makefile > Makefile_head.txt || true
  - [ -f package.json ] && jq '.scripts' package.json > package_scripts.json || true
  - rg --hidden --glob '!node_modules' "^\s*script" -n || true
- Artifacts: Makefile_head.txt, package_scripts.json

Task 9 — Dependency and license review
- Purpose: Capture dependencies and license file.
- Commands:
  - [ -f package-lock.json ] && head -n 500 package-lock.json > package-lock_head.json || true
  - [ -f yarn.lock ] && head -n 500 yarn.lock > yarn-lock_head.txt || true
  - [ -f LICENSE ] && sed -n '1,200p' LICENSE > LICENSE_head.txt || true
- Artifacts: dependency lock heads, LICENSE_head.txt

Task 10 — Produce analysis report
- Purpose: Synthesize artifacts into a short report.
- Instructions for the reporter: Use the collected artifacts to write report.md with sections:
  - Summary (primary language(s), package manager(s), CI)
  - Top-level structure (list top 10 dirs/files)
  - Key files (README, LICENSE, package manifests, Dockerfiles, workflows)
  - Tests and how to run them (from scripts)
  - Any red flags or recommended next steps
- Command example to create the report (manual step): assemble report.md from artifacts and include file references.
