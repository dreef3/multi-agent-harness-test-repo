Implementation Plan: Task-by-task plan to analyze codebase structure

Context

Repository: multi-agent-harness-test-repo
Repository ID: 1f0615d1-5163-4945-8e56-8852f4c5696c
Default branch: main

Goal

Run a reproducible, surface-level analysis that enumerates the repository structure, detects languages and package systems, finds tests and CI configurations, and outputs machine- and human-readable summary files.

High-level tasks

Task 1 — Prepare working copy (manual or sub-agent)
- Description: Clone the repository and checkout main. (Note: Not performed in planning phase.)
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git
  - cd multi-agent-harness-test-repo
  - git checkout main
- Output: local repository root for following tasks.

Task 2 — Top-level tree and file counts
- Description: Produce a high-level tree and count of files by extension and language.
- Commands (run at repo root):
  - mkdir -p analysis_output
  - find . -maxdepth 2 -mindepth 1 -type d -printf "%P\n" | sort > analysis_output/top_level_dirs.txt
  - find . -type f -not -path './.git/*' -printf "%p\n" | sed 's|\./||' > analysis_output/all_files.txt
  - (language detection) 
    - create a file extension frequency list:
      - awk -F. '!/\.git/ && NF>1{print $NF}' analysis_output/all_files.txt | sort | uniq -c | sort -rn > analysis_output/extension_counts.txt
  - Use 'cloc' if available for language summary:
    - cloc --json --out=analysis_output/cloc.json . || true
- Output files:
  - analysis_output/top_level_dirs.txt
  - analysis_output/all_files.txt
  - analysis_output/extension_counts.txt
  - analysis_output/cloc.json (optional)

Task 3 — Detect package manifests and lockfiles
- Description: Search for common package and manifest files for multiple languages.
- Commands:
  - rg -n --hidden --glob '!**/.git/**' "(^|/)PACKAGE|package.json|pyproject.toml|setup.py|requirements.txt|Pipfile|pom.xml|go.mod|Gopkg.toml|Cargo.toml|Gemfile|Makefile|build.gradle|composer.json|package-lock.json|yarn.lock|pnpm-lock.yaml|requirements.txt" || true
  - Specific checks (examples):
    - test -f package.json && echo "package.json found" || true
    - test -f pyproject.toml && echo "pyproject.toml found" || true
- Output: analysis_output/manifests.txt (copy the results)

Task 4 — Locate test directories and test frameworks
- Description: Find test directories and files (tests/, spec/, *_test.go, etc.)
- Commands:
  - rg -n --hidden --glob '!**/.git/**' "(^|/)(test|tests|spec|__tests__)/|_test\.go$|test_.*\.py$|\.spec\.js$|\.test\.(js|ts)$" || true
- Output: analysis_output/tests_found.txt

Task 5 — Identify CI configurations and workflows
- Description: Find CI config files (GitHub Actions, Travis, Circle, GitLab CI, etc.).
- Commands:
  - rg -n --hidden --glob '!**/.git/**' "\.github/workflows/|\.travis.yml$|circleci|gitlab-ci.yml|appveyor.yml|azure-pipelines.yml" || true
- Output: analysis_output/ci_configs.txt

Task 6 — Check for documentation and contribution files
- Description: Look for README, CONTRIBUTING, docs folder, CODEOWNERS, LICENSE.
- Commands:
  - ls -la | rg -i "README|CONTRIBUTING|LICENSE|CODEOWNERS" || true
  - test -d docs && echo "docs/ found" || true
- Output: analysis_output/docs_and_meta.txt

Task 7 — Produce CODEBASE_TREE.json (machine-readable)
- Description: Create a JSON summarizing top-level directories, file counts by extension, manifest paths, test paths, CI files, and a cloc summary if available.
- Example format (write to repo root: CODEBASE_TREE.json):
  {
    "top_level_dirs": ["src","tests","docs"],
    "file_counts_by_extension": {"py": 120, "js": 45},
    "manifests": ["package.json","pyproject.toml"],
    "tests": ["tests/","src/module/test_*.py"],
    "ci": [".github/workflows/ci.yml"],
    "cloc": { ... }
  }
- Commands (example using jq):
  - python3 - <<'PY'
import json,os
root='.'
# minimal example: gather top level dirs
entries=[e for e in os.listdir('.') if os.path.isdir(e) and e!='.git']
# read earlier outputs
ext_counts={}
if os.path.exists('analysis_output/extension_counts.txt'):
  with open('analysis_output/extension_counts.txt') as f:
    for line in f:
      parts=line.strip().split()
      if len(parts)>=2:
        ext=parts[1]
        cnt=int(parts[0])
        ext_counts[ext]=cnt
out={'top_level_dirs':entries,'file_counts_by_extension':ext_counts}
with open('CODEBASE_TREE.json','w') as f:
  json.dump(out,f,indent=2)
PY

Task 8 — Produce human-readable summary CODEBASE_SUMMARY.md
- Description: Create a concise report describing findings, interesting locations, and recommended next steps.
- Template content (to be created at repo root):
  - Short description of repository layout
  - Languages and counts
  - Package managers / manifests found
  - Tests and CI files found
  - Suggested next steps (run tests, dependency scan, run linter, review large/undocumented folders)
- Commands:
  - cp analysis_output/summary_template.md CODEBASE_SUMMARY.md || echo "write file with echo" > CODEBASE_SUMMARY.md
- Example summary excerpt (fill in automatically with earlier outputs).

Task 9 — Commit the outputs and open a PR
- Description: Commit analysis artifacts to a branch and open a PR for review.
- Commands:
  - git checkout -b analysis/codebase-structure
  - git add CODEBASE_SUMMARY.md CODEBASE_TREE.json analysis_output || true
  - git commit -m "Add codebase structure analysis outputs"
  - git push --set-upstream origin analysis/codebase-structure
  - (PR creation is handled by the harness; do not call gh here.)

Task 10 — Optional deeper passes (separate tasks)
- Dependency vulnerability scan (Snyk/OSV): run with repo token and produce report.
- Run tests and CI locally in containers where applicable.
- Static analysis (linters, type checks).
- Generate dependency graph (dependency-cruiser, pipdeptree, go mod graph).

Files to create (paths)
- CODEBASE_SUMMARY.md (repo root)
- CODEBASE_TREE.json (repo root)
- analysis_output/ (multiple helper output files listed above)

Acceptance criteria

- CODEBASE_SUMMARY.md exists and documents: top-level dirs, primary languages, manifest files, test locations, CI config files, and recommended next steps.
- CODEBASE_TREE.json exists and contains a machine-readable summary with top_level_dirs and file_counts_by_extension at minimum.
- analysis_output/ contains the raw outputs used to generate the summary.

Notes for implementers

- Use ripgrep (rg) where available; otherwise fall back to grep/find.
- The plan intentionally avoids running tests or modifying source files beyond adding analysis artifacts in their own folder.
- Ensure all commands ignore the .git directory and other VCS internals.
