Implementation Plan: Task-by-task instructions to analyze the codebase structure

Overview

This plan defines a set of precise, self-contained tasks for an implementing agent to execute against the repository https://github.com/dreef3/multi-agent-harness-test-repo.git. Each task includes the exact commands to run, the files to inspect, and the expected outputs. The agent will collect the outputs and assemble the final report at reports/codebase-structure-analysis.md.

Preparation (one-time)

1. Environment
   - A machine or container with git, bash, tree, jq, bat or cat, ripgrep (rg), and basic UNIX tooling installed.

2. Workspace
   - Working directory: /workspace or similar. All outputs will be written under /workspace/reports.

Commands and Tasks

Task 1 — Clone repository

- Purpose: get a local copy to inspect
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
- Expected outputs: success status and repository files in repo/

Task 2 — Top-level inventory

- Purpose: list top-level files and directories
- Commands:
  - cd repo
  - ls -la
  - tree -a -L 2 > /workspace/reports/raw/tree_top_level.txt
  - head -n 200 README* 2>/dev/null > /workspace/reports/raw/README_head.txt || true
- Deliverables: reports/raw/tree_top_level.txt, README_head.txt

Task 3 — Language and dependency detection

- Purpose: detect primary languages and dependency manifests
- Commands:
  - rg "^.*$" --hidden --glob "!.git" -n --type-add 'packagejson:*.json' -S --no-ignore-vcs --files | sed -n '1,200p' > /workspace/reports/raw/all_files.txt
  - rg --hidden --files --glob "!node_modules" --glob "!.git" -g "**/package.json" -S || true
  - rg --hidden --files --glob "!venv" --glob "!.git" -g "**/requirements*.txt" -S || true
  - rg --hidden --files --glob "!vendor" --glob "!.git" -g "**/go.mod" -S || true
  - rg --hidden --files --glob "!.git" -g "**/pyproject.toml" -S || true
  - rg --hidden --files --glob "!.git" -g "**/Gemfile" -S || true
  - (Run checks and save known manifests):
    - [ -f package.json ] && jq '. | {name: .name, version: .version, scripts: .scripts}' package.json > /workspace/reports/raw/package_json_summary.json || true
    - [ -f pyproject.toml ] && head -n 50 pyproject.toml > /workspace/reports/raw/pyproject_head.txt || true
    - [ -f go.mod ] && head -n 50 go.mod > /workspace/reports/raw/go_mod_head.txt || true
- Deliverables: reports/raw/all_files.txt and manifests summaries

Task 4 — CI/CD configuration

- Purpose: find CI workflows and CI-related files
- Commands:
  - rg --hidden --files --glob "!node_modules" --glob "!.git" -g ".github/workflows/**" -S > /workspace/reports/raw/ci_workflow_files.txt || true
  - ls -la .github/workflows || true > /workspace/reports/raw/github_workflows_ls.txt
  - head -n 200 .github/workflows/* 2>/dev/null > /workspace/reports/raw/github_workflows_head.txt || true
- Deliverables: reports/raw/github_workflows_*.txt

Task 5 — Tests, e2e suites, and coverage

- Purpose: identify test directories and how tests are invoked
- Commands:
  - rg --hidden --files --glob "!node_modules" --glob "!.git" -g "**/test*/**/*" -S > /workspace/reports/raw/test_files_list.txt || true
  - rg --hidden --files --glob "!.git" -g "**/*_test.*" -S >> /workspace/reports/raw/test_files_list.txt || true
  - [ -f package.json ] && jq '.scripts' package.json > /workspace/reports/raw/package_json_scripts.json || true
  - rg "coverage" .github -S > /workspace/reports/raw/ci_coverage_mentions.txt || true
- Deliverables: test_files_list.txt, package_json_scripts.json

Task 6 — Docker and container files

- Purpose: capture Dockerfiles and docker-compose
- Commands:
  - rg --hidden --files --glob "!.git" -g "**/Dockerfile*" -S > /workspace/reports/raw/dockerfiles.txt || true
  - rg --hidden --files --glob "!.git" -g "**/docker-compose*" -S > /workspace/reports/raw/docker_compose_files.txt || true
  - head -n 200 $(cat /workspace/reports/raw/dockerfiles.txt 2>/dev/null) > /workspace/reports/raw/dockerfile_heads.txt 2>/dev/null || true
- Deliverables: dockerfiles.txt, dockerfile_heads.txt

Task 7 — Static analysis and lints (if present)

- Purpose: find linters and static analysis config
- Commands:
  - rg --hidden --files --glob "!.git" -g "**/.eslintrc*" -S > /workspace/reports/raw/eslint_files.txt || true
  - rg --hidden --files --glob "!.git" -g "**/.prettierrc*" -S > /workspace/reports/raw/prettier_files.txt || true
  - rg --hidden --files --glob "!.git" -g "**/golangci.yml" -S > /workspace/reports/raw/golangci_lint_files.txt || true
- Deliverables: lists of linter config files

Task 8 — Large files and binary detection

- Purpose: detect any unexpectedly large files
- Commands:
  - git rev-list --objects --all | sort -k2 > /workspace/reports/raw/git_objects.txt || true
  - find . -type f -size +5M -exec ls -lh {} \; | sort -k5 -hr > /workspace/reports/raw/large_files.txt || true
- Deliverables: large_files.txt

Task 9 — Documentation, license, contribution

- Purpose: capture README, LICENSE, CONTRIBUTING
- Commands:
  - [ -f README.md ] && head -n 200 README.md > /workspace/reports/raw/README_head.txt || true
  - [ -f LICENSE ] && head -n 200 LICENSE > /workspace/reports/raw/LICENSE_head.txt || true
  - [ -f CONTRIBUTING.md ] && head -n 200 CONTRIBUTING.md > /workspace/reports/raw/CONTRIBUTING_head.txt || true
- Deliverables: README_head.txt, LICENSE_head.txt

Task 10 — Assemble the final report

- Purpose: synthesize the collected raw outputs into a human-readable analysis
- Steps for the implementing agent:
  - Create /workspace/reports/codebase-structure-analysis.md
  - Sections to include in the report:
    - Executive summary (1-3 paragraphs)
    - Top-level inventory (list of top-level files/dirs)
    - Language and dependency summary (languages, key files, main frameworks)
    - CI and test summary (workflows, test commands, coverage)
    - Build and runtime artifacts (Dockerfiles, Makefile)
    - Documentation and governance (README, LICENSE)
    - Notable files (large files, binary blobs)
    - Actionable recommendations (CI fixes, missing docs, cleanups)
  - Include excerpts from raw files (e.g., package.json scripts, first 50 lines of Dockerfile, first workflow) in code blocks and attach raw file paths in an appendix.

Expected output file path

- /workspace/reports/codebase-structure-analysis.md
- /workspace/reports/raw/* (all raw outputs)

Acceptance criteria

- The final markdown report exists at the path above and includes the sections listed.
- All raw files referenced in the report exist under /workspace/reports/raw/ and contain the captured command outputs.

Estimated effort

- Single implementing agent: ~20–45 minutes to run commands and assemble report (depends on repo size).

Notes and caveats

- The implementing agent must have network access to clone the repository.
- If the repository is private or restricted, the cloning step will fail; the agent should capture git clone errors and include them in reports/raw/clone_error.txt.
- The plan deliberately captures command outputs to keep the analysis auditable.
