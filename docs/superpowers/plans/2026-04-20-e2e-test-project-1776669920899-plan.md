Implementation plan: Task-by-task instructions to analyze the repository structure (repo id: 60c4e19b-4c50-4a6c-9b7f-9ecedfb2d288)

Overview
- This plan is a set of self-contained tasks appropriate for sub-agents or engineers. Each task describes exact commands to run, files to inspect, expected outputs, and acceptance criteria. No changes to repository code are required.
- Artifacts produced: CODEBASE_REPORT.json, README_ANALYSIS.md, FILE_TREE.txt, and an optional dependency graph (DEPENDENCY_GRAPH.*).

Environment assumptions
- Unix-like shell available (bash), git installed, cloc, tree, ripgrep (rg), jq.
- If cloc or tree not available, use fallback commands (find | xargs wc).

Tasks

Task 01 — Clone repository
- Purpose: get a working copy for offline analysis.
- Commands:
  git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  cd repo
- Outputs: local working tree at ./repo
- Accept when clone completes with exit code 0.

Task 02 — Shallow top-level inventory (fast)
- Purpose: capture top-level layout and quick stats.
- Commands (from repo root):
  ls -la
  tree -a -L 2 > FILE_TREE.txt  # if tree available
  find . -maxdepth 2 -type f | sed 's|^./||' | sort > FILE_LIST_LEVEL2.txt
  git rev-parse --abbrev-ref HEAD > GIT_BRANCH.txt
- Files to check manually: README*, LICENSE*, CONTRIBUTING*, CHANGES*, CODE_OF_CONDUCT*, .github/
- Outputs: FILE_TREE.txt, FILE_LIST_LEVEL2.txt, GIT_BRANCH.txt

Task 03 — Language and size breakdown
- Purpose: determine primary languages and source size.
- Commands:
  cloc --json . > cloc.json
  # Fallback if cloc not installed:
  find . -type f -name '*.py' -o -name '*.js' -o -name '*.ts' -o -name '*.go' -o -name '*.java' | wc -l
- Outputs: cloc.json (or manual counts)

Task 04 — Locate dependency and build files
- Purpose: find package managers and build systems.
- Commands:
  rg --hidden --files --glob '!node_modules' 'package.json|pyproject.toml|requirements.txt|Pipfile|setup.py|go.mod|Cargo.toml|pom.xml|build.gradle|Makefile|Dockerfile|composer.json' || true
  # or use find to search filenames explicitly
- Expected list: any files matching above patterns. Capture into DEP_FILES.txt

Task 05 — CI and workflow inspection
- Purpose: enumerate CI systems and workflow steps.
- Commands:
  ls -la .github/workflows || true
  rg --hidden --files --glob '.github/**' || true
  # For GitHub Actions, read each workflow YAML and extract job names and steps
  for f in .github/workflows/*.yml .github/workflows/*.yaml; do echo "--- $f"; sed -n '1,200p' "$f"; done > GITHUB_WORKFLOWS_RAW.txt
- Outputs: GITHUB_WORKFLOWS_RAW.txt

Task 06 — Test locations and scripts
- Purpose: find tests and test commands.
- Commands:
  rg --hidden --files --glob '!node_modules' 'test(s)?' || true
  rg --hidden 'pytest|unittest|mocha|jest|ava|vitest|go test|mvn test|gradle test' || true
  # Check package.json scripts
  if [ -f package.json ]; then jq '.scripts' package.json || true; fi > PACKAGE_SCRIPTS.json || true
- Outputs: list of test directories, PACKAGE_SCRIPTS.json

Task 07 — Docker and container artifacts
- Purpose: find Dockerfiles and image build contexts.
- Commands:
  rg --hidden --files 'Dockerfile' || true
  rg --hidden 'FROM ' --glob 'Dockerfile' || true > DOCKERFILES_CONTENT.txt
- Outputs: list and contents of Dockerfiles

Task 08 — Locate large and binary files
- Purpose: find files >5MB and large directories.
- Commands:
  find . -type f -size +5M -exec ls -lh {} \; | sort -k5 -h > LARGE_FILES.txt
  du -sh * | sort -h > TOP_DIR_SIZES.txt
- Outputs: LARGE_FILES.txt, TOP_DIR_SIZES.txt

Task 09 — Secrets and risky artifacts scan
- Purpose: quick check for accidental secrets (API keys, private certs).
- Commands (basic):
  rg --hidden "(AKIA|AIza|BEGIN RSA PRIVATE KEY|PRIVATE KEY|password=|secret=|API_KEY)" || true > SECRETS_POTENTIAL.txt
- Note: This is a heuristic; flag results for human review.

Task 10 — Produce CODEBASE_REPORT.json
- Purpose: aggregate all above outputs into one machine-readable JSON file.
- Suggested structure (example):
{
  "repo": "dreef3/multi-agent-harness-test-repo",
  "branch": "main",
  "top_level": ["README.md","package.json","src/"],
  "languages": { /* from cloc.json */ },
  "dependency_files": ["package.json","requirements.txt"],
  "ci": [".github/workflows/ci.yml"],
  "tests": { "locations": ["tests/"], "commands": ["npm test"] },
  "dockerfiles": ["Dockerfile"],
  "large_files": ["path/to/file 20MB"],
  "possible_secrets_count": 2
}

- Implementation note: assemble JSON with jq or a small script. Example jq pipeline:
  jq -n --arg repo "dreef3/multi-agent-harness-test-repo" \
    --slurpfile cloc cloc.json \
    '{repo:$repo, languages:$cloc[0]}' > CODEBASE_REPORT.json

Task 11 — Produce README_ANALYSIS.md (human summary)
- Purpose: write a concise summary with findings, quickstart, and recommendations.
- Sections to include:
  - Quick facts (languages, lines of code, size)
  - Top-level layout (short tree)
  - How to build/test locally (commands discovered)
  - CI summary
  - Risks and recommendations (missing docs, large files, secrets)
  - Next actions (e.g., run tests in CI, add contribution docs)

Task 12 — Optional: dependency graph and visualization
- Purpose: produce a graph of package dependencies (if package manager recognized).
- Commands examples:
  - For JS: npx madge --image dependency-graph.png src || true
  - For Python: pip install pydeps; pydeps --noshow -o pydeps.svg package_name || true
- Output: DEPENDENCY_GRAPH.png/.svg

Task 13 — Validation and acceptance
- Purpose: ensure all artifacts created and consistent.
- Checks:
  - CODEBASE_REPORT.json exists and parses with jq: jq . CODEBASE_REPORT.json
  - README_ANALYSIS.md exists and is < 2000 lines
  - FILE_TREE.txt exists and contains top-level dirs
