Implementation Plan: Analyze codebase structure for E2E Test Repo 0 (repo id: 149771ae-97f8-4251-82a5-a539eddd175a)

Overview
- This plan breaks the analysis into discrete tasks. Each task is self-contained and runnable by a sub-agent. The plan lists exact file paths to read and shell commands to run. The output artifacts are: summary_report.md and detailed_report.md placed in /reports/ in the repository root.

Artifacts
- /reports/summary_report.md — one-page summary: languages, main frameworks, top-level layout, CI present, tests present, quick recommendations.
- /reports/detailed_report.md — full listing with file trees, key file excerpts (first 50 lines), dependency lists, and findings per directory.

Tasks

Task 1 — Top-level listing
- Purpose: collect top-level files and directories.
- Commands:
  - ls -la
  - find . -maxdepth 2 -mindepth 1 -type d -printf "%P\n" | sort
  - find . -maxdepth 1 -type f -printf "%P\n" | sort
- Files to read (if present): README.md, CONTRIBUTING.md, CHANGELOG.md, LICENSE, CODE_OF_CONDUCT.md
- Output: /reports/top_level.txt (one-per-line list). Include presence/absence of README and LICENSE.

Task 2 — Detect languages and build systems
- Purpose: infer languages from manifest files.
- Files to look for (exact paths relative to repo root):
  - package.json, yarn.lock, pnpm-lock.yaml
  - pyproject.toml, requirements.txt, setup.py
  - go.mod, Gopkg.toml
  - Cargo.toml
  - pom.xml, build.gradle, build.gradle.kts
  - Gemfile
  - composer.json
- Commands: for each manifest found, output first 50 lines. Example commands:
  - [ -f package.json ] && head -n 50 package.json > /reports/package.json.head
- Output: /reports/detected_manifests.txt (list of detected manifest files and extracted snippets)

Task 3 — Source tree and code layout
- Purpose: locate source directories and test directories
- Directories to check (if present): src/, lib/, pkg/, cmd/, internal/, app/, server/, client/, web/, frontend/, backend/, tests/, test/
- Commands:
  - for d in src lib pkg cmd internal app server client web frontend backend tests test; do [ -d "$d" ] && find "$d" -maxdepth 2 -type f -printf "%P\n" | sed "s|^|$d/|" >> /reports/source_tree.txt; done
- Output: /reports/source_tree.txt

Task 4 — CI/CD and automation
- Purpose: detect CI config and common automation files
- Paths to check:
  - .github/workflows/*.yml
  - .gitlab-ci.yml
  - .circleci/config.yml
  - Jenkinsfile
  - Dockerfile (repo root and any Dockerfile in subdirs)
  - Makefile
- Commands: list files and head -n 200 their contents into /reports/ci_files/
- Output: /reports/ci_summary.txt and copies in /reports/ci_files/

Task 5 — Tests and coverage
- Purpose: detect tests and coverage config
- Patterns to search:
  - files matching *_test.go, **/tests/, **/__tests__/, pytest.ini, jest.config.js, coverage/.coverage, .nycrc
- Commands:
  - rg --hidden --glob '!node_modules' "_test\.go$|__tests__|pytest.ini|jest.config|coverage" -n || true
- Output: /reports/tests_summary.txt and file list /reports/test_files.txt

Task 6 — Dependency extraction
- Purpose: extract dependency lists for major languages
- For package.json: parse dependencies and devDependencies (first 200 lines okay)
- For pyproject.toml or requirements.txt: list dependencies
- For go.mod: list module and require lines
- Commands/examples:
  - [ -f package.json ] && jq '.dependencies, .devDependencies' package.json > /reports/package_deps.json || true
  - [ -f requirements.txt ] && cat requirements.txt > /reports/requirements.txt || true
  - [ -f go.mod ] && sed -n '1,200p' go.mod > /reports/go.mod.head || true
- Output: /reports/dependencies/

Task 7 — Documentation and metadata
- Purpose: extract README and license snippets and check contribution guidelines
- Files: README.md, docs/*, LICENSE, CODE_OF_CONDUCT.md, CONTRIBUTING.md
- Commands: head -n 200 README.md > /reports/README.head
- Output: /reports/docs_summary.txt

Task 8 — Generate summary and recommendations
- Purpose: synthesize collected artifacts into final reports
- Inputs: outputs from Tasks 1–7
- Produce /reports/summary_report.md containing:
  - One-paragraph repo description (inferred)
  - Languages and build systems
  - CI presence
  - Tests status
  - Key recommendations (e.g., add CI, add tests, add README improvements, lockfiles)
- Produce /reports/detailed_report.md including:
  - Top-level listing
  - Manifest snippets
  - Source tree
  - CI configs (snippets)
  - Tests listing
  - Dependency lists
  - Exact file excerpts collected

Task 9 — Quality checks and optional enhancements
- Lint the reports for clarity.
- Optionally create a JSON machine-readable summary at /reports/summary.json with fields:
  - repo_id, languages[], frameworks[], has_ci (bool), has_tests (bool), has_dockerfile (bool), has_readme (bool), license_present (bool)

Execution notes for implementer agents
- Use standard Unix commands (ls, find, head, sed, rg, jq) and ensure commands tolerate missing files (use [ -f ] checks).
- Place all outputs under /reports/ and ensure files are overwritten on rerun.
- Do not run dependency installs or tests.
- When extracting snippets, limit output lengths to keep reports compact (e.g., first 200 lines for configs).

Deliver the two report files and a short PR description: "Add repository structure analysis reports for E2E Test Repo 0."
