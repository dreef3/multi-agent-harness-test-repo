Implementation Plan: Tasks to Analyze the E2E Test Repo Structure

Overview
This plan defines a set of self-contained tasks for an executor (sub-agent or developer) to run locally or in CI to analyze the repository structure and produce the agreed deliverables. Each task includes exact commands, expected outputs (file paths), and acceptance criteria. The plan purposely avoids modifying source code unless writing to reports/analysis/ in a feature branch.

Repository
- URL: https://github.com/dreef3/multi-agent-harness-test-repo.git
- Default branch: main

Environment prerequisites
- git, bash, find, sed, awk
- ripgrep (rg) recommended
- cloc (recommended) for language/line counts
- jq for JSON manipulation
- graphviz (dot) optional for module graph
- Node.js and npm, Python, Go tools as needed to inspect manifests (optional)

Top-level tasks

Task 1 — Clone repository and create workspace
- Purpose: get a local copy and create a workspace branch if you intend to commit reports.
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git
  - cd multi-agent-harness-test-repo
  - git checkout -b analysis/repo-structure-report   # optional if committing
- Output: local repo at ./multi-agent-harness-test-repo
- Time: ~30s - 2m depending on repo size

Task 2 — Generate raw file listing
- Purpose: enumerate all files to start analysis
- Commands:
  - mkdir -p reports/analysis
  - find . -type f -not -path './.git/*' -print > reports/analysis/raw-file-list.txt
- Output: reports/analysis/raw-file-list.txt
- Acceptance: file exists and lists repo files

Task 3 — Compute language statistics and file counts
- Purpose: provide summary counts by language and size metrics
- Commands (recommended with cloc):
  - cloc --json . --exclude-dir=.git,node_modules > reports/analysis/cloc.json
  - jq . reports/analysis/cloc.json > reports/analysis/cloc-pretty.json
- Fallback without cloc:
  - rg --files | sed 's|^./||' | awk -F. '{print $NF}' | sort | uniq -c | sort -rn > reports/analysis/file-counts-by-ext.txt
- Outputs: reports/analysis/cloc.json (preferred) or file-counts-by-ext.txt
- Acceptance: JSON or text present with counts

Task 4 — Identify dependency manifests, package managers, and lockfiles
- Purpose: list dependency manifests (package.json, requirements.txt, Pipfile, go.mod, Gemfile, Cargo.toml, etc.)
- Commands:
  - rg -n "(^|/)package.json$|Pipfile$|requirements.txt$|go.mod$|Cargo.toml$|Gemfile$" --hidden --no-ignore -S --no-heading --line-number --glob '!.git' > reports/analysis/manifests.txt || true
  - # also explicitly check common files
  - for f in package.json requirements.txt Pipfile pyproject.toml go.mod Gemfile Cargo.toml; do [ -f "$f" ] && echo "$f" >> reports/analysis/manifests.txt; done
- Output: reports/analysis/manifests.txt
- Acceptance: file exists and lists manifests found

Task 5 — Detect CI/workflow files and important config
- Purpose: find CI config and top-level docs
- Commands:
  - rg -n "^" --hidden --no-ignore -S --glob '.github/**' .github || true
  - find .github -maxdepth 3 -type f -print > reports/analysis/ci-files.txt || true
  - for f in README.md README.rst LICENSE CONTRIBUTING.md CODE_OF_CONDUCT.md; do [ -f "$f" ] && echo "$f" >> reports/analysis/important-docs.txt; done
- Output: reports/analysis/ci-files.txt and important-docs.txt
- Acceptance: lists of discovered CI and doc files

Task 6 — Discover tests, test frameworks, and test directories
- Purpose: locate test folders and test runners
- Commands:
  - rg -n "(^|/)test(s)?/|pytest|unittest|mocha|jest|ava|go test|pytest.ini|jest.config|tox.ini|pytest\." -S --hidden --no-ignore || true > reports/analysis/test-hints.txt
  - find . -type d -name "test" -o -name "tests" > reports/analysis/test-dirs.txt || true
- Output: test-hints.txt, test-dirs.txt
- Acceptance: file lists test-related files/dirs

Task 7 — Parse top-level package manifests for main info
- Purpose: extract name, version, scripts, and test commands from package.json or pyproject.toml
- Commands (example for Node):
  - if [ -f package.json ]; then jq '{name,version,scripts,dependencies,devDependencies}' package.json > reports/analysis/package-summary.json; fi
  - if [ -f pyproject.toml ]; then python -c "import toml,sys;print(toml.load('pyproject.toml'))" > reports/analysis/pyproject-raw.txt || true; fi
- Output: package-summary.json or similar
- Acceptance: summary file with key fields

Task 8 — Summarize CI workflows (parse YAML for matrix, languages, steps)
- Purpose: compactly show what CI runs and what jobs exist
- Commands:
  - rg "^name:|steps:|runs-on:|matrix:" -n .github -S --hidden --no-ignore || true > reports/analysis/ci-hints.txt
  - # Optionally, create a small parser script to extract job names and runners
- Output: ci-hints.txt
- Acceptance: human-readable hints about workflows

Task 9 — Build summary JSON and Markdown report
- Purpose: consolidate outputs into reports/analysis/summary.json and summary.md
- Method: combine previous outputs. Example schema for summary.json:
  {
    "repo": "dreef3/multi-agent-harness-test-repo",
    "top_level_dirs": ["src","tests",".github",...],
    "languages": {"Python": 1200, "JS": 800},
    "manifests": ["package.json"],
    "ci_files": [".github/workflows/ci.yml"],
    "test_dirs": ["tests"],
    "noteworthy_files": ["README.md","LICENSE"]
  }
- Commands (example):
  - # Create a minimal summary by grepping earlier outputs
  - python - <<'PY'
import json,os
root='.'
summary={}
summary['repo']='dreef3/multi-agent-harness-test-repo'
summary['top_level_dirs']=[d for d in os.listdir('.') if os.path.isdir(d) and d!='.git']
# include cloc if exists
import pathlib
if pathlib.Path('reports/analysis/cloc.json').exists():
    summary['cloc']=json.load(open('reports/analysis/cloc.json'))
# other fields from files
for fname in ['manifests.txt','ci-files.txt','test-dirs.txt','important-docs.txt']:
    p='reports/analysis/'+fname
    if os.path.exists(p):
        summary[fname.replace('.txt','')]=[l.strip() for l in open(p).read().splitlines() if l.strip()]
open('reports/analysis/summary.json','w').write(json.dumps(summary,indent=2))
open('reports/analysis/summary.md','w').write('# Repo Structure Summary\n\nSee summary.json for structured output.\n')
PY
- Outputs: reports/analysis/summary.json, reports/analysis/summary.md
- Acceptance: both files present and readable

Task 10 — (Optional) Generate module dependency graph
- Purpose: visualize relationships between modules (e.g., for JS/TS or Python packages)
- Commands (example for Node):
  - npx madge --json src > reports/analysis/madge.json
  - jq -r 'to_entries|map("\(.key) -> \(.value[])")|.[]' reports/analysis/madge.json | dot -Tsvg -o reports/analysis/module-graph.svg
- Output: reports/analysis/module-graph.svg
- Acceptance: SVG produced or step skipped if not applicable

Task 11 — (Optional) Run linters/tests (with caution)
- Purpose: gather test results and simple quality metrics when environment allows
- Commands (example):
  - npm ci && npm test || true
  - pytest -q || true
  - go test ./... || true
- Note: these commands may modify local state (install deps). Only run if allowed.
- Outputs: reports/analysis/test-output.txt
- Acceptance: test outputs captured; this task is optional and gated

Task 12 — Commit results to branch and open PR (optional)
- Purpose: submit findings as a PR to the original repo
- Commands:
  - git add reports/analysis
  - git commit -m "Add repository structure analysis report"
  - git push --set-upstream origin analysis/repo-structure-report
- Then open a PR through GitHub UI or gha CLI
- Output: PR URL
