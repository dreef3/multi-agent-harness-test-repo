Implementation Plan: Task-by-task plan to analyze the codebase structure

Context
- Repository URL: https://github.com/dreef3/multi-agent-harness-test-repo.git
- Repository UUID: 6c093857-2fd1-4758-8cbc-1a563b139721
- Default branch: main
- Constraint: This planning agent will not clone the repo. The following plan defines tasks for implementation agents to run.

Overview
- Create a small automated analysis workflow (script-based) that clones the repo with depth=1, enumerates files, detects languages, parses common manifests, and writes two output files under the repo: analysis/structure-report.md and analysis/structure-manifest.json. All outputs should be committed to a branch named analysis/structure-analysis and a PR opened against main.

Deliverables
- analysis/structure-report.md (human readable)
- analysis/structure-manifest.json (structured JSON)
- A short log file: analysis/analysis-log.txt containing commands run and any notable warnings/errors

Tasks (each task is self-contained and can be dispatched to a worker)

Task 1 — Clone repository and prepare workspace
- Purpose: Acquire a local copy of the repo for inspection.
- Inputs: repository URL, default branch (main)
- Commands (exact):
  - git clone --depth=1 https://github.com/dreef3/multi-agent-harness-test-repo.git repo || exit 1
  - cd repo
  - git checkout main || true
- Outputs: local workspace at ./repo
- Failure modes: network or permission error -> exit with non-zero and write analysis/analysis-log.txt with error details.

Task 2 — Enumerate top-level files and directory tree
- Purpose: Produce a concise listing of top-level files and full tree up to depth 4.
- Commands:
  - cd repo
  - git ls-files > analysis/git-ls-files.txt
  - (if tree installed) tree -L 4 -a -I '.git' > analysis/tree.txt || find . -maxdepth 4 -print > analysis/tree.txt
- Output files: analysis/git-ls-files.txt, analysis/tree.txt

Task 3 — Detect languages and code metrics
- Purpose: Determine primary languages and approximate LOC per language.
- Commands:
  - cd repo
  - if command -v cloc >/dev/null 2>&1; then cloc --json . > analysis/cloc.json; else echo "cloc not installed" > analysis/cloc.txt; fi
  - Use fallback: detect by presence of manifest files: ls package.json pyproject.toml go.mod Cargo.toml requirements.txt Gemfile 2>/dev/null || true
- Output: analysis/cloc.json or analysis/cloc.txt

Task 4 — Identify package managers, build files, and key manifests
- Purpose: Parse package/dependency and build config files.
- Commands (examples):
  - cd repo
  - for f in package.json pyproject.toml setup.py requirements.txt Pipfile poetry.lock pyproject.toml; do [ -f "$f" ] && echo "$f" >> analysis/detected-manifests.txt; done
  - for f in go.mod Cargo.toml Gemfile composer.json Mix.exs; do [ -f "$f" ] && echo "$f" >> analysis/detected-manifests.txt; done
  - (If package.json exists) jq '{name: .name, scripts: .scripts, dependencies: .dependencies, devDependencies: .devDependencies}' package.json > analysis/package-json-summary.json 2>/dev/null || true
- Outputs: analysis/detected-manifests.txt, analysis/package-json-summary.json

Task 5 — Find CI/CD workflows and Dockerfiles
- Purpose: Locate common CI configs and container build files.
- Commands:
  - cd repo
  - find .github -maxdepth 3 -type f -name '*.yml' -o -name '*.yaml' > analysis/ci-workflows.txt 2>/dev/null || true
  - find . -type f -name 'Dockerfile*' > analysis/dockerfiles.txt || true
- Outputs: analysis/ci-workflows.txt, analysis/dockerfiles.txt

Task 6 — Locate tests, test frameworks, and coverage configs
- Purpose: Identify test directories and framework indicators.
- Commands:
  - cd repo
  - find . -type d -name 'test' -o -name 'tests' -o -name '__tests__' > analysis/test-dirs.txt || true
  - grep -R --line-number "pytest\|unittest\|mocha\|jest\|go test\|cargo test" -n -- . | head -n 200 > analysis/test-snippets.txt || true
- Outputs: analysis/test-dirs.txt, analysis/test-snippets.txt

Task 7 — Parse docs and README
- Purpose: Extract README summary (first 200 lines) and find docs/ or docs/**/*.md
- Commands:
  - cd repo
  - head -n 200 README.md > analysis/README-head.md 2>/dev/null || true
  - find docs -type f -name '*.md' > analysis/docs-list.txt 2>/dev/null || true
- Outputs: analysis/README-head.md, analysis/docs-list.txt

Task 8 — License detection
- Purpose: Identify LICENSE or COPYING files and extract SPDX if present.
- Commands:
  - cd repo
  - for f in LICENSE LICENSE.md COPYING; do [ -f "$f" ] && echo "$f" >> analysis/licenses.txt; done
  - (If license file found) head -n 50 LICENSE >> analysis/license-preview.txt || true
- Outputs: analysis/licenses.txt, analysis/license-preview.txt

Task 9 — Produce structured manifest JSON and human report
- Purpose: Consolidate findings into two outputs.
- Implementation notes: Use a small script (bash + jq or Python) to gather the prior analysis files and produce structure-manifest.json and structure-report.md.
- Example Python pseudo-commands (to be run in repo):
  - python3 - <<'PY'
import json,os
root='.'
manifest={}
manifest['top_level']=sorted([p for p in os.listdir('.') if p!='.git'])
# read cloc if exists
try:
    manifest['cloc']=json.load(open('analysis/cloc.json'))
except Exception:
    manifest['cloc']='not available'
# read detected manifests
manifest['detected_manifests']=open('analysis/detected-manifests.txt').read().splitlines() if os.path.exists('analysis/detected-manifests.txt') else []
# read ci
manifest['ci_files']=open('analysis/ci-workflows.txt').read().splitlines() if os.path.exists('analysis/ci-workflows.txt') else []
json.dump(manifest,open('analysis/structure-manifest.json','w'),indent=2)
# simple report
report = []
report.append('# Repository Structure Report')
report.append('\nTop-level files and dirs:\n')
report.append('\n'.join(manifest['top_level']))
report.append('\nDetected manifests:')
report.append('\n'.join(manifest['detected_manifests']))
open('analysis/structure-report.md','w').write('\n\n'.join(report))
PY

- Outputs: analysis/structure-manifest.json, analysis/structure-report.md

Notes for implementers
