Tasks (each task is self-contained and should be runnable on its own):

Task 1 — Clone repository and create analysis branch
- Purpose: Get a working copy and create branch for analysis artifacts.
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
  - git checkout -b analysis/codebase-structure
- Outputs: local repo clone ready for inspection.
- Expected runtime: < 2 min

Task 2 — Write raw directory tree
- Purpose: Provide full file/directory listing so humans can quickly scan repo layout.
- Commands:
  - cd repo
  - (if available) tree -a -I 'node_modules|venv|.venv|.git' > analysis/tree.txt || (find . -not -path './.git/*' -print | sed 's|^./||' > analysis/tree.txt)
- Output files:
  - analysis/tree.txt
- Notes: Exclude large common dependency directories for readability. If tree not installed, use find as fallback.

Task 3 — Detect languages and manifests
- Purpose: Identify primary languages and package manifests.
- Commands (run from repo root):
  - echo '{' > analysis/metadata.json && echo '  "manifests": {}' >> analysis/metadata.json && echo '}' >> analysis/metadata.json  # placeholder
  - Check for known manifests and copy them to analysis/manifests/:
    - if [ -f package.json ]; then mkdir -p analysis/manifests && cp package.json analysis/manifests/package.json; fi
    - if [ -f yarn.lock ]; then cp yarn.lock analysis/manifests/; fi
    - if [ -f pnpm-lock.yaml ]; then cp pnpm-lock.yaml analysis/manifests/; fi
    - if [ -f pyproject.toml ]; then cp pyproject.toml analysis/manifests/; fi
    - if [ -f requirements.txt ]; then cp requirements.txt analysis/manifests/; fi
    - if [ -f Pipfile ]; then cp Pipfile analysis/manifests/; fi
    - if [ -f go.mod ]; then cp go.mod analysis/manifests/; fi
    - if [ -f Cargo.toml ]; then cp Cargo.toml analysis/manifests/; fi
    - if [ -f Gemfile ]; then cp Gemfile analysis/manifests/; fi
    - if [ -f composer.json ]; then cp composer.json analysis/manifests/; fi
- Also run simple language detection:
  - rg --hidden --files --glob '!.git' --iglob 'package.json' --line-number || true
  - Use "file" and simple heuristics to count top-level source files, e.g., count *.js, *.ts, *.py, *.go, *.rs, *.java, *.cs using find | wc -l
- Output files:
  - analysis/manifests/* (copies of detected manifests)
  - analysis/metadata.json updated: include detected languages and manifest list.
- Notes: For package.json, later tasks will parse dependencies.

Task 4 — Copy important top-level docs and CI configs
- Purpose: Preserve README, LICENSE, CONTRIBUTING, and CI workflows for context.
- Commands:
  - mkdir -p analysis/copies
  - for f in README.md README rEADME.* LICENSE LICENSE.md CONTRIBUTING.md CODE_OF_CONDUCT.md; do if [ -f "$f" ]; then cp "$f" analysis/copies/; fi; done
  - mkdir -p analysis/ci && if [ -d .github/workflows ]; then cp -r .github/workflows analysis/ci/; fi
- Outputs:
  - analysis/copies/*
  - analysis/ci/*

Task 5 — Parse manifests into normalized dependency lists
- Purpose: Produce machine-readable dependency lists for quick review.
- Commands/examples:
  - For package.json (Node): jq '{name: .name, version: .version, dependencies: .dependencies, devDependencies: .devDependencies}' package.json > analysis/deps/package.json.metadata.json
  - For pyproject.toml: use tomlq or python -c to parse and write JSON: python - <<'PY'
import toml, json
p=toml.load('pyproject.toml')
print(json.dumps(p.get('tool',{}), indent=2))
PY
  - For go.mod: go list -m all > analysis/deps/go-mod-list.txt
  - For Cargo.toml: cargo metadata --format-version 1 --no-deps > analysis/deps/cargo-metadata.json
- Output files:
  - analysis/deps/* (one normalized file per detected manifest)
- Notes: If required tooling isn't available in runner, fallback to copying raw manifests.

Task 6 — Locate tests and run basic test discovery
- Purpose: Identify test frameworks and test directories/files.
- Commands:
  - rg --hidden --files --glob '!.git' -e '__tests__' -e 'test_' -e '.spec.' -n > analysis/tests/test_files.txt || true
  - List common test commands from package.json scripts (if package.json present): jq '.scripts' package.json > analysis/tests/package-scripts.json || true
- Output files:
  - analysis/tests/test_files.txt
  - analysis/tests/package-scripts.json
- Notes: Do not run tests by default — only discover.

Task 7 — CI/workflow inspection
- Purpose: Summarize CI workflows and identify triggers and tasks.
- Commands:
  - If .github/workflows exists, for f in .github/workflows/*.yml; do echo "--- $f"; yq eval '.' $f > analysis/ci/$(basename $f).yaml || cp $f analysis/ci/; done
  - Create analysis/ci/summary.md listing workflow names, triggers (push, pull_request), and jobs.
- Output files:
  - analysis/ci/* (copied workflows)
  - analysis/ci/summary.md
- Notes: Use yq if available; otherwise copy files and include the raw YAML.

Task 8 — License, contributors, and notable files summary
- Purpose: Identify license and contributor points-of-contact.
- Commands:
  - if [ -f LICENSE ]; then head -n 20 LICENSE > analysis/license_head.txt; fi
  - git log --format='%aN <%aE>' | sort -u > analysis/contributors.txt || true
- Output files:
  - analysis/license_head.txt
  - analysis/contributors.txt

Task 9 — Produce human-readable summary (structure.md)
- Purpose: Consolidate metadata.json, tree.txt, manifests, CI summary, test discovery into a short report.
- Content (structure.md):
  - One-paragraph repo summary (languages, top-level directories).
  - Bullet list: manifests found, CI workflows, test locations, Dockerfiles, build/packaging tools.
  - Short list of recommendations: run dependency scan, enable CI badges, run unit tests, check for large vendored directories.
- Example template (fill in fields programmatically):
  - Repository: <repo URL>
  - Primary languages: Node.js (package.json), Python (pyproject.toml), etc.
  - Top-level dirs: src/, lib/, tests/, docs/, scripts/
  - Manifests found: analysis/manifests/*
  - CI workflows: list from analysis/ci/summary.md
  - Tests: list from analysis/tests/test_files.txt
  - Recommendations: ...
