Implementation Plan: Tasks to analyze repository structure for E2E Test Repo 1 (repo id: ea078373-bcf2-4333-929b-e18ee8d78cb6)

Overview
- The plan is a sequence of self-contained tasks. Each task reads files and produces outputs under the `report/` directory. Tasks are independent where possible and include explicit commands and expected outputs. Use the repository root as the working directory.

Common constraints for all tasks
- Read-only operations only. Use `git` or filesystem listing commands; do not run code from the repo or install untrusted dependencies.
- Truncate file reads larger than 200 KB and skip binary files.
- Redact potential secrets by replacing long base64 blocks, RSA private key bodies, or values that match common secret patterns with "<REDACTED>" in outputs.

Task 1 — Root snapshot
- Purpose: capture top-level files and directories
- Commands:
  - ls -la
  - find . -maxdepth 2 -type f -printf "%P\n" | sort
  - For large repos: find . -maxdepth 1 -mindepth 1 -printf "%P\n" | sort
- Expected outputs:
  - report/root_files.txt (plain list)
  - report/root_tree.json (simple JSON array of entries)
- Files to inspect explicitly if present: README.md, CONTRIBUTING.md, LICENSE, .gitignore, CODE_OF_CONDUCT.md, .github/

Task 2 — Language and manifest detection
- Purpose: determine primary languages and package manifests
- Commands (try each):
  - test -f package.json && echo "node" || true
  - test -f pyproject.toml && echo "python-poetry" || true
  - test -f requirements.txt && echo "python-reqs" || true
  - test -f setup.py && echo "python-setup" || true
  - test -f go.mod && echo "go" || true
  - test -f Cargo.toml && echo "rust" || true
  - test -f Gemfile && echo "ruby" || true
  - test -f composer.json && echo "php" || true
  - test -f build.gradle && echo "java-gradle" || true
  - test -f pom.xml && echo "java-maven" || true
- Expected outputs:
  - report/languages.txt (one-per-line detections)
  - report/manifests/* — copy of each manifest truncated to 100 KB

Task 3 — Dependency and lockfile inventory
- Purpose: identify lockfiles and list top-level dependencies
- Files to look for: package-lock.json, yarn.lock, pnpm-lock.yaml, Pipfile.lock, poetry.lock, go.sum, Cargo.lock
- Commands:
  - For each lockfile present, output the top 50 lines and a parsed summary where possible (e.g., jq for package-lock.json)
  - Example: test -f package-lock.json && jq '.dependencies | keys' package-lock.json > report/manifests/package-lock-deps.json || true
- Expected outputs: report/manifests/*-lock-summary.txt

Task 4 — Source layout and entry points
- Purpose: find source directories, binaries, and likely entrypoints
- Look for directories: src/, lib/, cmd/, internal/, apps/, packages/, services/, server/, client/, agents/
- Commands:
  - find src lib cmd packages services -maxdepth 2 -type f -name '*.js' -o -name '*.ts' -o -name '*.py' -o -name '*.go' -o -name '*.rs' -print | sed 's|^./||' > report/source_files.txt
  - Search for common entrypoint filenames: main.py, index.js, server.js, app.js, bin/*
- Expected outputs: report/source_files.txt, report/entrypoints.txt

Task 5 — Tests and E2E locations
- Purpose: locate tests, test frameworks, and any e2e/functional test suites
- Directories to check: tests/, test/, __tests__/, e2e/, integration_tests/, spec/
- Commands:
  - find tests test e2e -maxdepth 3 -type f -name '*test*.py' -o -name '*test*.js' -o -name '*_spec.rb' -print > report/tests_files.txt
  - Parse package.json for test script: jq -r '.scripts.test // empty' package.json > report/test_script.txt || true
- Expected outputs: report/tests_files.txt, report/test_script.txt

Task 6 — CI / workflows
- Purpose: find CI configs and identify CI actions
- Paths: .github/workflows/*.yml, .gitlab-ci.yml, .circleci/config.yml, etc.
- Commands:
  - find .github/workflows -maxdepth 1 -type f -name '*.yml' -print -exec sed -n '1,200p' {} \; > report/ci_workflows.txt
  - test -f .gitlab-ci.yml && sed -n '1,200p' .gitlab-ci.yml >> report/ci_workflows.txt || true
- Expected outputs: report/ci_workflows.txt with list and top contents

Task 7 — Docker and containerization
- Purpose: find Dockerfiles and compose manifests
- Files: Dockerfile, docker-compose.yml, docker/**
- Commands:
  - find . -type f -iname 'Dockerfile' -print -exec sed -n '1,200p' {} \; > report/dockerfiles.txt || true
  - test -f docker-compose.yml && sed -n '1,200p' docker-compose.yml >> report/dockerfiles.txt || true

Task 8 — Scripts and tooling
- Purpose: detect helper scripts and dev tooling
- Look for scripts/ bin/ build scripts, Makefile, magefiles, and cli wrappers
- Commands:
  - test -f Makefile && sed -n '1,200p' Makefile > report/Makefile.txt || true
  - find scripts bin -type f -maxdepth 2 -print > report/scripts_list.txt || true

Task 9 — Static heuristics and red flags
- Purpose: identify secrets, very large files, or generated directories
- Commands:
  - find . -type f -size +5M -printf "%P %k KB\n" | sort -nrk2 | head -n 20 > report/large_files.txt
  - grep -IR --line-number --exclude-dir=.git -E "(API_KEY|SECRET_KEY|PRIVATE_KEY|password=|access_token)" . | head -n 200 > report/potential_secrets.txt || true
  - find . -path "*/node_modules/*" -prune -o -name 'node_modules' -print > report/ignored_node_modules.txt || true
- Redaction: for any potential secrets, replace RHS with <REDACTED> and flag location for manual review.

Task 10 — Produce final summaries
- Purpose: aggregate task outputs into final deliverables
- Commands (aggregation script example):
  - echo "High-level summary" > report/high-level.txt
  - echo "Root files:" >> report/high-level.txt
  - head -n 200 report/root_files.txt >> report/high-level.txt || true
  - jq -n '{root: (input | split("\n") ), languages: (input), ci: (input)}' > report/structure.json  # replace inputs with collected values in implementation
- Expected outputs:
  - report/high-level.txt
  - report/structure.json
  - report/details.md  (human-readable annotated findings)

Deliverables and review

Notes for implementers and operators

- If a manifest is unknown or contains a custom layout (monorepo with workspaces), include workspace top-level manifests in report/manifests/workspaces.txt
- Limit log verbosity and truncate long files; include a pointer to full file path for manual retrieval.
