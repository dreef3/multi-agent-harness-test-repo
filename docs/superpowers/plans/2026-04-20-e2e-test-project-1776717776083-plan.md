Implementation plan: Task-by-task instructions to analyze repository structure and produce analysis/codebase-structure.md

Repository

- Default branch: main
- Repository ID (for dispatching tasks later): bd03d9a6-1543-4d0c-9e28-2d49483cc288

High-level steps (one-run sequence)
1) Clone and checkout
   - git clone --depth 1 https://github.com/dreef3/multi-agent-harness-test-repo.git
   - cd multi-agent-harness-test-repo
   - git checkout main

2) Produce a file tree and top-level listing
   - tree -a -L 2 --dirsfirst -I 'node_modules|venv|__pycache__' > analysis/tree-L2.txt
   - ls -la > analysis/ls-root.txt
   - Deliverable: analysis/tree-L2.txt and analysis/ls-root.txt

3) Count languages and lines (LOC)
   - If cloc available: cloc --by-file --yaml --out=analysis/cloc.yaml .
   - Alternative: cloc --json --out=analysis/cloc.json .
   - If cloc is not installed, use: rg --hidden --files | sed -e 's|\./||' | xargs -n 100 file | cut -d: -f2 | sort | uniq -c > analysis/file-types.txt
   - Deliverable: analysis/cloc.yaml (or cloc.json) and analysis/file-types.txt

4) Identify primary manifest files and examine them
   - Search for common manifests: README.md, package.json, package-lock.json, yarn.lock, pnpm-lock.yaml, pyproject.toml, requirements.txt, setup.py, Pipfile, go.mod, Cargo.toml, Gemfile, composer.json, build.gradle, pom.xml
   - Commands:
     - rg -n "(^name:|^version:|^dependencies|^devDependencies|^tool.poetry|^project\.|^name\"\:|^version\"\:)" --hidden --no-ignore --glob '!node_modules' > analysis/manifest-grep.txt || true
     - For package.json: jq '.name, .version, .scripts, .dependencies, .devDependencies' package.json > analysis/package-json-summary.json || true
     - For pyproject.toml: sed -n '1,200p' pyproject.toml > analysis/pyproject-head.txt || true
     - For go.mod: sed -n '1,200p' go.mod > analysis/go.mod.head || true
   - Deliverable: analysis/manifest-grep.txt and per-manifest summaries under analysis/

5) Inspect CI and automation
   - Check .github/workflows, .travis.yml, .gitlab-ci.yml, .circleci/config.yml, Makefile, Dockerfile(s)
   - Commands:
     - tree -a .github/workflows -L 2 > analysis/github-workflows-tree.txt || true
     - sed -n '1,300p' .travis.yml > analysis/travis-head.txt || true
     - rg -n "dockerfile|Dockerfile|image:|actions/checkout" .github --hidden --no-ignore || true > analysis/ci-grep.txt
   - Deliverable: analysis/github-workflows-tree.txt, analysis/ci-grep.txt, and extracted heads of CI files

6) Tests, coverage, and quality tools
   - Search for tests directories and common test frameworks
     - rg -n "(^pytest|unittest|jest|mocha|ava|vitest|pytest.ini|tox.ini|jest.config|pytest)" --hidden --no-ignore > analysis/test-grep.txt || true
     - tree -a -L 3 tests spec test > analysis/tests-tree.txt || true
     - Check for coverage config: .coveragerc, coverage.xml, codecov.yml
   - Deliverable: analysis/test-grep.txt and analysis/tests-tree.txt

7) Containerization and deployment
   - Find Dockerfiles and Helm charts
     - rg -n "^FROM |^image: |kind: Deployment|apiVersion: apps/" --hidden --no-ignore > analysis/docker-helm-grep.txt || true
     - For each Dockerfile found, capture head: sed -n '1,200p' <Dockerfile> > analysis/Dockerfile-<path>-head.txt
   - Deliverable: analysis/docker-helm-grep.txt and Dockerfile heads

8) License and contribution metadata
   - Check files: LICENSE, CODE_OF_CONDUCT.md, CONTRIBUTING.md, .github/CONTRIBUTING
   - Commands: ls -la LICENSE* CONTRIBUTING* CODE* || true > analysis/license-contrib.txt
   - Deliverable: analysis/license-contrib.txt

9) Dependency surface (no install)
   - Node: if package.json present: jq '.dependencies, .devDependencies' package.json > analysis/node-deps.json || true
   - Python: if requirements.txt present: cat requirements.txt | sed -n '1,200p' > analysis/requirements-head.txt || true
   - Go: if go.mod present: sed -n '1,200p' go.mod > analysis/go-mod-head.txt || true
   - Deliverable: analysis/*-deps files

10) Produce the final report
   - Create analysis/codebase-structure.md with sections:
     - Repo metadata (URL, branch)
     - High-level summary (one-paragraph)
     - Top-level directories and their purpose
     - Languages and LOC (from cloc)
     - Key manifests and dependency summaries
     - CI/build/test automation
     - Containerization/deployment artifacts
     - License and contribution files
     - Notable risks and maintenance notes
     - Recommended next steps (e.g., run tests, run dependency audit, check security advisories)
   - Template (example header):
     # Codebase structure analysis
     - Repository: <url>
     - Branch: main
     ## Summary
     (one paragraph)
     ## Top-level layout
     - list dirs with a short explanation
     ## Languages & LOC
     - cloc table or summary
     ...
   - Save as analysis/codebase-structure.md and commit to a branch like analysis/codebase-structure (if modifying repo is allowed). If not allowed, return the file as the task output.

11) Package outputs
   - Ensure analysis/ directory contains all produced files:
     - analysis/tree-L2.txt
     - analysis/ls-root.txt
     - analysis/cloc.yaml (or json)
     - analysis/manifest-grep.txt
     - analysis/package-json-summary.json (if present)
     - analysis/github-workflows-tree.txt
     - analysis/test-grep.txt
     - analysis/docker-helm-grep.txt
     - analysis/license-contrib.txt
     - analysis/codebase-structure.md
