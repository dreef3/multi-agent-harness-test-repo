Implementation plan: Task-by-task plan to analyze repository structure for E2E Test Repo 1 (repo id: 80d2851d-a196-47e0-b79f-45256dbd1745)

Overview
- Produce two artifacts in the repo workspace:
  1) docs/codebase-structure.md — human-readable analysis
  2) reports/codebase-structure.json — structured metadata
- The analysis will be implemented as a sequence of independent tasks so they can be dispatched to sub-agents after the planning PR is merged.

Tasks (self-contained)

Task 1 — Top-level inventory
- Purpose: get the repository root layout and file counts.
- Commands to run in repo root:
  - ls -la
  - find . -maxdepth 2 -mindepth 1 -type d -print | sed 's|^./||' | sort
  - find . -maxdepth 1 -type f -print | sed 's|^./||' | sort
  - du -sh .
- Deliverable: write docs/codebase-structure.md section "Top-level layout" and add to JSON: top_level_dirs[], top_level_files[], repo_size.

Task 2 — Language and dependency detection
- Purpose: detect primary languages and dependency files.
- Files to check (if present): package.json, package-lock.json, yarn.lock, pnpm-lock.yaml, pyproject.toml, requirements.txt, setup.py, go.mod, Gemfile, Cargo.toml, composer.json, pom.xml, build.gradle
- Commands:
  - for file in package.json pyproject.toml go.mod Gemfile Cargo.toml pom.xml requirements.txt setup.py package-lock.json yarn.lock pnpm-lock.yaml; do [ -f "$file" ] && echo "FOUND:$file"; done
  - cat package.json | jq -r '.name, .version, .dependencies, .devDependencies' 2>/dev/null | head -n 50
  - head -n 200 pyproject.toml 2>/dev/null
- Deliverable: docs/codebase-structure.md section "Languages & Dependencies" and JSON fields: languages[], dependency_files[], primary_package_manager.

Task 3 — CI / workflows
- Purpose: list CI configurations and summarize jobs.
- Paths to scan: .github/workflows/*.yml or *.yaml, .gitlab-ci.yml, azure-pipelines.yml, circleci/config.yml, .travis.yml
- Commands:
  - ls -la .github/workflows || true
  - for f in .github/workflows/*.yml .github/workflows/*.yaml .gitlab-ci.yml .travis.yml azure-pipelines.yml .circleci/config.yml; do [ -f "$f" ] && echo "CI:$f"; done
  - for wf in .github/workflows/*.yml .github/workflows/*.yaml; do [ -f "$wf" ] && (echo "--- $wf" && sed -n '1,200p' "$wf"); done
- Deliverable: docs/codebase-structure.md "CI & workflows" and JSON ci_files[], ci_engines[].

Task 4 — Docker and containerization
- Purpose: find Dockerfiles and container-related configs.
- Paths to check: Dockerfile, docker-compose.yml, k8s manifests (k8s/, manifests/, charts/), buildpacks, pack.* files
- Commands:
  - rg -n --hidden "^FROM\b" --glob "**/Dockerfile" || true
  - ls -la Dockerfile || true; ls -la docker-compose.yml || true
- Deliverable: docs section "Docker & containers" and JSON dockerfiles[], compose_files[], k8s_manifests[].

Task 5 — Tests and test frameworks
- Purpose: locate tests and determine test commands.
- Paths to scan: tests/, spec/, e2e/, src/test, __tests__/
- Commands:
  - rg -n --hidden "describe\(|it\(|pytest|unittest|mocha|jest|ava|ginkgo|go test" || true
  - find tests spec e2e -maxdepth 3 -type f -print 2>/dev/null || true
  - parse package.json scripts.test if present: jq -r '.scripts.test' package.json 2>/dev/null
- Deliverable: docs section "Tests" with test dirs, test command, and JSON tests[].

Task 6 — Build and developer scripts
- Purpose: find Makefile, scripts/, bin/ and other developer tooling.
- Commands:
  - [ -f Makefile ] && sed -n '1,200p' Makefile
  - ls -la scripts || true
  - rg -n --hidden "#!/bin" scripts || true
- Deliverable: docs section "Build & scripts" and JSON build_files[], scripts[].

Task 7 — Documentation & contributor files
- Purpose: capture README, CONTRIBUTING, LICENSE, CHANGELOG, CODE_OF_CONDUCT
- Commands:
  - for f in README.md README.adoc CONTRIBUTING.md LICENSE CHANGELOG.md CODE_OF_CONDUCT.md; do [ -f "$f" ] && echo "DOC:$f"; done
  - head -n 200 README.md || true
- Deliverable: docs section "Docs" and JSON docs[].

Task 8 — Static size & complexity heuristics
- Purpose: provide quick metrics: total files, lines of code by language, largest files
- Commands:
  - cloc --json . > reports/cloc.json || rg --version >/dev/null && echo "cloc not installed"
  - find . -type f -name "*.py" -or -name "*.js" -or -name "*.ts" -or -name "*.go" | xargs wc -l | sort -rn | head -n 20
- Deliverable: include metrics in JSON: total_files, loc_by_language, largest_files[].

Task 9 — Generate outputs
- Purpose: synthesize findings and write artifacts.
- docs/codebase-structure.md structure (suggested content):
  - Title and generation timestamp
  - Top-level layout (list)
  - Languages and dependency summary
  - CI and workflows
  - Docker and containers
  - Tests and recommended test command
  - Build & scripts
  - Docs & governance
  - Quick metrics (size, loc)
  - Recommendations & next steps
- reports/codebase-structure.json example schema:
  {
    "repo": "multi-agent-harness-test-repo",
    "generated_at": "2026-04-19T...Z",
    "top_level_dirs": [...],
    "top_level_files": [...],
    "languages": ["javascript", "python"],
    "dependency_files": [...],
    "ci_files": [...],
    "dockerfiles": [...],
    "tests": [...],
    "metrics": { "total_files": 123, "loc_by_language": {"js": 1234} }
  }

Task 10 — Review and QA
- Purpose: automated checks to ensure required items are present and the files were written.
- Commands:
  - [ -f docs/codebase-structure.md ] && echo OK || echo MISSING
  - jq empty reports/codebase-structure.json || echo "JSON invalid"

Execution notes
- Each task is read-only.
- Use small head/sed excerpts when printing file content to keep outputs tractable.
- If cloc is unavailable, use rg/wc as fallback for rough LOC estimates.

Estimated effort
- Single agent: ~15–30 minutes (depending on repo size)
- Split into multiple agents: Task 1–4, 5–6, 7–8, 9–10 as parallelizable units.

Deliverables paths (to be created by implementers)
- docs/codebase-structure.md
- reports/codebase-structure.json
- reports/cloc.json (optional)

Assumptions & risks
- Assumes repository checkout is present in the agent workspace. Files and commands may vary for very large monorepos; increase head limits if needed.
