Implementation Plan: Task-by-task plan to analyze repository structure

Repository
- ID: 92bacce6-02f6-4ded-966b-d998662a05bc
- URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Overview
This plan consists of a sequence of self-contained tasks an implementation agent will run inside a workspace where the repository has been cloned. Each task is explicit about commands to run, the files to read, and the outputs to write. The final deliverable is docs/codebase-analysis.md plus artifacts in artifacts/.

Preparation (Task 0) — workspace setup
- Goal: ensure a clean workspace and shallow clone of the repository.
- Commands (run from an ephemeral CI agent or local machine):
  - git clone --depth 1 https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
- Output: repository checked out at ./repo

Task 1 — Top-level listing and file-tree
- Goal: produce a filtered file tree and top-level file list.
- Commands:
  - ls -la
  - tree -a -I 'node_modules|venv|\.venv|\.cache|dist|build|__pycache__' > artifacts/file-tree.txt
  - head -n 200 artifacts/file-tree.txt
- Files to read:
  - README.md (if present)
  - LICENSE
- Output files:
  - artifacts/file-tree.txt

Task 2 — Detect languages and manifests
- Goal: identify language ecosystems by presence of manifest files.
- Commands (POSIX shell):
  - echo "--- manifests ---" > artifacts/manifests-found.txt
  - for f in package.json pyproject.toml setup.py requirements.txt Pipfile Pipfile.lock go.mod Cargo.toml pom.xml build.gradle composer.json Gemfile mix.exs Cargo.lock; do if [ -f "$f" ]; then echo "$f" >> artifacts/manifests-found.txt; fi; done
  - rg -n "^name\b|^version\b|^dependencies|\bmodule\b" --hidden --glob '!node_modules' -S || true
- Expected detection targets and what they imply:
  - package.json -> Node.js / npm / yarn
  - pyproject.toml / requirements.txt -> Python
  - go.mod -> Go
  - Cargo.toml -> Rust
  - pom.xml / build.gradle -> Java
  - Gemfile -> Ruby
  - composer.json -> PHP
- Output: artifacts/manifests-found.txt

Task 3 — Read and parse manifest files
- Goal: extract top-level metadata (name, description, main entry, scripts, dependencies) for each manifest detected.
- Commands / examples:
  - If package.json exists: jq '{name,version,description,scripts,dependencies,devDependencies}' package.json > artifacts/package.json.summary.json || cat package.json | python -m json.tool > artifacts/package.json.pretty.json
  - If pyproject.toml exists: python -c "import tomllib,sys;print(tomllib.load(open('pyproject.toml', 'rb')))")" > artifacts/pyproject.parsed.txt  # or use toml library
  - If go.mod exists: sed -n '1,200p' go.mod > artifacts/go.mod.head.txt
- Output: artifacts/*.summary.json or parsed files, collected into artifacts/dependencies.json (see aggregation step)

Task 4 — Dependency enumeration by ecosystem
- Goal: list top-level dependencies and low-depth tree when safe.
- Commands (run only when appropriate and in ephemeral environment):
  - Node: if package.json exists: npm ci --ignore-scripts --no-audit --no-fund && npm ls --depth=0 --json > artifacts/npm-deps.json
  - Python: if requirements.txt or pyproject exists: python3 -m venv .venv && . .venv/bin/activate && pip install -r requirements.txt --no-deps -q || true && pip freeze > artifacts/pip-freeze.txt
  - Go: go list -m all > artifacts/go-mod-list.txt
  - Rust: cargo metadata --format-version 1 --no-deps > artifacts/cargo-metadata.json
- Note: these commands install or require network. If network disallowed, fall back to parsing manifest files only.
- Output: artifacts per-ecosystem dependency lists

Task 5 — Source layout mapping
- Goal: map primary source directories and test directories.
- Typical paths to check and read (if present):
  - src/, lib/, pkg/, cmd/, internal/, app/, main.go, main.py, server/, web/
  - test/, tests/, spec/, __tests__/
  - examples/
- Commands:
  - for d in src lib pkg cmd internal app tests test __tests__ examples; do if [ -d "$d" ]; then tree -L 2 "$d" >> artifacts/source-layout.txt; fi; done
- Output: artifacts/source-layout.txt

Task 6 — CI/CD and workflows
- Goal: summarize CI configuration and common automation.
- Files to read:
  - .github/workflows/*.yml
  - .gitlab-ci.yml
  - .circleci/config.yml
  - Jenkinsfile
- Commands:
  - if [ -d .github/workflows ]; then for y in .github/workflows/*.yml; do echo "== $y =="; sed -n '1,200p' "$y"; done > artifacts/ci-workflows.txt; fi
- Output: artifacts/ci-workflows.txt and a summary in the report

Task 7 — Docker, build, and packaging artifacts
- Goal: detect Dockerfile(s), images, build scripts.
- Files to read:
  - Dockerfile, docker/*, packer/*, Makefile, build.sh
- Commands:
  - rg -n "FROM |docker" -S --hidden --glob '!node_modules' || true
  - sed -n '1,200p' Dockerfile > artifacts/Dockerfile.head.txt || true
- Output: artifacts/Dockerfile.head.txt

Task 8 — Tests detection and recommended execution
- Goal: detect test frameworks and provide commands to run tests.
- Detection heuristics:
  - package.json scripts.test or jest config -> npm test / yarn test
  - pytest.ini or tests/ -> pytest
  - go test ./... for go
  - cargo test for Rust
- Provide safe commands with timeouts and CI-friendly flags.

Task 9 — Static analysis and lints
- Goal: run or recommend linters based on language.
- Examples:
  - Node: npx eslint --ext .js,.ts src/ --max-warnings=0
  - Python: flake8, mypy (if pyproject has config)
  - Go: go vet ./...
- These are optional steps; list commands in report and only run if explicitly requested.

Task 10 — Aggregate and write final report
- Goal: synthesize findings into docs/codebase-analysis.md
- File path: docs/codebase-analysis.md (create docs/ if missing)
- Report structure (markdown):
  - Title and repository link
  - High-level summary: languages, primary directories, size estimate
  - File tree excerpt
  - Manifests found and parsed metadata
  - Dependency summary and artifacts locations
  - CI/workflow summary
  - Build and run instructions per detected language/service
  - Test instructions
  - Security notes and recommended follow-ups (SCA, secrets scan, pinning)
  - Appendix: artifacts list and commands used
- Example snippet to write (replace placeholders after actual run):
  - "Languages detected: Node.js (package.json), Python (pyproject.toml), Go (go.mod)"
  - "Top-level layout: README.md, Dockerfile, src/, tests/, .github/workflows/ci.yml"
- Commands to create file:
  - mkdir -p docs artifacts
  - cat > docs/codebase-analysis.md <<'MD'
  - # Codebase analysis: multi-agent-harness-test-repo
  - Repository: https://github.com/dreef3/multi-agent-harness-test-repo
  - (rest of content synthesized from artifacts...)
  - MD

Task 11 — Create PR with analysis
- Goal: commit artifacts and the report to a new branch and open a PR (handled by harness rules).
- Commands:
  - git checkout -b analysis/codebase-structure
  - git add docs artifacts
  - git commit -m "Add codebase structure analysis"
  - git push --set-upstream origin analysis/codebase-structure
- Note: the harness will create the PR automatically when planning docs are written; this step is included for completeness.

Task 12 — Review and follow-ups
- Goal: list recommended next steps in the report: run full test suite under CI, add SCA job, fix lint errors, reduce dependency surface, set up reproducible builds, add CONTRIBUTING.md.

Deliverables
- docs/codebase-analysis.md
- artifacts/file-tree.txt
- artifacts/manifests-found.txt
- artifacts/*.{summary,parsed}.json
- artifacts/ci-workflows.txt
- A Git branch with the changes and a PR

Notes and constraints
- Network and package installation steps are optional and guarded: the agent should skip npm ci/pip installs if running in an environment that forbids network or package installs and should instead parse manifests only.
- The implementation will avoid writing any secrets to artifacts and will skip scanning common secret locations unless explicitly requested.
