Implementation Plan: Task-by-task actionable plan to analyze codebase structure

Overview
- Produce an automated analysis and a human review. Create a feature branch (e.g., analysis/codebase-structure) and commit report files under reports/. Do not change production code.
- Outputs (added to repository):
  - reports/codebase-structure.md  (human readable)
  - reports/codebase-structure.json (machine readable)
  - reports/README.md (how to regenerate)
  - scripts/analyze_codebase.sh (helper script to run the analysis locally/CI)

Task 1 — Prepare workspace & branch
- Goal: clone repo in sandbox and create a feature branch for changes.
- Steps (to run in an agent runner):
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git
  - cd multi-agent-harness-test-repo
  - git checkout -b analysis/codebase-structure
- Deliverables: working checkout, new branch.

Task 2 — Enumerate files & top-level layout
- Goal: produce a baseline file/directory tree and counts.
- Commands:
  - tree -a -I '.git' -L 2  # or find . -maxdepth 2 -mindepth 1 -print
  - find . -type f | sed 's|^./||' | wc -l
  - ls -la | sed -n '1,200p'
- Output file: reports/top-level-structure.txt

Task 3 — Detect languages and line counts
- Goal: identify languages and approximate LOC per language.
- Commands (preferred):
  - cloc --json . > reports/cloc.json
  - If cloc unavailable: use linguist or fallback to counting file extensions with find/awk.
- Use cloc.json to populate codebase-structure.json -> languages section.

Task 4 — Identify manifest & dependency files
- Goal: find package manifests and extract top-level dependencies.
- Files to look for (exact filenames/paths):
  - package.json
  - yarn.lock, package-lock.json
  - requirements.txt, Pipfile, pyproject.toml, setup.cfg, setup.py
  - go.mod
  - Cargo.toml
  - pom.xml
  - Gemfile
  - composer.json
- Commands/examples to extract dependencies:
  - For package.json: jq '.dependencies, .devDependencies' package.json
  - For pyproject.toml: use toml parser (python -c) or grep under [tool.poetry.dependencies]
  - For go.mod: sed -n '/^require/,/^$/p' go.mod
- Output files: reports/manifests-list.txt, reports/dependencies-extracted.json

Task 5 — Inspect CI, build & packaging
- Goal: locate CI workflows and build config.
- Files to inspect:
  - .github/workflows/*.yml
  - .gitlab-ci.yml
  - azure-pipelines.yml
  - .circleci/config.yml
  - Dockerfile(s), Makefile
- Commands:
  - jq -R -s -c 'split("\n")' .github/workflows/*.yml > reports/ci-workflows-raw.txt (or simply copy files)
- Populate: reports/ci-summary.md with which jobs exist, languages they run, test steps, and build steps.

Task 6 — Locate tests and test commands
- Goal: identify test directories, frameworks (pytest, jest, go test, etc.) and suggested commands.
- Actions:
  - Search for common test directory names: find . -type d \( -name test -o -name tests -o -name __tests__ \)
  - Inspect package.json scripts for 'test' script: jq '.scripts.test' package.json
  - For Python: look for pytest.ini, tox.ini, setup.cfg with [tool:pytest]
- Output: reports/tests-summary.md with commands to run and locations.

Task 7 — Static analysis & quick health checks (optional, opt-in)
- Goal: run linters/static analyzers without installing project deps.
- Recommended tools to run if present in repo or in runner image: eslint, flake8, golangci-lint, rubocop. If absent, skip.
- NOTE: These may require installing dependencies; mark as optional.
- Output: reports/static-analysis-summary.md

Task 8 — Security & secrets scan (optional)
- Goal: scan for accidentally committed secrets (API keys, .env files). Tools: truffleHog, git-secrets, detect-secrets. This is optional and should be run with care.
- Output: reports/secrets-scan.md (if run)

Task 9 — Assemble final report
- Goal: merge all collected pieces into two outputs.
- reports/codebase-structure.json (schema):
  {
    "repo": "dreef3/multi-agent-harness-test-repo",
    "top_level": ["src/", "tests/", "README.md", ...],
    "languages": { "Python": 1234, "JavaScript": 567 },
    "manifests": ["package.json", "pyproject.toml"],
    "dependencies": { "package.json": {"express": "^4.17.1"}, "pyproject.toml": {...} },
    "ci": [ {"path": ".github/workflows/ci.yml", "jobs": ["build","test"]}],
    "tests": {"frameworks": ["pytest"], "commands": ["pytest -q"]},
    "dockerfiles": ["Dockerfile"],
    "license": "MIT",
    "readme_summary": "first paragraph...",
    "recommendations": ["Add CI test coverage badge", "Add explicit CONTRIBUTING.md"]
  }

- reports/codebase-structure.md: human summary with sections: Executive summary, Top-level layout, Languages & LOC, Dependency manifests, CI & build, Tests, Docker/packaging, License, Risks & recommendations.

Task 10 — Commit & open PR
- Stage created reports and scripts: git add reports scripts && git commit -m "Add codebase structure analysis reports" && git push --set-upstream origin analysis/codebase-structure
- Open a PR from analysis/codebase-structure -> main (the harness will handle PR creation per process).

Implementation details & helpers
- scripts/analyze_codebase.sh (example outline to include in plan):
  - Check for required tools (cloc, jq). If missing, warn and attempt fallback commands.
  - Run cloc --json and write to reports/cloc.json
  - Find manifests and run extraction commands into reports/
  - Summarize CI workflows by listing files under .github/workflows and copying to reports/ci-*.yml
  - Produce codebase-structure.json by combining partial JSON pieces with jq.
- Ensure scripts are POSIX-friendly and idempotent. Use temporary directories for any unpacking.

Reporting & acceptance
- Review the generated reports to ensure they satisfy the Acceptance criteria in the design spec.
- Deliver final PR with the reports and scripts; include a short PR description summarizing findings.

Timeline estimate
- Automated script + run: 30–60 minutes
- Manual review & polish of report: 30–60 minutes

Notes & safety
- Do not run repository tests or install dependencies without explicit opt-in.
- Secrets scanning should be opt-in and conducted using approved tools.

Example commands to run locally (summary):
- git clone <repo>
- cd repo
- git checkout -b analysis/codebase-structure
- mkdir -p reports
- cloc --json . > reports/cloc.json
- find . -maxdepth 2 -type d -printf "%P\n" > reports/top-level.txt
- jq '.languages = (input | fromjson).SUM' reports/cloc.json > reports/codebase-structure.json (combine pieces with jq)
