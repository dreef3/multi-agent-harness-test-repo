Detailed tasks

Task 1 — Top-level inventory
- Purpose: Produce top-level listing and short descriptions
- Commands to run (read-only):
  - ls -la
  - tree -L 2 || rg --files | sed -n '1,200p'
  - head -n 50 README.md || true
- Files to read: README.md, LICENSE, CONTRIBUTING.md, .gitignore, top-level directories
- Output: analysis/inventory.md (table: name / type / short description)
- Acceptance: inventory.md lists every top-level entry and includes a one-line summary for the repo from README

Task 2 — Language & dependency detection
- Purpose: Detect languages and package manifests
- Files to inspect (if present): package.json, yarn.lock, package-lock.json, pnpm-lock.yaml, pyproject.toml, requirements.txt, Pipfile, setup.py, go.mod, Cargo.toml, Gemfile, composer.json, build.gradle, pom.xml
- Commands (read-only): head -n 200 <manifest>, jq -r '.name, .version, .dependencies' package.json || true
- Output: analysis/dependencies.md
- Acceptance: For each detected manifest include name, version (if available), top-level dependencies, and package manager used

Task 3 — CI/workflows and automation
- Purpose: Enumerate CI workflows and automation tools
- Files/paths: .github/workflows/*, .gitlab-ci.yml, .circleci/config.yml, Jenkinsfile, .github/dependabot.yml, .github/renovate.json
- Commands: ls -la .github/workflows || true; rg -n "on:|jobs:|workflow_dispatch|push:|pull_request:" -S .github/workflows || true
- Output: analysis/ci.md
- Acceptance: ci.md lists each workflow with trigger, main steps, and notes on building/testing/deploying

Task 4 — Tests and coverage
- Purpose: Locate tests and coverage configs
- Files to read: tests/ directories, package.json scripts, pytest.ini, jest.config.js, .coveragerc, coverage.yml
- Commands: rg -n "\bdescribe\b|\bit\b|pytest|unittest|mocha|jest|t\.test\(" -S || true; jq -r '.scripts.test' package.json || true
- Output: analysis/tests.md
- Acceptance: tests.md lists test frameworks and how to run tests locally (example command), and whether CI runs tests

Task 5 — Entrypoints and runtime
- Purpose: Identify entrypoints, scripts, Docker start commands
- Files: main.go, cmd/*, package.json scripts.start, Dockerfile, docker-compose.yml
- Commands: rg -n "ENTRYPOINT|CMD|EXPOSE|CMD \[" Dockerfile || true; jq -r .scripts package.json || true
- Output: analysis/entrypoints.md
- Acceptance: entrypoints.md lists executable artefacts and commands to run them locally

Task 6 — Linters, formatters, and pre-commit hooks
- Purpose: Detect linting and formatting configs
- Files: .eslintrc*, .prettierrc*, .flake8, pyproject.toml, .pre-commit-config.yaml
- Commands: rg -n "eslint|prettier|black|flake8|pre-commit|isort" -S || true
- Output: analysis/lint_and_hooks.md
- Acceptance: Document which formatting/linting tools are present and whether CI enforces them

Task 7 — Security quick-check
- Purpose: Detect dependency scanning, code analysis, and obvious secrets
- Files: .github/workflows/*codeql*, .github/dependabot.yml, .secrets*, .env*, .dockerignore
- Commands: rg -n "codeql|dependabot|secret|SECRET|API_KEY|TOKEN|PASSWORD" --hidden -S || true
- Output: analysis/security_and_lint.md
- Acceptance: Flag any automated scanners, and list any files that appear to contain secrets (do not expose secret contents in report; only file names and recommendation to rotate if real secrets found)

Task 8 — Metrics and size
- Purpose: Provide counts by language and rough LOC
- Commands (recommended): cloc --by-file-by-lang --quiet --out=analysis/cloc.csv . || use wc -l on files grouped by extension
- Output: analysis/metrics.md and analysis/cloc.csv (if cloc available)
- Acceptance: metrics.md includes top 5 languages by LOC and number of files

Task 9 — Final summary and recommendations
- Purpose: Combine findings into a single-page summary and recommended next steps
- Output: CODEBASE_ANALYSIS.md
- Content: one-paragraph summary, list of strengths, list of issues (priority: critical/high/medium/low), suggested next actions (e.g., add CI tests, enable dependabot, add README details, add CONTRIBUTING, modularize large packages)
- Acceptance: CODEBASE_ANALYSIS.md contains links to the analysis/*.md files
