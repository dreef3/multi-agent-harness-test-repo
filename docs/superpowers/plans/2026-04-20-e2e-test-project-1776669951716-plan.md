Implementation plan: Step-by-step tasks to analyze repository structure

Repository: E2E Test Repo 0
Repository ID: 7ce7f6c1-0572-422c-93f6-e254aca19563
Repository URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Summary
- The plan divides the analysis into discrete tasks. Each task operates only by reading repository files (no code execution). Each task will write a markdown artifact under analysis/ in the repo root. When all tasks are complete, produce CODEBASE_ANALYSIS.md summarizing findings and recommendations.

Task list (ordered, independent where possible)

Task 1 — Top-level inventory
- Purpose: Produce a listing of top-level files and directories with short descriptions.
- Files/paths to read: README.md, CONTRIBUTING.md, LICENSE, .gitignore, .gitattributes, any top-level directories.
- Commands/examples: ls -la, tree -L 2 .
- Output file: analysis/inventory.md
- Content structure:
  - Table of top-level entries (name, type, size if available, short description extracted from README or first file lines).
  - Notable absences (e.g., missing README or license).
- Estimated time: 10–15 minutes

Task 2 — Language and project type detection
- Purpose: Detect languages and packaging systems used.
- Files/paths to read (check presence): package.json, pnpm-lock.yaml, yarn.lock, package-lock.json, pyproject.toml, requirements.txt, Pipfile, setup.py, go.mod, Cargo.toml, Gemfile, composer.json, build.gradle, pom.xml, Makefile, Dockerfile
- Commands/examples: rg -n "^name|^version|^dependencies|^scripts" --hidden; head -n 200 <file>
- Output file: analysis/dependencies.md
- Content:
  - Languages detected and rationale (which files triggered detection)
  - For each detected package manifest, a short parsed summary (top-level fields: name, version, main scripts, key dependencies)
- Estimated time: 20–30 minutes

Task 3 — CI / workflows and automation
- Purpose: Detect CI configuration and release automation
- Files/paths: .github/workflows/*, .gitlab-ci.yml, .circleci/config.yml, Jenkinsfile, .github/dependabot.yml, .github/renovate.json
- Commands/examples: ls .github/workflows ; rg -n "on:|jobs:|workflow_dispatch|push:|pull_request:" -S .github/workflows || true
- Output file: analysis/ci.md
- Content:
  - List of workflows with filenames, triggers, and a brief one-line description for each (e.g., run tests, build image, release)
  - Presence of Dependabot or Renovate
- Estimated time: 20 minutes

Task 4 — Tests and test frameworks
- Purpose: Locate tests, detect frameworks, and find coverage configs
- Files/paths: tests/, test/, src/**/tests, package.json scripts.test, pytest.ini, jest.config.*, coverage.yml, .coveragerc
- Commands/examples: rg -n "describe\(|it\(|pytest|unittest|mocha|jest|t\.test\(" -S || true; jq -r .scripts.test package.json || true
- Output file: analysis/tests.md
- Content:
  - Test directories and example test files, frameworks detected, how tests are run (npm test, pytest -q), presence of CI test steps, and coverage tools/thresholds if any
- Estimated time: 20–30 minutes

Task 5 — Entrypoints, binaries, and Docker
- Purpose: Identify runtime entrypoints (main binaries, package entry_points, Docker CMD/ENTRYPOINT)
- Files/paths: main.go, cmd/, bin/, package.json (bin/main, scripts.start), setup.py entry_points, Dockerfile, compose files
- Commands/examples: rg -n "^ENTRYPOINT|^CMD|^EXPOSE" Dockerfile || true; jq -r .bin package.json || true
- Output file: analysis/entrypoints.md
- Content:
  - For each binary or service, path to the file, how to run it locally, example commands
- Estimated time: 15–25 minutes

Task 6 — Linters, formatters, pre-commit, and repo checks
- Purpose: Detect linting and formatting configurations and pre-commit hooks
- Files/paths: .eslintrc*, .prettierrc*, .flake8, pyproject.toml [tool.black], .pre-commit-config.yaml
- Commands/examples: rg -n "eslint|prettier|black|flake8|pre-commit" -S || true
- Output file: analysis/lint_and_hooks.md
- Content:
  - Which tools are present, whether they are enforced via CI, suggested improvements
- Estimated time: 15 minutes

Task 7 — Security quick-check
- Purpose: Identify presence of dependency scanners and secret files; flag obvious secrets
- Files/paths: .github/workflows/codeql-analysis.yml, dependabot config, .secrets.* (if any), .env.example
