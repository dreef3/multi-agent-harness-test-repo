Implementation Plan: Step-by-step tasks to analyze repository structure

Overview
- This implementation plan is a set of independent, executable tasks that inspect the repository contents, collect structured outputs, and write a concise Markdown report. Each task is self-contained and writes artifacts under the repository paths listed. Do not rely on network access to external services; all inspection is local to the repository working copy.

Artifacts to be produced (file paths inside repository)
- docs/codebase-analysis.md (human-readable summary)
- artifacts/structure.json (machine-readable discovery)
- artifacts/raw/tree.txt (partial file tree)
- artifacts/raw/manifest-files.txt (list of known manifest files found)
- artifacts/raw/loc.txt (lines-of-code summary)
- artifacts/raw/ci-files.txt (CI configuration file list)
- artifacts/raw/dockerfiles.txt (Dockerfile paths)
- artifacts/raw/test-summary.txt (test frameworks and counts)
- artifacts/raw/dependency-outputs/* (language-specific dependency command outputs)

Task 1 — Create output directories and baseline tree
- Description: Ensure directories exist and write a baseline partial tree + top-level file listing.
- Commands (to run at repo root):
  mkdir -p docs artifacts/raw
  find . -maxdepth 3 -type f -not -path './.git/*' | sed 's|^\./||' > artifacts/raw/tree.txt
  head -n 200 artifacts/raw/tree.txt > artifacts/raw/tree-summary.txt
- Expected outputs: artifacts/raw/tree.txt, artifacts/raw/tree-summary.txt

Task 2 — Detect common manifest files and package managers
- Description: Find common package/manifest files.
- Commands:
  rg --hidden --files --glob '!**/.git/**' "package.json|pyproject.toml|requirements.txt|Pipfile|poetry.lock|go.mod|go.sum|pom.xml|build.gradle|Cargo.toml|Gemfile|composer.json|setup.py" || true
  (If ripgrep (rg) not present, use: find . -maxdepth 4 -type f \( -name package.json -o -name pyproject.toml -o -name requirements.txt -o -name Pipfile -o -name poetry.lock -o -name go.mod -o -name pom.xml -o -name build.gradle -o -name Cargo.toml -o -name Gemfile -o -name composer.json -o -name setup.py \) -print)
  Save output to artifacts/raw/manifest-files.txt
- Expected outputs: artifacts/raw/manifest-files.txt

Task 3 — Identify CI configuration files
- Description: Look for CI definitions (.github/workflows, .gitlab-ci.yml, circleci/config.yml, azure-pipelines.yml)
- Commands:
  find . -maxdepth 4 \( -path './.github/workflows' -o -name '.gitlab-ci.yml' -o -path './.circleci' -o -name 'azure-pipelines.yml' \) -print > artifacts/raw/ci-files.txt || true
- Expected outputs: artifacts/raw/ci-files.txt

Task 4 — Find Dockerfiles and container config
- Description: Search for Dockerfiles.
- Commands:
  rg --hidden --files --glob '!**/.git/**' "Dockerfile" > artifacts/raw/dockerfiles.txt || find . -type f -iname Dockerfile -print > artifacts/raw/dockerfiles.txt || true
- Expected outputs: artifacts/raw/dockerfiles.txt

Task 5 — Run language-specific introspection where possible
- Description: For each manifest discovered, run safe dependency listing commands and store outputs under artifacts/raw/dependency-outputs/. If the language tool is not present, note that and store the manifest path only.
- Per-manifest commands (run only if manifest exists):
  - NPM / Node (package.json present): npm --version && npm ls --json --all > artifacts/raw/dependency-outputs/npm-ls.json || npm ls --depth=0 --json > artifacts/raw/dependency-outputs/npm-ls.json
  - Python (pyproject.toml, requirements.txt): python3 -V && pip3 freeze > artifacts/raw/dependency-outputs/pip-freeze.txt (if virtualenv not active, pip3 freeze will list environment)
  - Go (go.mod): go version && go list -m all > artifacts/raw/dependency-outputs/go-list.txt
  - Maven (pom.xml): mvn -v && mvn dependency:tree -DoutputFile=artifacts/raw/dependency-outputs/mvn-dep-tree.txt -DoutputType=text || true
  - Cargo (Cargo.toml): cargo --version && cargo tree --all-features > artifacts/raw/dependency-outputs/cargo-tree.txt || true
- Notes: Each command should be guarded and proceed only if manifest file exists. If command fails or tool not installed, write a short note file indicating the failure.

Task 6 — Tests discovery and simple counts
- Description: Identify test directories and try to count tests for common frameworks.
- Commands:
  rg --hidden --files --glob '!**/.git/**' "__tests__|tests/|spec/|test_*.py|*_test.go|*_spec.rb" || true > artifacts/raw/test-files-list.txt
  # Example test counts (run only if frameworks present):
  - Node/Jest (package.json containing jest): npx --no-install jest --listTests > artifacts/raw/dependency-outputs/jest-list.txt
  - Python (pytest): pytest --collect-only -q > artifacts/raw/dependency-outputs/pytest-collect.txt || true
  - Go: go test ./... -run ^$ -json > artifacts/raw/dependency-outputs/go-test-json.txt || true
- Expected outputs: artifacts/raw/test-files-list.txt and any framework-specific outputs.

Task 7 — Lines of code summary
- Description: Produce a simple LOC by extension report.
- Commands:
  rg --hidden --files --glob '!**/.git/**' -g '!node_modules' -g '!vendor' -g '!venv' -S -n . | awk -F. '{print $NF}' | sort | uniq -c > artifacts/raw/loc-by-ext.txt
  # Alternatively use cloc if available:
  cloc --json . --quiet --out=artifacts/raw/loc.json || true
- Expected outputs: artifacts/raw/loc-by-ext.txt and/or artifacts/raw/loc.json

Task 8 — Assemble machine-readable structure.json and human Markdown
- Description: Collect the raw artifacts and synthesize a summary report.
- Steps to synthesize (shell or small script):
  - Read artifacts/raw/manifest-files.txt to identify languages.
  - Summarize counts and top-level directories from artifacts/raw/tree-summary.txt.
  - Include CI, Dockerfile, test summary, and LOC summary.
  - Build a JSON with keys: repository_id, summary_timestamp, top_level_files (first 50), manifest_files, ci_files, dockerfiles, languages (guessed), loc (from cloc), tests (counts/paths), recommendations (list).
  - Save to artifacts/structure.json and docs/codebase-analysis.md
- Example high-level outline for docs/codebase-analysis.md:
  - Title, date, repo id
  - Primary languages and manifests found
  - Top-level layout (src/, cmd/, pkg/, scripts/, .github/)
  - CI and Containerization
  - Tests and test frameworks
  - Lines of code summary
  - Immediate recommendations (e.g., add CODEOWNERS, increase test coverage, pin dependency ranges)

Task 9 — Commit artifacts and open PR (performed by harness)
- Description: Once artifacts are created in the working tree, the harness will create a PR for review. Ensure artifacts are reasonably sized (truncate huge files) and raw outputs are under artifacts/raw/.

Acceptance criteria (repeated)
- docs/codebase-analysis.md present and readable; summarizes high-level structure, languages, CI, Docker, tests, and recommendations.
- artifacts/structure.json present and machine-readable.
- Raw outputs in artifacts/raw/ for reproducibility.

Notes for implementers
- Guard language-tool commands with existence checks so the job does not fail when a tool is absent.
- Keep artifact sizes bounded (truncate outputs > 1MB and add a note).
- Use UTF-8 encoding; sanitize binary files out.
