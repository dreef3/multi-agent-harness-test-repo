Implementation Plan: Task-by-Task Repository Structure Analysis

Overview
This plan breaks the work into self-contained tasks that produce the analysis artifacts and add optional CI automation. Each task states the exact file paths to add/modify and the expected commands to run. Tasks are small and testable independently.

Repository: https://github.com/dreef3/multi-agent-harness-test-repo.git
Repository ID: 0b2d3c73-ba9d-4aad-9107-fbb86c7fa954

Task 1 — Create analysis script (scripts/analyze_repo.sh)
- Goal: Add a single, well-documented bash script that inspects the checked-out repository and emits: reports/structure.json and docs/repo-analysis.md.
- File path to create: scripts/analyze_repo.sh
- Responsibilities:
  - Verify common tools (find, wc, file, git). Optionally use cloc or scc if installed to gather language line counts.
  - Generate a list of top-level directories and counts of files per directory.
  - Detect common manifest files (package.json, requirements.txt, pyproject.toml, setup.py, go.mod, Cargo.toml, pom.xml, build.gradle, Dockerfile, .github/workflows/*).
  - Find test directories (test, tests, __tests__, spec) and common test files (pytest, jest, go test files, etc.).
  - Identify large files (size > 1MB by default) and list top N (e.g., top 20) largest files.
  - Write structured JSON to reports/structure.json with fields: repo, snapshot_time, top_level_dirs, files_by_type, languages (if cloc present), manifests, ci_files, tests, large_files.
  - Produce docs/repo-analysis.md summarizing the structured data in human-readable form and highlighting 3-5 recommended actions.
- Commands to run (example):
  - git clone <repo-url> .
  - bash scripts/analyze_repo.sh
- Estimated time: 1-2 hours for a small repo, 2-4 hours for larger monorepos.

Task 2 — Add human-readable report (docs/repo-analysis.md)
- Goal: Ensure the generated markdown is committed under docs/ so maintainers see results in the repo UI.
- File path: docs/repo-analysis.md (generated by scripts/analyze_repo.sh)
- Responsibilities:
  - Include sections: Summary, Languages & Size, Dependency Manifests, CI & Workflows, Tests, Large Files, Recommendations, Next Steps.
  - Use consistent headings and include JSON block showing top-level structure (small snippet from reports/structure.json).
- Estimated time: 15-30 minutes (auto-generated by script).

Task 3 — Add machine-readable report directory (reports/structure.json)
- Goal: Store the structured output for downstream automation, tests, or dashboards.
- File path: reports/structure.json (generated by the script)
- Responsibilities:
  - Schema example (keys only, not implementation code):
    - repo: string
    - snapshot_time: ISO8601
    - top_level_dirs: [{name, file_count, total_bytes}]
    - files_by_type: [{extension, count, bytes}]
    - languages: [{language, files, blank, comment, code}] or null
    - manifests: [{path, type}]
    - ci_files: [{path}]
    - tests: [{path, framework_hint}]
    - large_files: [{path, size_bytes}]
- Estimated time: few minutes (script output).

Task 4 — Add optional CI workflow (.github/workflows/repo-analysis.yml)
- Goal: Add a GitHub Actions workflow that can run the analyzer on push/PR and commit the results to the branch or upload as an artifact. This is optional and can be gated behind a repository policy.
- File path to create: .github/workflows/repo-analysis.yml
- Workflow responsibilities:
  - Run on push and pull_request (or be manual dispatch only).
  - Set up job: checkout, install cloc (optional), run scripts/analyze_repo.sh, upload reports/structure.json and docs/repo-analysis.md as artifacts.
- Note: The workflow should not commit back to main automatically. It can upload artifacts for manual inspection.
- Estimated time: 30-60 minutes.

Task 5 — Manual review and refinement
- Goal: After running the analyzer, a reviewer inspects docs/repo-analysis.md and adjusts recommendations, clarifies ambiguous areas (e.g., generated directories that should be ignored by analysis), and ensures no sensitive data was exposed.
- Responsibilities:
  - Check for large binary files that may be accidentally committed.
  - Verify detection of monorepo workspaces (package.json workspaces, go modules, etc.).
  - Adjust thresholds (e.g., large file size) if repo needs it.
- Estimated time: 1 hour.

Task 6 — Open PR with artifacts
- Goal: Open a PR that adds scripts/analyze_repo.sh, .github/workflows/repo-analysis.yml (optional), and the generated docs/repo-analysis.md and reports/structure.json to the repo.
- PR contents (file list):
  - scripts/analyze_repo.sh
  - docs/repo-analysis.md
  - reports/structure.json
  - .github/workflows/repo-analysis.yml (optional)
- PR description template: include short summary, commands to reproduce locally, and notes about tools used (cloc optional).
- Estimated time: 30 minutes.

Task 7 — Follow-up actions (post-merge)
- If the PR is merged, consider dispatching sub-agents to implement recommendations (e.g., add CI test steps, split large files, add .gitattributes to mark generated files, or add license/CONTRIBUTING docs).
- Provide explicit tasks for each recommendation, ensuring they are self-contained.

Execution Notes and Example Commands
- Example local reproduction steps (to run locally; these are not executed here):
  1. git clone https://github.com/dreef3/multi-agent-harness-test-repo.git
  2. cd multi-agent-harness-test-repo
  3. bash scripts/analyze_repo.sh
  4. less docs/repo-analysis.md
  5. cat reports/structure.json | jq '.'

- Recommended optional tools: cloc or scc for language metrics, jq for JSON formatting.

Deliverable Locations (explicit file paths)
- scripts/analyze_repo.sh — the analyzer script to run locally or in CI
- docs/repo-analysis.md — generated narrative report committed to repo
- reports/structure.json — generated structured report
- .github/workflows/repo-analysis.yml — optional CI workflow to produce artifacts

Acceptance Criteria
- The PR contains the script and generated artifacts and the outputs correctly reflect the repo content.
- The docs/repo-analysis.md is readable and provides at least 3 actionable recommendations.
- The structured JSON contains the schema described above and can be parsed by jq.

Timeline (rough)
- Implement analyzer script and generate artifacts: 1-3 hours
- Create PR and workflow: 30-60 minutes
- Manual review and refinement: 1 hour

Notes about constraints
- I did not access or enumerate the repository contents while writing this plan. The plan uses common manifest names and standard CLI tools. If the target repository uses an unusual structure, the analyzer script will be adjusted in a follow-up pass after one run.
