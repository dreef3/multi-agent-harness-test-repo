Implementation Plan: Step-by-step tasks to analyze repository structure

Repository: https://github.com/dreef3/multi-agent-harness-test-repo.git
Repository ID: cafcdb01-d111-4d1a-8191-20a46704edd2
Working directory (target for clone): /workspace/repo (sub-agents may choose a different path but update commands accordingly)

Overview
This plan breaks the work into self-contained tasks. Each task is safe to run in isolation, does not modify repository files, and writes outputs under analysis/ inside the repo. Tools used: git, ls, find, tree, rg (ripgrep) or grep, cloc, jq, npm/pnpm/yarn (if present), pip/poetry, go, mvn/gradle, lizard/radon (optional). When a tool is not available, task should record the reason in analysis/log.txt and continue.

Tasks

Task 1 — Clone repository and create analysis directory
- Commands:
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git /workspace/repo
  - cd /workspace/repo
  - mkdir -p analysis metrics analysis/dependencies
- Expected outputs:
  - /workspace/repo populated
  - /workspace/repo/analysis/ exists
- Files produced: analysis/log.txt (initial log entry)

Task 2 — Capture top-level file tree and counts
- Commands:
  - cd /workspace/repo
  - tree -L 3 -a > analysis/tree.txt || (find . -maxdepth 3 -print > analysis/tree.txt)
  - (ls -la | wc -l) > analysis/top_level_count.txt
- Expected outputs:
  - analysis/tree.txt (top-level layout)
  - analysis/top_level_count.txt

Task 3 — Detect languages and LOC
- Commands:
  - cd /workspace/repo
  - cloc --json . > analysis/languages.json || (if cloc missing: curl -sS https://raw.githubusercontent.com/AlDanial/cloc/master/build/cloc > cloc && perl cloc ... )
  - If cloc not available, run: find . -type f -name "*.py" | wc -l etc. and record in analysis/languages_fallback.txt
- Expected outputs:
  - analysis/languages.json (cloc results)

Task 4 — Identify package manifests and key files
- Commands:
  - cd /workspace/repo
  - rg --hidden --no-ignore-vcs "(^package.json$|^pyproject.toml$|requirements.txt$|Pipfile$|go.mod$|pom.xml$|build.gradle$|Cargo.toml$|Gemfile$|composer.json$)" -S -g '!*node_modules/*' --files-with-matches > analysis/manifest_files.txt || use grep/find
  - rg --hidden --no-ignore-vcs "README|LICENSE|CONTRIBUTING|CHANGELOG|Makefile" -S --files-with-matches > analysis/key_files.txt
- Expected outputs:
  - analysis/manifest_files.txt
  - analysis/key_files.txt

Task 5 — Export dependency lists per language
- For JavaScript/Node (if package.json present):
  - cd /workspace/repo
  - jq '.' package.json > analysis/dependencies/package.json.json
  - if npm available: npm --silent ci --ignore-scripts --no-audit --no-fund || true (do not install globally; run in ephemeral environment) — but to avoid installs, prefer: npm ls --prod --json > analysis/dependencies/npm_list.json || (npm may need install; if so, skip and record in log)
- For Python (if requirements or pyproject present):
  - cat requirements.txt > analysis/dependencies/requirements.txt (copy manifest)
  - if pyproject.toml: cp pyproject.toml analysis/dependencies/
  - if pipdeptree available: pipdeptree --warn silence --packages . --json-tree > analysis/dependencies/pipdeptree.json (optional)
- For Go (if go.mod present):
  - go list -m all > analysis/dependencies/go_modules.txt
- For Java/Maven (if pom.xml present):
  - mvn -q -DskipTests dependency:tree -DoutputType=text -DoutputFile=analysis/dependencies/maven_dependency_tree.txt || copy pom.xml
- For Rust (Cargo.toml):
  - cargo metadata --format-version 1 --no-deps > analysis/dependencies/cargo_metadata.json
- If any step fails due to missing tool, add a note to analysis/log.txt

Task 6 — Find CI/CD and workflow files
- Commands:
  - cd /workspace/repo
  - rg "\.github|\.gitlab-ci|circleci|azure-pipelines|wercker|Jenkinsfile" -S --files > analysis/ci_files.txt || find . -maxdepth 3 -type f -name '*.yml' -print | rg "workflow|github|gitlab|azure|circleci" > analysis/ci_files.txt
  - cp -r .github analysis/ci_github || true (only copy for quick view)
- Expected outputs:
  - analysis/ci_files.txt
  - analysis/ci_github/ (if exists)

Task 7 — Discover tests and summarize test commands
- Commands:
  - cd /workspace/repo
  - rg "pytest|unittest|mocha|jest|ava|nose|go test|mvn test|gradle test|cargo test" -S --files-with-matches > analysis/tests_files_with_matches.txt
  - For Node: check package.json scripts.test: jq -r '.scripts.test' package.json >> analysis/tests_summary.md 2>/dev/null || true
  - For Python: if setup.cfg or pyproject: rg "test" setup.cfg pyproject.toml || true
  - Try discovering tests without executing: pytest --collect-only -q > analysis/tests_pytest_collect.txt (run only if pytest installed locally and environment safe)
- Expected outputs:
  - analysis/tests_summary.md
  - analysis/tests_files_with_matches.txt

Task 8 — Lightweight static metrics and complexity
- Commands:
  - cd /workspace/repo
  - cloc results already captured; additionally run lizard for C/C++/Java/C#/Objective-C if installed: lizard . -l py -o analysis/metrics/lizard_py.txt || true
  - For Python: radon cc -s -j . > analysis/metrics/radon_cc.json (optional)
  - For JavaScript: if eslint configured, run eslint --print-config . || skip
- Expected outputs:
  - analysis/metrics/* (complexity and quick lint outputs where possible)

Task 9 — Assemble REPORT.md
- Commands:
  - Create analysis/REPORT.md with sections:
    - Repo summary (url, clone date, top-level structure summary from tree.txt)
    - Languages and LOC summary (from languages.json)
    - Key manifests and files (manifest_files.txt, key_files.txt)
    - CI and workflow summary (ci_files.txt)
    - Dependencies summary (list and paths in analysis/dependencies)
    - Tests summary and recommended next steps
    - Metrics summary and any tool failures logged in analysis/log.txt
  - A template snippet to populate REPORT.md will be generated by the runner script below (example content provided to the implementer but not executed here).
- Output:
  - analysis/REPORT.md

Task 10 — Wrap-up and upload artifacts
- Commands:
  - Ensure all analysis/* outputs are saved.
  - tar -czf analysis_artifacts.tar.gz analysis/ || true
  - Print a final log to analysis/log.txt with command list and any errors.

Execution notes and safety
- Do not run untrusted repository code. Avoid running repository binaries or scripts.
- Prefer manifest inspection over installing dependencies. If installation is required for dependency tree extraction, it must be done in an isolated ephemeral environment and only with explicit approval.
- Each task must log success/failure to analysis/log.txt for traceability.
