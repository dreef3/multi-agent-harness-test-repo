Implementation plan: repository analysis for E2E Test Repo 1 (id: ac94d952-b75d-4dfb-8445-f3bde56b444c)

Overview
This plan contains a sequence of self-contained tasks that an executor (agent or human) can run in the workspace to produce a complete structural analysis of the repository. Each task includes exact shell commands, expected output file paths (all under analysis/), and notes about optional follow-ups.

Prerequisites
- A machine with: git, bash, find, rg (ripgrep) or grep, tree (optional), cloc (optional), jq (optional), node/npm (optional), python (optional), go (optional).
- Network access to clone the repository.

Workspace policy
- All outputs should be written under analysis/ at the repository root so they can be reviewed and committed.

Tasks

Task 1 — Clone repository
- Purpose: get a local copy to analyze
- Command (run from an empty workspace directory):
  git clone --depth 1 https://github.com/dreef3/multi-agent-harness-test-repo.git repo
- Workdir: repo/
- Outputs: none (prerequisite)

Task 2 — Top-level file listing and minimal file tree
- Purpose: capture repository top-level layout
- Commands (run inside repo/):
  mkdir -p analysis
  ls -la > analysis/top-level-ls.txt
  # tree if available; limit to 3 levels to avoid huge output
  if command -v tree >/dev/null 2>&1; then
    tree -a -L 3 > analysis/file-tree.txt
  else
    find . -maxdepth 3 -print | sed 's|^./||' > analysis/file-tree.txt
  fi
- Outputs:
  analysis/top-level-ls.txt
  analysis/file-tree.txt

Task 3 — Detect language breakdown
- Purpose: determine primary languages and approximate LOC per language
- Commands:
  if command -v cloc >/dev/null 2>&1; then
    cloc --by-file --include-lang= --json . > analysis/language-breakdown.json
    cloc --csv . | sed '1,2d' > analysis/language-breakdown.csv
  else
    # Fallback: simple file-extension heuristic
    find . -type f | sed 's/.*\.//' | sort | uniq -c | sort -rn > analysis/language-extensions.txt
  fi
- Outputs:
  analysis/language-breakdown.json (preferred)
  analysis/language-breakdown.csv or analysis/language-extensions.txt (fallback)

Task 4 — Find manifest and dependency files
- Purpose: locate package manifests and lockfiles
- Commands (inside repo):
  rg --hidden --files --glob '!node_modules' 'package.json|pyproject.toml|requirements.txt|Pipfile|Pipfile.lock|poetry.lock|pyproject.toml|setup.py|go.mod|Cargo.toml|Cargo.lock|Gemfile|pom.xml|build.gradle|yarn.lock|package-lock.json|pnpm-lock.yaml|composer.json' -n --no-ignore-vcs > analysis/manifests-found.txt || true
  # Save full contents of any manifests found:
  mkdir -p analysis/manifests
  while read -r f; do
    mkdir -p "analysis/manifests/$(dirname "$f")"
    sed -n '1,400p' "$f" > "analysis/manifests/$(echo "$f" | sed 's|[/.]|_|g').txt" || true
  done < analysis/manifests-found.txt || true
- Outputs:
  analysis/manifests-found.txt
  analysis/manifests/*.txt

Task 5 — CI and workflow configs
- Purpose: find CI config files and capture basic metadata
- Commands:
  rg --hidden --files --glob '!node_modules' '\.github/workflows/*|\.gitlab-ci.yml|circleci|\.travis.yml|azure-pipelines.yml' -n --no-ignore-vcs > analysis/ci-configs-found.txt || true
  mkdir -p analysis/ci
  while read -r f; do
    echo "***** $f *****" >> analysis/ci/ci-configs.txt
    sed -n '1,400p' "$f" >> analysis/ci/ci-configs.txt || true
    echo "\n\n" >> analysis/ci/ci-configs.txt
  done < analysis/ci-configs-found.txt || true
- Outputs:
  analysis/ci-configs-found.txt
  analysis/ci/ci-configs.txt

Task 6 — Test discovery
- Purpose: find tests and suggest commands to run them
- Commands:
  rg --hidden -n --files --glob '!node_modules' '(^|/)test(s)?(/|$)|(^|/)spec(/|$)|__tests__|\.test\.|\.spec\.' --no-ignore-vcs > analysis/tests-discovered.txt || true
  # Summarize by language using manifests if present
  echo "Suggested test commands:" > analysis/tests-suggested.txt
  if [ -f package.json ]; then
    echo "node/npm: npm test (or check scripts.test in package.json)" >> analysis/tests-suggested.txt
  fi
  if [ -f pyproject.toml ] || [ -f requirements.txt ] || rg -n "pytest" package.json >/dev/null 2>&1; then
    echo "python: pytest (or python -m pytest)" >> analysis/tests-suggested.txt
  fi
  if [ -f go.mod ]; then
    echo "go: go test ./..." >> analysis/tests-suggested.txt
  fi
- Outputs:
  analysis/tests-discovered.txt
  analysis/tests-suggested.txt

Task 7 — Dependency snapshots (per-language)
- Purpose: capture high-level dependency lists without installing
- Commands:
  mkdir -p analysis/dependencies
  # Node
  if [ -f package.json ]; then
    jq '.dependencies, .devDependencies' package.json > analysis/dependencies/package.json-deps.json 2>/dev/null || cat package.json > analysis/dependencies/package.json-raw.json
  fi
  # Python
  if [ -f requirements.txt ]; then
    cp requirements.txt analysis/dependencies/requirements.txt
  elif [ -f pyproject.toml ]; then
    sed -n '1,400p' pyproject.toml > analysis/dependencies/pyproject.toml
  fi
  # Go
  if [ -f go.mod ]; then
    sed -n '1,400p' go.mod > analysis/dependencies/go.mod
  fi
  # Rust
  if [ -f Cargo.toml ]; then
    sed -n '1,400p' Cargo.toml > analysis/dependencies/Cargo.toml
  fi
- Outputs:
  analysis/dependencies/*

  rg --hidden --files --glob '!node_modules' '\.eslintrc|\.eslintrc\.js|\.prettierrc|\.prettierrc\.json|\.prettierrc\.js|\.stylelintrc|\.flake8|pyproject.toml|\.editorconfig|tox.ini' -n --no-ignore-vcs > analysis/lint-configs-found.txt || true
  mkdir -p analysis/lint

    sed -n '1,400p' "$f" >> analysis/lint/lint-configs.txt || true

  done < analysis/lint-configs-found.txt || true
- Outputs:
  analysis/lint-configs-found.txt
  analysis/lint/lint-configs.txt

  rg --hidden --files --glob '!node_modules' 'Dockerfile|docker-compose.yml|k8s|helm|Terraform|\.tf$|cloudformation' -n --no-ignore-vcs > analysis/deploy-found.txt || true

Task 10 — Produce human-readable summary and recommended next steps
- Purpose: synthesize findings into analysis/summary.md
- Commands (example template to fill automatically from collected data):
  cat > analysis/summary.md <<'EOF'
  # Repository structural analysis

  Repository: https://github.com/dreef3/multi-agent-harness-test-repo.git

  Summary (auto-generated)
  - Top-level files: see analysis/top-level-ls.txt
  - File tree (top 3 levels): analysis/file-tree.txt
  - Languages: analysis/language-breakdown.json (or language-extensions.txt)
  - Manifests found: analysis/manifests-found.txt
  - CI config(s): analysis/ci-configs-found.txt
  - Tests discovered: analysis/tests-discovered.txt

  Quick observations
  - (Populate after running the data-collection tasks)

  Recommended next steps
  1. Run the test commands suggested in analysis/tests-suggested.txt in an isolated environment.
  2. Run dependency vulnerability scans (npm audit, pip-audit, cargo-audit) as appropriate.
  3. Add/enable CI if missing; if present, run the workflows to confirm they pass.
  4. If the repo lacks a CONTRIBUTING.md or clear README sections for setup/testing, add them.

  EOF

- Outputs:
  analysis/summary.md

Optional Task 11 — Run tests (manual/explicit)
- Purpose: run test suites using appropriate tooling. This is optional because it may install dependencies and take time.
- Example commands (do in a container or clean environment):
  # Node
  if [ -f package.json ]; then
    npm ci && npm test
  fi
  # Python
  if [ -f pyproject.toml ] || [ -f requirements.txt ]; then
    python -m venv .venv && . .venv/bin/activate; pip install -r requirements.txt || pip install -e .; pytest -q
  fi
  # Go
  if [ -f go.mod ]; then
    go test ./...
  fi
