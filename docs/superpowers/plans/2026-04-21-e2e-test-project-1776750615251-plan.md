Implementation Plan: Codebase Structural Analysis for E2E Test Repo 1

Repository: multi-agent-harness-test-repo
Repo ID: 0e3d8ac6-333a-4b53-b987-30dd94883eae
Target branch: planning/codebase-analysis (harness will create PR)

Overview
- Break the work into discrete, self-contained tasks that a sub-agent can run inside a checked-out copy of the repository. Each task writes its outputs into artifacts/codebase_analysis/ so they can be reviewed and included in the PR.
- Tasks avoid network access and do not modify repository content (except adding artifact files).

Deliverables (files to be created under repository root):
- artifacts/codebase_analysis/report.md  (human-readable analysis)
- artifacts/codebase_analysis/summary.json (machine-readable summary)
- artifacts/codebase_analysis/file_list.csv (optional)
- artifacts/codebase_analysis/commands.log (exact commands run)

Task list (each task is self-contained):

Task 1 — Workspace preparation
- Purpose: Ensure artifact directory exists and record environment.
- Steps:
  1. Create artifacts/codebase_analysis/ directory.
  2. Record `uname -a`, `cat /etc/os-release`, `bash --version` into artifacts/codebase_analysis/commands.log header.
- Commands (example):
  mkdir -p artifacts/codebase_analysis
  echo "# Environment" > artifacts/codebase_analysis/commands.log
  uname -a >> artifacts/codebase_analysis/commands.log 2>&1
  cat /etc/os-release >> artifacts/codebase_analysis/commands.log 2>&1 || true
  bash --version >> artifacts/codebase_analysis/commands.log 2>&1
- Expected output: artifacts/codebase_analysis/commands.log

Task 2 — Top-level tree snapshot
- Purpose: produce a deterministic listing of top-level files and a recursive file tree limited to depth 4.
- Steps/commands:
  (use tree if available, otherwise find)
  if command -v tree >/dev/null; then tree -L 4 -a -I 'node_modules|.venv|venv|__pycache__' > artifacts/codebase_analysis/tree.txt; else find . -maxdepth 4 -path './.git' -prune -o -print | sed 's|^\./||' > artifacts/codebase_analysis/tree.txt; fi
  wc -l artifacts/codebase_analysis/tree.txt >> artifacts/codebase_analysis/commands.log
- Expected output: artifacts/codebase_analysis/tree.txt

Task 3 — File list CSV and counts
- Purpose: produce a CSV of all files with size and extension and simple counts per extension.
- Steps/commands:
  find . -type f -not -path './.git/*' -print0 | xargs -0 stat -c '%n,%s' | sed 's|^\./||' > artifacts/codebase_analysis/file_sizes.csv
  awk -F, '{n=substr($1,match($1,"\\.[^./]*$")+1); if(n=="") n="<none>"; count[n]++} END{for (e in count) print e "," count[e]}' artifacts/codebase_analysis/file_sizes.csv > artifacts/codebase_analysis/counts_by_ext.csv
- Expected outputs: file_sizes.csv, counts_by_ext.csv

Task 4 — Language and LOC estimation
- Purpose: Calculate lines-of-code per language using cloc if available, otherwise fallback to simple wc.
- Steps/commands:
  if command -v cloc >/dev/null; then cloc --json --quiet . > artifacts/codebase_analysis/cloc.json; else
  # fallback: aggregate by extension
  awk -F, '{print $1}' artifacts/codebase_analysis/file_sizes.csv | sed 's/.*\.//' | sort | uniq -c > artifacts/codebase_analysis/extension_counts.txt
  # per-extension LOC: for ext in $(awk -F, '{print $1}' file_sizes.csv | sed 's/.*\.//' | sort -u); do find . -name "*.$ext" -not -path './.git/*' -print0 | xargs -0 cat | wc -l; done > artifacts/codebase_analysis/loc_by_ext.txt
  fi
- Expected outputs: cloc.json or loc_by_ext.txt

Task 5 — Manifest discovery and parsing
- Purpose: Locate and parse package manifests and lockfiles to extract declared dependencies.
- Files to check (non-exhaustive): package.json, package-lock.json, yarn.lock, pyproject.toml, setup.cfg, requirements*.txt, Pipfile, go.mod, Cargo.toml, Gemfile, composer.json, pom.xml, build.gradle, Dockerfile(s).
- Steps/commands (examples):
  rg --hidden --glob '!**/.git/**' "^(name|version|dependencies|devDependencies|requires|package)" --pretty -n || true
  for f in package.json package-lock.json yarn.lock pyproject.toml setup.cfg requirements.txt requirements-dev.txt Pipfile Pipfile.lock go.mod Cargo.toml Gemfile composer.json pom.xml build.gradle; do if [ -f "$f" ]; then cp "$f" artifacts/codebase_analysis/; fi; done
  # Extract minimal dependency lists via jq/toml/json parsing if available
  if command -v jq >/dev/null && [ -f package.json ]; then jq '{name: .name, version: .version, dependencies: .dependencies, devDependencies: .devDependencies}' package.json > artifacts/codebase_analysis/package.json.summary.json; fi
- Expected outputs: copies of manifest files and any extracted summaries under artifacts/codebase_analysis/

Task 6 — CI and workflow inspection
- Purpose: Summarize CI configuration (.github/workflows, .gitlab-ci.yml, other CI infra) and Dockerfiles.
- Steps/commands:
  if [ -d .github/workflows ]; then find .github/workflows -type f -maxdepth 2 -print0 | xargs -0 -I{} sh -c "echo --- {}; sed -n '1,200p' {}" > artifacts/codebase_analysis/ci_workflows.txt; fi
  if rg -n --hidden "FROM .*" --glob '!**/.git/**' Dockerfile* || true; then rg -n "FROM" Dockerfile* > artifacts/codebase_analysis/dockerfiles.txt || true; fi
- Expected outputs: ci_workflows.txt, dockerfiles.txt

Task 7 — Tests and test commands
- Purpose: Detect test directories and test frameworks, extract npm/test scripts, pytest sections, gradle/maven test tasks.
- Steps/commands:
  rg --hidden --glob '!**/.git/**' "\b(pytest|unittest|jest|mocha|pytest.ini|tox.ini|karma|pytest|go test|cargo test)\b" || true
  if [ -f package.json ]; then jq '.scripts' package.json > artifacts/codebase_analysis/npm_scripts.json || true; fi
  rg --hidden -n "\bpytest\b|\bnpx jest\b|\bnpm test\b|\bmvn test\b|\bgradle test\b" || true > artifacts/codebase_analysis/test_command_hits.txt
- Expected outputs: npm_scripts.json, test_command_hits.txt

Task 8 — Static scan for sensitive files
- Purpose: Find potentially sensitive files but do NOT output their contents. Only list their paths and flag severity.
- Steps/commands:
  rg --hidden --glob '!**/.git/**' -n "(?i)aws_access_key_id|aws_secret_access_key|private_key|-----BEGIN RSA PRIVATE KEY-----|BEGIN PRIVATE KEY|cookie.secret|SECRET_KEY|ENV\s*=|password" || true > artifacts/codebase_analysis/sensitive_hits.txt
  # Post-process: redact or omit file content; only list path and matching snippet (max 120 chars) if safe.
- Expected outputs: sensitive_hits.txt

Task 10 — Produce human-readable report.md
- Purpose: Create artifacts/codebase_analysis/report.md with the sections: Executive summary, Repo snapshot, Languages & LOC, Key manifests and dependencies, CI and tests summary, Sensitive findings, Notable files/dirs, Recommendations and next steps, Commands run appendix.

Execution notes for sub-agents

- Each task must be implemented as a standalone script/step so it can be retried without side-effects.
- Do not run dependency installation or tests.
- If optional tools are missing (cloc, jq, tree), fall back to provided alternative commands.
- Respect .gitignore patterns for performance but still scan hidden files unless explicitly excluded (we exclude .git).
