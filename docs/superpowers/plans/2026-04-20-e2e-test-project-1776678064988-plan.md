Implementation Plan: Task-by-Task

Context
Repository: multi-agent-harness-test-repo
Repo ID: b53cde93-4c4b-4b20-8bf0-14137ecd2a50
URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Overview
We will perform an automated inventory then a manual inspection to create the final Markdown report. Each task is self-contained and produces artifacts under reports/analysis/ and the final report at docs/codebase-structure.md (or reports/codebase-structure.md if docs/ not present). Tasks are written so an implementer or sub-agent can run them without additional orchestration.

Task 1 — Clone repository
- Purpose: Obtain repository contents for analysis.
- Commands (run from workspace):
  git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  cd repo
- Output: local copy at ./repo/

Task 2 — Top-level inventory
- Purpose: List top-level files/dirs and record filesystem metadata.
- Commands:
  cd repo
  ls -la
  echo "" > reports/analysis/file-inventory.txt
  (ls -la | sed 's/^/TOP: /') > reports/analysis/file-inventory.txt
  tree -a -L 2 > reports/analysis/tree-level2.txt || find . -maxdepth 2 -print > reports/analysis/tree-level2.txt
- Output files:
  - reports/analysis/file-inventory.txt
  - reports/analysis/tree-level2.txt

Task 3 — Detect languages & size
- Purpose: Determine primary languages and approximate LOC.
- Commands (preferred):
  cloc --json . > reports/analysis/cloc.json
  # fallback if cloc missing:
  rg --files | sed 's/^/FILE: /' > reports/analysis/all-files.txt
  # or use github-linguist gem if available
- Output:
  - reports/analysis/cloc.json (or reports/analysis/all-files.txt)

Task 4 — Detect package manifests & build scripts
- Purpose: Find package manager files and common build/test scripts.
- Files to look for (top-level and in subdirs): package.json, pyproject.toml, requirements.txt, Pipfile, setup.py, Gemfile, go.mod, Cargo.toml, Makefile, Dockerfile, docker-compose.yml, nx.json, workspace.json, angular.json, pom.xml, build.gradle, requirements-dev.txt
- Commands:
  rg --hidden --ignore-file .gitignore --files -g '!node_modules' --glob 'package.json|pyproject.toml|requirements.txt|Pipfile|setup.py|Gemfile|go.mod|Cargo.toml|Makefile|Dockerfile|docker-compose.yml|pom.xml|build.gradle' > reports/analysis/manifests.txt
  # Inspect package.json scripts if present
  if [ -f package.json ]; then jq '{scripts: .scripts, name: .name, version: .version}' package.json > reports/analysis/package-json-summary.json || true; fi
- Output:
  - reports/analysis/manifests.txt
  - reports/analysis/package-json-summary.json (if applicable)

Task 5 — Detect CI and workflows
- Purpose: Find CI configs (.github/workflows, .gitlab-ci.yml, circleci config).
- Commands:
  rg --hidden --files --glob '.github/**' --glob '.gitlab-ci.yml' --glob '.circleci/**' > reports/analysis/ci-files.txt || true
  tree -a .github/workflows -L 1 > reports/analysis/github-workflows.txt || true
- Output:
  - reports/analysis/ci-files.txt
  - reports/analysis/github-workflows.txt

Task 6 — Locate tests and e2e directories
- Purpose: Find tests, e2e, spec, integration directories and test frameworks used.
- Commands:
  rg --hidden --files --glob 'tests/**' --glob 'test/**' --glob 'spec/**' --glob 'e2e/**' > reports/analysis/test-files.txt || true
  # Inspect package.json devDependencies for test frameworks
  if [ -f package.json ]; then jq '.devDependencies, .dependencies' package.json > reports/analysis/package-deps.json || true; fi
- Output:
  - reports/analysis/test-files.txt
  - reports/analysis/package-deps.json

Task 7 — Detect containerization and deployment descriptors
- Purpose: Find Dockerfile(s), docker-compose.yml, k8s manifests, helm charts.
- Commands:
  rg --hidden --files --glob 'Dockerfile' --glob 'docker-compose.yml' --glob 'helm/**' --glob 'charts/**' --glob 'k8s/**' > reports/analysis/container-files.txt || true
- Output:
  - reports/analysis/container-files.txt

Task 8 — Summarize notable scripts and entry points
- Purpose: Identify main entry points (main.py, index.js, cmd/ directories, README run commands).
- Commands:
  rg --hidden --files --glob 'main.*' --glob 'index.*' --glob 'cmd/**' > reports/analysis/entrypoints.txt || true
  head -n 200 README.md > reports/analysis/readme-head.txt || true
- Output:
  - reports/analysis/entrypoints.txt
  - reports/analysis/readme-head.txt

Task 9 — Compile Markdown report
- Purpose: Convert all findings into docs/codebase-structure.md
- Structure for docs/codebase-structure.md:
  - Title, repo url & commit hash used for analysis
  - Top-level inventory (excerpt)
  - Languages and LOC summary (cloc.json excerpt)
  - Package manifests found with paths
  - CI/workflow summary
  - Tests and test frameworks
  - Containerization and deployment files
  - Notable scripts, build & run commands
  - Recommendations and next steps (dependency updates, add CI badge, add CONTRIBUTING, test coverage)

- Commands:
  mkdir -p docs || true
  python3 - <<'PY'
import json
from pathlib import Path
r=Path('reports/analysis')
out=Path('docs/codebase-structure.md')
# Create a simple compiled report (example template)
out.write_text('# Codebase structure analysis\n\n')
# Append sections based on files that exist
for fname in ['file-inventory.txt','tree-level2.txt','cloc.json','manifests.txt','ci-files.txt','test-files.txt','container-files.txt','entrypoints.txt']:
    p=r/fname if isinstance(fname,str) else None
PY
  # (Implementer should fill template assembling the pieces; this step is non-destructive.)

- Output:
  - docs/codebase-structure.md (final human-readable report)

Task 10 — Commit report and artifacts
- Purpose: Add the reports and report markdown to a branch and open PR.
- Commands:
  git checkout -b analysis/codebase-structure
  git add reports/analysis/* docs/codebase-structure.md || true
  git commit -m "Add codebase structure analysis report" || true
  git push -u origin analysis/codebase-structure
- Note: The harness will create the PR automatically (per project workflow). If not, maintainers can create PR from the pushed branch.
