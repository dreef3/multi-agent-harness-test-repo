Task 1 — Checkout repository (precondition)
- Purpose: create a local workspace of the target repo for analysis.
- Steps (example commands):
  - git clone https://github.com/dreef3/multi-agent-harness-test-repo.git repo
  - cd repo
- Outputs: working copy for subsequent tasks.
- Note: The harness/dispatcher will handle credentials; include repoId when dispatching.

Task 2 — Produce full file tree
- Purpose: record the top-level and full tree of files with file sizes and types.
- Commands:
  - find . -path ./\.git -prune -o -type f -printf '%P|%s|%TY-%Tm-%Td %TH:%TM:%TS\n' > analysis/file-list.txt
  - python3 - <<'PY'
import json,sys
with open('analysis/file-list.txt') as f:
    items=[l.strip().split('|') for l in f if l.strip()]
json.dump([{'path':p,'size':int(s),'mtime':m} for p,s,m in items], open('analysis/file-tree.json','w'), indent=2)
PY
- Expected outputs: analysis/file-tree.json and analysis/file-list.txt

Task 3 — Detect languages and line counts
- Purpose: identify languages and approximate code size per language.
- Preferred: cloc (if available). Fallback: simple file extension grouping and wc -l.
- Commands (cloc):
  - cloc --json . > analysis/cloc.json || true
- Fallback commands:
  - python3 - <<'PY'
import os,sys,subprocess,json
ext_map={}
for root,_,files in os.walk('.'):
    if '.git' in root.split(os.sep):
        continue
    for f in files:
        p=os.path.join(root,f)
        try:
            lines=sum(1 for _ in open(p,'rb'))
        except Exception:
            continue
        ext=os.path.splitext(f)[1].lower() or '<noext>'
        ext_map.setdefault(ext,0)
        ext_map[ext]+=lines
print(json.dumps({'by_extension':ext_map},indent=2))
PY > analysis/languages.json
- Expected outputs: analysis/cloc.json or analysis/languages.json

Task 4 — Discover package/manifest files and parse basic metadata
- Purpose: find and parse common manifest files to identify package managers and top-level scripts.
- Targets: package.json, pyproject.toml, requirements.txt, Pipfile, setup.py, go.mod, Cargo.toml, Gemfile, composer.json, build.gradle, pom.xml
- Commands (examples):
  - for f in package.json pyproject.toml requirements.txt Pipfile setup.py go.mod Cargo.toml Gemfile composer.json build.gradle pom.xml; do if [ -f "$f" ]; then mkdir -p analysis/manifests; cp "$f" analysis/manifests/; fi; done
  - python3 - <<'PY'
import json,glob
out={}
for path in glob.glob('analysis/manifests/*'):
    name=path.split('/')[-1]
    try:
        with open(path,'r',encoding='utf-8') as f:
            out[name]=f.read()
    except Exception as e:
        out[name]=None
json.dump(out,open('analysis/manifests.json','w'),indent=2)
PY
- Expected outputs: analysis/manifests.json and copies under analysis/manifests/

Task 5 — Dependency extraction per ecosystem
- Purpose: extract declared dependencies for major ecosystems.
- Commands (examples):
  - If package.json exists:
    - jq '{name: .name, version: .version, dependencies:.dependencies, devDependencies:.devDependencies, scripts:.scripts}' package.json > analysis/package.json.summary.json || true
  - If pyproject.toml exists:
    - python3 - <<'PY'
import tomllib, json
p='pyproject.toml'
if os.path.exists(p):
    with open(p,'rb') as f:
        data=tomllib.load(f)
    json.dump(data, open('analysis/pyproject.json','w'), indent=2)
PY
  - For requirements*.txt: cat into analysis/requirements-files.txt
  - For go.mod: cp go.mod analysis/manifests/ and run 'go list -m all' if go toolchain available.
- Expected outputs: analysis/deps.json and per-ecosystem summaries (e.g., analysis/package.json.summary.json)

Task 6 — Test discovery and summary
- Purpose: find test directories and common test files, infer test commands.
- Commands:
  - find . -type d \( -name test -o -name tests -o -name spec -o -name __tests__ \) -print > analysis/test-dirs.txt
  - find . -type f \( -name '*_test.go' -o -name 'test_*.py' -o -name '*_spec.rb' -o -name '*test.js' \) -print > analysis/test-files.txt
  - Summarize package.json "test" script if present (see Task 5). Create analysis/tests-summary.json with fields: test_dirs, test_files, suggested_test_command
- Expected outputs: analysis/tests-summary.json

Task 7 — CI and workflow inspection
- Purpose: detect CI definitions and summarize jobs and triggers.
- Commands:
  - if [ -d .github/workflows ]; then jq -s '.' .github/workflows/*.yml > analysis/ci-workflows.json || true; fi
  - list other CI files: CircleCI (.circleci/config.yml), GitLab (.gitlab-ci.yml), Travis (.travis.yml)
  - cp present files into analysis/ci-manifests/
- Expected outputs: analysis/ci-summary.json summarizing detected workflows and top-level job names and triggers.

Task 8 — Size and hotspot analysis
- Purpose: identify large files and directories (by size) and top contributors to code size.
- Commands:
  - du -sh * | sort -h > analysis/top-level-sizes.txt
  - find . -type f -not -path './.git/*' -printf '%s %p\n' | sort -rn | head -n 50 > analysis/largest-files.txt
  - Convert to JSON: python3 script to build sizes.json
- Expected outputs: analysis/sizes.json, analysis/largest-files.txt

- Expected outputs: docs/codebase-analysis.md
