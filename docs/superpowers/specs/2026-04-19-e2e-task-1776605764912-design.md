# Design Spec: Add repository marker file

Goal

- Add a simple marker file to the repository to indicate it has been processed/seeded by the multi-agent harness. This file is intentionally minimal and non-invasive.

Approach

- Add a single text marker file at the repository root named `PI_MARKER`.
- The file will contain metadata: repository identifier, a short purpose note, and a placeholder for creation timestamp. The marker is human-readable and requires no code execution or configuration changes.

Key decisions

- Filename: `PI_MARKER` at the repository root. Chosen to avoid hidden-dot-file handling differences and to be clearly visible to humans and scripts.
- Content: small, static metadata block containing repository UUID (98a13732-555a-434f-84f3-a3870f2070dc), purpose, and a creation timestamp placeholder. Keeping content minimal reduces maintenance.
- Branching/PR: Implementation will be made on a short-lived branch (`planning/add-pi-marker`) with a clear commit message and PR title. No other files will be modified.

Constraints

- Do not modify any source code, CI configuration, or tests.
- Keep the change atomic and readable.

Success criteria

- A file named `PI_MARKER` exists at the repository root with the specified metadata.
- A PR is opened with a clear description; the change is small and ready to merge.
