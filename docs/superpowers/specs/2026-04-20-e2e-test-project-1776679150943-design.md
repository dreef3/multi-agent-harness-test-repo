Design spec: Analyze the codebase structure for the E2E Test Repo (id: 17d2f7cf-8afd-4c9e-a631-3c349f50207d)

Goal
- Produce a thorough, actionable analysis of the repository structure and contents. Deliver a concise executive summary and a detailed breakdown of files, directories, languages, dependency manifests, CI, tests, build scripts, and potential risk/maintenance issues.

Approach
- Static, non-destructive analysis of the repo filesystem. The analysis will be automated by a set of tasks that inspect the top-level tree, detect languages and sizes (by LOC), parse common package manifests (package.json, pyproject.toml, go.mod, etc.), locate CI and Docker artifacts, identify test suites and coverage config, and highlight notable files (README, LICENSE, CONTRIBUTING, CODEOWNERS). Produce machine-readable artifacts (JSON) and human-readable reports (Markdown). Create a short executive summary plus appendices.

Key decisions
- Non-invasive: no changes to the repo; read-only analysis and report generation.
- Prioritize common language ecosystems (JavaScript/TypeScript, Python, Go, Java, Rust) but keep generic fallbacks.
- Generate both summary (for humans) and artifact files (for downstream tools to consume).
- Flag potential issues (missing license, no CI, no tests, large binary blobs, secrets) but do not attempt to remediate.

Outputs
- reports/structure_summary.md — human-readable summary and findings.
- reports/structure_details.json — structured data: top-level dirs/files, language breakdown, key manifests, CI files list, tests found, sizes.
- reports/top_files_by_size.txt — largest files.
- reports/dependency_manifests.md — extracted dependency snapshots (package.json, pyproject, go.mod, etc.).
- Optional: reports/gitleaks_report.json if secrets scan is enabled.

Constraints
- This planning document is not the analysis itself. It defines the tasks and outputs so implementation agents can run them against the repo.
