# Design spec: Add simple repository marker file

Goal

Create a simple, human- and machine-readable marker file in the repository root so automated agents and humans can quickly detect that the repository has been touched by the harness.

Approach

- Add a single hidden marker file at the repository root named `.repo-marker`.
- The file will contain a tiny structured payload (3 lines) with the repository UUID, the agent/tool that created it, and an ISO-8601 timestamp placeholder to be filled by the implementing agent.
- Keep the change minimal: no other files modified, no code changes, single small commit and PR.

Key decisions

- File name: `.repo-marker` (hidden dotfile to avoid clutter, but obvious purpose).
- Location: repository root to make it easy to find programmatically.
- Content format: plain text with a minimal key: value layout for readability and simple parsing. Example:

  repo_id: 094b88e4-3ac6-4e3f-a1a3-781f5102a0d9
  created_by: planning-agent
  created_at: <ISO-8601 timestamp>

- Commit and PR: create a dedicated branch (e.g., `add-repo-marker`), commit with message `Add .repo-marker`, open a PR with title `Add .repo-marker marker file`.

Constraints

- Do not modify any other repository files.
- Keep file content minimal and ASCII-safe.
- Let the implementation task populate the actual timestamp at commit time.
