Design spec: Analyze codebase structure for repository "multi-agent-harness-test-repo" (id: 8f508779-4d1f-4e5c-a9a1-0c646a59d68d)

Goal
- Produce a clear, actionable report describing the repository's structure: top-level files and directories, languages & primary packages, build/test/CI configuration, notable code areas (src, tests, examples), documentation (README, docs), license, and any immediate issues (missing CI, monorepo indicators, large binary blobs).

Approach
- Perform a read-only static analysis of the repository contents and metadata. The planned implementation will:
  - Fetch repository metadata (default branch, commit SHA, topics) via GitHub API.
  - Clone the repository (read-only) to a workspace for file-system inspection.
  - Generate a directory tree and inventory of files by type and language.
  - Detect package managers, build files, CI workflows, and test frameworks (package.json, requirements.txt, pyproject.toml, go.mod, pom.xml, .github/workflows/ etc.).
  - Run lightweight heuristics to detect monorepos, submodules, git-lfs usage, large files, and presence of end-to-end or integration tests.
  - Produce a human-readable report (Markdown) plus a machine-readable summary (JSON) listing languages, sizes, and locations.

Key decisions
- Read-only analysis: no changes will be made to the target repository.
- Use cloning in a disposable workspace for complete file-system analysis (this handles large trees reliably); alternatively, GitHub tree API can be used when cloning is not allowed.
- Prioritize quick heuristics and structural discovery over deep static analysis (no building or running tests).

Deliverables
- Analysis report: reports/8f508779-4d1f-4e5c-a9a1-0c646a59d68d-analysis.md
- Machine-readable summary: reports/8f508779-4d1f-4e5c-a9a1-0c646a59d68d-summary.json
- Tooling: tools/repo-inspector/ (scripts + README) to reproduce the analysis
- CI job (optional) to run the inspector on demand

Risks and mitigations
- Very large repositories or LFS objects: detect via git-lfs config and avoid attempting to download large objects; use GitHub API fallback.
- Private or rate-limited GitHub API access: document required token scopes and fall back to unauthenticated tree endpoints where possible.

Acceptance criteria
- A Markdown report describing: top-level layout, languages with byte counts, key config files found (with file paths), tests and CI presence, and recommendations for maintainers.
- A JSON summary with a list of file counts by extension, directory sizes, and detected package manifests.
