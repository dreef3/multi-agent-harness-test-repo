# Design Spec: Add repository marker file

Goal
- Create a simple marker file at the repository root so automated systems and humans can detect that the planning step ran and left a fingerprint in the repository.

Approach
- Add a lightweight text marker file at the repository root named `MARKER.txt` containing a short, single-line message identifying the repository and the purpose of the file. The file will be created on a dedicated branch and submitted as a small, single-purpose PR.

Key decisions
- File name: `MARKER.txt` (root-level) — visible and unobtrusive; not hidden so reviewers can easily find it.
- File contents: a short, human-readable one-line statement such as:
  "marker: created by planning agent for multi-agent-harness-test-repo"
- Commit/PR: create a branch named `add/marker-file` and open a PR titled "chore: add MARKER.txt" with a concise description.
- No code changes beyond adding this single file; no tests required.

Security & Policy
- The file contains no secrets or executable content.
- The change is low-risk and intended for bookkeeping/CI detection.

Success criteria
- PR is opened and contains exactly one new file `MARKER.txt` at repository root with the agreed content.
- After merge, the repository root contains `MARKER.txt` with content matching the plan.
