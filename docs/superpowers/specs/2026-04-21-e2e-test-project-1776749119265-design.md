Design Spec: Codebase Structure Analysis

Goal

Perform a high-level analysis of the repository "multi-agent-harness-test-repo" (repo id: 1f0615d1-5163-4945-8e56-8852f4c5696c) and produce a concise report describing the codebase structure, languages, package managers, tests, CI, notable files, and suggested next steps for deeper inspection.

Approach

1. Produce a reproducible plan that an automated sub-agent (or human) can execute without manual cloning steps in this planning phase. The plan will list exact commands and files to inspect.
2. Identify top-level directories and files, detect primary programming languages and build systems, detect package manifest files, and locate tests and CI configuration files.
3. Produce output artifacts in the repository root: CODEBASE_SUMMARY.md (human-readable summary) and CODEBASE_TREE.json (machine-readable tree and metadata) so downstream agents can consume them.
4. Define acceptance criteria for the analysis: summary includes languages, dependency managers, test frameworks, CI files, top-level structure, and a list of files/locations requiring manual review.

Key decisions

- Non-invasive: The spec does not modify source code. It only reads and produces summary files.
- Deterministic commands: The plan contains exact shell commands (ls, find, rg, file globs) so an implementer can reproduce the analysis reliably.
- Output artifacts and their locations: CODEBASE_SUMMARY.md and CODEBASE_TREE.json at repository root. These will be created by implementation tasks.
- Scope: This first pass is structural and surface-level (file types and locations). Deeper tasks (dependency vulnerability scanning, running tests, building docs) are listed as optional follow-ups but not executed by this plan.

Deliverables

- CODEBASE_SUMMARY.md: human-readable summary with findings and recommended next steps.
- CODEBASE_TREE.json: JSON describing the top-level filesystem tree, counts of source files per language, and paths to manifest/CI/test files.
- Implementation plan (this document plus the plan below) written to the planning branch and available as a PR for review.

Constraints

- Do not clone repositories or execute commands against remote hosts in this planning phase — the plan is to be saved and used by implementers or sub-agents.
- The plan must be self-contained and reproducible given the repository ID above.
