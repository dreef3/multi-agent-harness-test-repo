# Design Spec: Add a simple marker file to repository

## Goal
Create a small, clearly identifiable marker file at the root of the repository to signal that the repository has been processed/annotated by the multi-agent harness. The marker is intentionally simple and non-invasive.

## Approach
- Add a single text file named E2E_MARKER.txt at the repository root.
- The file will contain a one-line human-readable message and metadata: repository name (E2E Test Repo 1), repository id (786cca37-f950-4c24-8773-3183518f2732), and an ISO-8601 timestamp placeholder.
- Commit the new file on a short-lived feature branch. The harness will create the PR from the planning branch after this spec/plan is accepted.

## Key decisions
- File name: E2E_MARKER.txt (explicit and visible at the repository root). Using a visible filename rather than a hidden dotfile makes the marker easy to locate.
- Content: minimal informational text including repo id so automated systems can validate the marker belongs to this repository.
- Location: repository root to maximize discoverability and avoid affecting subdirectories or build systems.

## Acceptance criteria
- A new file exists at repository root: E2E_MARKER.txt
- The file contains the repository id: 786cca37-f950-4c24-8773-3183518f2732
- The file contains a brief human-readable message indicating purpose

## Risks & mitigations
- Risk: Some repositories may have policies preventing new files in the root. Mitigation: The file is small and non-executable; if this is a concern the implementer can instead place the marker under a non-invasive directory (e.g., .pi/) — but current plan uses the repository root per instruction.
