# Design Spec: Add a simple repository marker file

Goal

Create a simple marker file in the repository to indicate the presence of this automated planning step. The marker will be a single, small text file placed at the repository root so it is easy to find and verify.

Approach

- Create a file named REPO_MARKER at the repository root. The file will be plain text and contain a short, human-readable message including the repository name and the repository UUID provided in the task description. The file will be committed to the repository default branch (main).
- The marker content will include a creation date placeholder (YYYY-MM-DD) so the implementer can stamp the actual date when committing.

Key decisions

- Filename: REPO_MARKER (no leading dot) so it's visible in listings and not hidden by default.
- Content: a one- to three-line human-readable message containing the repository display name, repository UUID, and creation date placeholder.
- Branch: target the repository default branch (main). If repository policies require a feature branch + PR, the implementer should create a short-lived branch and open a PR; otherwise commit directly if CI/policy allows. The planning harness will open the planning PR for this spec/plan; the implementation task should follow repository conventions.
- If a file named REPO_MARKER already exists, the implementation should not overwrite it silently: either leave it as-is or append a new marker line indicating this second attempt, depending on repository policy. The plan includes an idempotent safe path: create file only if missing, otherwise add a companion file REPO_MARKER.1 or update with a timestamped line.

Success criteria

- A file named REPO_MARKER exists at the repository root on the target branch.
- The file contains the repository name and UUID and the creation date (YYYY-MM-DD replaced by the actual date).
