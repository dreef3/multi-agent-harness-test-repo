Design spec: Analyze codebase structure for repository "multi-agent-harness-test-repo" (id: 0fdf7599-6b35-4ed5-99e5-1adaade7bb71).

Goal
- Produce a concise, machine- and human-readable analysis of the repository structure showing: top-level layout, primary languages and frameworks, dependency manifests, CI/workflow configuration, test suites (unit/e2e), build/run entrypoints, notable scripts, and areas requiring attention (e.g., missing README, no CI, unclear run instructions).

Approach
- Perform a non-invasive static inventory of the repository filesystem (no code execution).
- Detect language and package managers by presence of common manifest files (package.json, pyproject.toml, go.mod, etc.).
- Identify CI and automation by enumerating .github/workflows/, .gitlab-ci.yml, Dockerfile(s), and Makefile or scripts/.
- Locate tests by searching for tests/, e2e/, spec/, *Test files, and framework-specific test markers.
- Extract run/build commands from README, package scripts, and common CI job steps.
- Produce two deliverables: a Markdown human report and a JSON machine-readable inventory.

Key decisions
- Report format: primary deliverable will be Markdown (easy to read) plus a JSON file for tooling. Markdown will include a directory tree at depth 2, language detection summary, CI status, test inventory, and suggestions.
- Scope: limit to static analysis only (no running tests or building). This keeps analysis fast and safe.
- Automation: provide exact bash commands in the implementation plan so sub-agents can run the analysis reproducibly.

Deliverables
- codebase-analysis.md (Markdown report)
- codebase-inventory.json (structured inventory)
- Optional: short summary to paste in PR description or issue.

Success criteria
- The Markdown contains: top-level file list, top-level directories, likely primary language(s), dependency files found, CI/workflows found, test folders found, build/run entrypoints, and suggested next steps.
- The JSON contains keys: files, directories, languages, manifests, ci_files, test_paths, dockerfiles, entrypoints, recommendations.
