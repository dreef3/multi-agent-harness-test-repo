Design spec: Analyze codebase structure for the repository "E2E Test Repo" (id: 86c2a38d-26ed-4e65-856e-eb7eee498e14, URL: https://github.com/dreef3/multi-agent-harness-test-repo.git)

Goal
- Produce a concise, actionable analysis of the repository's structure: top-level directories and files, languages and major frameworks used, package and build tooling, test layout and CI, Docker/containers, important scripts, and any potential red flags (missing license, no CI, unclear tests).
- Deliver both a human-readable report (report.md) and a machine-readable summary (report.json).

Approach
- Because the planning agent cannot clone or inspect the repository here, the plan defines a sequence of reproducible tasks for an implementation agent to run in a workspace where the repository is available.
- The implementation will: clone the repository, enumerate files and directories, detect languages and package manifests, locate CI and test configurations, detect Dockerfiles and container config, and summarize scripts and dependencies.
- Produce a prioritized summary and suggested next steps (e.g., run tests, add missing CI, add license) plus raw artifacts (file lists, dependency manifests).

Key decisions
- Output formats: markdown summary for humans (report.md) and JSON for tooling (report.json). Also save raw file lists and a snapshot of package manifests.
- Detection heuristics: look for common files (package.json, pyproject.toml, go.mod, Cargo.toml, pom.xml), CI folders (.github/workflows), test folders (test, tests, cypress, e2e), Dockerfile(s), and top-level scripts (Makefile, scripts/).
- The plan emphasizes reproducible commands (git clone, tree, rg/rg, cloc) and separation of tasks so sub-agents can run them independently.

Deliverables
- report.md (human summary)
- report.json (structured summary)
- artifacts/: file-list.txt, top-files.txt, dependency-manifests/*, ci-files.txt, test-files.txt
- commands and logs used to produce the report

Constraints
- The planning agent does not access the repository; the plan must be executed by an implementation agent that has network/clone permissions.
