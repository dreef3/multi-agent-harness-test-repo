Design spec: Create a simple repository marker file

Goal
- Add a minimal marker file to the repository to indicate presence of this project and to serve as a predictable change for CI/automation tests.

Approach
- Create a single plain-text marker file at the repository root named MARKER.md. The file will contain one short, human-readable line indicating that the file is a marker and the date the plan was created.
- Open a short-lived feature branch and a PR that adds the marker file. The PR title and body will explain the purpose.

Key decisions
- File name and location: MARKER.md at the repository root (visible and easy to find). A dotfile (e.g., .marker) was considered but MARKER.md is more visible in web UI.
- Content: a single line identifying the repository and the creation date (UTC). This keeps the change minimal and non-executable.
- Branch/PR naming: branch name add/marker-file or add-marker-file; PR title: "Add simple repository marker file".
- No code changes or tests required.

Success criteria
- A PR exists adding the file MARKER.md to the repository root with the agreed content.
- The PR is mergeable (no additional changes required) and the file appears in main after merge.

Risks and mitigations
- Very low risk: adding a plain Markdown file does not affect code or CI. If repository has branch protections, follow required review/CI steps.
