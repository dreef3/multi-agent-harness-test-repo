# Design Spec: Add repository marker file

Goal

- Create a simple, single marker file in the repository to indicate that the planning step has run and to make it easy for automated checks to detect the change.

Approach

- Add a plain text marker file at the repository root named `REPO_MARKER.txt`.
- The file will contain a brief single-line statement identifying the repository, purpose, and creation date.
- Changes will be proposed on a feature branch `add-repo-marker` with a clear commit message and PR title.

Key Decisions

- File name: `REPO_MARKER.txt` chosen for clarity and to avoid hidden-file issues on some platforms.
- Location: repository root to make discovery trivial for tests and scripts.
- Contents: a single human-readable line including repository name and creation date (e.g., "E2E Test Repo 1 marker — created by planning agent on 2026-04-20").
- Branch and PR metadata: branch `add-repo-marker`, commit message `Add repository marker file`, PR title `Add REPO_MARKER.txt` with a short description.

Non-goals

- No code changes, tests, or CI modifications.
- No additional files or automation beyond the single marker file.
