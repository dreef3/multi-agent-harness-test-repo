Design spec: Codebase structure analysis for repository af589922-5126-4711-934b-37c7d5619136 (https://github.com/dreef3/multi-agent-harness-test-repo)

Goal
- Produce a clear, reproducible analysis of the repository's structure, languages, key manifests and build tooling, CI configuration, tests, and notable files (README, LICENSE, docs). Deliver a short human-readable summary plus machine-friendly artifacts (JSON) that downstream agents or humans can use.

Approach
- Non‑destructive, read‑only analysis performed in-repo.
- Collect a full file list, top-level tree, and targeted scans for common language manifests and CI config files.
- Identify primary languages, package managers, build systems, and test frameworks by looking for standard files (package.json, pyproject.toml, go.mod, Cargo.toml, pom.xml, etc.).
- Locate CI configurations (.github/workflows, .circleci, Jenkinsfile) and containerization (Dockerfile, docker-compose.yml).
- Enumerate test directories and known test file patterns (tests/, spec/, __tests__, *.spec.*, cypress/, e2e/).
- Extract dependency lists from manifests where possible, but do not attempt to install or resolve packages.
- Produce outputs: human summary (analysis/summary.md), file tree (analysis/tree.txt), language and manifest summary (analysis/lang-manifest.json), CI and test inventory (analysis/ci-tests.json), dependency snippets (analysis/dependencies.json) and a short list of recommended next steps.

Key decisions
- Prioritize common languages and manifest files; do not attempt to parse every exotic or generated file type.
- Use simple CLI commands (find, rg/grep, head) for robustness in diverse environments rather than requiring specific tools (cloc, linguist) that may not be available. If tools exist, optional use is documented.
- Provide both human summary and structured JSON outputs so other agents can consume findings.

Deliverables
- analysis/summary.md — concise human-readable summary of structure, languages, CI, tests, and recommendations.
- analysis/tree.txt — top-level tree (depth 3) of repo files.
- analysis/lang-manifest.json — list of detected languages and manifest files.
- analysis/ci-tests.json — list of CI configs and test directories/patterns.
- analysis/dependencies.json — extracted dependency lists or manifest excerpts.
- README note describing how the analysis was produced and commands used.

Success criteria
- All deliverables created in the repository under analysis/.
- Summary correctly identifies at least the primary language(s), presence/absence of CI, and test directories.
- Outputs are machine-readable and include the commands run to generate them.

Constraints
- Do not modify repository source files (analysis files may be added in analysis/).
- No network calls or package installs; analysis is purely file-based.

Estimated effort
- Quick scan: ~15–30 minutes for a medium-sized repo; longer if multiple languages or large dependency manifests need parsing.
