Design spec: Analyze codebase structure for repository "E2E Test Repo 1"

Repository
- Name: E2E Test Repo 1
- ID: 136ad42f-fdc8-41fd-a515-d23075abe901
- URL: https://github.com/dreef3/multi-agent-harness-test-repo.git
- Default branch: main

Goal
- Produce a clear, reproducible analysis of the repository's codebase structure and high-level characteristics so humans and automated agents can quickly understand: top-level layout, languages used, dependency manifests, build/test scripts, CI/Docker usage, notable sizes and hotspots, and suggestions for next actions.

Approach
- Run an automated, repeatable scan of the repository using standard command-line tools and language/package-specific inspectors.
- Produce both raw artifacts (tree listings, cloc/tokei output, dependency lists, CI and Dockerfile detection) and a concise human-readable summary report and a machine-readable JSON summary.
- Store results under an analysis/ directory in the repository to keep outputs versioned and easy to review.

Key decisions
- Non-destructive: the analysis will not change source files. It will create new files under analysis/ only.
- Language-agnostic: start with general tools (tree/ls/find, cloc/tokei, ripgrep) then branch into language-specific dependency inspectors only if corresponding manifest files are present (package.json, requirements.txt, Pipfile, pyproject.toml, go.mod, pom.xml, build.gradle, Cargo.toml, Gemfile, etc.).
- CI and Docker detection: look for common CI config files (.github/workflows, .gitlab-ci.yml, .circleci/config.yml) and Dockerfile(s) at repo root and in subdirectories.
- Test detection: search for common test directories and test commands in package manifests/CI files.
- Produce visual artifacts (dependency tree text files, optional module graph SVG) only if the relevant tools are available and safe to run.

Deliverables
- analysis/structure_report.md — human-readable summary and recommendations.
- analysis/structure_raw/ — raw outputs: tree.txt, cloc.json, language_summary.txt, file_manifest.txt.
- analysis/dependencies/ — dependency listings per language (e.g., npm-deps.txt, pip-deps.txt, maven-deps.txt).
- analysis/ci_and_docker.txt — detected CI configs and Dockerfiles.
- analysis/summary.json — machine-readable summary: top-level dirs, languages, sizes, tests present, recommended next steps.

Constraints
- The planning agent will not perform the scan itself; the implementation plan defines self-contained tasks for sub-agents to run against the repository. The plan assumes standard UNIX utilities and commonly-available language tools (cloc/tokei, npm, pip, mvn, go) but includes fallbacks (just list files) if language tools are not available.

Success criteria
- A completed analysis/ directory committed to a branch with the above artifacts and a short PR explaining findings.
