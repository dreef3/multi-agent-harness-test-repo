Design Spec: Add a simple repository marker file

Goal
- Create a small marker file at the repository root to indicate that the repository has been touched by the multi-agent harness. This file will be human-readable and include minimal metadata (purpose, repository id, creation timestamp placeholder).

Approach
- Add a single file named REPO_MARKER.md at the repository root. The file will contain a short description, the repository id (d4d1a0d6-7f00-4a1b-b2f4-67cdaa66afde), and a created-by line indicating the multi-agent harness and a placeholder for an ISO-8601 timestamp.
- A short commit will add the file on a feature branch named add/repo-marker and open a PR against main. The PR title and body will explain the purpose and include the marker contents.

Key decisions
- Use README-style Markdown (REPO_MARKER.md) for readability across tools and viewers.
- Keep contents minimal and stable to avoid churn: include purpose, repository id, and a timestamp placeholder to be set at commit time by the implementing agent.
- Place file at repository root so it is easy to find.

Success criteria
- A new file REPO_MARKER.md exists on a branch and a PR is opened against main.
- The file contains: a one-line description, the repository id, and a created timestamp line.

Notes
- This spec only describes the change; implementation will be performed in a follow-up task/PR.
