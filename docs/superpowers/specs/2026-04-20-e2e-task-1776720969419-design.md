# Design Spec: Add simple repository marker file

## Goal
Create a small, human-readable marker file in the repository root to signal that the repository has been processed by the multi-agent harness. The marker is intentionally simple and non-invasive.

## Approach
- Add a single text file at the repository root named `REPO_MARKER.md`.
- The file will contain a short message including the repository id, the purpose of the marker, and an ISO-8601 creation timestamp.
- The change will be made on a dedicated branch and submitted as a pull request for review.

## Key decisions
- File name: `REPO_MARKER.md` (explicit, visible in repo listing; easier for humans to find than dotfiles).
- Location: repository root (so it's easy to locate programmatically and manually).
- Contents: minimal metadata: repository id, purpose string, creation timestamp. No secrets or generated data.
- Commit & PR conventions: branch `add-repo-marker`, commit message `Add REPO_MARKER.md`, PR title `Add simple repository marker file`.

## Acceptance criteria
- A file `REPO_MARKER.md` exists at the repository root in the main branch after merge.
- The file contains the repository ID and a creation timestamp.
- PR exists with the specified title and description.
