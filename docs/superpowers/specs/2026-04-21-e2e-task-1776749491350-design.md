# Design Spec: Add simple marker file to repository

Goal
- Add a simple marker file to the repository `E2E Test Repo 1` to indicate the presence of the multi-agent harness/PI planning step. The marker is intentionally minimal and non-invasive.

Approach
- Create a single file at the repository root named `.PI_MARKER` (hidden file). The file will contain a few lines of plain text: repository identifier, creation timestamp placeholder, and a short purpose statement. No changes to source code, tests, or build configuration.

Key decisions
- Filename: `.PI_MARKER` (hidden) to avoid cluttering repo listings while still being easily findable.
- File location: repository root so CI and tools can easily locate it.
- Content: plain text with structured, human-readable fields. Example:
  - Repository: f3f15f7e-3c94-47ff-bafd-2fc19291be77
  - Created-by: pi-planning-agent
  - Created-at: <ISO-8601 timestamp>
  - Purpose: Marker file indicating planning step completed by the multi-agent harness.
- Branching and PR: Create a short-lived feature branch from `main` and open a pull request describing the addition. CI should be unaffected.

Risks and mitigations
- Risk: accidental commit to main. Mitigation: use a named feature branch (e.g., `feature/add-pi-marker`).
- Risk: name collision with existing files. Mitigation: choose a unique, dot-prefixed filename `.PI_MARKER` and check for existence before creating (implementation task should be idempotent).

Success criteria
- A file named `.PI_MARKER` exists at the repository root in the default branch after PR merge with the expected content.
