Design Spec: Codebase Structure Analysis for E2E Test Repo

Goal

- Produce a concise, actionable analysis of the repository "multi-agent-harness-test-repo" (repo id: 28358939-723f-4488-8ed7-b27d77f8845d) describing the top-level layout, primary languages and frameworks, build/test/CI setup, source and test directories, dependency files, and notable project artifacts (Dockerfiles, Makefiles, docs, scripts).

Approach

- Run a sequence of read-only repository-inspection commands to enumerate files and directories, detect languages, and extract key configuration files. Commands will include folder listings (ls/tree/find), language counters (cloc or linguist-like heuristics), and targeted reads of manifest and CI files (package.json, pyproject.toml, go.mod, Cargo.toml, Gemfile, requirements.txt, Pipfile, package-lock.json/yarn.lock, pnpm-lock.yaml), CI configs (.github/workflows, .gitlab-ci.yml), Dockerfile/Makefile, and top-level README/CONTRIBUTING.
- Summarize findings into a small Markdown report (ANALYSIS.md) that lists: repository overview, language breakdown, important files, source layout, tests, CI/build flows, and recommended next steps for maintainers or integrators.

Key decisions and assumptions

- The analysis will be non-destructive and read-only.
- Because the planning stage cannot clone or inspect the repository directly, the plan will be a precise, reproducible set of commands and file reads for an implementing agent with repository access to produce the report.
- The implementation will prioritize common manifest and layout patterns (Node, Python, Go, Rust, mono-repos with packages/, services/, apps/, etc.). The plan will also cover how to detect monorepo managers (lerna, pnpm, turborepo, nx) and package managers (npm, yarn, pnpm).
- Output will be a single Markdown report at ANALYSIS/README or ANALYSIS.md at repo root and a short machine-readable JSON summary (analysis.json) containing the language breakdown and key file list.

Success criteria

- The repository root contains an ANALYSIS.md summarizing the structure and listing key files and directories.
- The repository root contains analysis.json with fields: languages, top_level_dirs, manifests, ci_files, dockerfiles, test_dirs, src_dirs.
- The report includes recommended next steps (e.g., tests to run, CI to inspect, possible integration points).

Deliverables

- ANALYSIS.md (human-readable summary)
- analysis.json (machine-readable summary)
- A short log commands.txt showing the exact commands executed

Security and privacy

- No secrets will be printed. If .env or similar files exist, the analysis will note their presence but not dump contents.

Timeline and effort

- Single implementing agent: ~15-30 minutes for a normal-sized repo (hundreds of files). Larger repos may require more time.
