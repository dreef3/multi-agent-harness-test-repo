Design spec: Add a simple repository marker file

Goal
- Create a small, human-readable marker file in the repository root to signal that the planning agent has recorded an intent to modify the repo (used by the multi-agent harness e2e tests).

Approach
- Add a single text file at the repository root named REPO_MARKER.txt containing a few metadata lines: repository name, repository id, creation date, and a short purpose note. The file is intentionally minimal and non-executable.

Key decisions
- File name: REPO_MARKER.txt (uppercase, clear semantic meaning).
- Location: repository root to make it easy to find and verify.
- Content: plain text with repository name and id, created-by tag, date, and purpose. No sensitive data.
- Commit and PR: create a dedicated branch (pi/add-repo-marker) and open a PR titled "Add repository marker file" with a brief description.

Rationale
- A single marker file is low-risk, transparent, and simple for automated verification by CI or harness tests.
