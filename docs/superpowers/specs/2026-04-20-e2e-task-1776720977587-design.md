# Design Spec: Add simple marker file

Goal

Create a single, lightweight marker file at the repository root of the E2E Test Repo 1. The marker file serves as an unambiguous artifact indicating that the repository has been touched by the planning process (useful for CI/e2e test detection and bookkeeping).

Approach

- Add a single text file named MARKER.txt at the repository root.
- The file will contain minimal metadata: repository UUID, creating agent identifier, and an ISO-8601 timestamp placeholder (to be filled at commit time).
- Keep the file content intentionally small and human-readable so it can be inspected in the repo browser or by simple tooling.

Key decisions

- Filename: MARKER.txt (explicit, visible in repo listings; avoids dotfiles that some tools hide).
- Location: repository root so it's easy to find and consistent across repositories.
- Content: plain text with 3–4 lines (repository id, creator, timestamp, and purpose). Include the repository UUID provided in the task description so the artifact is uniquely tied to this planning run.
- Branching & Commit: create a short-lived branch (e.g., add-marker-file) and a single commit with a clear message such as "Add simple marker file". Open a PR from that branch to main.

Minimal acceptance criteria

- MARKER.txt exists at the repository root in the PR and contains the repository UUID provided in the task description.
- PR is opened against the default branch (main).
- File is < 1 KB and contains only plain text metadata.
