# Design Spec: Add a simple repository marker file

Goal
- Add a small, human- and automation-friendly marker file at the repository root to indicate that the harness created a marker. This is intentionally minimal: a single file containing a short message and a timestamp.

Approach
- Create a new file named REPO_MARKER.txt at the repository root.
- File content will be a few lines: a one-line identifier, a creation timestamp (UTC), and an optional short note (who/what created it).
- The planning branch will include only this new file; no other changes.

Key decisions
- Filename: REPO_MARKER.txt (plain text and visible in repository listings).
- Location: repository root for maximum discoverability.
- Content format: 2–3 lines, e.g.:
  REPO_MARKER
  Created: 2026-04-19T00:00:00Z
  Created by: multi-agent-harness
- The actual timestamp will be set when the implementation task runs.

Acceptance criteria
- A file named REPO_MARKER.txt exists at the repository root in the planned change.
- File contains an identifier and a creation timestamp.
- Change is limited to adding this single file.

Risks and mitigations
- Risk: filename collision with existing repository files. Mitigation: chosen name is generic but unlikely to exist; implementer should check for existing file and fail the task if it already exists (or choose alternative name with suffix).  
- Risk: timezone confusion. Mitigation: use UTC ISO 8601 timestamp.
