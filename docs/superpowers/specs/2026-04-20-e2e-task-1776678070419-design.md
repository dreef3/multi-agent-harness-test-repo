Design spec: Create a simple marker file in the repository

Goal
- Add a single, minimal marker file to the repository root so CI / tooling and humans can detect that the planning PR was created and the repository was touched by the harness.

Approach
- Create a new file at the repository root named REPO_MARKER.txt containing one line identifying the repository and creation timestamp placeholder.
- Keep the file content minimal and non-sensitive; it should not affect builds or code.

Key decisions
- File name: REPO_MARKER.txt — chosen because it's visible in directory listings and avoids hidden-file problems.
- Location: repository root — simplest and most discoverable location.
- Content: single-line text: "Marker: created by planning agent" with optional repository identifier and ISO8601 timestamp placeholder. The planning agent will include the exact text to write in the implementation plan, but will not modify repo itself.
- Commit metadata: branch name "add/repo-marker", commit message "Add repository marker file", PR title "Add repository marker file".

Success criteria
- A new file REPO_MARKER.txt exists at the repo root with the specified content in the created PR.
- The file is small, human-readable, and harmless to build processes.

Constraints and assumptions
- Do not run or modify any other repository files.
- This planning document will be committed to the planning branch by the harness.
