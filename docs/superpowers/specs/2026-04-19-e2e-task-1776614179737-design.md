# Design Spec: Add Repository Marker File

Goal

- Add a simple, human-readable marker file to the repository root to indicate that an automated planning agent produced a planning artifact. The file provides a minimal, non-intrusive signal for later automation or auditing.

Approach

- Create a plain text file at the repository root named REPO_MARKER.txt containing a short structured message with the repository name and id, the agent name, and the creation date (ISO 8601).
- Keep the file minimal so it won't affect builds or tooling. Use plain ASCII content and a short filename (no leading dot) so it is visible in directory listings.

Key decisions

- File path and name: /REPO_MARKER.txt (root of the repository). Chosen to be obvious and easy to find.
- Content format: 3–5 lines of plain text with fields: Repository, Repository ID, Created-by, Created-on. This keeps it simple and suitable for scripts.
- Creation date: use the planning agent's date at the time the plan is created (ISO 8601, e.g., 2026-04-19). If implementer prefers, they may update the date to the actual commit date when creating the file.

Minimal scope

- This is intentionally small: only one new file added, no code changes, no tests required.
