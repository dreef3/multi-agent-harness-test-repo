# Design Spec: Add simple repository marker file

Goal
- Create a simple marker file in the repository to indicate the planning agent has run and to serve as a lightweight sentinel for automation/CI.

Approach
- Add a single hidden marker file at the repository root named `.repo-marker` containing a short, single-line message and a timestamp.
- Keep the file minimal (one or two lines) so it has no functional impact on the project and will not affect builds.

Key decisions
- File path: `.repo-marker` at repository root. Hidden file reduces clutter in listings.
- File content: A single human-readable line with a brief description and ISO 8601 date (2026-04-19). This is sufficient as a marker and easy for scripts to detect.
- Commit metadata: the implementation task will create a branch (e.g. `add-repo-marker`) and a single commit with message: "Add repository marker file".

Constraints
- Per instructions, this document only describes the change; the actual implementation will be performed by the CI or a sub-agent after planning PR is created by the harness.
