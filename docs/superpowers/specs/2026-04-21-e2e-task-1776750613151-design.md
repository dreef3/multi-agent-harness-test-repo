Title: Add simple repository marker file

Goal
- Add a simple, lightweight marker file to the repository to signal that this repository has been processed by the multi-agent harness test flow. The marker is intended for downstream automation and human reviewers.

Approach
- Create a single hidden marker file at the repository root named .e2e-marker. The file will contain a small, static set of metadata lines including the repository name and repository UUID from the provided instructions.
- The change will be committed in a single commit with a clear message so the harness can open a PR against the repository's default branch.

Key decisions
- Filename: use a hidden file name ".e2e-marker" to keep the repo clean and avoid cluttering top-level files.
- Content: include the repository display name and repository id exactly as provided: "E2E Test Repo 1" and "ea078373-bcf2-4333-929b-e18ee8d78cb6". Keep the content minimal and human-readable.
- Location: repository root so the harness and other agents can find it predictably.

Acceptance criteria
- A file exists at /.e2e-marker in the repository with the expected content.
- The change is present in a single commit and a PR is created by the harness.
- The file is small (one to a few lines) and encoded UTF-8.

Notes
- This spec is intentionally minimal. No code changes to project sources are needed.
- Implementation will be performed by the automated implementation agent after this planning PR is created and merged.
