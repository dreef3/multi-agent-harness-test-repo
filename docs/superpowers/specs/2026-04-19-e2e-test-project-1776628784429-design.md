Design Spec: Analyze codebase structure for E2E Test Repo (id: c4447e85-ad42-4d8d-ab33-f5e286fe6b1f)

Goal
- Produce a concise, actionable analysis of the repository structure, languages, top-level components, test suites, CI configuration, and notable files (README, LICENSE, package manifests). Deliverable: a structured report that can be used by engineers to orient themselves and to drive further work (tests, refactors, CI changes).

Approach
- Because I was instructed not to clone or explore the repository from this planning agent, the plan below specifies a set of small, deterministic tasks that a sub-agent (or human) will run inside the repository working directory. Each task is self-contained, references exact file paths and commands, and collects artifacts (file lists, manifest contents, test outputs, CI workflow files). After the tasks run, a follow-up aggregation task will produce the final analysis report.

Key decisions
- Tasks are small and independent so they can be executed in parallel by sub-agents.
- Focus first on discovery (file tree, language manifests), then on dynamic checks (run tests, examine CI), then on metadata (README, license, contributors) and metrics (size, cloc).
- Avoid making assumptions about the repository contents; the plan enumerates exactly what to check so findings are reproducible.

Success criteria
- A final report that lists: top-level directories/files, detected languages and package managers, test frameworks and how to run tests, CI workflows, notable scripts (npm scripts, Makefile targets), and basic stats (lines of code, repo size). If tests are present, include test command outputs and failure summaries.

Constraints
- This spec does not execute any commands. It defines the actions that must be executed by implementation tasks which will inspect the repository at c4447e85-ad42-4d8d-ab33-f5e286fe6b1f and produce artifacts and the final report.
