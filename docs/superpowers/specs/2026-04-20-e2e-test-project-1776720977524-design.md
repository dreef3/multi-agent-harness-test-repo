Design spec: Analyze repository "multi-agent-harness-test-repo" (id: f38a09ea-cf13-4846-8501-16f41cf338ff)

Goal
- Provide a clear, actionable analysis of the repository's codebase structure, technologies, and configuration, so maintainers and automation agents can understand the project layout and next steps for tests, CI, and development.

Approach
- Because I must not clone or explore the repository, this spec describes a non-invasive, reproducible analysis workflow that an implementer or sub-agent can execute against the repository. The workflow is a sequence of deterministic file inspections and lightweight commands (ls, rg, cat/read, cloc where available) that produce a structured report: top-level file tree, detected languages and dependency manifests, CI/workflow files, test layout and commands, Docker/infra assets, and a summary of missing or risky items.
- The analysis will be structured into discrete tasks so they can be dispatched to sub-agents or run locally. Each task lists exact paths to inspect, CLI commands to run, and the expected parsing/outputs to generate.

Key decisions
- Non-invasive: no write/commit operations or tests run in this phase; analysis is read-only.
- Conservative detection: rely on the presence of canonical files (package.json, pyproject.toml, go.mod, Cargo.toml, etc.) and common directories (src/, tests/, .github/workflows/) to infer languages, frameworks, and CI.
- Produce a standard deliverable: (1) short textual summary (languages, frameworks, top-level layout); (2) file tree snapshot (top 3 levels); (3) dependency files and their parsed contents (name + version where available); (4) CI/workflow summary; (5) test and coverage status hints; (6) recommended next steps and potential issues.

Out-of-scope
- Running tests, installing dependencies, or modifying repository files. This spec assumes an agent with read access to the repository will run the commands and collect outputs.

Deliverables
- This design spec (saved).
- Implementation plan with step-by-step tasks and exact file paths and commands (saved).
- Resulting analysis report (to be produced by the implementer after running the plan).
