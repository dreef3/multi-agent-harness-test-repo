# Design Spec: Add repository marker file

Goal

Create a simple marker file in the repository to indicate that the planning PR was created and to provide an easily-findable artifact for CI, automation, or humans.

Approach

- Add a single new file at the repository root named REPOSITORY_MARKER.md.
- The file will contain a one-line human-readable message identifying the repository and the purpose of the file, plus the creation date.
- Keep the file minimal to avoid introducing any behavior changes or runtime impact.

Key decisions

- File name & location: REPOSITORY_MARKER.md at the repository root. Using an explicit .md name ensures visibility in GitHub UI and avoids hidden-file semantics.
- Content: A short message including repository identifier (if available), purpose, and ISO date. Example content: "Repository marker: created by planning agent on 2026-04-20." This keeps it stable and machine-readable if needed.
- Commit & PR: The harness will create the commit and open a planning branch / PR. Commit message should be: "Add repository marker file".
- No tests or code changes are required; this is a pure content addition.

Risks & Mitigations

- Risk: accidental overwrite of an existing file. Mitigation: check for an existing REPOSITORY_MARKER.md in PR generation (harness will show diff); if an existing file exists, the implementation task should abort and choose a different filename such as REPOSITORY_MARKER_<timestamp>.md.

Success criteria

- A new file REPOSITORY_MARKER.md appears in the repository root with the content specified.
- The planning PR is opened and includes only this file addition.
