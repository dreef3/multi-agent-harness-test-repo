# Design Spec: Add e2e-marker.md

Goal

Add a minimal repository marker file to indicate that an end-to-end test step has passed. This is a non-functional artifact intended for CI checks, automation, or human discoverability.

Approach

- Create a single file at repository root named e2e-marker.md
- The file will contain exactly one line: "# E2E Test Passed"
- Commit the new file on a new branch so changes are isolated and can be reviewed in a PR

Key decisions

- Filename: e2e-marker.md to match repository naming conventions and be visible in GitHub UI.
- Content: a single Markdown header line to be easily machine- and human-readable.
- Branch: create a new branch for the commit to allow code review and avoid modifying main directly.

Risks & mitigations

- If the file already exists, the implementer should create e2e-marker_<timestamp>.md instead.
- The change is content-only and will not affect runtime.

Success criteria

- A new branch is opened with a single commit adding e2e-marker.md containing the exact line "# E2E Test Passed".
