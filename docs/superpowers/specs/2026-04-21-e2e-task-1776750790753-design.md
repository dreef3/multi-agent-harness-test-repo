Design Spec: Add e2e-marker.md repository marker

Goal
- Add a single-line marker file named e2e-marker.md to the repository root containing the exact text "# E2E Test Passed". This file is intended to be used by automated end-to-end test harnesses to indicate a passing state.

Approach
- Create a new branch (e.g., add/e2e-marker) and add the file e2e-marker.md at the repository root with the single line content. Commit the change and open a PR to the default branch (main).

Key decisions
- File name: e2e-marker.md (explicitly requested).
- Content: Exactly one line containing the markdown heading: # E2E Test Passed (no additional whitespace or lines).
- Location: repository root for easy discovery by automation.
- Branch: create a short-lived feature branch to avoid direct commits to main.

Acceptance criteria
- e2e-marker.md exists at repository root.
- The file contains exactly one line: # E2E Test Passed
- A PR exists that adds the file on a new branch targeting main.
