Design Spec: Analyze codebase structure for E2E Test Repo 0 (repo id: 149771ae-97f8-4251-82a5-a539eddd175a)

Goal
- Produce a concise, actionable structural analysis of the repository that identifies: top-level layout, languages and frameworks, key configuration and build files, test suites, CI and CD configuration, dependency manifests, containerization, and documentation. Highlight potential areas of technical debt or missing elements (e.g., absent tests, missing README, no license).

Approach
- Perform static repository inspection (no runtime). Tasks include listing top-level directories and files, detecting language and build systems by presence of common manifest files (package.json, pyproject.toml, go.mod, Cargo.toml, pom.xml, etc.), identifying CI config files (.github/workflows, .gitlab-ci.yml), locating Dockerfiles, Makefiles, and common CI/CD scripts.
- Inspect directories for source code layout (src/, lib/, pkg/, cmd/, app/, internal/), test directories (test/, tests/, __tests__, *_test.go), and examples or docs directories.
- Extract dependency information from manifests (name, version, major dependencies) and note absence if manifests not found.
- Identify presence and contents of README, CONTRIBUTING, CHANGELOG, LICENSE.
- Summarize findings into a report with recommendations and next steps.

Key decisions
- Static-only analysis: do not run tests or install dependencies.
- Use manifest-file heuristics to detect language and build systems.
- Provide both a quick summary and a detailed listing to serve different audiences (maintainers vs. automated sub-agents).

Deliverables
- A Summary report (1 page) listing languages, primary frameworks, and top-level layout.
- Detailed report including file lists, key file excerpts (first 50 lines), and a task checklist for remediation or further investigation.

Constraints
- Do not clone or read the repository contents in this planning step. The plan enumerates exact file paths and commands for later execution by implementation agents.

Success criteria
- The plan enables an implementation agent to produce the summary and detailed reports with no further clarification.
