Design spec: Analyze repository structure for "multi-agent-harness-test-repo" (id: e720f506-f75a-4409-9393-2e6b7a72a300)

Goal
- Produce a thorough, reproducible inventory and analysis of the repository structure: file layout, languages, package managers, test suites, CI/CD, Docker artifacts, scripts, and dependency surface (including vulnerabilities). Deliver a single human-readable summary plus machine-readable artifacts (JSON/TXT) suitable for follow-up automation.

Approach
- Use a small set of standard CLI tools and commands to create deterministic artifacts: a file inventory, language breakdown (cloc), package-manager manifests, dependency listings, CI workflow inventory, test discovery and test-run reports, Dockerfile analysis, and a vulnerability scan where supported by language ecosystem tools.
- Organize outputs under a reports/ directory in the working copy; keep the commands and the exact outputs so results are reproducible.
- Produce a final summary report (reports/structure_report.md) that highlights key findings and remediation pointers.

Key decisions
- Rely on filesystem inspection plus ecosystem-native tooling (npm/yarn, pip/pip-audit, go, cargo, bundler, cargo) rather than attempting to infer structure only from filenames.
- Save all raw outputs (JSON/TXT) for later programmatic consumption; produce a concise summary for humans.
- Prioritize non-destructive operations: where commands modify the environment (npm install, pip install), prefer read-only or ephemeral environments (npm ci in a clean environment, virtualenv) and record the commands so a reviewer can reproduce them.

Deliverables
- reports/file_inventory.txt — full file list
- reports/cloc.json — language metric output
- reports/package_manifests.txt — detected package manifest files and their locations
- reports/dependency_* — per-ecosystem dependency lists/audits
- reports/ci_workflows.md — CI/CD workflow files and summary
- reports/test_summary.md — test discovery and run summary
- reports/docker_summary.md — Dockerfile inventory and notes
- reports/vulnerability_report.md — summarized findings from audits
- reports/structure_report.md — final human-readable summary with recommendations

Constraints and assumptions
- This planning spec assumes access to the repository clone and typical CLI tools installed (git, find, rg/rg, cloc, npm, pip, go, cargo, docker). If a tool is missing, the plan lists alternatives.
- The plan is non-invasive and will not commit changes to the repository by itself; it instructs how to create the reports. Implementation will be performed by the execution environment or sub-agents.
