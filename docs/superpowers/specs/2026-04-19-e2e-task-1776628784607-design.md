Design Spec: Create repository marker file

Goal
- Add a simple, human- and machine-readable marker file at the repository root to indicate that the planning PR touched the repo. The marker will help automated processes detect that the repo has been processed by the multi-agent harness.

Approach
- Create a plain text file at the repository root named REPO_MARKER.txt.
- The file will contain a single-line marker and some metadata: creation date (UTC) and the target repository id.
- Keep the file minimal and non-intrusive (no executable bits, no changes to build). The marker file should be safe to include in any repo and easy to find with a simple filename match.

Key decisions
- Filename: REPO_MARKER.txt. Chosen for visibility and cross-platform ease (not hidden). "REPO_MARKER.txt" is explicit and unlikely to conflict with common repo files.
- File contents: 2-3 lines consisting of a clear marker, ISO-8601 UTC timestamp, and the repository UUID from the task description (3742fa55-d77a-47e8-9b84-0b32d70fac26). Example content:
  REPO_MARKER
  created: 2026-04-19T00:00:00Z
  repo-id: 3742fa55-d77a-47e8-9b84-0b32d70fac26
- Placement: repository root.
- Branch and PR: Implementation will be authored on a short-lived feature branch (e.g., add/repo-marker) and opened as a PR against main. The planning harness will create the PR automatically; implementation tasks should not run git/gh themselves.

Non-goals / constraints
- Do not modify any existing source files, build scripts, or CI configuration.
- Do not run or include any executable code. The file is purely informational.

Success criteria
- A file named REPO_MARKER.txt exists at the repository root with the expected contents.
- The PR adds only this file and no other changes.
