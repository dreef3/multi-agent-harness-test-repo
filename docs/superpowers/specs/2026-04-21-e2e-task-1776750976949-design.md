# Design Spec: Add repository marker file

## Goal
Create a simple, human-readable marker file at the repository root that indicates this repository has been processed by the multi-agent harness. The marker will be a small text file that is easy to find and verify.

## Approach
- Add a single file at the repository root named `REPO_MARKER.txt`.
- The file will contain a short, fixed message identifying the repository and include a placeholder for the creation timestamp and repository identifier.
- Commit the new file on a dedicated branch (`add-repo-marker`) and open a PR for review and merge.

## File details
- Path: `REPO_MARKER.txt` (placed at repository root)
- Content (template):

  Marker: E2E Test Repo 1
  Repository ID: 65e3f553-8dce-4693-90a0-e11ebf637ddf
  Created-By: multi-agent-harness
  Created-At: <ISO-8601 UTC timestamp>

- Content is intentionally minimal and plain text to keep it compatible with automation and human inspection.

## Key decisions
- Use a clear filename (`REPO_MARKER.txt`) rather than a hidden file so maintainers and automated checks can easily find it.
- Place at repository root for discoverability.
- Include repository identifier in the file to tie the marker to the repository explicitly.
- Use a placeholder for the timestamp so the implementation step can insert the actual creation time.

## Security and policy
- The marker file contains only non-sensitive metadata.
- No credentials or secrets will be written.

## Success criteria
- `REPO_MARKER.txt` exists in the repository root with the content template filled (timestamp may be added by the implementer).
- A PR is opened from branch `add-repo-marker` with a clear commit message.
