# Design Spec: Add simple marker file

Goal
- Add a simple marker file to the repository to indicate the presence of the E2E test repo and to serve as a lightweight artifact that automated systems or humans can check for.

Approach
- Create a single text file at the repository root named `E2E_MARKER.txt` containing a short, unchanging payload: repository identifier, purpose note, and a placeholder timestamp. This is intentionally minimal and non-invasive: no source code changes, no build changes, no scripts.

Key decisions
- Filename: `E2E_MARKER.txt` (chosen to be visible in repo listings and avoid hidden-dot behavior).
- Location: repository root for discoverability.
- Content: plain UTF-8 text with three lines: a human-readable description, the repository UUID provided in the task, and a stable tag indicating the file was added by the planning agent. A timestamp line will be added at commit time by the implementation step if desired.
- Implementation risk: None — adding a single text file has no runtime implications.

Success criteria
- A new file `E2E_MARKER.txt` exists at repository root after the planning PR is merged, containing the specified content including the repository id listed in this spec.
