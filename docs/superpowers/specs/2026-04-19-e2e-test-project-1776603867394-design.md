Design Spec: Analyze codebase structure of repository "multi-agent-harness-test-repo" (E2E Test Repo, id: ebf38f5c-42d9-4f9f-b549-664880d67743)

Goal
- Produce a clear, structured analysis of the repository: languages, top-level directories, package / dependency files, build/test tooling, CI configuration, source layout, and potential areas requiring attention (missing README, failing tests, large binaries, mono-repo layout, etc.).

Approach
- Because I must not clone or explore the repository, this spec defines the analysis scope and the exact investigator tasks an execution agent should run in the repository working copy. The output will be a human-readable report and a machine-readable JSON summary suitable for automated consumption.
- The analysis will be accomplished by a series of small, self-contained tasks that inspect common files and directories, detect languages and package managers, parse CI and GitHub workflows, list tests, and summarize sizes and complexity indicators (lines of code by language, large files).

Key decisions
- Non-invasive: The analysis will only read files and run static inspection commands; it will not modify files or run tests/compilation.
- File-first detection: Use a prioritized list of files to detect language and tooling (package.json, pyproject.toml, setup.py, requirements.txt, go.mod, Cargo.toml, Gemfile, etc.).
- Deliverables: (1) A short human summary (top findings), (2) a detailed file-/directory-level listing, (3) a JSON report including language breakdown, top-level directories, CI files, test locations, and potential concerns.
- Implementation will be delivered as a plan of discrete tasks that can be dispatched to execution agents; each task is self-contained and includes exact file paths and shell commands to run.

Success criteria
- A complete report that identifies: primary languages, package managers, top-level layout, CI/config files, test frameworks presence, README/CONTRIBUTING, Docker/Makefile presence, and large/interesting files. The JSON summary should be schema-complete and include file counts and LOC estimates by language.
