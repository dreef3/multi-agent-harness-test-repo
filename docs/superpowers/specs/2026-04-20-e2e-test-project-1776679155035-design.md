Design specification: Analyze the codebase structure for repository "E2E Test Repo 1" (id: c6ca4ca8-5660-4067-a980-6f25fd520cf7, URL: https://github.com/dreef3/multi-agent-harness-test-repo.git).

Goal
- Produce a clear, actionable summary of the repository's structure, languages, major components, entry points, dependency files, build/test/CI configuration, and any notable patterns or missing artifacts (e.g., README, license, tests).

Approach
- This planning document outlines a non-invasive, repeatable analysis workflow that a sub-agent (or human) will run against the remote repository. The analysis will:
  - Enumerate top-level files and directories.
  - Identify common manifest and config files (package.json, pyproject.toml, go.mod, Cargo.toml, Dockerfile, .github/workflows, CI config files).
  - Detect primary languages and estimate code size by language.
  - Locate main entry points (src/index.*, main.go, cmd/, etc.).
  - Collect test directories and test runner configuration.
  - Extract dependency and build/test commands.
  - Produce deliverables: textual summary (README-like), JSON manifest of files, dependency list, and suggested next steps.

Key decisions
- The analysis will be implemented as a set of small, audit-style tasks (file enumeration, targeted file reads, dependency extraction) so results are easy to review and re-run.
- The output will be both human-readable (markdown summary) and machine-readable (JSON manifest) so other agents or CI can use it.
- The plan assumes no destructive actions and avoids making code changes.

Success criteria
- A short summary (1–2 pages) describing the repo layout and important files.
- A JSON manifest listing discovered files and their types.
- A dependency summary enumerating package manager files and commands to reproduce the environment.
- Clear recommendations for missing documentation, test gaps, or CI improvements.

Notes
- This spec only describes the analysis; it does not execute it. The next document (implementation plan) lists exact tasks, file paths, commands, and expected outputs so a worker agent can perform the analysis and produce the deliverables.
