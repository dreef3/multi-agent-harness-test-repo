Design spec: Analyze codebase structure for repository "E2E Test Repo 1" (id: ac94d952-b75d-4dfb-8445-f3bde56b444c, URL: https://github.com/dreef3/multi-agent-harness-test-repo.git).

Goal
- Provide a concise, reproducible analysis of the repository's codebase structure: top-level layout, languages and frameworks used, dependency manifests, CI configuration, tests, build tooling, and notable files (README, LICENSE, Dockerfiles, etc.).
- Produce machine-readable artifacts that downstream implementer agents can use to run deeper checks (file tree, language breakdown, dependency lists, CI list, test discovery, and a human-readable summary with recommendations).

Approach
- Create a set of automated tasks that clone the repo and collect structural information using standard command-line tools (find, rg, tree, cloc, package-manager introspection). Save outputs under an analysis/ directory in the repository workspace so results can be committed or reviewed.
- Look for common language manifests and configuration files (package.json, pyproject.toml, requirements.txt, go.mod, Cargo.toml, Gemfile, build.gradle, pom.xml), CI config (.github/workflows, .gitlab-ci.yml, circleci, etc.), test directories, linter configs, Dockerfiles, and README/license files.
- Provide summary.md that synthesizes the data and offers immediate observations and recommended next steps (e.g., run tests, update dependencies, add missing CI, improve README).

Key decisions
- Do not run full test suites or install dependencies automatically in the analysis step; collect metadata and commands to run tests instead (to avoid side effects and long runs). Make test runs optional tasks.
- Use cloc for language breakdown if available; fall back to file-extension heuristics using find + sed/awk if cloc isn't installed.
- Save raw outputs (file-tree.txt, manifests-found.txt, dependency lists) to analysis/ so sub-agents can pick them up.
- Tasks will be self-contained and include exact commands and output file paths so they can be run in the harness.

Deliverables
- analysis/summary.md — human summary and recommendations
- analysis/file-tree.txt — full repo tree (top 3 levels by default)
- analysis/manifests-found.txt — list of important manifest/config files found
- analysis/language-breakdown.txt — cloc or heuristic results
- analysis/ci-configs.txt — list and content excerpts of CI files
- analysis/tests-discovered.txt — discovered test files and suggested commands to run
- analysis/dependencies/ — per-language dependency snapshots (e.g., package.json dependencies, requirements.txt, go.mod)
