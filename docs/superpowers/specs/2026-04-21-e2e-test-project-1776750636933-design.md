Design spec: Analyze codebase structure for repository "multi-agent-harness-test-repo" (id: 731d7059-8b54-40a5-89be-6ad22438ab06)

Goal
- Produce a concise, actionable analysis of the repository's codebase structure covering languages, layout, build and test systems, CI, dependency surfaces, and potential risk/maintenance issues.

Approach
- Because I cannot clone or inspect the repository in this planning phase, this spec defines the analysis scope, the methods and commands an implementation agent should run against the repository, and the expected deliverables. The actual repository introspection will be executed in the implementation phase by follow-up tasks.

Scope and outputs
- A top-level file/directory tree (1–3 levels) and counts of files by language.
- Identification of primary languages and build systems (package.json, pyproject.toml, go.mod, Cargo.toml, etc.).
- CI and automation files (.github/workflows, .gitlab-ci.yml, Dockerfile, Makefile).
- Test setup and test-running commands, plus basic test coverage if available.
- Dependency list (direct dependencies) and flags for outdated or risky packages.
- Notable files: README.md, LICENSE, CONTRIBUTING.md, CODE_OF_CONDUCT.md, CHANGELOG.md.
- Potential issues: large generated folders, missing CI, no license, fragile scripts, binary artifacts checked in.

Key decisions
- Focus on high-impact information that helps new contributors and maintainers: how to build, test, run, and where core logic lives.
- Report will avoid changing code; it will only inspect files and produce recommendations.
- Use automated, repeatable commands so the analysis can be reproduced or re-run by CI/agents.

Deliverables
- Markdown report including tree, languages, CI summary, test instructions, dependency snapshot, and prioritized recommendations.
- Raw artifact files: file list, dependency extraction, test run output (if tests are executed later).

Constraints
- No repository cloning/inspection in this phase. The next phase will include concrete commands to run against the repository.
