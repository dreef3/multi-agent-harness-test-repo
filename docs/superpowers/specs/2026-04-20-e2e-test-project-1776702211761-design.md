Design spec: Analyze repository structure for "E2E Test Repo 0" (id: 019595aa-3665-4a36-82b5-b06c9b785fbd, https://github.com/dreef3/multi-agent-harness-test-repo.git)

Goal
- Produce a concise, actionable static analysis of the repository structure: top-level layout, primary languages, package/manifest files, CI/workflows, test layout, build/entry points, important configs (Dockerfile, Makefile), and sizes.
- Produce artifacts: a human-readable summary, a file tree, language and line-count breakdown, list of dependency manifests, CI/workflow summary, list of tests, and recommended next steps or risks.

Approach
- Non-invasive static analysis (no code execution beyond reading files). Clone the repository locally into a working directory for inspection.
- Use standard CLI tools (find, du, wc, sed, grep), and language counting utilities when available (cloc, tokei). Parse known manifest files for dependency lists.
- Generate structured outputs saved under analysis/ in the cloned repo: tree.txt, languages.json (or languages.txt), manifests.md, ci.md, tests.md, sizes.csv, summary.md.

Key decisions
- Read but do not execute project code, tests, or CI scripts.
- Target common ecosystem files by name: README*, LICENSE*, package.json/yarn.lock/package-lock.json, pyproject.toml/requirements.txt/setup.py, go.mod, Cargo.toml, Gemfile, composer.json, Dockerfile, Makefile, .github/workflows/*, .gitlab-ci.yml, .circleci/, tests/, src/, pkg/, cmd/ etc.
- Use cloc/tokei when available for language breakdown, with fallbacks to wc/grep.
- Produce both machine-friendly artifacts (CSV/JSON) and a concise human summary highlighting likely entry points and potential issues (missing README, no CI, no tests, multiple languages, large vendor directories).

Deliverables
- analysis/summary.md: human summary and recommendations
- analysis/tree.txt: repository tree (top 4 levels)
- analysis/languages.csv or languages.json: language to LOC mapping
- analysis/manifests.md: list of dependency and manifest files with short contents or parsed dependency lists
- analysis/ci.md: CI/workflows found and quick summary
- analysis/tests.md: tests layout and test command suggestions
- analysis/sizes.csv: top N directories and sizes
- analysis/report.json: combined metadata (optional)

Constraints & assumptions
- The spec assumes we can clone and read the repository. The plan will include the exact commands to do that and to produce the artifacts.

Next step
- Execute the implementation plan to gather the artifacts and produce the summary report.
