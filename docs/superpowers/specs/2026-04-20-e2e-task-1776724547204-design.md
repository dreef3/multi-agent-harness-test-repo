# Design Spec: Add Simple Marker File

## Goal
Create a simple marker file at the root of the repository to indicate that the planning PR was created for this task. The marker will be minimal, human-readable, and non-intrusive.

Repository: E2E Test Repo 1
Repository ID: 01f654cd-db55-449e-8d8e-56f0298a87eb

## Approach
- Add a new file at the repository root named `E2E_MARKER.txt`.
- The file will contain a one-line marker string identifying the repository and the purpose, for example:

  E2E Marker: multi-agent-harness-test-repo (01f654cd-db55-449e-8d8e-56f0298a87eb)

- Create a single commit that adds this file and open a PR against the default branch (main).

## Key Decisions
- Filename: `E2E_MARKER.txt` — chosen to be explicit and visible in the repo root.
- Location: repository root — easiest to find and verify.
- Content: minimal single-line marker with repository name and ID.
- Commit message: "Add E2E marker file".
- PR title: "Add simple E2E marker file".

This change is intentionally small to reduce review friction and to serve as a clear marker for downstream automation or manual verification.
