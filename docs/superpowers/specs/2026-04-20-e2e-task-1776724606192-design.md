# Design Spec: Add a simple repository marker file

Goal

- Add a minimal, easily-detectable marker file to the repository root to indicate that the planning agent has executed and to serve as a simple artifact for CI/integration tests.

Approach

- Create a single plain-text file at the repository root named MARKER.txt containing a one-line descriptive message and metadata (creation date and repository identifier).
- Keep the file content minimal and stable so it can be checked by automation or humans.

Key Decisions

- Filename: MARKER.txt — chosen for visibility and cross-platform readability.
- Location: repository root to make detection simple.
- Content: a short human-readable description including an ISO 8601 date and the repository id provided in the task description.
- Commit/PR: changes will be added on a feature branch with a concise commit message and a PR titled "chore: add MARKER.txt". The planning harness will open the PR automatically.

Success Criteria

- A file named MARKER.txt exists at the repository root.
- The file contains the agreed-upon content format.
- A PR is opened against the repository's default branch containing the new file.

Risks and Mitigations

- Low risk: this is a trivial addition. If filename collides with an existing file, use MARKER-1.txt and note the change in PR description.
