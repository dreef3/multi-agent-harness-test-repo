Design Spec: Codebase Structure Analysis

Goal

Perform a comprehensive, reproducible analysis of the E2E Test Repo (https://github.com/dreef3/multi-agent-harness-test-repo.git, branch: main) to produce a concise report describing the repository structure, primary languages and frameworks, entry points, build/test commands, CI configuration, notable files and directories, and risk/maintenance hotspots. The outcome will be a single report file (CODEBASE_ANALYSIS.md) that can be reviewed and used to prioritize follow-up engineering tasks.

Approach

Because the current planning run must not clone or explore the repository, this design documents the exact, reproducible steps that an implementer (or an automated sub-agent) should run to analyze the codebase. The analysis will be performed by running a small set of shell commands to list files, detect languages and manifests, inspect key files (README, package manifests, CI config, Dockerfile, main source directories), and run static checks for tests and scripts. The implementer will gather metadata (file sizes, LOC counts, top-level directories) and extract important details such as dependency manifests, test frameworks, build targets, and CI workflows.

Key decisions

- Focus on manifest files and common language indicators (package.json, pyproject.toml, go.mod, Cargo.toml, pom.xml, build.gradle, Dockerfile, .github/workflows, tests/) because they reveal most of the repository's structure and toolchain.
- Produce a human-readable report (CODEBASE_ANALYSIS.md) that includes: summary, language breakdown, top-level tree, important files, build/test commands, CI summary, recommended next steps, and risks.
- Use simple shell tooling (find, rg/grep, wc, sed, jq) to keep the analysis portable. Include exact commands to allow automation in a CI job or by a sub-agent.
- Provide acceptance criteria: report exists at CODEBASE_ANALYSIS.md and contains required sections with evidence (command outputs or file excerpts where appropriate).

Deliverable

- CODEBASE_ANALYSIS.md at the repository root containing the analysis and recommended next steps.
