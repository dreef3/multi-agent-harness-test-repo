# Design Spec: Add simple repository marker file

Goal

- Create a simple marker file in the repository to indicate this repo has been operated on by the multi-agent harness. The file should be minimal, discoverable at the repository root, and stable across platforms.

Approach

- Add a single text marker file at the repository root named: MARKER.txt
- The file will contain a one-line human-readable message and metadata comments (creation timestamp and generator identity). Keep contents short.
- Create this change on a feature branch and open a PR against the main branch.

Key decisions

- File name: MARKER.txt (plain filename, visible in UI, no hidden-dot prefix to avoid accidental ignoring)
- Location: repository root for easy discovery.
- Contents: minimal single-line marker plus optional metadata lines (timestamp, generator string). Example content documented in the implementation plan.
- Branch/PR: create a short-lived branch (e.g., add/marker-file) and PR with title "Add repository marker file".

Rationale

- MARKER.txt is obvious in repo listings (unlike hidden dotfiles) and meets the "simple marker file" requirement.
- Metadata helps traceability and debugging if needed later.

Scope

- This spec only covers adding a single file; it does not include additional automation, tests, or CI changes.
