Design Spec: Add a simple marker file to the repository

Goal
- Create a minimal, visible marker file in the repository root to indicate the planning agent has prepared an implementation plan. This file is intentionally simple and self-contained.

Approach
- Add a plain-text marker file at the repository root named AGENT_MARKER.txt.
- The file will contain a short, human-readable message and a few metadata fields: repository UUID (from the task description), agent name, and a date placeholder. The implementation step will fill the date at commit time.
- The change will be committed on a short-lived feature branch (e.g., add/agent-marker) and opened as a PR titled "Add agent marker file" with a concise description.

Key decisions
- File name: AGENT_MARKER.txt (visible at the repository root so it's easy to find).
- Content: A 3–5 line textual marker including repository id, agent name, and created date placeholder. No executable content or configuration changes.
- Permissions: regular file (0644).
- Branch/PR conventions: branch add/agent-marker, commit message "Add AGENT_MARKER.txt (planning agent marker)", PR title "Add agent marker file".

Why this minimal approach
- The requirement is to create a simple marker file. This design keeps the change tiny, low-risk, and easy to review. It will be straightforward to verify and revert if desired.
