Design Spec: Add e2e-marker.md

Goal

- Create a simple marker file named e2e-marker.md at the repository root containing a single exact line: "# E2E Test Passed". This serves as a lightweight artifact for E2E verification.

Approach

- Create a short-lived branch (add-e2e-marker), add the file e2e-marker.md with the exact content, commit with an explicit message, push the branch and open a PR. No changes to existing code; additive only.

Key decisions

- Filename and location: e2e-marker.md in the repository root for visibility.
- Content: the exact single line "# E2E Test Passed" (including leading # and single space) to match test expectations.
- Branch name: add-e2e-marker
- Commit message: chore: add e2e-marker.md

Success criteria

- A PR is opened that adds e2e-marker.md with the exact content.
- After merging, the file exists on the main branch with the expected content.
