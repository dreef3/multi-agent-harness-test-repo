# Design Spec: Add Repository Marker File

Goal

- Create a simple marker file in the repository to indicate the planning agent has executed and to provide a lightweight, visible artifact for E2E tests.

Approach

- Add a single plaintext file at the repository root named MARKER.txt containing a short one-line marker string and a timestamp. The file will be committed on a short-lived branch and opened as a PR for review.

Key decisions

- File name: MARKER.txt — chosen for visibility (not hidden) and simplicity.
- Location: repository root — makes it easy for tests and humans to find the marker.
- Content: a single-line string with repository identifier and creation timestamp, e.g. "MARKER: Created by planning agent — E2E Test Repo 1 — 2026-04-20T00:00:00Z". Using an ISO timestamp for clarity.
- Commit and PR metadata: Branch name `add-marker-file`, commit message `chore: add repository marker file MARKER.txt`, PR title `Add repository marker file`.
- No changes to existing code or tests; purely an additive, non-functional artifact.

Risks and mitigations

- Low risk: adding a new text file should not affect builds or runtime. If any repo-specific rules disallow new files at root, reviewers can request relocation; the plan keeps the content minimal to reduce objections.

Success criteria

- A PR is opened that adds `MARKER.txt` at the repository root with the specified content.
- The PR passes repository CI (if any) and can be merged.
