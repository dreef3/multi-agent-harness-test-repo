Design Spec: Add a simple marker file to the repository

Goal

Add a single, plainly visible marker file to the repository root that signals this repository has been processed by the planning agent. The marker is small, human-readable, and safe for any repository (no code changes, no side effects).

Approach

Create a new file at the repository root named MARKER.md. The file will contain a short description, the creation date, and an author line. MARKER.md is chosen to be visible in repository listings and to avoid hidden-file edge cases (.dotfiles) while remaining non-intrusive.

Key decisions

- Filename: MARKER.md (visible in directory listings and GitHub UI). Alternative considered: .marker (hidden) or PI_MARKER.txt; MARKER.md chosen for clarity.
- Location: repository root to make discovery trivial.
- Content: minimal, human-readable text with creation date and a short purpose string.
- No code changes or build implications: this file will only document presence; it will not be imported or referenced by the source code.

Security and repository hygiene

- The marker contains no secrets or executable content.
- The file is safe to add to all branches; planning harness will open a PR to the default branch (main) containing the new file.

Deliverable

- A PR that adds MARKER.md at repository root with a concise content and commit message: "chore: add MARKER.md marker file".
