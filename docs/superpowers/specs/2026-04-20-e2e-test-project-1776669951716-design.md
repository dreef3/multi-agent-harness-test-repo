Design spec: Codebase Structure Analysis for E2E Test Repo 0

Goal
- Produce a clear, actionable analysis of the codebase structure for repository "E2E Test Repo 0" (id: 7ce7f6c1-0572-422c-93f6-e254aca19563, URL: https://github.com/dreef3/multi-agent-harness-test-repo.git). The analysis will inventory top-level files and folders, detect languages and dependency manifests, inspect CI/workflow files and tests, identify entry points and build/runtime artifacts, flag obvious maintenance/security issues, and deliver a concise recommendations summary.

Approach
- Non-invasive repository scan (no code changes). Use a set of deterministic heuristics and file checks to classify files and subsystems. The process will be divided into small, testable tasks: inventory, language detection, dependency analysis, CI/workflows inspection, tests/coverage review, entrypoint detection, static configuration/lint/security checks, and metrics (file counts/LOC). Each task will produce a short markdown artifact under analysis/ for easy review and a final CODEBASE_ANALYSIS.md summarizing results and recommendations.

Key decisions and constraints
- We will rely on common filenames and directory conventions (README.md, package.json, pyproject.toml, go.mod, Cargo.toml, Dockerfile, .github/workflows/, tests/, src/ or pkg/, etc.) to detect technologies. This is robust for most repos but may miss custom layouts.
- No runtime execution of the code or tests will be performed in this analysis step. If dynamic detection is needed (e.g., running package managers), that will be listed as a follow-up task.
- All outputs will be saved as markdown under analysis/ to keep results version-controlled and reviewable.

Deliverables
- analysis/inventory.md: top-level files and directories with short descriptions
- analysis/dependencies.md: detected dependency files and a summarized dependency list
- analysis/ci.md: detected CI workflows and brief notes on coverage and deployment hooks
- analysis/tests.md: test locations, frameworks detected, and coverage config (if any)
- analysis/entrypoints.md: binaries, scripts, Docker entrypoints, and package entry points
- analysis/security_and_lint.md: notable security/lint config (Dependabot, CodeQL, eslint, pre-commit, etc.)
- analysis/metrics.md: file counts, rough LOC by language (via cloc or wc heuristics)
- CODEBASE_ANALYSIS.md: single-page executive summary and recommendations

Risk/Edge cases
- Repos with unconventional layouts or generated code may require human review. Multi-language monorepos will be split per language in the report.
- Private submodules or git subtrees will not be expanded by these tasks.

Next step
- Run the implementation plan which defines concrete commands and files to inspect. After the plan is executed by worker agents, collect the markdown outputs and produce the final summary.
