# Design Spec: Codebase Structure Analysis for multi-agent-harness-test-repo

Goal

- Produce a clear, concise analysis of the repository "multi-agent-harness-test-repo" (id: d57e9fa6-23e5-4eb2-b09f-59be0357a706). The deliverable is a markdown report that summarizes the codebase structure, languages, package managers, CI, tests, key folders and files, and suggested next steps.

Approach

- Because this planning agent is not allowed to clone or explore the repository directly, the spec defines an automated analysis workflow that a follow-on implementation agent will run against the repository.
- The analysis workflow will be implemented as a set of reproducible shell commands and file inspections that produce a human-readable report at docs/codebase-analysis.md in the repository and open a PR.
- Analysis will include: top-level tree, language detection, dependency manifest discovery, build/test commands, CI config inspection, key source directories, docs and examples, license and contribution files, and test coverage (if present).

Key decisions and constraints

- Non-interactive, scriptable approach: produce a single shell script with commands and outputs so the analysis can be reviewed and re-run.
- Focus on static repository structure and files; no runtime execution of project tests or build steps that alter state or require credentials.
- Output will be a markdown report (docs/codebase-analysis.md) and a supporting raw command output file (docs/codebase-analysis-raw.txt) committed to a branch and opened as a PR.
- The plan will reference standard files to inspect (README*, package.json, pyproject.toml, go.mod, Cargo.toml, build.gradle, pom.xml, .github/, .gitlab-ci.yml, Dockerfile, Makefile, tests, src/) but will remain robust if files are missing.

Deliverables

- docs/codebase-analysis.md — high-level human-readable analysis and recommendations.
- docs/codebase-analysis-raw.txt — raw command outputs (ls, tree, rg summaries) for traceability.
- scripts/generate-codebase-analysis.sh — script that runs the analysis and writes the files above.
- PR opened on the repository with these files added on a feature branch (handled by the harness).

Success criteria

- The report lists repository languages, top-level directories, key files, package managers, CI config, test presence, and recommended next steps.
- The analysis script runs without requiring credentials and exits with a non-zero status only on unexpected errors.
