Design Spec: Add e2e-marker.md to E2E Test Repo

Goal
- Add a minimal marker file at the repository root named e2e-marker.md containing exactly the single line:

# E2E Test Passed

(including a trailing newline). This serves as a non-invasive marker that automated systems or human reviewers can check to confirm E2E completion.

Approach
- Create a small, single-file change on a new feature branch. The change must be limited to adding the file only; no other files or code should be modified.
- Open a pull request from the feature branch into main. After review, merge to main. Validate the file exists on main with the exact content.

Key decisions
- Filename and location: e2e-marker.md placed at repository root for discoverability.
- Exact content: a single markdown header line "# E2E Test Passed" followed by a newline, and nothing else (no timestamp, metadata, or additional text).
- Branch naming: add/e2e-marker. If that name is already in use, create a unique branch name add/e2e-marker-<timestamp>.
- Commit and PR metadata: Commit message "Add e2e-marker.md" and PR title "Add e2e-marker.md" with a short body describing the single-line change.

Success criteria
- After merge, main contains /e2e-marker.md with exact content "# E2E Test Passed\n".

Risks and mitigations
- Risk: feature branch or PR with the same name already exists. Mitigation: detect and reuse existing PR if it matches the desired change, or create a unique branch name.
- Risk: content drift. Mitigation: enforce the exact required content in the implementation task and reviewer checklist.

Constraints
- Do not modify other repository files.
- No timestamps or additional metadata in the file; content must be exact.
