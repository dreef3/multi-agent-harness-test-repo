Design Spec: Create a simple marker file in the repository

Goal
- Add a minimal marker file to the repository to signal that the E2E Test Repo 0 has been prepared/identified. The file should be easily findable at the repository root and contain a small amount of identifying metadata (repository name and id).

Approach
- Create a single text file at the repository root named E2E_MARKER.txt. The file will contain one or two lines: a short human-readable description and the canonical repository identifier (id: dd8d0426-5f12-48de-8a21-cf4a10f8c1d7). Optionally include an ISO 8601 timestamp indicating when the marker was created.
- Keep content minimal to avoid affecting project behavior and to make automated checks easy.

Key Decisions
- Filename: E2E_MARKER.txt (explicit, visible in repo root). Avoid hidden file names so tests and humans can find it easily.
- Contents: include repository id and a brief description. A sample content line: "E2E Test Repo 0 - dd8d0426-5f12-48de-8a21-cf4a10f8c1d7" followed by an ISO timestamp.
- No changes to existing files, no executable code, no tests required.
- Commit and open a short-lived branch named add/e2e-marker with commit message: "Add E2E marker file". PR base: main.

Risks and Mitigations
- Risk: file name may conflict with existing repository files. Mitigation: chosen name is generic but unlikely to conflict; reviewers should confirm. If a conflict exists, choose E2E_MARKER_<unique>.txt.
- Risk: repository policies about added files. Mitigation: marker content is non-sensitive and small.

Success Criteria
- A file named E2E_MARKER.txt exists at repository root containing the repository id and optional timestamp.
- PR merging the change into main.
