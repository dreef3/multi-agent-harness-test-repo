# Design Spec: Add a simple repository marker file

Goal

- Add a single, small marker file to the root of the repository to indicate that the planning/agent workflow ran and to provide a predictable place for CI/e2e tests to look for a marker.

Approach

- Create one plain-text marker file at the repository root named REPO_MARKER.txt containing a short human-readable message and the repository UUID. The file is intentionally minimal, machine- and human-friendly, and safe to commit to any branch.

Key decisions

- Filename: REPO_MARKER.txt (keeps it visible in repo root; avoids hidden-dot files to make discovery easier).
- Content: one to three lines: a short message, timestamp placeholder (optional), and the repository UUID provided in the task description. Including the repo UUID helps tie the marker to the planning task.
- Branching/PR: Implementation will be done on a short-lived branch (e.g., plan/add-repo-marker) with a clear commit message and a PR for review and merge.
- No code changes besides adding this single file to keep the change minimal and risk-free.

Acceptance criteria

- The repository contains REPO_MARKER.txt at the root on the default branch after the PR is merged.
- File content matches the agreed text in the implementation plan.
