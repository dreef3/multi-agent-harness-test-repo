# Design Spec: Add repository marker file

Goal

Create a simple, visible marker file at the repository root to indicate this repository has been touched by the multi-agent planning process. The marker will be intentionally small and non-invasive.

Approach

- Add a single new file at the repository root named MARKER.txt.
- The file will contain a one-paragraph human-readable message and a reference to the repository id used by the planning harness.
- Keep the file content minimal so it does not affect builds or tooling.

Key decisions

- File name: MARKER.txt — chosen for high visibility (not hidden) and simplicity.
- Location: repository root — easiest for humans and automation to find.
- Content: a short message with the repository UUID and a note not to modify without review.
- Commit: single commit with a clear message "Add MARKER.txt marker file" on a feature/planning branch. No other files will be changed.

Risks and mitigations

- Risk: Some CI or tooling might detect unexpected files. Mitigation: the file is a plain text artifact with no executable content and minimal size; include it in top-level README if required later.

Deliverable

- A new file at /MARKER.txt with the agreed content and a corresponding commit/PR opened by the planning harness.
