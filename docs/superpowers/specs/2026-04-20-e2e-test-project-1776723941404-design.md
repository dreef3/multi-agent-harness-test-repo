Design Spec: Analyze codebase structure for E2E Test Repo (08088a6f-4199-4f4d-8084-1e6b2eb6ccf9)

Goal
- Produce a concise, actionable analysis of the repository structure, languages, key components (src, tests, CI), dependency surfaces, and potential risk/attention areas.

Approach
- Perform an automated inventory and summarization of the repository contents using file-system and language detection tools.
- Identify top-level directories and files, common build files, CI configs, Docker/config infra, and test suites.
- Extract dependency manifests and produce a summary of declared dependencies.
- Summarize developer-facing docs (README, CONTRIBUTING) and licenses.
- Produce outputs: file tree, language statistics, dependency lists, CI summary, test command discovery, and risk/notes.

Key decisions
- Focus on artifacts that influence maintenance and security first: dependency manifests, CI pipelines, Dockerfiles, and tests.
- Use commonly available CLI tools (find, rg, jq, cloc, tree) and language/package-specific tools (npm/yarn, pip/pipdeptree, pip-tools, cargo, mvn/gradle) where manifests are present—fall back to static parsing if package manager not installed.
- Save structured artifacts (JSON/text) that can be reviewed or used by downstream agents.
- Keep analysis reproducible: each task lists exact commands and output file paths.

Outputs
- spec saved in planning branch (this document).
- artifacts expected from implementation (file_tree.txt, language_stats.json, dependency_manifest.* , ci_summary.md, tests_summary.md, risk_report.md, optional architecture diagram).

Constraints
- The plan does not clone or access the repo itself (per instructions). The plan is an operator for sub-agents that will run in an environment with network/clone access.
