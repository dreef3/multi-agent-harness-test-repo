Design Spec: Codebase Structure Analysis for E2E Test Repo

Goal
- Produce a clear, actionable static analysis of the repository at https://github.com/dreef3/multi-agent-harness-test-repo.git (repo id: 4cb446ae-9b19-4348-999f-730272ee9ae3). The output should summarize language(s), layout, CI, tests, dependency management, Docker/Kubernetes artifacts, docs, notable tooling, and high-level recommendations for onboarding and further automated checks.

Approach
- Perform a static repository scan (no build or test execution) to enumerate files and directories, detect language(s) and package managers, locate CI and automation, and collect evidence for tests, linting, and deployment artifacts.
- Produce a human-readable report at the repository root (CODEBASE_ANALYSIS.md) containing a file tree summary, language breakdown, CI/workflow summary, test layout and commands, dependency files, developer tooling, and prioritized recommendations.

Key Decisions
- Analysis is static-only. We will not run tests, install dependencies, or execute project code to avoid side effects and long-running operations.
- The plan targets common file locations and patterns (README.md, package.json, pyproject.toml, .github/workflows, src/, tests/, Dockerfile, etc.) to be robust across many projects.
- The final deliverable is a single markdown report plus a short machine-readable JSON summary for downstream tools.

Outputs
- CODEBASE_ANALYSIS.md (human friendly) at repo root
- analysis_summary.json (machine-readable) at repo root
- A short set of recommended next steps (CI checks, test runs, dependency pinning, coverage measurement)

Success Criteria
- Report created and committed to the repository describing the structure and containing evidence (file lists and excerpts) for the main findings.
- Clear action items for maintainers and follow-up automated tasks.

Notes
- This spec is intentionally conservative: it avoids executing repository code or network calls. If maintainers want dynamic checks (test runs, dependency installs), those will be proposed in the recommendations but not performed as part of the static analysis.
