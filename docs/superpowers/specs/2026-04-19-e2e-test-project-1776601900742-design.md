Design spec: Analyze codebase structure for "multi-agent-harness-test-repo" (id: 31be50fc-d386-4b6f-9a20-eaf00a0b8a8b)

Goal
- Produce a concise, actionable analysis of the repository structure, contents, and tooling. Deliverables: a written report that identifies top-level layout, languages and package managers, build and test files, CI/workflow configuration, key entry points, and recommended next steps (tests, linting, dependency checks).

Approach
- Perform an automated, stepwise inspection of the repository: clone the repo, enumerate top-level files and directories, detect languages and package managers, inspect key manifest and configuration files (README, package.json, pyproject.toml, go.mod, Cargo.toml, Gemfile, Pipfile, setup.cfg/setup.py), inspect .github/workflows and Dockerfiles, locate tests and test frameworks, and run (optionally) linters and unit tests in a sandbox.
- Generate a structured report: top-level tree, language and dependency summary, CI matrix, test coverage of repo and commands to run tests, and any notable issues (missing docs, monorepo layout complexity, or unusual configuration).
- Produce artifacts: REPORT.md in the repo root (for human consumption), optional dependency graph (output/deps.png), and a checklist of follow-up tasks (e.g., fix failing tests, add CI badges).

Key decisions and constraints
- Non-invasive: operations will only clone and inspect files; changes are limited to creating the analysis report and any reproducible artifacts.
- Priority: identify what a new developer or maintainer needs to know to build, test, and contribute to the repo.
- Provide exact commands so results are reproducible and automatable in CI.

Risks
- Repository may be private or inaccessible; cloning may fail.
- Some tests may be long-running or flaky; plan commands should allow optional timeouts.
- Large repositories may produce very long listings; output will be summarized when necessary.

Success criteria
- A REPORT.md that lists: top-level directories and purpose, detected languages and package managers, build and test commands, CI/workflow summary, test locations and frameworks, entry points, Docker/containers, and suggested next steps.
