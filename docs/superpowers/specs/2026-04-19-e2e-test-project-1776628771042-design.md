Design Spec: Codebase Structure Analysis for E2E Test Repo 1

Goal
- Produce a clear, actionable analysis of the repository at https://github.com/dreef3/multi-agent-harness-test-repo.git (id: c4703286-4e3b-49a2-8ee6-4add82819367). The analysis should summarize the top-level layout, primary languages and frameworks, dependency manifests, CI config, test suite presence, and high-level risks or blockers for downstream tasks (tests, builds, or feature work).

Approach
- Since this planning PR cannot run commands or inspect the repository directly, the plan will define an automated, repeatable procedure for an executor (CI job, sub-agent, or developer) to run locally or in the harness. The executor will collect structured outputs (file tree, language detection, manifests, CI files, tests) and a short narrative summary.
- The analysis will be performed in three passes:
  1. Inventory: list top-level files and directories, important manifests (README, LICENSE, package.json, pyproject.toml, go.mod, Cargo.toml), and CI configs (.github/workflows, .gitlab-ci.yml).
  2. Content probes: open and extract key fields from manifests (name, version, dependencies, scripts), detect languages (via file extensions and common tools), detect test directories and test runners.
  3. Heuristic assessment: report large files, missing or inconsistent metadata (no README, no license), presence/absence of tests and CI, potential dependency risks, and recommended next steps.

Key Decisions
- Output format: single Markdown report (analysis/structure-report.md) containing:
  - A top-level summary (1–2 paragraphs)
  - A representative tree (top 3 levels)
  - Languages and percentages (based on file extension counts)
  - Important files (manifests, CI, docs) with extracted metadata
  - Tests and CI results presence
  - Issues and recommendations
- Tools: use standard shell utilities (ls, find, sed), ripgrep (rg) when available, and small helper scripts (Python) to parse JSON/TOML. These are non-invasive and portable.
- Deliverables: the markdown report and a compressed raw-outputs archive (analysis/raw-outputs.tar.gz) containing the full text of probed files and command outputs.

Constraints
- Do not modify the repository contents.
- The plan must be executable without elevated privileges and should avoid network access except git clone (if executor chooses to clone).

Success Criteria
- The executor can run the provided task list and produce the Markdown report and raw outputs.
- The report clearly identifies the repository structure and any missing or problematic elements for further work.
