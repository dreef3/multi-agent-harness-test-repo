Design Spec: Create a simple marker file in the repository

Goal
- Add a single, small marker file at the repository root to indicate the repository has been processed by the planning agent. This file should be human-readable and include minimal metadata (marker origin and timestamp).

Approach
- Create a new file at the repository root named REPO_MARKER with a one-paragraph content identifying the repository, the planning agent, and the timestamp of creation.
- Commit the new file on a short-lived branch and open a pull request that can be reviewed and merged by the maintainers.

Key decisions
- Filename: REPO_MARKER (plain filename at repo root). Rationale: visible without listing hidden files and unlikely to conflict with common source files.
- Content: a concise line or two with the marker origin ("multi-agent-harness planning agent"), repository ID (0b2d3c73-ba9d-4aad-9107-fbb86c7fa954) and an ISO8601 timestamp placeholder. The implementation will generate the actual timestamp at commit time.
- Branching: create branch add-repo-marker for the change.

Success criteria
- A file named REPO_MARKER exists at the repository root in the main branch after the PR is merged and contains the expected metadata line.

Constraints
- Do not modify any other files.
- Keep the change minimal and easily reviewable.

Notes
- No tests are required for this change.
- This spec will be followed immediately by a detailed implementation plan.
