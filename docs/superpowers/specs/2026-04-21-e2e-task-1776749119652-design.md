Design Spec: Add a simple marker file to the repository

Goal
- Add a simple, human-readable marker file at the repository root to indicate that the planning step has created a marker. This is a tiny, low-risk change intended to provide an explicit artifact that other automation or reviewers can detect.

Approach
- Create a single file at the repository root named AGENT_MARKER.txt.
- The file will contain a short message identifying the purpose and the agent that created the marker, and a placeholder for a creation timestamp.
- The change is intentionally minimal: a single new file, no changes to code or tests.

Key Decisions
- File name: AGENT_MARKER.txt chosen for clarity and visibility (not hidden). If the project prefers hidden files, switching to .agent-marker is trivial.
- Location: repository root so it is easy to find and detect in CI or by humans.
- Content: a one-line description plus an ISO-8601 timestamp placeholder. Example content is provided in the implementation plan.

Risks and Rollback
- Very low risk. If undesired, the file can be removed in a follow-up commit.

Success Criteria
- A new file AGENT_MARKER.txt exists at the repository root with the agreed content.
