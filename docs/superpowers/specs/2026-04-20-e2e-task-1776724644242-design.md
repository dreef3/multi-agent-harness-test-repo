Design Spec: Add a simple repository marker file

Goal
- Create a single, simple marker file in the repository to indicate the presence of this change and to serve as a lightweight artifact for automated checks or manual verification.

Approach
- Add a plain text file at the repository root named REPO_MARKER.txt.
- The file will contain a short, human-readable single-line marker that includes the repository friendly name and the repository UUID provided in the task description. This keeps the file non-controversial, easy to find, and machine-parseable if needed.

Key decisions
- File name: REPO_MARKER.txt (placed at repository root). A .txt file is universally readable and visible in directory listings and in GitHub UI.
- File contents: single-line marker with repository name and repository id to uniquely identify the target repository for auditing.
- No code or tests required; this is a content-only change.

Risks and mitigations
- Risk: naming conflict if file already exists. Mitigation: the implementer should check for an existing file and, if present, either update it or create REPO_MARKER_1.txt per repository conventions. The planned implementation will assume the file does not yet exist; if it does, maintainers can merge accordingly.

Success criteria
- A file named REPO_MARKER.txt is present at the repository root with the specified content after the PR is merged.

Notes
- This spec is intentionally minimal and low risk. It is designed to be straightforward to implement and review.
