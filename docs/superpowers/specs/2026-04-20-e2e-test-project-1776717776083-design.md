Design spec: Analyze the codebase structure and produce a concise report summarizing the repository's layout, languages, key manifests, CI/build/test configuration, and potential maintenance/risks.

Goal
- Provide an actionable, human-readable summary of the repository named "E2E Test Repo 1" (https://github.com/dreef3/multi-agent-harness-test-repo.git, default branch: main). The report should identify top-level directories, primary languages and approximate LOC, package/manifest files, CI and Docker configuration, test suites, and suggested follow-ups.

Approach
- Without cloning in this planning step, prepare a hands-on plan that a runner can execute to gather an authoritative analysis. The runner will: clone the repository, generate file and directory trees, calculate language LOC (cloc), inspect manifest files (package.json, pyproject.toml, go.mod, Cargo.toml, etc.), enumerate CI workflows (.github/workflows, .travis.yml, .gitlab-ci.yml), check for Dockerfiles and Makefiles, and list tests and coverage configuration. The runner will also look for CONTRIBUTING.md, README.md, LICENSE, and other metadata.

Key decisions and scope
- Focus on repository structure and metadata (files and directories), build/test configuration, and dependency manifests. Do not modify the repository.
- Use common tools: git, tree, cloc, ripgrep (rg), jq, and appropriate language package managers (npm/pnpm/yarn, pip, go) only to read manifests and list top-level dependencies (depth=0). Dependency installation is not required.
- Produce a final analysis document (analysis/codebase-structure.md) in Markdown with sections: Summary, Top-level layout, Languages & LOC, Manifests & Dependencies, CI & Automation, Tests & Coverage, Docker/Containerization, Risk & Maintenance notes, and Recommendations.

Deliverables
- A design spec (this document) committed to planning branch (saved via write_planning_document type="spec").
- A detailed implementation plan saved as a second planning document (type="plan") that enumerates exact commands, file paths to inspect, and expected outputs for creating analysis/codebase-structure.md.

Constraints
- The plan will not execute commands or clone the repo; it will instruct a runner (or sub-agents) how to perform the analysis and what to collect.
