# Design Spec: Add a simple repository marker file

Goal
- Add a minimal, easily-discoverable marker file to the repository so automated harnesses and humans can detect that the planned change was applied.

Approach
- Create a single text file at the repository root named REPO_MARKER.txt.
- The file will contain a short, static message with identifying metadata (repository name and ID and creation date). No code changes or behavioural changes in the repo.

Key decisions
- File name: REPO_MARKER.txt (plain text, visible at repo root). This avoids hidden files and ensures easy discovery.
- Location: repository root to make it immediately visible.
- Content: static metadata lines including repository name and repository UUID provided in the task description, plus a brief purpose statement and ISO 8601 creation date. This is short and non-actionable.
- No tests or build changes required.

Security and safety
- The file contains only non-sensitive metadata provided in the task. No secrets or executable content.

Deliverables
- A single file added at: REPO_MARKER.txt
- Commit message, branch, and PR description provided in the implementation plan.
