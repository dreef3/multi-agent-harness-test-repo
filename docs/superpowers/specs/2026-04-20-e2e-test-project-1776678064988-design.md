Design Spec: Codebase Structure Analysis for E2E Test Repo 1

Goal
- Produce a concise, actionable analysis of the codebase structure for repository "multi-agent-harness-test-repo" (id: b53cde93-4c4b-4b20-8bf0-14137ecd2a50, URL: https://github.com/dreef3/multi-agent-harness-test-repo.git). Deliver a Markdown report describing top-level layout, languages, package/manifest files, build & test commands, CI, containerization, notable scripts, and a recommended next-steps checklist.

Approach
- Clone the repository locally (or use the harness workspace) and run an automated inventory followed by targeted inspections. The automated inventory will list top-level files, detect language(s) and package manifests, identify CI/workflows, find Dockerfiles and Docker Compose, and locate test directories and scripts. Static-analysis helpers (cloc, rg/rg, git ls-files, and file/detect-helpers) will produce machine-readable outputs which will be summarized into a human-readable Markdown report.

Key Decisions / Scope
- Focus on structure and developer-facing surface (entrypoints, build/test dev commands, CI, dependency manifest). Avoid modifying code. Produce an actionable report and a set of follow-up tasks (dependency graph, test coverage, complexity) as optional deeper analyses.
- Output will be saved to docs/codebase-structure.md (or reports/codebase-structure.md if docs/ not present) inside the repository so it can be reviewed in a PR.

Deliverables
- A Markdown report reporting: top-level files/directories, detected languages and sizes, package manifests, build/test commands, CI pipeline files, containerization, tests location, notable scripts, and recommendations.
- Raw data files: file-inventory.json, language-summary.txt, ci-files.txt saved under reports/analysis/.

Constraints
- This spec assumes the executor has network access to clone the repository and standard CLI tools installed: git, tree, cloc (or equivalent), ripgrep (rg), jq. If cloc is unavailable, use wc/linguist fallback.

Success criteria
- A Markdown report committed to the repository at docs/codebase-structure.md (or reports/) describing the codebase structure and at least three actionable recommendations.


Key stakeholders
- Requester (you) and repository maintainers.
