# Design Spec: Add repository marker file

Goal
- Create a simple, human-readable marker file at the root of the repository to indicate the repository has been touched by the multi-agent harness exercise. This file will serve as a deterministic artifact for e2e checks and bookkeeping.

Approach
- Add a single plain-text file named REPO_MARKER.txt at the repository root. The file will contain a short header and a few metadata lines: repository name, repository UUID, purpose, created-by, and created-at timestamp in ISO 8601 format.
- Keep the contents minimal and stable so that automated tests can locate and validate the marker reliably.

Key decisions
- File name: REPO_MARKER.txt (chosen because it's visible, unlikely to be ignored, and clearly indicates purpose).
- Location: repository root to make discovery trivial.
- Content: plain-text with a fixed set of fields (Repository, Repository ID, Purpose, Created-by, Created-at). This allows simple grep/regex validation in CI.
- Idempotency: the plan will create the file if missing. If the file already exists, the implementation task will either skip creation or update the file depending on the implementer's preference; the plan will recommend creating a new branch and a commit that adds the file.

Success criteria
- A file at the repository root named REPO_MARKER.txt exists in the default branch (after PR merge).
- The file contains the repository UUID and a created-at timestamp.
