Design Spec: Add a simple repository marker file

Goal
- Add a minimal, human- and machine-readable marker file to the repository so automated harnesses and humans can quickly detect this repo/branch contains the test marker.

Approach
- Create a single text file at the repository root named E2E_MARKER.txt containing a short, stable identifier and the repository UUID provided in the task. The file will be intentionally minimal (a few lines) so it is safe to include in any branch or PR.

Key decisions
- Filename: E2E_MARKER.txt (plain text, visible at repo root)
- Content: a couple of lines including a human-friendly label and the repository id (09e82d21-9913-41b4-bd51-4181405b8645). Optionally include a Created-By line.
- No code or tests are added—this is solely a marker file. The implementation will be one small file addition and a commit on a feature branch with a descriptive commit message.

Rationale
- A simple and predictable file name and content makes automated detection robust and avoids changing code or CI configuration. Keeping it minimal reduces risk.

Acceptance criteria
- A file named E2E_MARKER.txt exists at the repository root in the PR branch.
- The file contains the repository id string: 09e82d21-9913-41b4-bd51-4181405b8645
- A PR is opened against the default branch (main) adding this file; CI is not expected to be changed by this PR.

Notes
- This spec is intentionally minimal as requested.
