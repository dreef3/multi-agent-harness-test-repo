Design Spec: Codebase Structure Analysis for E2E Test Repo

Goal
- Produce a concise, reproducible analysis of the codebase structure for the repository "multi-agent-harness-test-repo" (repo id: f0511a0a-da77-405f-aa54-92cd9514187b). The analysis should identify languages, packages, top-level modules, test suites, CI/CD configuration, build or run scripts, Docker artifacts, key entry points, and any obvious maintenance concerns (missing README, no license, brittle dependency files, etc.).

Approach
- Perform a static repository inspection using a sequence of deterministic command-line queries and file reads executed from the repository root. Do not run tests or build steps. Collect structured outputs (file lists, parsed package manifests, CI files, Dockerfiles, test files, and README/license presence). Summarize findings in a human-readable report.

Scope
- Top-level directory layout and notable subdirectories (src, lib, tests, e2e, examples, infra)
- Language detection using file extensions and common manifest files (package.json, pyproject.toml/setup.py/requirements.txt, go.mod, Cargo.toml, Gemfile, composer.json)
- Dependency manifests and lockfiles
- Test frameworks and test locations (unit, integration, e2e)
- CI/CD configuration (.github/workflows, GitLab CI, CircleCI, Azure pipelines)
- Docker artifacts (Dockerfile(s), docker-compose.yml)
- Entry points (main scripts, CLI, server startup files)
- Linting / formatting / pre-commit and developer tooling (eslint, black, pre-commit)
- Documentation / README / contributing / license

Key Decisions and Constraints
- The analysis will be purely static (no builds, no network calls, no git operations beyond file reads). Results will be saved under ./analysis/codebase_report.md and supporting artifacts under ./analysis/artifacts/. The plan avoids modifying source files.
- Use small, idempotent shell commands and targeted file reads so results are reproducible by other agents.
- Produce both a machine-friendly JSON summary and a human-readable Markdown report to serve different consumers.

Deliverables
- analysis/codebase_report.md — human-readable summary with findings and recommendations
- analysis/artifacts/file_tree.txt — curated file tree listing
- analysis/artifacts/language_summary.json — simple language/manifest summary
- analysis/artifacts/ci_files.txt, docker_files.txt, tests_list.txt, manifests_parsed.json

Estimated effort
- Single-agent run: ~10–20 minutes depending on repo size (static only).
