Design spec: Create a simple marker file in the repository

Goal
- Add a single marker file at the repository root to indicate the presence of this planning agent's change. This is intentionally minimal: a plain-text file that can be committed in its own PR.

Approach
- Create a new file named MARKER.md at the repository root. The file will contain a short human-readable message with the repository name/ID and the date the planning spec was created.
- Keep the file content minimal and non-functional (no build changes, no code modifications).
- The planning harness will commit the file on a new branch and open a PR for review.

Key decisions
- Filename: MARKER.md at the repository root. This is visible and easy to locate.
- Content: brief text including repository name and ID, creation date, and purpose line. No metadata or secrets.
- No modifications to existing files or tests. Single-file addition ensures a small, easy review.

Repository
- E2E Test Repo 1 (id: c6ca4ca8-5660-4067-a980-6f25fd520cf7)

Expected outcome
- A new file MARKER.md exists in the repository root containing the marker text. A PR will be opened on the repository's default branch to add the file.

Notes
- This spec only describes the change; implementation will be created in a separate commit/PR as per the harness workflow.
