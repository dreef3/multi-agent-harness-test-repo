Design Spec: Analyze codebase structure for repository

Repository:
- Name: E2E Test Repo 0
- ID: 92bacce6-02f6-4ded-966b-d998662a05bc
- URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Goal
- Produce a concise, actionable analysis of the repository's codebase structure: languages and runtimes used, top-level layout (src, pkg, cmd, tests, scripts, docs), important files (package manifests, CI configs, Dockerfiles), test and build instructions, dependency footprint, and recommended follow-up actions.

Approach
- Because I cannot clone or inspect the repository from this planning agent, this spec defines the analysis goals and the precise checklist and commands that an implementation agent will run inside the repo to produce the full analysis report. The implementation will perform filesystem inspection, parse manifest files, run dependency and test discovery commands, and output a final report file.

Key design decisions
- Non-invasive: the analysis will be read-only — it reads files and runs build/test commands only when safe and in ephemeral environments.
- Multi-language support: analysis steps cover common polyglot files (Node, Python, Go, Rust, Java, Ruby, PHP). The implementation will detect and adapt to detected languages.
- Reproducible report: output a single markdown report at docs/codebase-analysis.md and include machine-readable artifacts (file-tree.txt, dependency-list.json).
- CI/workflow awareness: include scanning of .github/workflows and common CI files to summarize pipelines.

Expected outputs
- docs/codebase-analysis.md — human-readable analysis and recommendations.
- artifacts/file-tree.txt — tree output of repo (filtered).
- artifacts/dependencies.json — found dependencies by language/manifest.

Security and safety
- No secrets will be exfiltrated. The implementation will skip scanning files in .git if configured and will not upload artifacts externally.
