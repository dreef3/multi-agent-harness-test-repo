# Design Spec: Add simple marker file to repository

Goal

Add a single, small marker file to the repository to indicate that the planning PR for this change exists and to make it easy for automation and humans to detect that this repo received the planning work.

Approach

- Create a plain-text marker file at the repository root named `MARKER.txt`.
- The file will contain a short human-readable template: repository name, repository id, purpose of the file, and a one-line note indicating it was added by the planning agent.
- The file is intentionally minimal and safe: no executable content, no configuration, and no code changes.

Key decisions

- Filename: `MARKER.txt` at repository root. A simple top-level filename maximizes visibility.
- Content: plain text with repository id and a brief note. Keep content generic so the file is non-sensitive and safe to add to any repository.
- Branch and PR: the harness will create a planning branch and PR automatically. The planning document will be committed to the planning branch as required by the harness.

Success criteria

- A new file `MARKER.txt` appears at the repository root with the described content in the merged commit on the default branch.
- The file contains the repository id and a short description so automation and humans can identify it.
