Design Spec: Add simple marker file to repository

Goal
- Create a simple marker file at the repository root to indicate that the planning stage has created a marker. The marker will be intentionally minimal and non-functional.

Approach
- Add a plain-text file at the repository root named MARKER.txt containing a short human-readable message and a UTC timestamp placeholder. The file will be created on a new branch and submitted as a small PR to the repository's default branch (main).

Key decisions
- Filename: MARKER.txt (uppercase, easy to spot in repo root).
- Location: repository root for maximum visibility.
- Content: 2–3 lines with a one-line description and an ISO-8601 UTC timestamp placeholder (the implementer will insert the actual timestamp when creating the file). Example content is provided in the implementation plan.
- Branching & PR: a short-lived branch named pi/add-marker-file and a PR titled "Add MARKER.txt marker file". The PR description will include the purpose and content of the marker file.

Acceptance criteria
- MARKER.txt exists at the repository root on main after the PR is merged.
- The file contains the provided human-readable message and a valid timestamp.
- The change is limited to adding this single file; no other files are modified.

Risks & mitigations
- Risk: accidental changes to other files. Mitigation: implementation task explicitly adds only MARKER.txt and the PR should show a single-file addition.

Notes
- This is intentionally minimal and has no runtime impact.
