Design spec: Analyze codebase structure for repository "E2E Test Repo 1" (id: e6b7e30e-3d6e-47d4-b853-b0471b7ad216)

Goal
- Produce a clear, actionable structural analysis of the codebase at the repository root: top-level layout, primary languages, package and dependency manifests, important directories (src, lib, tests, examples, docs), CI and build pipelines, and any notable patterns (monorepo, microservices, packages).

Approach
- Inventory: enumerate top-level files and directories, identify language ecosystems by detecting standard manifest files (package.json, pyproject.toml, go.mod, Cargo.toml, etc.), and recognize monorepo/workspace configs.
- Dependency & Build: detect dependency manifests and build scripts (Makefile, Dockerfile, task runners), and list CI configuration files (.github/workflows, .gitlab-ci.yml). Extract dependency lists for major ecosystems.
- Code Metrics: compute per-language LOC, file counts, largest files and directories, and tests coverage surface (presence of test frameworks and test directories).
- Architecture Mapping: find probable entrypoints (main files), service or package boundaries, and document the layout (library vs application, packages, CLI, web app).
- Quality & Governance: check for LICENSE, CONTRIBUTING, CODE_OF_CONDUCT, SECURITY.md, and presence of a changelog or releases.
- Deliverables: structured report (Markdown) summarizing findings, JSON artifact with inventory and metrics, and a short recommendations list.

Key decisions
- Non-invasive read-only inspection: plan will use repository checkout or API access to read files; no code changes.
- Produce both human-read (report.md) and machine-readable (summary.json) outputs so follow-up automation tasks can consume results.
- Prioritize high-level layout and dependency manifests first — these provide the most insight quickly.

Estimate
- Small repo (<50 files): ~10–20 minutes
- Medium repo (50–1000 files): ~30–90 minutes
- Large repo (>1000 files): 1–3 hours

Success criteria
- A Markdown report that explains the repository structure, lists key files and directories, enumerates languages and dependency manifests, and provides suggested next steps.
- A JSON summary containing inventory and metrics.
