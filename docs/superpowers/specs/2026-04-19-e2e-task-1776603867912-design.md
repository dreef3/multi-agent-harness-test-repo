Design Spec: Create a simple repository marker file

Goal
- Add a small, easily discoverable marker file to the repository to indicate that the multi-agent harness planning step has run and to provide minimal repository metadata.

Approach
- Create a single text file at the repository root named REPO_MARKER.txt containing minimal metadata: repository name, repository UUID (from the task), author (planning agent), and the creation date. Keep the file short, human-readable, and encoded as UTF-8.

Key decisions
- File name and location: REPO_MARKER.txt at the repository root so it's simple to find.
- Content: A few labeled lines (repository name, repository id, created-by, created-on) to be useful for automation and humans.
- Commit metadata: a single commit with message "Add repository marker file" and a PR titled "Add repository marker file". Branch name: add-repo-marker.
- No changes to code or tests: this is strictly a marker file; no functional changes.

Why
- A stable, minimal marker file is useful for E2E test repos and automation to confirm provisioning or pipeline steps completed.

No implementation is performed in this spec. The next document (implementation plan) contains exact steps and the file contents to be added.
