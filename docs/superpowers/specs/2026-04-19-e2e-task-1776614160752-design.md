# Design Spec: Add simple repository marker file

Goal

Create a simple, human-readable marker file in the repository root to indicate this repository was processed by the planning agent. The marker will be small, deterministic, and safe (no code changes or build/test side-effects).

Approach

- Add a single file at the repository root named REPO_MARKER.txt.
- The file will contain the repository UUID (fe849074-e4c4-42dc-b3a5-a6ff1fc770bd), a brief message, and the creation date (2026-04-19).
- The change is limited to adding this one file; no existing files will be modified.

Key decisions

- Filename: REPO_MARKER.txt — clear, visible in git status, avoids hidden files.
- Location: repository root — easy to find and consistent across repos.
- Content: minimal metadata (repository id, creation date, short message) so it is both machine-parseable and human-friendly.
- Commit/PR: single commit with message "Add repository marker file"; PR title "Add REPO_MARKER.txt marker file".

Rationale

This approach is low-risk, easy to review, and satisfies the requirement to "create a simple marker file in the repository" without touching any code or tests.
