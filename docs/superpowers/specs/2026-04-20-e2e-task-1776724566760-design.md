# Design Spec: Add simple marker file to repository

Goal

- Create a simple, visible marker file in the repository root to indicate the repository has been processed by the planning agent.

Approach

- Add a single Markdown file at the repository root named MARKER.md. The file will contain a short human-readable one-paragraph message plus metadata lines (agent name, date, repository identifier). This keeps the change minimal, visible, reversible, and safe (no code changes).

Key decisions

- Filename: MARKER.md (chosen because it is visible in GitHub UI and not hidden).
- Location: repository root (maximizes discoverability).
- Content: small, non-executable Markdown with explicit metadata (agent, date, repository id).
- Commit/branch: create a short-lived feature branch (e.g., add/marker-file) and commit with a clear message (e.g., "chore: add MARKER.md marker file").
- No test changes or CI adjustments are needed.

Rationale

- Using a plain Markdown marker avoids platform-specific hidden files and is easy for maintainers to review and remove if desired.

Potential follow-ups

- If a standardized marker format is required across multiple repositories, we can define a convention (filename, JSON/YAML structure) and add automated tooling to populate it.
