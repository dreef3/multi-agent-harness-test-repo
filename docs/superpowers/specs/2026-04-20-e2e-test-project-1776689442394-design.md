Design Spec: Codebase Structure Analysis for E2E Test Repo 1

Goal
- Produce a comprehensive, non‑invasive analysis of the codebase structure for the repository at https://github.com/dreef3/multi-agent-harness-test-repo.git (repo id: cafcdb01-d111-4d1a-8191-20a46704edd2). Deliverables will include a human-readable REPORT.md summarizing findings and machine-readable artifacts (file tree, language summary, dependency manifests, CI/config list, test coverage of test suites, and basic code metrics).

Approach
- Clone the repository into an isolated workspace and inspect files and directories without modifying source files.
- Identify top-level layout: src/, pkg/, cmd/, tests/, scripts/, docs/, .github/, etc.
- Detect languages and frameworks via manifest files (package.json, pyproject.toml, requirements.txt, go.mod, pom.xml, Cargo.toml, Gemfile, etc.) and via cloc to count LOC per language.
- Enumerate package managers, build/test scripts, CI configs (.github/workflows, .gitlab-ci.yml, circleci/, etc.).
- Extract dependency lists from manifests (npm/yarn/pnpm, pip, go, maven, cargo) into JSON summaries.
- Locate test suites and run basic discovery commands (e.g., pytest -q --collect-only, npm test --silent --if-present) but do not run full test suites unless asked — only discover and summarize.
- Run static analysis and metrics tools where safe and quick: cloc for LOC, lizard/radon for complexity (language-dependent), and eslint/pylint quick checks only if configured and fast. These checks will be read-only and optional if tools are unavailable.
- Produce outputs: REPORT.md (human summary), tree.txt (file tree), languages.json (cloc output), dependencies/ (per-language dependency exports), ci_files.txt, tests_summary.md, metrics/ (cloc and simple complexity results).

Key Decisions
- Non-invasive: do not modify repository files. All work will be read-only.
- Reproducible: provide exact commands used and store outputs in the repo's analysis/ directory.
- Incremental and tolerant: if a tool (e.g., radon, lizard) is missing or a command fails for a language, record the failure in the report and continue other analyses.
- Security: run commands in an isolated environment and avoid executing arbitrary code from the repository. Test discovery only; no test execution unless explicitly requested.

Deliverables
- analysis/REPORT.md: summary with findings (languages, top-level layout, key files, CI, tests, dependencies, recommended next steps).
- analysis/tree.txt: top-level file tree (depth 3 by default).
- analysis/languages.json: cloc output or equivalent.
- analysis/dependencies/: per-language dependency exports (package.json parsed, requirements.txt copy, go.mod list, etc.).
- analysis/ci_files.txt and analysis/tests_summary.md.
- analysis/metrics/: lightweight complexity and LOC outputs.

Estimate
- Small repo: ~10–30 minutes. Medium: 30–90 minutes. Large/monorepo: 1–3 hours (dependent on installed analysis tooling).

Constraints
- I will produce a reproducible plan that a sub-agent can execute; I will not run the analysis myself in this planning step.
