Design Spec: Analyze codebase structure for E2E Test Repo (f648b34b-f510-4880-9f8e-9f254b32ab5d)

Goal
- Produce a clear, reproducible analysis of the repository's structure and contents for the provided E2E Test Repo. The analysis should identify languages, package managers, top-level directories, key manifests (package.json, pyproject.toml, go.mod, Cargo.toml, etc.), CI config, test frameworks, and other important project signals. Deliverables will be machine-readable artifacts (JSON) and a human-readable summary (Markdown).

Approach
- Use non-destructive filesystem inspection commands to enumerate files and directories, read important manifest files, and detect languages and tooling. No code changes will be made.
- Generate structured reports saved in a reports/ directory in the repo: a) file index (JSON), b) language/tooling summary (JSON), c) dependencies snapshot (JSON or text), and d) a human-readable repository-structure.md.
- Use common heuristics for language detection: presence of manifest files (package.json, pyproject.toml, setup.py, requirements.txt, go.mod, Cargo.toml, composer.json, build.gradle, pom.xml), file extensions, and CI config files (.github/workflows, .gitlab-ci.yml). Also inspect README and license files.

Key Decisions
- Non-invasive: only read files and run lightweight commands (find, rg/grep, head/cat, tree) so the analysis is safe to run in CI or locally.
- Keep outputs in a dedicated reports/ folder so results are discoverable and separate from source.
- Produce both JSON and Markdown outputs to satisfy automated and human consumers.
- Provide exact commands to be executed by implementers (sub-agents) so the plan is actionable and self-contained.

Success Criteria
- A reports/ directory containing file-index.json, language-summary.json, dependencies.txt (or JSON), and repository-structure.md describing the repo structure, key files, and recommended next steps.
- No modifications to repository source files.
