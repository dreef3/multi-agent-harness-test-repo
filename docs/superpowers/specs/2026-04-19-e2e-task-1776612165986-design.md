# Design Spec: Add repository marker file

Goal

Create a simple marker file in the repository to indicate that the multi-agent harness has recorded or touched the repository. This file is intentionally minimal and non-invasive.

Approach

- Add a single, visible marker file at the repository root named PI_MARKER.md.
- The file will contain a one-line description and metadata: repository UUID, intent/purpose, and a placeholder for creation timestamp and author. Keep the content human-readable and safe for all platforms.

Key decisions

- Filename and location: PI_MARKER.md at the repository root. A visible Markdown file avoids hidden-file issues and is easy for humans and tools to find.
- Content: small structured block with repository id (7f3337f7-5a2f-48f9-b19a-e9a64494bad6), purpose statement, and placeholders for created-by and created-at. Example content will be included in the implementation plan.
- Git workflow: create a feature branch named pi/add-marker-file, commit the new file with a clear commit message, and open a PR titled "chore: add PI_MARKER.md marker file" with a brief description.

Constraints and assumptions

- Do not modify other repository files.
- Keep the change minimal to reduce review overhead.
- This spec only describes the change; the actual file creation and commit will be performed by implementers (sub-agents) after the plan is accepted and PR created by the harness.
