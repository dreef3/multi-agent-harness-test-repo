Design spec: Analyze the codebase structure of the E2E Test Repo (https://github.com/dreef3/multi-agent-harness-test-repo.git, id: 96ad3d3d-7633-4afa-9f75-f779007952f1)

Goal
- Produce a clear, actionable analysis of the repository structure: top-level layout, language(s), package manager(s), key config files, test framework(s), CI setup, build/run scripts, Dockerfiles, and any obvious risks or maintenance concerns.

Approach
- Because I must not clone or explore the repository from this agent, the plan will be a sequence of self-contained inspection tasks intended for sub-agents or CI runners that do have repository access. Each task will be explicit about which files to read and which commands to run, the exact output to capture, and the format for the expected findings.
- The output of the plan will be a short written report that contains: (1) Top-level file and directory listing, (2) Detected languages and primary package manifests, (3) CI and workflow details, (4) Test suite location and test runner, (5) Build/run instructions (from scripts and containers), (6) Noted technical risks or areas that need attention.

Key decisions and constraints
- No code changes will be made; analysis is read-only.
- Tasks will prefer safe commands (ls, find, sed/grep/read) and avoid running tests or builds that could be side-effecting. If test execution is required later, that will be an explicit optional task.
- Report will surface actionable items (missing CI, outdated manifests, monorepo indicators, etc.) and will include precise file references for follow-up.

Deliverables
- A single coherent analysis report (Markdown) summarizing findings.
- A set of artifact files: full file-tree (up to depth 4), list of package manifests with their contents, CI workflow files, Dockerfiles, and tests index.

Success criteria
- The report identifies the repo's primary languages, build tools, test frameworks, CI, and notable files.
- The artifacts collected are sufficient for a developer to understand the project layout and where to start work.
