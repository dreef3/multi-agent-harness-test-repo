Design Spec: Codebase Structure Analysis for E2E Test Repo (a9ff8893-eb13-47f6-a51a-1fad29e0fecb)

Goal
- Produce a clear, actionable analysis of the codebase structure for the repository at https://github.com/dreef3/multi-agent-harness-test-repo.git (default branch: main). The analysis should identify top-level layout, primary languages, package/manifest files, CI and workflow configuration, test suites, Docker/build assets, key scripts, dependency surface, and any notable smells or missing artifacts.

Approach
- Create an automated, repeatable repository scan that produces a Markdown report. The scan will be executed by a sub-agent (or CI job) that clones the repository and runs a set of deterministic commands to enumerate files, detect languages, inspect manifests, and summarize CI/test configuration.
- Produce both a human-friendly summary and machine-readable artifacts (JSON inventory and a dependency list) so downstream automation can act on the findings.

Key decisions
- Non-invasive analysis: read repository files only (no code modifications, no test execution). Where dependency resolution or install would be useful, capture manifests and lockfiles but do not run installs.
- Standardized checklist: the report will follow a structured checklist so that all repos get the same coverage (README, license, contributing, languages, package managers, CI, tests, Docker, build, examples, artifacts, risk areas).
- Deliverables placed inside the repository under docs/codebase-analysis.md and analysis/code-inventory.json so they can be reviewed and revised in a PR.

Outputs
- docs/codebase-analysis.md — human-readable summary and recommendations.
- analysis/code-inventory.json — machine-readable inventory (top-level tree, manifests, CI files, Dockerfiles, test dirs, lockfiles, language breakdown).
- checklist.txt — short pass/fail checklist.

Constraints
- This planning step does not clone or inspect the repository; it only defines the design and implementation plan. The actual scan will be performed by one or more sub-agent tasks that will clone the repository and run the commands listed in the implementation plan.

Success criteria
- A PR in the repository contains the two files above and a small report showing the codebase structure and recommended next steps.
