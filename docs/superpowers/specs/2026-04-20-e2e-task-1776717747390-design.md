Design Spec: Add e2e-marker.md

Goal

Add a simple, single-line marker file named `e2e-marker.md` to the repository root to indicate that an end-to-end test or automated harness run completed successfully. The file should be trivial and safe for all environments.

Approach

- Create a new plaintext Markdown file at the repository root called `e2e-marker.md`.
- The file will contain exactly one line: `# E2E Test Passed`.
- Commit the new file on a dedicated branch (e.g., `chore/add-e2e-marker`) and open a PR against the default branch (`main`).

Key decisions

- Filename: `e2e-marker.md` (explicit, visible, and uses Markdown for easy rendering in GitHub).
- Content: a single Markdown heading line so it is human-readable and machine-detectable.
- Location: repository root for immediate visibility.

Constraints

- This planning step will not modify the repository; it only records the intended change.
- Implementation should be a single-file addition with minimal risk to CI or tooling.
