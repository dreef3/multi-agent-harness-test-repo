# Design Spec: Add a simple repository marker file

## Goal
Create a simple marker file in the repository to indicate that the repository has been processed by the multi-agent harness. The marker should be minimal, human-readable, and include basic metadata (repository id and date). The file will live at the repository root so it is easy to find.

Repository: E2E Test Repo 0
Repository ID: 7b8848a5-d387-4160-89c1-5480f755ab23

## Approach
- Add a plain text file named REPO_MARKER.txt at the repository root.
- File contents will include a short message, the repository id, and the date the marker was created.
- Create a branch for the change, commit the new file, and open a pull request against the default branch (main).

This change is intentionally small and non-invasive (adds only one new file). No code changes or CI updates are required.

## Key decisions
- Filename: REPO_MARKER.txt — chosen for clarity and visibility at repository root.
- Content: short message + repository id + ISO date. Example content is provided in the implementation plan.
- Branch and PR: use branch name add/repo-marker and PR title "Add repository marker file".

## Acceptance criteria
- A file named REPO_MARKER.txt exists at the repository root in the PR.
- The file contains the repository id (7b8848a5-d387-4160-89c1-5480f755ab23) and an ISO date.
- PR targets main and uses the prescribed branch and commit message.
