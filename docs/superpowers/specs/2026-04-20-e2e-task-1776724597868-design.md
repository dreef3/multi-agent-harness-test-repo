# Design Spec: Add simple marker file

Goal

Create a small, explicit marker file in the repository root to serve as an easily-identifiable artifact for CI / E2E harness tests.

Approach

- Add a single plain-text file at the repository root named MARKER.txt.
- The file will contain a short machine- and human-readable message identifying the repository, creation timestamp, and purpose.
- Commit the new file on a short-lived feature branch and open a PR to the repository's default branch (main).

Key decisions

- Filename: MARKER.txt (plain text, visible by default on GitHub). Chosen for simplicity and discoverability.
- Content: 3–4 lines including repository identifier, creation timestamp (UTC), and a short purpose statement. Example content is provided in the implementation plan.
- Encoding: UTF-8, LF line endings.
- Branch naming: feature/add-marker-file
- Commit message: "chore: add MARKER.txt marker file"
- PR title: "Add MARKER.txt marker file"

Acceptance criteria

- A new file MARKER.txt exists at the repository root with the specified content.
- The PR is created against main and shows only the new file addition.

Security & scope

- This change only adds a non-sensitive text file; no code or configuration changes.
- No external dependencies or secrets are required.
