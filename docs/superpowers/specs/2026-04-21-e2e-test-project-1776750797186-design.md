Design spec: Analyze repository "multi-agent-harness-test-repo" (id: 188ee74c-5ec7-4042-acf2-cf8d6b63cff1, URL: https://github.com/dreef3/multi-agent-harness-test-repo.git) and produce a concise but thorough codebase-structure analysis.

Goal
- Produce a clear, actionable summary of the repository structure, primary languages and frameworks, dependency surfaces, CI/CD configuration, test layout, and potential quick wins (missing docs, risky dependencies, obvious quality issues).

Approach
- Perform a non-invasive static scan of the repository (clone + read), collect: root file list, top-level directories, language indicators (package.json, pyproject.toml, go.mod, Cargo.toml, Gemfile, etc.), CI configuration (.github/workflows, .gitlab-ci.yml), Dockerfiles, Makefiles, test directories and commands, README and docs, license, and contribution guide.
- Use small command-line tools where available to augment inspection: tree (or find), ripgrep (rg) for quick patterns, cloc or scc for language breakdown, grep and jq to extract CI workflow steps and test commands.
- Produce two deliverables: (1) short textual summary for humans (README-style analysis); (2) machine-friendly manifest (JSON) listing key files, language breakdown, test commands, and CI workflows.

Key decisions
- Prioritize explicit files that declare dependencies and test commands (package.json, requirements.txt, pyproject.toml, setup.py, Pipfile, go.mod, Cargo.toml, Gemfile, etc.).
- Report both what exists and what is missing (e.g., no tests, no CI, no license) as potential risks.
- Keep the initial pass read-only and reproducible by sharing exact commands used so other agents can re-run the scan.

Success criteria
- A one-page summary listing main directories, top languages, dependency files, CI presence, test status, and recommended next steps.
- A JSON manifest with absolute paths (relative to repo root) of key files and detected test commands.
- A short list of recommended follow-ups (run tests, run linters, refresh dependencies, add CI if missing).
