Design Spec: Create a simple marker file in the repository

Goal
- Add a small, clearly-named marker file to the repository root to signal that the planning agent has prepared a change. This file will be minimal and non-intrusive so it can be used by CI or humans to detect the presence of the planned change.

Approach
- Create a single text file at the repository root named E2E_MARKER.txt.
- File contents will be a short, fixed message that includes the repository identifier from the task description so it is unambiguous which repo the marker belongs to.
- The change will be produced in a short-lived branch and submitted as a PR by the harness (this planning document will be attached to the planning branch by the harness automatically).

Key Decisions
- File name: E2E_MARKER.txt (explicit, visible, avoids hidden-file issues). If a different convention is later preferred, the plan can be adjusted.
- File content: single-line human-readable marker that includes repository id: e4579eb3-4c59-49cf-90d9-553b23a8b8b7.
- No code or executable content will be added; file is text-only.
- No tests required.

Success criteria
- A new file E2E_MARKER.txt exists in the repository root with the specified contents.
- A PR is opened by the harness for review/merge.
