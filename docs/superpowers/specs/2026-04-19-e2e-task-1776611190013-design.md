Design Spec: Add a simple repository marker file

Goal

- Add a minimal, human-readable marker file in the repository root that signals the planning agent has placed a marker in the repo. This file is purely informational (no runtime impact) and will be used by automated and human reviewers to verify the planning/implementation workflow.

Approach

- Create a single file named REPO_MARKER.md at the repository root containing a short description, the creation date, and the creator (planning agent) identity. The file will be committed on a dedicated branch and opened as a pull request against the repository default branch.

Key decisions

- File name and location: REPO_MARKER.md in the repository root. A Markdown file is chosen for readability in GitHub web UI.
- File content: a one-paragraph description and metadata (created-by and date). Keep content stable and non-sensitive.
- Branching and PR: create a short-lived branch named add/repo-marker-<short-id> and open a PR to main (default branch). Commit message: "Add repository marker file REPO_MARKER.md".
- Permissions: file should be normal text file with default file mode (0644).

Acceptance criteria

- REPO_MARKER.md exists at repository root in the target branch or merged into default branch.
- File contains the expected description and metadata (created-by and date).
- A PR was opened for code review (unless the change is applied directly by a maintainer).

Notes

- This change is intentionally minimal. No build or test changes, no code execution.
- The implementation will be performed by a separate implementation task that creates the file, commits it on a branch, and opens a PR.
