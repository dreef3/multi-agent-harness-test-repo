Title: Add a simple E2E marker file

Goal

Create a minimal, human-readable marker file at the repository root that signals the repo was prepared for E2E testing. This file is intended to be lightweight, visible in the repo browser, and stable across CI runs.

Approach

- Add a single text file named E2E_TEST_MARKER.txt at the repository root.
- The file will contain a short header and a small set of metadata fields (repository id, repository URL, created-by, created-at). The created-at field will be filled when the file is added.
- The marker file will be UTF-8 plain text, not executable, and intentionally small.

Key decisions

- Filename: E2E_TEST_MARKER.txt (explicit, non-hidden so it's easy to find).
- Location: repository root to make discovery straightforward from the repo view.
- Content: a few lines of plain-text metadata rather than JSON/YAML to keep it minimal and human-readable.
- No changes to existing code, no tests required. Single commit and a lightweight PR.

Repository

- Target repository: E2E Test Repo 0
- Repository ID: 9ded3c6b-f190-4066-bdac-2c8f5be62149
- Repository URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Acceptance criteria

- A file named E2E_TEST_MARKER.txt exists at the repository root.
- The file contains the repository id and repository URL and a created-at timestamp.
- The file is committed on a new branch and opened as a PR with an appropriate title and description.
