# Design Spec: Codebase Structure Analysis

Goal
- Produce a clear, reproducible analysis of the codebase structure for the repository:
  - ID: 71f28d54-dfd5-4446-923a-c1d15bf4686a
  - URL: https://github.com/dreef3/multi-agent-harness-test-repo.git

Approach
- Perform a non-invasive static analysis (no runtime builds) to discover the repository layout, languages, dependency manifests, test layout, CI configs, and notable scripts. The analysis will be automated so it can be re-run by CI or sub-agents and will produce both a human-friendly markdown report and machine-readable artifacts (JSON) for downstream tooling.

Key decisions
- Output formats: human-readable report at docs/codebase-analysis.md and a set of JSON artifacts under analysis/ (file-tree.json, languages.json, manifests.json, deps.json, tests-summary.json, ci-summary.json, sizes.json).
- Detection strategy: use filesystem listing (find), language line-count (cloc or git-linguist fallback), parse known manifest files (package.json, pyproject.toml, requirements*.txt, go.mod, Cargo.toml, build files), and inspect .github/workflows and CI config files.
- Test and script discovery: look for test folders (tests/, spec/, *_test.go), test runners in package manifests, and npm/test scripts.
- Deliverables: a summary README-like report with recommendations for onboarding, CI, and technical debt hotspots.

Constraints
- This spec does not access or inspect the repository contents itself; it describes the planned automated analysis tasks to be executed by implementation agents.

Success criteria
- A generated docs/codebase-analysis.md summarizing: top-level layout, detected languages, primary packages/services, test coverage locations, CI config summary, dependency types and notable large dependencies, and suggested next actions.
- Machine-readable artifacts under analysis/ for programmatic consumption.

Next step
- Execute the implementation plan (detailed tasks) to run the analysis and produce the artifacts described above.
